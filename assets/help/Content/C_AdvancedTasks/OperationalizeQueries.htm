Operationalizing queries

Xcalar Insight provides you with built-in functions for you to cleanse data and transform data to a useful format. In addition, you can use UDFs to extend the Xcalar Insight features to suit your data analysis methodology. Executing these functions interactively enables you to see results almost instantaneously. However, it does not show the query results for the entire dataset. To gain insight into the entire dataset instead of a sample of your data, you must operationalize your queries.

Operationalizing queries provides these additional advantages:

  • You can perform the same query at regular intervals. As the data in your data source changes, you can develop insights into your data over time.
  • Because the operations in an operationalized query are automated, you can avoid human errors that might happen if you had to manually execute the operations for each query.

Understanding dataflow groups

A dataflow is a set of elementary operations. A dataflow group consists of one or multiple dataflows that can be run at the same time, either immediately or according to a schedule you define. Creating a dataflow group is a step in query operationalization.

Preparing for a query

Follow these steps to prepare for query operationalization:

  1. Open the query graph viewer as described in Using the dataflow graph.
  2. Locate the query graph for the table for which you want to operationalize queries. For information about the components of a query graph, see Query graph viewer

  3. Click the Add Data Flow icon in the query graph for the table. The Add to Data Flow Group modal window is displayed, showing a preview of the dataflow.

  4. To create a dataflow group, follow these steps:

    1. Select the New Data Flow Group button, which creates a dataflow group containing the dataflow in the preview.
    2. Click Next.

    To include the dataflow in an existing dataflow group, follow these steps:

    1. Select the Append to Existing Data Flow button.
    2. To the left of the preview, a list of dataflow groups is displayed. Select a dataflow group. The dataflow group in the preview becomes a member of the selected dataflow group.
    3. Click Next.

  5. Select the columns you want to include in the file resulting from the dataflow.

    EXAMPLE: Suppose the table named Inventory contains the columns named ProductName, Code, Price, and Department. You can operationalize a query for the table and specify that only Code and Price be included in the table created by the query.
  6. Click SAVE. Now you have a dataflow that can be executed immediately or at scheduled times.

Creating a schedule for a query

After you create a dataflow group, the query can be started on demand or at regular intervals as described in these steps:

  1. Click Scheduler in the top menu.

  2. Perform one of the following steps:

    • Click Run Now to start the query immediately.

    • Click SCHEDULES in the left panel. Fill out the scheduling information as follows:

      Field Description
      Schedule Name Naming a schedule enables you to refer to it when you need to modify it.
      Start Time The local time (your browser time) at which the query starts. 
      Frequency How often the query is executed.
      Number of Runs How many time the query is executed.
  3. Click SAVE.

Modifying a query schedule

Follow these steps to modify a query schedule:

  1. Click the schedule listed in the Scheduler panel. Information about the schedule is displayed in the DETAIL panel.
  2. Click MODIFY SCHEDULE and then enter the scheduling information.
  3. Click SAVE.

You can operationalize a query instead of executing operations interactively. Operationalized queries can be scheduled to take place regularly.
