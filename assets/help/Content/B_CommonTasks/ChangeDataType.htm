Changing the data type for a column

When you point a dataset to a data source, each column in the dataset is assigned a data type, which can be one of the following:

  • Array
  • Boolean
  • Float
  • Integer
  • Mixed
  • Object
  • String
  • Unknown

A column's initial data type is Mixed if there are different types of data in the column. A column's initial data type is Unknown if Xcalar Insight cannot determine the data type or if all the values in the column are null or FNF (field not found).

NOTE: A table cell contains FNF when the row does not have a column that other rows in the table have. For example, if a column named EmlopyeeID exists for all rows except row 1, the value on row 1 under the EmployeeID column header is FNF.
IMPORTANT: The initial data type for a value generated by a UDF is always String. If your UDF produces a value that needs to be a type other than String, it is important to follow the instructions in this section to change its data type before performing operations on the value.

Restrictions on data type changes

Remember the following restrictions when you attempt to change a column's data type:

  • Changing a column's data type to Array, Object, or Unknown is not allowed.
  • You can change the data type for a column only if the column contains values. If you create a column with the Add a column option in the column drop-down menu, you cannot set the data type for the new, empty column.

Supported integer types and floating point values

The following integer types are supported:

  • Signed and unsigned 32-bit integers
  • Signed and unsigned 64-bit integers

Both 32-bit and 64-bit floating-point values are supported.

Different ways to change the data type

You can use one of the following methods to change the data type:

  • Use the Change data type option in the column drop-down menu. This method changes the data type for one column, and does not create a column.
  • Select the Map… option in the column drop-down menu. This method keeps the existing column intact and creates a column with the new data type.
  • Use the Smart Type Casting feature. This method changes the data type for one or multiple columns. It does not create new columns.

Using the Change data type option

Follow these steps to change the data type for one column:

  1. Right click the column header to display the column drop-down menu.
  2. Select the Change data type option in the column drop-down menu.

Using the map function to change the data type

After you select the Map… option in the column drop-down menu, the Map modal window is displayed.

Expand and read the following list that describes the rules and tips for entering information in a modal window launched by a Xcalar Insight function.
  • By default, the name of the column from which you start the function is displayed as an argument. If you want to use another column as an argument, you can change the argument to the name of another column.
  • When typing a column name as an argument, always start with the dollar sign ($).
  • If you provide a name for the column created by the function, simply type the column name without the dollar sign ($).
  • To avoid mistakes when typing a column name as an argument, when your insertion point is in the argument field, move the cursor to the active table and click anywhere in the desired column. If necessary, use the scrollbar to move the table to view. The column's name is automatically entered in the argument field.
  • You can undo any typing with the undo key or key combination that you normally use in your browser.
  • If the argument list includes check boxes in the Empty column, leave the check boxes unselected unless the argument is an empty string. Selecting the check box indicates that you indeed want to pass an empty string as an argument to the function.

Follow these steps to change the data type:

  1. In the Category field, select Type-Casting Functions.
  2. In the Functions field, select the desired data type.

  3. In the Argument list, you can accept the default name or enter a name for the new column that is created with the new data type.

    If the desired data type is integer, you have the option of entering the base. For example, for binary, enter 2 as the base.

    NOTE: There are two check boxes in the Empty column. When changing the data type of a column, leave the check boxes unselected. These check boxes are for operations that manipulate strings.
  4. Click Apply. A new column is created with the new data type.

Using the Smart Type Casting feature

Xcalar Insight provides data type suggestions for columns in your table. Follow these steps to change the data type for one or multiple columns:

  1. Right click the table title to display the drop-down menu.
  2. Select Smart Type Casting…. The Smart Cast modal window is displayed.

  3. The Smart Cast Result table in the Smart Cast modal window shows the data types suggested by Xcalar Insight for particular columns. The columns with suggested data types are highlighted in blue. Follow one of these steps to accept or reject the suggestions:

    • To accept all suggestions, click ACCEPT ALL SELECTED. The data type of each column is changed according to the suggestion.
    • To reject all suggestions, click CANCEL to return to the worksheet.

    • To reject the suggestion for a column and keep the original data type, click the column header. The column is removed from the Smart Cast Result table, and the column is no longer highlighted in blue.

      TIP: To quickly locate a column, click the row in the Smart Cast Result table that corresponds to the column.
  4. (Optional)For any column that is not in the Smart Cast Result table, you can click its header to display a list of possible data types. Select the desired data type. The Smart Cast Result table is renamed to Cast Result because one of the new data types is specified by you. Click CAST ALL SELECTED to change the data types for columns listed in the Cast Result table.
TIP: To display the suggestions again (for example, after you remove or replace the suggested data types), click SMART CAST. Remember, however, that doing so removes data types specified by you. The Smart Cast Result table displayed contains only columns with new data types suggested by Xcalar Insight.

You can change the data type of a column (for example, from string to integer) in one of several ways. You can change the data type without creating a column or you can keep the existing column and create a column with the desired data type.
