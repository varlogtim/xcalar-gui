Grouping data in a table

You can use the Group By function in a similar way as you would with the Group By clause in SQL. You can group by one column or multiple columns (up to 96 columns).

Aggregate functions supported

The following aggregate functions are supported:

  • Average
  • Count
  • List aggregate
  • Maximum
  • Maximum integer
  • Minimum
  • Minimum integer
  • Sum
  • Sum integer

Grouping data by columns

Follow these steps to group data by one or multiple columns:

  1. Right click the header of the column that you want to group by to display the drop-down menu. Then select Group By….

    Expand and read the following list that describes the rules and tips for entering information in a modal window launched by a Xcalar Insight function.
    • By default, the name of the column from which you start the function is displayed as an argument. If you want to use another column as an argument, you can change the argument to the name of another column.
    • When typing a column name as an argument, always start with the dollar sign ($).
    • If you provide a name for the column created by the function, simply type the column name without the dollar sign ($).
    • To avoid mistakes when typing a column name as an argument, when your insertion point is in the argument field, move the cursor to the active table and click anywhere in the desired column. If necessary, use the scrollbar to move the table to view. The column's name is automatically entered in the argument field.
    • You can undo any typing with the undo key or key combination that you normally use in your browser.
    • If the argument list includes check boxes in the Empty column, leave the check boxes unselected unless the argument is an empty string. Selecting the check box indicates that you indeed want to pass an empty string as an argument to the function.

  2. In the Group By… modal window, follow these steps:

    1. Select the aggregate function from the Function drop-down menu.
    2. In the Argument List, enter the name of the column for which you want to perform the aggregate function.
    3. In the Argument List, enter the name of the column to group by. You can enter multiple column names.

    4. If you group by one column, you can select Include Sample to include other columns not aggregated in the newly created table. Otherwise, the new table contains only the new column resulting from the aggregate function and the column you group by.

      If you group by multiple columns, you cannot select Include Sample.

    5. Accept the suggested name for the new column or type the desired name.

  3. Select Join Back if you want to join the resultant table to the original table before the group by operation. This performs a left outer join, with the original table being the left table. Joining back the table provides a convenient way to compare the values in the column resulting from the group by operation to values in the original column.

  4. Click Apply.

Example of joining back a group-by table

You have a Flights table listing the daily delay times of each airline carrier in the current year. You can perform a group by operation to determine each carrier's average delay time. In the new table, a column shows the average delay times grouped by airline carriers. If you select Join Back, the new table is joined to the original table, which enables you compare the daily delay time to the average delay time for each carrier.

Example of a Group-by operation on multiple columns

You have a Flights table with these columns: Carrier, ArrivalDelay, Date. You can perform the avg function and then group the result by Carrier and then Date. The result shows that the average arrival delay time for each carrier on each day.

You can use the Group By function on a table column in a similar way as you would with the Group By clause in SQL.
