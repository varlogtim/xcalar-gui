Query graph viewer

The query graph is a tool that provides an audit trail for your data. It helps you trace data lineage through all stages of your analytics pipeline. Over time, the data in your table might become increasingly complex due to the operations performed on the table. You can use the query graph to determine how the data has been transformed since the table was created. For example, the query graph helps you determine when a value in a column was filtered out, or what operation was used to create a particular column.

Overview of the query graph viewer

The query graph viewer presents the query graphs for all tables in the same worksheet. The following screen shows a viewer with two query graphs, one for each table. The tables are named airlines3 and airlines4.

Example of a query graph

In the following example, a series of operations take place on a table with a column named DayOfWeek, as explained in this list:

  1. A table named airlines is created.
  2. A map function is executed to change the data type of the DayOfWeek column to Integer. The map function is Int(DayOfWeek).
  3. A map function is executed to create a column that shows whether it is true or false that the DayOfWeek value is greater than 3. The map function is gt(DayOfWeek_integer, 3).
  4. The table is filtered to exclude all rows with a false value in the newly created column. The table now contains only rows with DayOfWeek greater than 3.
  5. The table is sorted by DayOfWeek in ascending order.

Task you can perform by hovering over a table icon

In each query graph, you can hover over a table icon to display a pop-up, which lists the fields in the table. You can click a field to trace its lineage. For example, clicking DayOfWeek_integer_gt6 highlights all the tables and the dataset in the query graph. This is because the column results from a map function executed on the DayOfWeek column, which exists in every table and in the dataset. To remove the highlight, click any highlighted table icon. The following screen shows the highlighted tables in a query graph.

Tasks you can perform by clicking the table icon

You can click a table icon in the query graph to display a pop-up that lists the actions you can take. The possible actions depend on whether the selected table is in the worksheet or is a temporary table, as explained in the following list:

  • Adding a table to the worksheet (if the table is a temporary table): You can click a table icon in the query graph and add the table to the worksheet. After you add the table to the worksheet, the added table has its own query graph.

  • Reverting to the selected table (if the table is a temporary table): If you want to restore the table to a particular state, click a table icon in the query graph and revert to it. The revert causes the operation icons and the table icons to the right of the selected table icon to disappear.

    NOTE: Tables removed from the query graph due to a revert are not dropped. They are categorized as Temporary in the Tables list. Temporary tables can be added back to the worksheet.
  • Finding the table in the worksheet (if the table is in the worksheet): You can quickly locate a particular table in a worksheet by clicking its icon in a query graph. This feature is useful if you have many tables in a worksheet. Remember not to maximize the query graph viewer so that the table you try to locate is visible.
  • Archiving the table (if the table is in the worksheet): You can archive a table that you do not use often. The archived table does not show up in the worksheet, but can be added back at any time from the Tables list. Also, the query graph of an archived table is removed from the query graph viewer.

  • Deleting a table: You can click a table icon in the query graph to drop the table. The table icon remains in the query graph but is displayed in gray. If the table is a part of other query graphs, its icons in those query graphs become gray as well.

    NOTE: You cannot undo a table deletion.

For more information about different table statuses such as temporary and archived, see Understanding and changing table statuses.

Saving or displaying a query graph as an image file

If you want to save a query graph as an image (for example, to include the image in a presentation or to share the image with others), click the white space in the query graph and select Save Image in the pop-up. The image is saved in the .png format.

Alternatively, you can select Open Image in New Tab, which displays the query graph as an .png image file in a new tab of your browser.
