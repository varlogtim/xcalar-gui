First steps

This section describes the steps up to the point where you can start performing operations in tables presented in a Xcalar Insight workbook.

Starting Xcalar Insight

Follow these steps to start Xcalar Insight:

  1. Point the browser to the location where Xcalar Insight is installed. For example, enter the following URL that represents the location of Xcalar Insight:

    http://xcalar.mycompany.com/index.html

  2. Log in by using the login name and password given to you by the Xcalar Compute Engine administrator. If this is your first login, the Workbook Browser is displayed. Otherwise, Xcalar Insight resumes where you left off the last time you used it.
NOTE: Each Xcalar Insight user can have one login at a given time. Closing the browser window does not automatically log you out of Xcalar Insight. If you are logged in to Xcalar Insight and you try to log in again, a message informs you that you cannot have concurrent logins. You have the option of canceling the login attempt or continue the login, which causes Xcalar Insight to terminate your other connection.

Creating a workbook as your workspace

Follow these steps to create a workbook:

  1. In the Workbook Browser modal window, click the New Workbook button.

  2. In the New Workbook Name field, type the name of the workbook.

    Workbook names are case sensitive and must be from 1 to 255 characters. There are no restrictions on special characters.

  3. Click the CREATE button. After the workbook is created, it contains one worksheet named Sheet 1. The Worksheet tab is automatically selected, but the worksheet contains no contents until you point to a dataset.

For more information about the Workbook Browser, see Workbook Browser.

Pointing to your data source

Pointing to a data source means telling Xcalar Insight to derive metadata from a dataset that Xcalar Insight can access via the NFS or HDFS protocol. The collection of metadata is called dataset reference.

Follow these steps to point to your data source:

  1. Click Data Stores in the top menu to display a three-panel screen as shown in the following example:

  2. In the left panel, the IN heading is automatically selected, indicating that you can ingest data from a data source. In the center panel, select the protocol for accessing the data source. It can be NFS or HDFS.

  3. Click BROWSE to navigate to the location of the data source.

  4. The data source can be a file or a directory. If you point to a directory, data from all records in all files in the directory is ingested. If the directory contains nested directories, you can select Point to data source recursively to traverse all the subdirectories.

    IMPORTANT: Before pointing to a directory, verify that all files in the directory are in the same format and have the same schema. Otherwise, the result might be unpredictable.

  5. When the Browse Dataset modal window opens, select the data source and click SELECT DATA SOURCE. The Browse Dataset modal window closes.

    TIP: If you have a large number of files in this modal window, use the File Format field to filter out irrelevant files. For example, select CSV in the File Format field if you are only interested in selecting from CSV files.
    NOTE: Unicode is supported for both delimiter-separated values and JSON that is UTF-8 encoded. UTF-16 or other encoding systems are not supported without UDFs.

  6. In the center panel , you can accept the suggested dataset reference name or type the desired name.

    Dataset reference names are case-sensitive. The following list describes the dataset reference naming rules:

    • Length can be from 1 to 255 characters.
    • Characters can be A-Z, a-z, 0-9, and underscore (_).

    For example, you can name one dataset my_test_dataset1 and another My_test_dataset1.

  7. If the data format is CSV or text, follow these steps to preview the data and then save the settings in the worksheet. Previewing data means you can see the suggested delimiters and promote the first row to header in the Preview panel. If the format of the data source is not CSV or text (for example, JSON), click POINT TO DATA. (No delimiter is suggested for JSON data. Therefore the SMART PREVIEW button is not applicable to a JSON data source.)

    If the SMART PREVIEW button is applicable to your dataset, it is strongly recommended that you follow these steps to create an easy-to-read preview. However, if the suggested field delimiter and line delimiter are acceptable, you can select Promote First Row As Header and then click POINT TO DATA.

    IMPORTANT: If you point to the data without promoting the first row at this point, you will not be able to change the first row into the header row at a later time.
    1. Click SMART PREVIEW.

    2. In the Preview panel, accept the suggested delimiter or choose another delimiter. For example, click Apply tab as delimiter as shown in the following sample screen:

      If you do not want to use the default character as the delimiter, specify the desired characters. Use the cursor to highlight an occurrence of the characters in the preview, and Xcalar Insight will automatically highlight all instances of the selected characters. Click Apply highlighted characters as delimiter.

      After you apply the delimiter, data appears in a columnar, easy to read format with the appropriate delimiter separating the columns.

    3. Promote the first row as header in one of two ways:

      • Click the Promote first row as header button.
      • Click the Promote Header icon on the left of the first row.
      IMPORTANT: If you point to a directory, and each file in it has its own header row, you might see multiple header rows in the preview. After you save as described in the next step, Xcalar Insight will remove the duplicate headers.

      A preview of the dataset formatted for readability is displayed.

    4. Click Save & Exit.

You now have a dataset reference pointing to a data source through the selected protocol. The data reference is listed in the Dataset Refs section of the left panel.

Managing dataset references

In the Dataset Refs section of the left panel, you can organize dataset references in folders. For example, you can create a folder named Finance for dataset references pointing to datasets in the Finance department, another folder named Engineering for dataset references pointing to datasets in the Engineering department, and so on.

You can see other users' dataset references. You can preview and create tables from these dataset references. However, you cannot perform the following tasks if you are not the owner of the dataset reference:

  • Renaming the dataset reference
  • Moving the dataset reference to another folder
  • Deleting the dataset reference
  • Creating a folder in another user's dataset reference folder
