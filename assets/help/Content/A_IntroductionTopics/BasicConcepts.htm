Basic Xcalar Insight concepts

This section explains basic concepts you must understand to use Xcalar Insight efficiently.

Understanding the purpose of Xcalar Insight

Xcalar Insight is a visual tool that enables you to interactively and intuitively design queries and algorithms, and to synthesize schema. The queries generated when you use Xcalar Insight interactively can be saved and operationalized. Xcalar insight can also be used to schedule operationalized queries.

The data being queried can be arbitrary data in arbitrary formats. It can reside on any arbitrary data source such as an HDFS, Amazon S3, or local ext4 file system.

Understanding workbooks and worksheets

You can create workbooks in Xcalar Insight, which can be regarded as files. Within a workbook, you can create as many worksheets as you want. Worksheets in the same workbook usually contain data related to the same project. You can create tables in each worksheet.

The relationship between a worksheet and a workbook is similar to the relationship between an Excel worksheet and workbook.

Understanding tables

You create a table from the dataset that points to a data source. With Xcalar's Data-In-Place architecture, data is pointed to at the source and analyzed without an explicit load or ETL (Extract, Transform and Load) processing.

You can manipulate data within a table (for example, by sorting) or you can manipulate multiple tables to create a new table (for example, by joining). You use table operations to implement an algorithm that you design for deriving meaning from your data.

Data formats supported

Without any user-defined functions (UDFs), you can use Xcalar Insight to point to data in CSV or JSON.

NOTE: The CSV format includes not only comma-separated values but also other delimiter-separated values. You can select the delimiter for separating fields and for separating records. The default field delimiter is tab and the default record delimiter is newline (\n).

Data encoding supported

Unicode is supported for both delimiter-separated values and JSON that is UTF-8 encoded. UTF-16 or other encodings are not supported without UDFs.

How to enter a string in Xcalar Insight

In the Xcalar Insight user interface, when you need to type a string value (for example, when you want to filter out rows containing the string "John"), always surround the string with quote marks.

If the string contains a backslash (\) for escaping a special character, you must use double quote marks. If the string contains a backslash that should be interpreted as a regular character, you must use single quote marks. Either single or double quote marks can be used if the string contains no backslashes.

To use Xcalar Insight efficiently, you must understand some basic concepts such as the purpose of Xcalar Insight, meanings of workbooks, worksheets, and tables, and what types of data encoding and data formats are supported.
