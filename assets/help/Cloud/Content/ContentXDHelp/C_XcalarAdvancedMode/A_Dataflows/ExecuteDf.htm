Executing a dataflow

This topic describes the steps for executing a dataflow. It assumes that you have already created a dataflow as described in Creating and editing a dataflow.

You can execute a partial dataflow or an entire dataflow. In addition, if the dataflow is built for optimized execution, you can execute the dataflow with or without optimization. For more information about optimized execution, see Understanding a dataflow with optimization.

Requirements for a dataflow execution

For a dataflow to be executed, all Nodes in the dataflow or the selected part of the dataflow must be executable. The following conditions must be met for a Node to be executable:

  • The Node must be configured.
  • Unless the Node is the source Node (the first Node of a dataflow or a sub-dataflow), the Node must have a parent Node. In addition, all the parent Nodes must be executable.

  • The Node must not depend on a Single Value Node that descends from that Node. For example, you cannot execute the dataflow in the following screenshot because the Single Value Node calculates the aggregate value from the result set of the Map Node, which in turn uses the aggregate value in the Xcalar defined function (XDF). If the Single Value Node is linked to Node 4 (Cast Node) instead of the Map Node, you can execute the dataflow.

Executing a partial dataflow

Executing a partial dataflow is helpful in debugging the dataflow in smaller parts before moving on to the next part. To execute a partial dataflow:

  1. Display the dataflow in the dataflow editor.

  2. Select one or multiple Nodes. For more information about making multiple Node selections, see Multi-selecting Nodes.

  3. Right click anywhere in the canvas (but not on an unselected Node) and click Execute.

    The dataflow is executed and the results are displayed as described in Results of executing a dataflow.

Executing an entire dataflow

Executing an entire dataflow means executing all the Nodes of a dataflow including all the sub-dataflows in the current dataflow tab. To execute an entire dataflow:

  1. Display the dataflow in the dataflow editor.
  2. Do one of the following:
    • Click the Execute All icon as shown in the following screenshot:

    • Deselect all Nodes in the dataflow. Right click on an empty space in the dataflow canvas and select Execute All.

    • If there is a Link Out or Export Node in the dataflow, deselect all Nodes, right click on an empty space in the dataflow canvas and select Create Optimized Dataflow. This creates and executes an optimized copy of the dataflow.

      The dataflow is executed and the results are displayed as described in Results of executing a dataflow.

  3. While the dataflow is executing:

    • To stop the execution, click on the red Cancel Execution icon on the top.
    • To make sure that the executing Node is always visible, right click on the dataflow canvas and select Focus on running node. This is useful when your dataflow is large and does not fit in the screen.

Results of executing a dataflow

The following list describes what happens when you execute a dataflow:

  • The execution starts at the first Node of the dataflow or the partial dataflow.
  • When the execution is in progress, the percentage of completion is displayed for each Node.
  • After the execution of a Node is finished, a statistics box is displayed above the Node with the following information:
    • Number of rows in the result set (if applicable to the Node).
    • Time taken to complete the execution.
    • Skew of the result set (if applicable to the Node). Clicking on the skew number displays the details of the largest skew number of all the interim operations performed during the execution of the Node. For example, executing a Group By Node may internally perform an index operation and a group by operation. Both the operations generate a skew number and the larger skew number is displayed in the statistics box.
    NOTE: Executing a Link Out Node does not produce a statistics box.

  • If applicable, a result set for each executed Node is available for you to view by right clicking and selecting View Result on that Node. If you execute a Single Value Node, you can view the resultant value. For understanding the result set window, see Understanding the result set view in Dataflow Mode.

  • As each Node executes, its status indicator changes color. For more information about the status indicator, see Understanding the Node status indicator.
  • To drop the results of all Nodes, right click on the dataflow canvas and select Reset All. This is useful if you changed something in the dataflow and want to execute all the Nodes again. As a result of resetting, the status indicator of all Nodes turn yellow.

Understanding the Node status indicator

Every Node in the dataflow has a status indicator. The following table explains the various states of the Node status indicator:

Status indicator color

Description

Clear

The Node is not configured. For example, immediately after you add a Node to a dataflow, the status indicator is clear. You must configure the Node before it is executable. To execute the entire dataflow, all Nodes must be configured.

Yellow

The Node is configured, but it does not have a result set. After you execute the Node, a result set is created.

Green

The Node was successfully executed and you can view its result set. However, if the result set has been dropped, the indicator remains green, and you must generate the result set again to view it.

Red

The Node is incorrectly configured. For example, if a Node in the middle of a dataflow does not have a link to a parent Node, or if you entered invalid information during the Node configuration.

Tracking the dataflow operations

You can track the processing details of your dataflow operations from the Operations panel of Xcalar Design. To learn more about displaying and canceling an operation, see Displaying the processing details of an operation.

Listing of dataflow executions

All dataflows in Xcalar Design are listed in the Dataflows panel. Clicking on any of these dataflows displays the last state of the dataflow, including the execution status. However, there are a few cases in which the dataflow execution is listed in one of the following sub-folders:

  • Optimized Dataflows: In Xcalar Design, if you execute a dataflow as optimized, a new optimized version of the dataflow is created and listed in the Optimized Dataflows folder.
  • Abandoned executions: If you start a dataflow execution in Xcalar Design and close or refresh your browser window while the dataflow is still executing, the dataflow execution is considered abandoned and keeps on executing in the background. When you log into Xcalar Design next time, if the system is unable to correlate the abandoned dataflow execution with an existing dataflow, it is listed in a folder called Abandoned executions.
  • SDK Dataflows: If a dataflow is executed through Xcalar SDK, it is listed in a dataflow folder called SDK Dataflows. If the dataflow is executed as optimized, it is listed in the SDK Dataflows\Optimized SDK Dataflows folder.

Each dataflow in these folders has a status indicator, which can be in one of the following states:

  • Yellow status indicator indicates that the dataflow is still executing.
  • Green status indicator indicates that the dataflow finished executing successfully.
  • Red status indicator indicates that the dataflow execution was canceled or it failed.
  • Clear status indicator is a rare phenomenon and it indicates that the execution did not start.

To cancel the execution of a dataflow in these folders:

  1. Select the dataflow you want to cancel. Its status indicator must be yellow.
  2. Click the red Cancel Execution icon on top of the dataflow window.
    The dataflow execution stops and its status indicator turns red.

What you can or cannot do with a dataflow in these folders:

  • You can view, cancel execution, publish, duplicate and delete a dataflow.

  • You cannot modify, configure or start execution of a dataflow.

If you delete all the dataflows from any of these folders, the folder is automatically deleted.
