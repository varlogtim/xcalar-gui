Understanding the result set view in Dataflow Mode

If you execute a dataflow or a Node, Xcalar creates a result set for each executed Node and you can view the result set in a table format. Viewing the result set helps you determine the operations you want to add for developing your dataflow further.

NOTE: When you execute a dataflow with optimization, the execution does not create a result set for each Node. For more information about optimized dataflows, see Understanding a dataflow with optimization.

Viewing the result set of a Node

To view the result set of an executed Node, right click the Node and select View Result. The Result Set view appears above the dataflow canvas. View Result may be disabled if the Node is not executed.

You can view only one result set at a time. To view the result set of another Node, repeat the steps in this procedure for that Node.

NOTE: You can configure the dataflow setting to automatically display the result set after execution. For more information, see Configuring dataflow settings.

Navigating the result set view

In the Dataflow Mode, the result set view consists of the results generated from the execution of a Node, arranged in a tabular format. The originating Node name is displayed at the top of the result set. The following list describes how you can navigate the result set view:

  • If you have a large number of columns and you need to search for a particular one, enter a partial or complete column name in the result set view search bar. The search is case-insensitive. If there is a match, the column header is highlighted. Press Enter to see the next match.
  • You can enter the row number in the Skip to rows field to go to a particular row.
  • You can scroll horizontally and vertically.

By default, the result set view contains a special column called DATA. This column enables you to display a modal window named Data Browser. The main purpose of the Data Browser is to show you all the fields in your result set even if they are hidden. For more information about how to use the Data Browser, see Performing tasks in the Data Browser.

Using cell option in the result set view

In the result set view of the Dataflow Mode, you can right click any cell in the result set to perform the following tasks:

  • Add a filter operation. For more information about filter operations, see Filtering in a result set column.
  • Copy the value in the cell to the clipboard.

Understanding the result set menu

The result set view in the Dataflow Mode provides a menu with the following options:

  • Export for exporting data from the result set as described in Exporting the result of a dataflow.
  • Change data type for changing the data types of the desired columns as described in Changing the data type and name of a column.
  • Correlation for displaying the correlation between numeric columns and for displaying the common aggregates as described in Determining the correlation between two columns.
  • Convert to Jupyter DF for creating a Jupyter Notebook from the result set as described in Using Xcalar Design with Jupyter Notebook.
  • Sort columns by name for sorting the columns by column name.

  • Sort columns by type for sorting the columns by data type.

  • Sort columns by prefix for sorting the columns by prefix.
  • Resize all coulmns for resizing all columns to fit the size of the headers, size of the content, or size of the headers and the content.
  • Advanced Options for:
    • Creating an Erroneous Rows result set as described in Creating a result set with erroneous rows only.
    • Creating a Complement result set for a Filter Node result set, which creates a new Filter Node with the same parent Node as the original Filter Node but complements the filter logic. This is useful when you want to see the rows that are excluded from the Filter Node result set.

    • Viewing the skew details. The skew value is an indicator for how evenly the data in a result set is distributed across cluster nodes. The value ranges from 0 to 100. If the skew value is 0, the data is perfectly distributed across all cluster nodes. If the skew value is 100, all the data is on one cluster node.

      NOTE: Data distribution affects the cluster performance. The more evenly the data is distributed, the faster the operations run on a cluster. If you experience slowness in your result set, check to see if a high skew is the cause of low performance. If so, and if you run this operation frequently, contact Xcalar for information about improving the performance. You can also refer to the Xcalar community forum and knowledge base to obtain tips on how to reduce the skew.

Menu item Description

For more information about how to use the DATA column, see Performing tasks in the Data Browser.

To align the column text to the left, center, or right. You also have the option to wrap the text.
