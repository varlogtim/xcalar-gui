Managing datasets

This topic describes the tasks you can perform to manage your datasets.

To manage your datasets, you must be in the Dataflow Mode. If not, from the top menu bar of Xcalar Design, click the mode switch to display the Dataflow Mode user interface.

Understanding dataset names

Xcalar Design provides a default name for each dataset when you import data. The name is based on the name of the source file or directory. You can enter a different name or accept the default. You cannot rename the dataset at a later time. If you import the same data source twice, the system will add a running number to the dataset name.

Understanding the Datasets panel

From the sidebar, click the Sources icon to display the Datasets panel. The Datasets panel displays all the datasets that you can use.

You can switch to view the datasets as a list or as a grid by clicking the List View and Grid View icons in the Datasets panel.

Acknowledging a dataset

Suppose you are previewing one dataset while another one is being created. After Xcalar finishes creating the other dataset, it lists the newly created dataset in the Datasets panel and displays an icon for the new dataset. The icon has a check mark associated with it, which indicates the addition of the dataset to the cluster. To acknowledge it, click the dataset icon, and the check mark is removed.

Organizing datasets in folders

In the Datasets panel, you can organize datasets by folders. For example, you can create a folder named Finance for datasets that import data sources from the Finance department, another folder named Engineering for datasets that import data sources from the Engineering department, and so on.

Follow these steps to place datasets in folders:

  1. Right click anywhere in the Datasets panel.
  2. Select New Folder in the pop-up menu. A New Folder is created. Alternatively, click the Add New Folder icon on top of the dataset panel and a new folder is created.

  3. Right click on the folder name and select Rename menu to rename the folder name.

    The following list describes the naming conventions:
    • The name is case sensitive.
    • Length must not exceed 255 characters.
    • Characters must be A-Z, a-z, 0-9, hyphen (-), or underscore (_).

    4. Do not specify Shared as the folder name because it is a reserved name.

  4. If you enter a name that already exists, an error message is displayed.

  5. Drag and drop any datasets to the newly created folder.

Sorting datasets

The Datasets panel displays a SORT BY menu for you to sort the datasets by Name, Type or Size. By default datasets and dataset folders are not sorted in any order.

  • Clicking NAME sorts the datasets and folders alphabetically in the ascending order.
  • Clicking TYPE sorts the datasets alphabetically in the ascending order but all the folders are listed first.
  • Clicking on SIZE sorts the datasets in the increasing order of size.

Alternatively, right click anywhere in the Datasets panel to display a drop-down menu. Select Sort by on the menu to sort by Name, Type, Size, or None.

Understanding dataset deletion

To delete a dataset, you need to deactivate it first. You can deactivate a dataset if it is not being used by any of the dataflows and any of the users. If you try to deactivate a dataset which is in use, you get an error message and the deactivation fails.

Deleting a dataset

To delete a dataset:

  1. In the Datasets panel, locate the dataset.
  2. Right click on the dataset name and select Deactivate dataset menu item from the drop-down. If the dataset was not being used by any workbook and any user, a confirmation message is displayed. Click Confirm and the dataset is deactivated.
  3. Right click on the dataset name again and select the Delete menu item from the drop-down.
  4. When a confirmation message is displayed, click Confirm if you are sure that you no longer need the dataset. The dataset is deleted.

Effect of a cluster restart on datasets

After the cluster is restarted, some datasets previously in your list of datasets might not be displayed. Only the datasets used by an active workbook on the cluster are shown immediately.

A dataset appears in the list again in one of these situations:

  • A user activates a workbook containing a result set or aggregate that depends on the dataset.
  • You activate the workbook that was active when you created the dataset.
EXAMPLE: You create a dataset called airlines, which is used in workbook1, and a dataset called carrier, which is used in workbook2. Both datasets are visible in the list of datasets, regardless of which workbook is active. After a cluster restart, if workbook1 is active, only the dataset named airlines is displayed. To also display the dataset named carrier, activate workbook2.

Activating a dataset

To activate an inactive dataset:

  1. Click on the Datasets icon under Sources icon in the sidebar. The Datasets panel is displayed.
  2. In the Datasets panel, locate the dataset you want to activate.
  3. Right click on the dataset name and select Activate dataset menu item from the drop-down. A confirmation message is displayed. Click Confirm and the dataset is activated.
