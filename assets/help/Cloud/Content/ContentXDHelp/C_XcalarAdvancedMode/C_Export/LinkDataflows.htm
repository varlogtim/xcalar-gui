Modularizing dataflows using linking

Big dataflows are hard to manage. To keep the dataflows manageable, modularize them by dividing them into many smaller parts. This enables you to focus on smaller parts and build a big dataflow gradually.

For example, when you read data into dataset Nodes, the first thing you may want to do is to curate the data. Such as, convert dates to the required format, split a column data into multiple columns, handle invalid data and so on. All this data curation on each dataset can be developed as separate dataflows and the output of these can be used as input in the other parts of the dataflow.

Linking dataflows enables you to modularize your dataflows. Xcalar Design provides two Nodes for linking the dataflows - Link In and Link Out. The Link In Node is a source Node and the Link Out Node is a terminal Node. A Link Out Node is used as the last Node in a dataflow to feed a Link In Node in another dataflow.

To use the Link In and the Link Out Nodes, verify that you are in Dataflow Mode. If not, from the top menu bar of Xcalar Design, click the mode switch to display the Dataflow Mode workspace. From the sidebar, click the Dataflow Panel icon. The Dataflow Mode workspace opens.

Using the Link Out Node

To use a Link Out Node:

  1. Create a dataflow.
  2. Click the Out category of operations in the Dataflow Mode toolbar to display the Link Out Node.
  3. Drag the Link Out Node to the dataflow canvas.
  4. Connect the output of the last Node of the dataflow to the Link Out Node.

  5. Right click on the Link Out Node and select Configure.

    The Link Out configuration panel is displayed.

  6. Enter a unique Link Out reference name for the Link Out Node. This is the name that a Link In Node will use to link to the Link Out Node.
  7. In the Column to export section, check the columns you want in your result set. The column selection is needed only if you create and execute this dataflow as optimized. You can check Select All to select all columns.
  8. If the Link In Node triggers the execution of this Node check box is checked (default state), the referring Link In Node executes the dataflow containing this Link Out Node to generate the result set. If the check box is unchecked, the Link In Node uses the existing result set of the Link Out Node and gives an error if the result set does not exist.

  9. Click Save.

    The Link Out Node configuration is saved with a name that can be used as an input in a Link In Node in another dataflow.

  10. To create and execute the dataflow as optimized, right click on a configured Link Out Node and select Create Optimized Dataflow.

Using the Link In Node

To use a Link In Node:

  1. Click the In category of operations in the Dataflow Mode toolbar to display the Link In Node.
  2. Drag the Link In Node to the dataflow canvas.

  3. Right click the Link In Node and select Configure.

    The Link In configuration panel is displayed.

  4. There are 2 options for selecting a source for the Link In Node - a Link Out Node and a result set:
    • To use a Link Out Node as the source:
      1. In the Source Type section, select the Link Out Node radio button.
      2. Click the Dataflow Name field to view the list of dataflows in the workbook and select a dataflow that contains the Link Out Node that you want to use as input in this Link In Node.
      3. Click the Link Out Reference Name field to view the list of Link Out Node names available in the selected dataflow and select the name you want to use as input in this Link In Node.
    • To use a result set as the source:
      1. In the Source Type section, select the Result Set radio button.

      2. In the Result Set Name field, paste the result set name. To get a result set name, find the Node in the dataflow whose result set you want to use, click on the Node to display its configuration and copy the result set value from the JSON.

      The Column Schema of the selected Link Out Node or the entered result set is displayed.

  5. (Optional) To remove a column from the Column Schema, click X before the Column Name.

  6. (Optional) To add a column in the Column Schema, click Add Column.

    This adds an empty column name and column type in the Column Schema.

    Click the empty Column Name to view the list of available column names and select a column name you want to add to the result set.

    The corresponding Column Type field is automatically filled in. If you add a Column Name that already exists in the Column Schema, the existing duplicate column name is wiped out.

  7. Once you are satisfied with the Column Schema, click Save.

    If a Column Name is left empty, an error message is displayed. You can either fill it up or remove the empty column and click Save. The Link In configuration is saved.

Finding the Link Out Node connected to a Link In Node

To find out the Link Out Node connected to a Link In Node in a dataflow, right click on a Link In Node and select Find the linked node. It opens up the dataflow with the connected Link Out Node highlighted.
