Modularizing dataflows using a Custom Node

Custom Node is a convenient way of modularizing your dataflows. If you have a set of operations in your dataflow which is used at many places, you can create a Custom Node for those operations and share it. Once a Custom Node is shared, it is available in the Dataflow Mode toolbar as a compound operator for you to use across all your workbooks. This new Node can be used like any other Node in a dataflow.

NOTE: The shared Custom Node is only visible to the user who created it.

Restrictions in creating a Custom Node

There are certain restrictions on what a Custom Node can include:

  • It cannot include Link In, Export and Link Out Nodes.
  • It cannot have more than one output Nodes.

  • It cannot include disconnected dataflows. For example, in the following dataflow, you cannot create a Custom Node by selecting Node1, Node2, Node4, and Node5 because that is equivalent to having two disconnected dataflows, one with Node1 and Node2 and the other with Node4 and Node5.

Creating and sharing a Custom Node

To create and share a Custom Node:

  1. Create a dataflow and test it.
  2. Once you are satisfied with the dataflow, select all the Nodes of the dataflow.
  3. Right click on any of the selected Nodes and select Create Custom Node.
  4. The selected dataflow Nodes are compacted in a single Custom Node.

  5. Right click the Custom Node and select Share Custom Node.

    A new Custom Node appears in the Custom category of the Dataflow Mode toolbar.

  6. To use this Custom Node in any dataflow, drag it into the dataflow canvas from the Dataflow Mode toolbar.
NOTE: See Restrictions in creating a Custom Node for more information on what can constitute a Custom Node.

Expanding a Custom Node

If you are using a Custom Node in a dataflow and need to see all the Nodes that constitute that Custom Node:

  1. Right click the Custom Node and select Expand.

    The Node is expanded in place.

  2. You can click the Undo icon in the top left corner of Xcalar Design to go back to the previous state.

Editing a Custom Node

To edit the configuration of the individual Nodes of a Custom Node:

  1. Right click the Custom Node in a dataflow and select Edit.

    A new tab opens up with the expanded Custom Node.

  2. Right click on any of the expanded Nodes (except the Nodes named Input and the Output) and reconfigure it.

    NOTE: You cannot add or remove any Node from the expanded Custom Node.

Renaming a shared Custom Node

When you create or share a Custom Node, system generates a default name for it. You cannot rename a Custom Node in the dataflow but you can rename a shared Custom Node in the Dataflow Mode toolbar. Follow these steps to rename a shared Custom Node:

  1. Select the Custom Node you want to rename in the Dataflow Mode toolbar.

  2. Click the pencil icon on the left.

    A Rename modal window is displayed.

  3. Enter a new name and click Save.

    The Node is renamed.

NOTE: Renaming the Custom Node in the toolbar has no effect on already used Custom Nodes in a dataflow.

Deleting a shared Custom Node

To delete a shared Custom Node from the toolbar:

  1. Select the Custom Node you want to delete in the toolbar.

  2. Click on the trash icon on the left.

    A DELETE modal window is displayed.

  3. Click Confirm and the Node is deleted.
NOTE: Deleting a Custom Node from the toolbar has no effect on already used Custom Nodes in a dataflow.

Editing a shared Custom Node

A shared Custom Node cannot be edited. You can drag a shared Custom Node to the dataflow canvas and edit the Node instance in the dataflow but the edits are not saved back to the shared Custom Node in the toolbar. It stays in the dataflow where you edit it, the shared Custom Node in the toolbar stays unchanged.
