Effect of a result set on memory consumption

Each result set in your workbook consumes memory. Keeping only the result sets required for modeling your queries helps conserve memory.

Dropping a result set

Xcalar is designed to automatically drop the result sets after certain period of inactivity but you can proactively conserve memory by dropping the result sets you don't need. To drop a result set:

  1. From the sidebar, click the System icon and then the Monitor icon .
  2. Click Release Memory, which displays the Drop Result Sets modal window.

  3. In the Drop Result Sets modal window, use the following list to determine which result set to drop:

    • Check the amount of memory consumed by each result set. The size information helps you decide which result set to drop.
    • You might have multiple active result sets in your workbook. It is least likely that you want to drop active result sets because they are the ones that you are currently working with or worked with recently. However, it is acceptable to drop an active result set if it is not needed anymore.
    NOTE: Before selecting a result set, you can sort the result sets by name, size, or date modified.
  4. Click Drop and Confirm.

Generating a result set

After a result set is dropped, if you need to view the result set, you can re-generate it. To re-generate a result set:

  1. Right click a Node in the dataflow for which the result set has been dropped.
  2. In the drop-down, select Generate Result. The result set, which was dropped, is available again.

Locking and unlocking a result set

If you do not want a result set to be dropped, either by Xcalar due to memory pressure or by you, you can lock it. However, in the following conditions a result set is dropped even if it is locked:

  • When you remove the Node from the dataflow
  • When you deactivate the workbook

To lock and unlock a result set, whether or not you are currently viewing it:

  1. Right click a Node in the dataflow that has a result set.
  2. In the drop-down, select Lock Result. A lock icon appears on the Node.
  3. To unlock a result set, click Unlock Result in the drop-down. The lock icon disappears from the Node.
NOTE: You can still perform tasks to manipulate the locked result set. For example, you can use the result set menu to sort or resize columns in a locked result set.

For more information on minimizing the memory usage by reducing the fields in a result set, see Reducing memory usage using a Project operation.
