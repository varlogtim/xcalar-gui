Creating connectors

This section describes how to create a Xcalar Connector, which enables you to access your source data.

You can create a connector whether you are in a SQL Mode or Dataflow Mode workspace.

To create a connector:

  1. From the sidebar, click the Sources icon , then the Connectors icon , and then from the Connectors panel, click Create Connector, as shown in the following screenshot:

  2. From the Create Connector window, in the Name field, enter a name for the connector.

  3. From the Connector Type list, select the connector type that best suites your source data's storage repository and access authentication method. For more information, see Understanding the connector types.

  4. Based on the selected connector type, entry fields appear that require access credentials, and the location of the connector's source data storage:

    • azure blob storage account with sas token: Enter the sas_token and account_name.
    • Confluent Kafka: In the kafka_url field, enter the REST API endpoint URL for the Kafka server.
    • Database: Enter the dbtype (MSSQL/MYSQL/Oracle/PG/Hive/DB2), dbname, hostname of the database server, port of the database server, and the user id.

    • HttpFs: Enter the Httpfs node in the following form: <FQDN of httpfs node>:<port number>

      Multiple HttpFs nodes can be entered separated by a semicolon. For example:

      http://httpfsnode1.example.com;http://httpfsnode2.example.com:14001;http://httpfsnode3.example.com

      For protocol, you can use https or http. The default protocol is http, and the default port is 4000.

    • HttpFs with Kerberos: Enter the Kerberos configuration file path, the keytab, and the HttFs node.

      To create a keytab, refer to https://kb.iu.edu/d/aumh.

      Enter the Httpfs node in the following form: <FQDN of httpfs node>:<port number>

      Multiple HttpFs nodes can be entered separated by a semicolon. For example:

      http://httpfsnode1.example.com;http://httpfsnode2.example.com:14001;http://httpfsnode3.example.com

      For protocol, you can use https or http. The default protocol is http, and the default port is 4000.

    • S3 Account with Specified Credentials: Fields secret_access_key and access_key are the required fields.

    • WebHDFS: Enter the namenode in the following form: <FQDN of namenode>:<port number>

      Multiple namenodes can be entered separated by a semicolon. For example,

      http://namenode1.example.com;http://namenode2.example.com:50071;http://namenode3.example.com

      For protocol, you can use https or http. The default protocol is http, and the default port is 50070.

    • WebHDFS with Kerberos: Enter the Kerberos configuration file path, the keytab, and the namenode.

      To create a keytab, refer to https://kb.iu.edu/d/aumh.

      Enter the namenode in the following form: <FQDN of namenode>:<port number>

      Multiple namenodes can be entered separated by a semicolon. For example,

      http://namenode1.example.com;http://namenode2.example.com:50071;http://namenode3.example.com

      For protocol, you can use https or http. The default protocol is http, and the default port is 50070.

    The following connector types do not need any other info:

    • Azure Blob Storage Account with Environmental Credentials
    • Generated
    • Google Cloud Storage Account with Environmental Credentials
    • S3 Account with Environmental Credentials

  5. (Optional) Verify each field's entry values.

    IMPORTANT: Once the connector is created, you cannot make changes, such as modifying a connector's name or required credentials. If the connector entry values change, a new connector must be created.

  6. Click Add.

    The SUCCESS message appears, informing you that the connector was created and added into the Connectors lists.

IMPORTANT: Before deleting a connector, verify that there are no workbooks depending on the datasets that are imported from the connector. When a connector is deleted, workbooks containing data from the deleted connector can no longer be activated after a cluster restart.

Next step

Prepare the data source located at the connector by following the instructions in Understanding the data source format requirements.
