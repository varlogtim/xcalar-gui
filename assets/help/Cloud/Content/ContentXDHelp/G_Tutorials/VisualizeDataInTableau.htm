Visualizing an Xcalar table in Tableau

Many times, after you have processed your data in Xcalar, you want to visualize it using specialized Business Intelligence (BI) software like Tableau. This tutorial shows how you can write an application in Xcalar so that it's result set is accessible from Tableau.

The first step in creating an application in Xcalar is to create a project. A project is a container where you create, design, troubleshoot, and execute your application. Open Xcalar Design in a browser and log in. Create a new project called VisualizeDataInTableauand open it by clicking on the project card.

Xcalar has two modes - the SQL Mode and the Developer Mode. In SQL Mode, you develop your queries using the SQL query statements and user-defined functions (UDFs) written in Python. In Developer Mode, you develop your applications using visual components, SQL query statements, and UDFs written in Python.

We will be using visual components in this application; therefore, we will work in Developer Mode. If you are not in Developer Mode, from the top menu bar of Xcalar Design, click the mode switch to Developer Mode.

Next, we will import the wine reviews file (winemag-data.json) into Xcalar. If you have already created a dataset using this file, skip to the second step.

1. Importing the wine reviews file

To import the wine reviews file, click on the Sources icon in the sidebar and then in the Datasets panel, click Import data Source. Since our file resides on Xcalar’s public S3 bucket (/xcmarketplace-us-east-1/tutorial-datasets/), for the Connector, select Public S3 and in the Data Source Path, enter /xcmarketplace-us-east-1/tutorial-datasets/. Click Browse to browse the directory structure and select winemag-data.json. Click Next to preview the file.

Xcalar automatically detects the format of the file and displays its preview. Since this is a simple json file, we don't need to pull out any nested fields. Let's accept all default options and click Create Dataset. The dataset winemag-data is created with 129,971 rows and 13 columns. The next step is to read the dataset into a module.

2. Creating a module and reading the dataset

To read the dataset, we need to create a module. In Xcalar, a module is a graphical representation of a series of operations performed on data. Each operation is represented by a graphical component called Node.

To create a module, click on the Application Builder icon in the sidebar and click Create new module or New module tab. A new module is created. From the In category of Nodes, add a Dataset Node to the module. To configure the Dataset Node, click it and select Configure. Select the winemag-data dataset in the configuration panel and click Save. By default, the Node gets executed and the result set is displayed:

Next, we will group the wines by their ratings, stored in the points column.

3. Grouping rows

To group the wines by their ratings, right click the points column header and select Group By. A Group By Node is added to the canvas and its configuration panel opens up. Configure the Node as follows:

Fields to group on: $winemag_data::points

Function to apply to group: count

Field name to count occurrences of: $winemag_data::title

New column name for the group by resultant column: wine_count

Save and execute the Node. A new result set is displayed with 21 rows:

4. Changing data types

If you look carefully at the data type of each column in the column header, you will notice that the points column is of type string (). Let's convert it to integer by right clicking the points column header and selecting Change data type -> Integer. A Cast Node is added to the canvas and it's configuration panel is displayed. The points column is at the top of the panel displaying a New Name and New Type. If you want to change the data type of another column, you can click on another column name at the bottom of the panel and enter its New Name and New Type. We want to change the data type of points column only.

Click Save and execute the Node.

A new result set is displayed with points column as Integer.

5. Saving the result set as a table

To ensure a seamless client-side application compatibility, Xcalar emulates the Spark Java Database Connectivity (JDBC) and Open Database Connectivity (ODBC) server protocols. Therefore, any BI software that supports JDBC or ODBC can access Xcalar as long as the data is stored in standard tables.

To save our result set in a table, from the Out category of Nodes, drag an ddrop the Publish Table Node to the canvas. Connect the Cast Node to the Publish Table Node. To configure the Publish Table Node, click on it and select Configure. In the configuration panel, enter a table name and check Select All. Leave other fields as default.

NOTE: Xcalar automatically converts the name of the table to the upper case.

Save and execute the Node. The WINE_RATINGS table is created.

To verify, from the sidebar, click on the Sources icon, and then the Tables icon. The Tables panel gets displayed. Look for WINE_RATINGS and click on it to see its preview:

5. Visualizing an Xcalar table in Tableau

To visualize an Xcalar table in Tableau, you need to connect to an Xcalar instance and then select the table of interest.

Configuring Tableau to work with Xcalar

  1. Open Tableau desktop

  2. In the Connect panel, under the To a Server section, locate and select Spark SQL.

    The Spark SQL modal appears.

  3. In the Server field, enter the hostname or IP address of the Xcalar cluster.
  4. Verify that the value of the Port field is 10000.
  5. From the Type list, select SparkThriftServer.
  6. From the Authentication list, select Username and Password.

    7. Based on this selection the Username and Password fields are made editable.

  7. In the Transport drop-down, select SASL.
  8. In the Username field, enter a valid Xcalar user name.
  9. In the Password field, enter the password for the valid Xcalar user name.
  10. (Optional) If you require a secure internet connection, select the Require SSL check box.
  11. Click Sign In, which takes you back to the Tableau desktop and the data source page.

Viewing Xcalar tables in Tableau

  1. In the data source page, from the Schema list, click the search icon and locate and select the default database.

  2. In the Table section, click the search icon in the search field to populate your Xcalar tables.

    NOTE: Only the active Xcalar tables show up in this list.
  3. Drag the WINE_RATINGS table to the canvas, and then select the sheet tab to view or start analysis.
