Example of importing an XML data source

An XML data source can be imported into a dataset as well as into a table. However, the process of importing into datasets and tables is slightly different. This topic describes the procedure for both.

The partial content of the XML file (books.xml) used in the example:

<?xml version="1.0"?>

<catalog>

<book id="bk101">

<author>Gambardella, Matthew</author>

<title>XML Developer's Guide</title>

<genre>Computer</genre>

<price>44.95</price>

<publish_date>2000-10-01</publish_date>

<description>An in-depth look at creating applications

with XML.</description>

</book>

<book id="bk102">

<author>Ralls, Kim</author>

<title>Midnight Rain</title>

<genre>Fantasy</genre>

<price>5.95</price>

<publish_date>2000-12-16</publish_date>

<description>A former architect battles corporate zombies,

an evil sorceress, and her own childhood to become queen

of the world.</description>

</book>

Importing an XML into a dataset

To import an XML data source file named books.xml into a dataset:

  1. Verify that you are in Dataflow Mode. If not, from the top menu bar of Xcalar Design, click the mode switch to display the Dataflow Mode workspace.
  2. From the sidebar, click on the Sources icon.
  3. Click on the Import Data Source icon in the Datasets panel.

  4. Browse and select books.xml in the Select Data Source Location window and click Next.

    The Import Data Source window is displayed with the preview of the file. Xcalar detects the format of the file as XML.

  5. The XML preview shows that the file contains a catalog of books. If you wish to extract all the books with their details, enter /catalog/book in the XPath field:

  6. Click the refresh button to preview the resultant dataset:

    The preview shows a dataset with a single column called book and each cell in this column contains the details of one book:

  7. (Optional) Select both Matched XPaths and Element XPath.

  8. Click Create Dataset.

    The dataset is generated with a single column called book and each cell in the column contains a book object. For details on how to pull out the individual elements from an object, see Pulling out all elements of an array or an object from a column.

Pulling out all elements of an array or an object from a column

To pull out all the elements of an array or an object from a dataset column into separate columns:

  1. Add a Dataset Node to a dataflow.
  2. Configure the Node to read the dataset with an array or an object column.

  3. Save and execute the Node.

    The result of the Node is displayed.

  4. Locate the array or the object column that you wish to separate.

    NOTE: The notation for an object is {} and the notation for an array is [].

  5. Click one of the cells in the column and select Pull all:

    All elements of the object are separated into new columns:

NOTE: If the original column with an object contained nested arrays or nested objects, some of the new columns may still contain an array or an object. You can keep on pulling those columns into new columns till the time there are no more array or object columns left.

Importing an XML into a table

To import an XML file into a table:

  1. From the sidebar, click on the Sources icon and then click the Tables icon.
  2. In the Tables panel, click on the Import Data Source icon.

  3. Browse and select an XML file in the Select Data Source Location window and click Next. We are using books.xml as an example.

    The Import Data Source window is displayed with the preview of the file. Xcalar detects the file format as XML.

  4. The XML preview shows that it's a catalog of books. If you wish to extract all the books with their details, enter the following in the XPath field:
    /catalog/book

  5. Click on the Refresh icon to see the table preview. Notice that the Schema text box is still empty because the book column is an object and it cannot be imported into a table directly. You have to specify the schema to extract all elements.

  6. Enter the name, newName and type for each of the name value pairs of the book object in the Schema text box as shown below:
    [{"name":"book._id", "newName": "id", "type":"string"},{"name":"book.author", "newName": "author", "type":"string"},{"name":"book.description", "newName": "description", "type":"string"},{"name":"book.genre", "newName": "genre", "type":"string"},{"name":"book.price", "newName": "price", "type":"string"},{"name":"book.publish_date", "newName": "publish_date", "type":"string"},{"name":"book.title", "newName": "title", "type":"string"}]

    The name field represents the element to extract, the newName field represents the new column name for that element and the type field represents the type of the column.

    NOTE: If the original column contains an array, access its elements using a subscript. For example, the first element of a category array can be accessed as category[0], second element can be accessed as category[1], and so on.

  7. Click Create Table.

    A table is created with the following columns - id, author, description, genre, price, publish_date and title:
