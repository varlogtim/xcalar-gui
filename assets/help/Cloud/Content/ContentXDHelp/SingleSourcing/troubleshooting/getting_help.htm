Getting Help from Xcalar

This topic describes how to obtain technical support from Xcalar by either filling out a support ticket or by starting a live chat.

NOTE: You can follow the steps in this topic even when Xcalar is not up and running.
IMPORTANT: Sending a ticket from the Xcalar cluster, requires an internet connection.

Reporting problems

You can send a support ticket to report a problem or to send a comment to Xcalar.

Before reporting a problem, write down the following pertinent information:

  • A clear and concise description of the issue, including when it began to occur.
  • Relevant log files, including the relevant error message sections in the files.
  • Xcalar product information:
    • The number of nodes in your Xcalar cluster and their node identification number, including:
      • The CPU usage as a percentage
      • The amount of memory being used as a percentage.
      • The amount of memory being used in bytes.
      • The amount of available memory in bytes.
      • The number of bytes sent to the network.
      • The number of bytes received from the network.
    • Your user name.
    • The name of the Workbook in which the issue appears.

When you are ready, report the problem to Xcalar Technical Support, as described in Generating a support ticket.

Generating a support ticket

TIP: You can also submit a support ticket through the Xcalar Customer Portal at https://myxcalar.zendesk.com.

To generate a support ticket:

  1. As a Xcalar Design user, log in to Xcalar Design.
  2. From the Xcalar Design menu bar, click the username icon, and then select File Support Ticket.
    The File Support Ticket modal window opens.
  3. From the Type of issue list, do one of the following:
    • If the problem is new:
      1. Select New.
      2. From the Severity list, select the impact level of the problem, as follows:
        • If this is a feature request or a comment about Xcalar, select 4- Feature request.
        • If this is a general information request, select 3 - General information request.
        • If the Xcalar cluster is functioning but the performance or functionality is compromised, select 2 - Functionality is compromised.
        • If the Xcalar cluster is down or unresponsive, or if a critical feature is not functioning, and this is a production environment, select 1 - Cluster is down.
      3. In the Additional (new) Information section, add the information you collected as follows:
        1. In the Subject field, enter a short description that identifies the problem.
        2. In the Comments field, enter details about the problem.
    • If you have more information for an existing ticket or the problem relates to an existing ticket:
      1. Select Existing.
      2. From the Select Ticket section, do one of the following:
        • From the Ticket ID column, locate your ticket identification number and select the check circle.
        • In the Or enter an existing ticket ID field, enter your ticket identification number.
      3. In the Additional (new) Information section, enter new details about the problem.
  4. (Optional) To Include additional information by forwarding a support bundle, select the Generate Support Bundle check box. For example, if you are reporting a performance issue or a crash, including a support bundle provides Xcalar Technical Support with additional information that helps diagnose the cause.
    5. NOTE: Generating a support bundle may take a long time.
  5. According to the Type of issue selected, click File Ticket or Update Ticket.
TIP: During the support ticket generation, the File Support Ticket icon turns gray. After generation, the icon resumes its original white color.

Manually sending a support ticket

When there is a transmission issue, an error message appears, notifying you of the error, and Generate Ticket is replaced with Download. Manually send the support ticket using the following task.

To manually send a support ticket:

  1. In the File Support Ticket modal window, click Download.

    2. A ticket file is created and downloaded to your computer, excluding the support bundle.

  2. On a computer with browser access, log in to the Xcalar Customer Portal at:

    3. https://myxcalar.zendesk.com/

  3. From the main menu bar, click Submit a request.

    4. The Submit a request page opens.

  4. In the Subject field, enter a short description that identifies the problem.
  5. From the Category list, select a category that matches the problem.
  6. In the Description text box, enter see the attached support file.
  7. In the Attachments field, click Add file.
  8. From the Find File window, locate and select the support file and then click Open.
  9. Back in the Submit a request page, click Submit.

Starting a live conversation with Xcalar technical support

To start a live conversation with Xcalar technical support:

  1. As a Xcalar Design user, log in to Xcalar Design.
  2. From the Xcalar Design menu bar, click the username icon, and then select Live Chat.
    The Live Chat modal window opens.
  3. In the Name and Email fields, enter your name and email address. This ensures that the chat log is automatically sent to your email address for your records.
  4. Click START and wait for a Xcalar technical support engineer to respond.

    5. As soon as the Xcalar technical support engineer starts a conversation with you, a support ticket is generated and its ticket identification number is saved to your Xcalar Customer Portal account.

TIP: At any time during the conversation, you can send the chat log to your email address by clicking the chat icon in the left corner of the Live Chat modal window.
