Working with Xcalar Defined Functions

Xcalar SQL supports Xcalar Defined Functions (XDF) inserted directly in the SQL query statement.

Understanding the XDF data type data format syntax

When including an XDF in your query you must include the function's output data type format.

The data type is prefixed before the function name using the first initial of the data type, followed by xdf and then an underscore (_), as shown in the below table:

Table 1 XDF prefix function name syntax

Function Name Prefix

Data Type

Example

bxdf_

boolean

bxdf_exists

fxdf_

float

fxdf_sum

ixdf_

integer

ixdf_add

mxdf_

money

mxdf_addNumeric

sxdf_

string

sxdf_concat

For example, here is the syntax for the wordCount XDF whose output data format is integer:

ixdf_wordCount(column_name)

Example of an XDF in SQL

Below is a SQL query statement that will count the number of words in a column using the wordCount XDF.

SELECT ixdf_wordCount(table.colname) FROM table;

Where,

  • table is the name of the table that contains the column whose words are to be counted.
  • colname is the name of the column whose words are to be counted.

For more information, see Map functions.

NOTE: When appropriate, Xcalar recommends using XDFs rather than UDFs in your SQL query statements. This is because they process faster and are able to return more data type formats than UDFs.
