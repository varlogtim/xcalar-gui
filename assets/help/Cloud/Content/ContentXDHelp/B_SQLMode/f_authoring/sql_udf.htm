Working with UDF modules in SQL

Xcalar SQL supports python functions inserted as user-defined functions (UDFs) directly in the SQL query statement. This enables you to invoke python functions as part of your SQL queries.

NOTE: The data type output for values returned by UDF functions is string.

When creating the UDF you must ensure the following:

  • Each UDF function name is unique and is not the name of an existing SQL function.
  • The name of the UDF module containing your functions is sql.
    Only UDF definitions from the sql module are invoked from a SQL query, which can contain multiple UDF definitions.
    • NOTE: Xcalar recommends using lowercase for your UDF function names.

For example, in the following procedure a test UDF is created using a Python string concatenation operator to merge the row string values from two columns, separated by a colon. The test UDF is inserted into a SQL query, which names the SQL table and what columns to combine.

  1. From the Xcalar Design sidebar, click the UDF icon.

    2. The User Defined Function panel opens in the SQL WORKSPACE.

  2. In the User Defined Function code editor, enter the following: 
    3. def test(col1, col2):
      return str(col1) + ":" +str(col2)
  3. From the User Defined Function menu, click the Save File icon .

    4. The SAVE AS modal window opens.

  4. In the Save As field, enter sql.
    5. IMPORTANT: The name of the UDF module must be sql.
  5. Click Save.
  6. In the SQL Editor code editor, enter the following: 
    7. SELECT test(col_A, col_B) FROM test_table

    Where,

    • col_A and col_B are the columns to combine.
    • test_table is the table that contains the columns to combine.
  7. Click Execute, which creates the result set.

For more information about UDFs and how to create them, see Understanding user-defined functions.
