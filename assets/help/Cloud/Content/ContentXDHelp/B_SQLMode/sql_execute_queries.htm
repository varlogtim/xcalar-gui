Understanding the SQL execution features

This sub-section discusses the SQL Mode execution features for applying query statements to your data and monitoring their execution.

Executing code from snippets containing multiple statements

Xcalar Design enables you create code snippets that contain multiple SQL query statements, which are run sequentially and as separate query statements on your data.

When executing multiple statements, Xcalar applies each statement to the latest version of the table.

For example, you have two query statements in a code snippet that are being applied to table A and a new version of tableA is updated after the execution of your first statement. Xcalar applies your second query statement to the updated table A.

Monitoring the statement execution

Xcalar Design enables you to visually monitor the query execution process from the Preview section.

When you run your SQL query statement on your data, by either clicking Execute or pressing F5, Xcalar Design enables you to visually see the phases of the statement's process in the Preview section. The final result displays an optimized dataflow diagram containing individual node components that represent each phase of the query. Where each executed input, output, and query operation is represented by an operation node.

Canceling the processing of a statement

If you notice any issues during the running of your statement, from the SQL History menu, click the Cancel SQL in execution icon .

Tracking the SQL operation details

You can track the processing details of your SQL query statement from the Xcalar Design Operations panel.

To track the SQL operation details:

  1. From the sidebar, click the System icon, and then click the Operations icon.
  2. In the Operations panel, click the Show all operations icon .

    3. The Show all operations list opens to the last operation entry at the bottom of the list.

  3. From the Show all operations list, select sql.

    4. The Operations Details page opens, displaying the progress and the processing details of your SQL query.

    NOTE: When applying multiple SQL queries, multiple sql operations will appear in the Show all operations list.

For more information, see Displaying the processing details of an operation.

Re-displaying a SQL error message

Xcalar Design verifies your SQL query statements for errors. When errors are encountered, such as incorrect SQL syntax, an incorrect table name, or applying queries on an inactive table, an error message appears. You can also view the error message from the SQL History section.

To re-display the error message:

From the SQL History section, locate the row containing the Failed SQL query that was applied to your statement and then select the Results entry field. The SQL ERROR message window appears displaying the error message of your query for you to troubleshoot.
