Working with temporal data

NOTE: By default the IMD feature is not enabled in Xcalar Design.
To enable the IMD feature, log into Xcalar Design and from the sidebar, expand the System icon, click the Settings icon, and then in the in the General Settings page, select the Show IMD screen check box and click Save.

This section provides information about using the Xcalar Design IMD feature to analyze data that is continuously changing.

For information on how to bring your base and delta data into Xcalar and for more information on how to work with temporal data, contact Xcalar at support@xcalar.com.

Working with mutable tables

IMD stands for insert, modify (update), and delete, which are commonly used data manipulation operations for making changes to data. The IMD feature in Xcalar Design enables you to process, track changes, and view the data changes from these operations on your most current and historical temporal data for comparison and analysis.

The IMD feature tracks data changes through base tables, known as Published Tables, which support the insert, update, and delete data manipulation operations and enables you to revert to any data state in a Published Table at any given time.

NOTE: Though Xcalar Published Tables are generally updated as part of an optimized dataflow pipeline that triggers data changes in real-time, you can also update Published tables from standalone datasets or multiple datasets that contain data timestamps. For more information about configuring a Xcalar dataflow pipeline for your real-time data changes, contact Xcalar at support@xcalar.com.

Xcalar IMD table pre-requisite tasks

To ensure that your data includes parameters that track data manipulation actions, the following tasks, if applicable, must be completed before generating your Published Table or applying updates from your delta table.

  • Verify that the Xcalar table contains a column that can be used as a unique primary key identifier. This column identifies your data records and is used to track changes.
  • (Optional) If your Xcalar table contains a column that is used to sort the data in ascending or descending order, known as an order by column, run a Xcalar Design Sort operation.
  • Verify that the Xcalar table contains an opcode column. For more information, see Understanding the Xcalar data manipulation action type column.

Understanding the Xcalar data manipulation action type column

The base table and each delta table must contain a data manipulation action type column, known as the opcode column, that identifies and tracks which IMD operations are performed during a data update.

You can either, create an opcode column from a user-defined function (UDF) or use the opcode column created by your change data capture (CDC) software. Whichever method you choose, you must verify that the data manipulation action types and values are identical to the Xcalar data manipulation action types and values below:

  • 0, which represents a row that has been deleted.
  • 1, which represents a row that has been inserted.
  • 2, which represents a row that has been updated.
TIP: To change incorrect values in your opcode column, use the Map function in Xcalar Design.

Working with the IMD screen

This topic describes how to use the IMD screen to view and analyze your mutable data.

Displaying the IMD screen

NOTE: By default the IMD feature is not enabled in Xcalar Design.
To enable the IMD feature, log into Xcalar Design and from the sidebar, expand the System icon, click the Settings icon, and then in the in the General Settings page, select the Show IMD screen check box and click Save.

To display the IMD screen:

  1. Verify that the IMD feature is enabled.
  2. From the Xcalar Design sidebar, click the Sources icon, then the Tables icon, and then in the Tables panel menu, click the View IMD icon , which displays the IMD screen as shown in the following image.

Understanding the IMD screen

The Xcalar Design IMD screen contains the following sections for viewing and working with your Published Tables:

  • Published Tables, which lists the Published tables that you are currently working with.

  • Inactive Tables, which lists the Published tables that you are currently not working with.
    • NOTE: You can apply data changes to Inactive Published tables.
    RECOMMENDATION: Inactive Published tables use very little memory. Therefore, move all Published tables that you are not working with into the Inactive Tables section.
  • Details about section, which provides the Published table's characteristics, including entries for each data update. For more information, see Understanding the Details section.
  • Published Tables Timeline section, which displays vertical time identifier lines for the Published table's creation date and subsequent data updates.

Coalesce icon

This icon appears when you select the check box of a Published table.

Clicking this icon combines all the delta table data changes in the Published table. This action saves memory and improves refresh times.

IMPORTANT: Though this action saves memory and significantly reduces the amount of time to refresh your tables, it also removes all the delta table data updates from the Published table and prevents you from analyzing your data from a specific Point in time.

Activate icon

This icon appears when you select the check box of an Inactive Published table.

Clicking this icon changes the state of the Published table to active and moves it to the Published Tables section.

Sort icon

This icon sorts the Published or Inactive tables list.

Clicking this icon toggles between ascending or descending order.

Filter field

This field enables you to search for a specific Published table or filter out Published tables that are not of interest.

Refresh icon

Clicking this icon refreshes the IMD screen.

Trash can icon

This icon appears when you select the check box of either a Published table or an Inactive Published table.

Clicking this icon deletes a Published Table.

WARNING: When a Published Table is deleted, all delta table updates and any descendant tables generated from a delta table or the Published table are no longer viewable from the IMD screen.

To ensure that the data in the Published Table is available at a later time, do one of the following:
- Save memory by changing its state to Inactive.
- Generate a Xcalar table with either the Latest or Coalesce function before deletion, which ensures that a Xcalar table is available for creating a new Published table.
IMPORTANT: Applying any new updates from the time you deleted the Published table will fail.

Understanding the Generate Tables popup

The Generate tables popup enables you to generate a Xcalar table from the Published table using one of the following output types:

  • Point in time, which generates a Xcalar table for the selected active Published Table comprising all data changes up to that point.
  • Latest, which generates a Xcalar table for the selected active Published Table comprising all data changes up to the current time.

Generating a Xcalar table to analyze mutable data

This topic describes how to generate a Xcalar table that displays current or historical results of the data updates in your Published table.

To generate a Xcalar table to analyze mutable data:

  1. Verify that the IMD feature is enabled.
  2. From the Xcalar Design sidebar, click the Sources icon, then the Tables icon, and then in the Tables panel menu, click the View IMD icon .

    3. The IMD screen appears.

  3. In the Published Tables section, select the row of the active Published table that is required to produce a Xcalar table.
  4. Depending on which output type you require, do one of the following:
    • To generate Point in time output, click on the Time-bar of the Published table's Batch ID that contains the ending time, and then from the Generate Tables pop-up, click Point in time.
    • To generate Latest output, verify that the row of the Published table that is required to produce a Xcalar table is highlighted, click on the row, which opens the Generate Tables pop-up element, and then click Latest.

    A Table Node is created in the Dataflow Mode canvas and is displayed in the Preview section for analysis.

  5. (Optional) To generate a Xcalar table containing specific columns and/or rows, from the Generate Tables pop-up, select the ellipses menu by the side of the Xcalar table button option and do one the following:
    • To generate a Xcalar table containing rows and columns based on a condition:
      1. In the Filter String field, enter the condition as filter text.

        2. For example, to generate a Latest Xcalar table that contains only rows of data that was inserted, enter eq(XcalarOpCode, 1), where the filter syntax for the equal function is function(operand1, operand2).

      2. Click the Xcalar table option.

        3. A Table Node is created in the Dataflow Mode canvas and is displayed in the Preview section for analysis.

      For more information on working with filtering in Xcalar Design, see Filtering in a result set column.

    • To generate a Xcalar table containing specific columns:
      1. In the Columns (comma separated) field, enter each column name required in the output followed by a comma.
      2. Click the Xcalar table option.

        3. A Table Node is created in the Dataflow Mode canvas and is displayed in the Preview section for analysis.

Conserving memory by compressing updates

The IMD Coalesce function is similar to the Latest function, in that all data changes are combined within the Published table. But unlike the Latest function, the Coalesce function removes all the delta table updates from the Published table's Timeline, which reduces the Published table's memory consumption. If you have a Published table that contains delta data that is no longer required for analysis, Xcalar recommends applying the IMD Coalesce function on the Published table.

NOTE: As all previous delta table updates are removed from the Published table after applying the Coalesce function, temporal data is no longer available for analysis.

To conserve memory by compressing updates:

  1. Verify that the IMD feature is enabled.
  2. From the Xcalar Design sidebar, click the Sources icon, then the Tables icon, and then in the Tables panel menu, click the View IMD icon .

    3. The IMD screen appears.

  3. In the Published Tables section, select the Published Table's check box.
  4. Click the Coalesce icon .

    5. A COALESCE warning message appears, which states that the operation cannot be undone and asks you to confirm the action.

  5. Click CONFIRM, which:
    • Combines all the delta table updates in the Published table.
    • Removes all the delta table updates from the Published table and the Published table's Timeline.
RECOMMENDATION: Inactive Published tables use very little memory. Therefore, move all Published tables that you are not working with into the Inactive Tables section.

Conserving memory by displaying less update time-bars

Xcalar conserves memory by limiting the number of viewable update time-bars to 128. When the Published table exceeds this limit, a Load More Updates tooltip is displayed next to the last updated time-bar:

  • To display more instances of updated Published tables, click the Load More Updates element.

    • Repeat until the Load More Updates tooltip disappears.

  • To revert to the update time-bar limit and conserve memory, click the refresh icon .

IMD Published table naming conventions

The following list describes the naming conventions:

To avoid any issues from the case sensitivity column naming differences between Xcalar and Xcalar SQL, column names are automatically processed into uppercase characters.

  • Length must not exceed 255 characters.
  • First character must be a letter of the alphabet.
  • Characters must be A-Z, a-z, 0-9, or underscore (_).
