Managing datasets

When you import data, XDP creates a dataset (typically in memory), which is the representation of raw data and metadata. You can preview the data in the dataset and create tables with all or selected fields in the dataset.

Xcalar Design provides a default name for each dataset when you import data. The name is based on the name of the source file or directory. You can enter a different name or accept the default. You cannot rename the dataset at a later time.

Datasets are shared by all users on the same cluster. The Datasets panel displays all the datasets. The following partial screenshot shows an example of the Datasets list.

Organizing datasets in folders

In the Datasets panel, you can organize datasets in folders. For example, you can create a folder named Finance for datasets importing data sources in the Finance department, another folder named Engineering for datasets importing data sources in the Engineering department, and so on.

Understanding dataset deletion

Your attempt to delete a dataset fails and the error message states that the dataset is in use if one or both of the following conditions are true:

  • Your workbook contains a table (hidden, temporary, or active) that was created from the dataset, or
  • You have an aggregate created from the dataset.
NOTE: A dropped table that is a part of a batch dataflow is not considered a table dependent on a dataset. As a result, such table does not prevent the dataset deletion. For example, after you drop a table named Airlines1#ue3 that is created from a dataset named Airlines, you can successful delete Airlines even if Airlines1#ue3 appears in a batch dataflow (provided that no other tables or aggregates use the dataset). For information about dataflows, see Understanding and creating batch dataflows.

If you do not have a table or aggregate created from the dataset, your attempt to delete a dataset has various effects depending on various factors.

If you are the creator of the dataset

If no other users have tables or aggregates created from the dataset, the dataset is deleted immediately.

Otherwise, the dataset is marked for deletion. You cannot take any action for this dataset. When all tables and aggregates from this dataset are dropped, this dataset is deleted automatically. The associated dataset is also removed from memory.

If you are not the creator of the dataset

You cannot delete a dataset if it is in another user's folder. However, you can mark it for deletion by selecting Delete when right-clicking its icon. The following list describes the effects of marking another user's dataset for deletion:

  • You cannot take any action on the dataset.
  • Other users who do not have tables or aggregates created from the dataset cannot start using the dataset.
  • Other users who already have tables or aggregates created from the dataset can see that the dataset is marked for deletion, but can continue using it.
  • The creator of the dataset can see that the dataset is marked for deletion. The creator can choose to continue using it or to delete it. The dataset continues to exist until the creator deletes it.
  • If the creator chooses to delete the dataset, the effect is the same as that described in If you are the creator of the dataset

How the dataset icon changes

The dataset icon changes depending on the status of the dataset. For more information about how to determine the status of the dataset, see Example of dataset deletion.

Deleting a dataset

Follow these steps to delete a dataset:

  1. In the Datasets list, navigate to the dataset that you want to delete. It can be located in your folder or another user's folder.
  2. Right click the icon or name of the dataset.
  3. In the drop-down menu, click Delete.
  4. When a confirmation message is displayed, click CONFIRM if you are sure that you no longer need the dataset.

What you cannot do to other users' datasets

You can view or mark other users' datasets for deletion. You can preview and create tables from other users' datasets. However, you cannot perform the following tasks:

  • Renaming the folder containing other users' datasets
  • Moving other users' datasets to another folder
  • Creating a folder in another user's dataset folder

Example of dataset deletion

In this example, user1 creates a dataset named airlines, from which a table is created. This dataset is visible to all other users on the cluster. The following screenshot shows the dataset list as seen by user1.

Suppose user2 navigates to user1's dataset folder, right-click the icon for airlines, and select Delete. The airlines dataset is not deleted immediately but is marked for deletion. User2 can change the status of airlines because user2 does not have any tables or aggregates created from it. From this point on, user2 can no longer use airlines. After the airlines dataset is marked for deletion by user2, the appearance of the dataset icon changes as well for user1, as illustrated in the following screenshot:

Now user1 and other users who have tables and aggregates from the dataset can continue to use airlines. User1 can delete it when all users no longer have tables and aggregates from the dataset. Alternatively, user1 can mark it for deletion. In this way, after the last table or aggregate from the dataset is dropped, the dataset is deleted automatically.

NOTE: To see the correct status of the dataset, you must click Refresh. In this example, as soon as user2 marks the airlines dataset for deletion, its icon disappears. Without clicking Refresh, it seems to user2 that the dataset is deleted, but actually it remains for other users who currently have tables or aggregates from the dataset.

Effect of cluster restart on datasets

After a cluster is restarted, some datasets previously in your list of datasets might no longer be displayed. Only the datasets used in your active workbook are shown immediately after the restart.

NOTE: The datasets still exist; they are available for you to use again when needed.

A dataset appears in the list again when:

  • you activate a workbook that uses the dataset, or
  • a batch dataflow is run against the dataset
EXAMPLE: You create a dataset called airlines, which is used in workbook1, and a dataset called carrier, which is used in workbook2. Both datasets are visible in the list of datasets, regardless of which workbook is active. After a cluster restart, if workbook1 is active, only the dataset named airlines is displayed. To also display the dataset named carrier, activate workbook2.
