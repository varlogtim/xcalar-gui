Steps for Xcalar Cloud Preview

This section describes the steps up to the point where you can start performing operations in tables presented in a workbook.

IMPORTANT: This topic is not applicable if you use Xcalar 1.2. Information here is for users of the early-access version of Xcalar. To get started with Xcalar 1.2, see Easy steps for Xcalar 1.2 (with default settings) or Detailed steps for Xcalar 1.2 (with custom settings).

Starting Xcalar Design

Point your browser to the URL given to you by Xcalar to start Xcalar Design. Then use the username and password provided by Xcalar to log in.

Creating a workbook as your workspace

Follow these steps to create a workbook:

  1. The Workbook Browser is automatically displayed after you log in to Xcalar Design for the first time. Enter the workbook name and click CREATE WORKBOOK on the New Workbook card.

    The following list describes the naming conventions:
    • The name is case-sensitive.
    • Length can be from 1 to 255 characters.
    • Characters can be A-Z, a-z, 0-9, hyphen (-), and underscore (_).
    • First character must be a letter.
  2. Click to activate the workbook. The icon is the first one on the menu when you hover over the workbook card.

Importing the data source

If the following assumptions are true, the easy steps in this section are sufficient for you to get started with Xcalar Design. Otherwise, follow additional steps in Importing the data source (optional steps).

  • The data format of your data source is natively supported by Xcalar.

    NOTE: Unicode is supported for both delimiter-separated values and JSON that is UTF-8 encoded. UTF-16 or other encoding systems are not supported without UDFs.

  • Field names in the data source do not contain illegal characters, which are the following characters.

    • caret (^)
    • single quote mark (')
    • double quote mark (")
    • left or right parenthesis (( ))
    • left or right square bracket ([ ])
    • left or right brace ({ })
    • backslash (\)
    • colons (:)
    • comma (,)
    • period (.)
  • You accept all the default settings suggested by Xcalar Design.

Importing data source (easy steps)

IMPORTANT: The data source for Xcalar Cloud Preview must be a file. It cannot be a directory. Also, the file size limit is 2 GB. If you try to upload a file larger than this limit, an error message is displayed.

Follow these steps to import data:

  1. In the Add File / Create Dataset window, upload a file to the Xcalar instance in one of two ways:

    • Click BROWSE as shown in the following screenshot. Then navigate to the data source file and select the file.

    • Locate the data source file on your computer. Then drag and drop it into the Add File / Create Dataset window. The following screenshot shows an example of copying a file named MyData to the Xcalar instance.

    NOTE: If a network problem happens during the upload, Xcalar Design can resume the upload process from where it left off as soon as network access becomes available again.

  2. In the list of uploaded data source files, select the file you want Xcalar Design to import as shown in the following screenshot. You can click only one file at a time.

    The Import Data Source window is displayed.

    NOTE: The preview in this window enables you to verify that you have selected the intended data source file. You can position your mouse on a column header and drag it to resize the column. You can also use your mouse to scroll the rows.

    If you accept the default values in this window, continue to the next step. To override the default values, skip the next step; instead, go to Importing the data source (optional steps)

  3. In the Import Data Source window, create a dataset by clicking CREATE DATASET. The following screenshot shows the Import Data Source window.

  4. In the Dataset Preview window, select the columns to send to a worksheet, and then click CREATE TABLE.
    5. NOTE: Under certain circumstances, not all columns are included automatically in the Dataset Preview. For example, if a dataset contains 62 rows and the last 2 rows have a column that the first 60 rows do not have, this column is not included in the table in this step. You can, however, add this column later according to Adding columns from a dataset to a table.

Importing the data source (optional steps)

If you do not want to accept all the default values when importing the data source, or if your data needs parsing with a UDF, follow these additional steps:

  1. In the Dataset Name field, accept the suggested name or type the desired name. The name cannot be changed later. The dataset enables you to create tables from the data source. For more information about datasets, see Managing datasets.

    The following list describes the naming conventions:
    • The name is case-sensitive.
    • Length can be from 1 to 255 characters.
    • Characters can be A-Z, a-z, 0-9, hyphen (-), and underscore (_).
    • First character must be a letter.

  2. If you want Xcalar Design to invoke a user-defined function (UDF) when parsing the data in the data source, select Parse Data With UDF, and then the module name and function name. If no UDFs are needed for parsing the data, skip to the next step.

    Parsing a data source with a streaming UDF is required if your field names in your data source contains illegal characters, which are listed in the following list:

    • caret (^)
    • single quote mark (')
    • double quote mark (")
    • left or right parenthesis (( ))
    • left or right square bracket ([ ])
    • left or right brace ({ })
    • backslash (\)
    • colons (:)
    • comma (,)
    • period (.)

    A UDF for parsing data when Xcalar Design imports the data is called a streaming UDF. You can preview the results of the streaming UDF by clicking REFRESH PREVIEW. If you see unexpected results, you can select another UDF or no UDFs. For more information about UDFs, see Understanding UDFs and Using UDFs.

    Recommendation: A streaming UDF causes columns to be ordered differently than they are in the data source. See Creating a first row as column headers for preparing your data source to ensure that you can easily locate columns in a table in Xcalar Design.

  3. The fields in the rest of the Import Data Source window tell Xcalar Design about the data format and layout. Xcalar Design automatically detects the format and layout of the data source, but you can override the default values.

    TIP: After you enter values in these fields, if you want to see the values recommended by Xcalar Design again, click REDETECT FORMAT.

    The following table describes the fields.

    Field Description
    Format

    It can be separated values, JSON, Excel, or Text.

    Depending on the format you select, some fields in this table might not be available. For example, for the Excel format, only Promote First Row as Header is available.
    Promote First Row as Header

    Depending on the data format, Xcalar Design automatically selects or deselects this option. For example, if the data source is a CSV file, Xcalar Design recommends that you promote the first row as header.

    If you use the Skip Rows field to skip rows that do not require analysis, Xcalar Design promotes the first row after the row-skipping. 

    EXAMPLE: The file employees.csv contains comments in the first row, and the strings Name and Salary in cells on the second row. You can specify 1 in the Skip Rows field to eliminate the first row and promote the row containing Name and Salary as header.
    IMPORTANT: You can promote the first row only at this point. You will not be able to change the first row into the header row at a later time.
    Record Delimiter The character for separating records, which can be \n or a null character.
    Field Delimiter The character separating fields, which can be a tab or a comma.
    Quoting Character The character surrounding a string in the data source. The double quote mark (") is the default quoting character in Xcalar Design. If the data source uses another character to enclose a string, enter the character in this field.
    Skip Rows

    The number of rows to omit at the beginning of each file when Xcalar imports the data source. Typically you omit rows if they do not contain data requiring analysis (for example, rows containing comments). If you import a directory with multiple files, the same number of rows are skipped in each file.

    If you use a UDF when importing a data source, the UDF parses the data before the rows are skipped.

  4. Click CREATE DATASET.

Next step

You are now ready to create a table according to the instructions in Creating a table.
