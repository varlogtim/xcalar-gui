Troubleshooting

This section describes how to resolve problems that you might encounter when using Xcalar Design.

Workbook activation

When you try to activate a workbook that you worked on previously, the following error message is displayed:

Failed to execute user-defined function/application

The following list explains the possible causes for the error:

  • One or more UDFs used to create tables in the workbook have been renamed or removed.
  • One or more source data files imported by Xcalar have been renamed or removed.

Changes to UDFs or source data files do not cause an error if the cluster has been running continuously. However, if the cluster is restarted after the changes are made, Xcalar Design cannot locate the UDFs or source data files to re-create the tables in the workbook. As a result, you cannot re-activate the workbook.

Solution

Restore the UDFs and source data files to their original location under the same name. Then re-activate the workbook.

Importing a data source

Fields in data source not appearing

When importing a CSV or text file, fields in the data source do not show up as fields in the dataset created by Xcalar Design. For example, you expect to see four fields in the dataset but only one column is displayed in the dataset preview.

This problem happens if a character used literally in the source file is interpreted by Xcalar Design as the quoting character for denoting a string.

Solution

In the Import Data Source window, leave the Quoting Character field empty.

Dataset creation failure

After you click CREATE DATASET or CREATE TABLE in the Import Data Source window, an error message appears.

The following list describes the possible causes for this error:

  • Dataset size specified is too small compared to the size of the data source. For example, if you try to import a 2 GB file but you specify a 1 GB dataset size.
  • An inappropriate field delimiter is specified. This might cause a record to exceed the maximum length.
  • The first row being promoted to the header exceeds the 255-character limit.

Solution

Follow these steps to import the data source again:

  1. Click CLOSE in the modal window to go to the Dataset Preview window.

    The Dataset Preview window shows the error messages similar to the following if the specified dataset size is too small:

    Importing data source failed. Error: Sample size 1024 is less than the smallest file size 8647786

    Please try to import data with different parameters(for example, a larger dataset size)

    The Dataset Preview window shows the error messages similar to the following if an incorrect field delimiter is specified, which results in records exceeding the maximum length:

    Importing data source failed. Error: failed to parse data from file, Error File: /datasets/airlines.csv

    Please try to import data with different parameters(for example, a larger dataset size)

  2. Click Retry in the Dataset Preview window.
  3. In the Import Data Source window, verify the settings in various fields and make appropriate changes. For example, under Advanced Options, specify a dataset size that is at least the same size as the source data file or select Entire file/folder.
  4. Create the dataset.

Dataset creation failure

After you click CREATE DATASET or CREATE TABLE in the Import Data Source window, an error message appears.

This error is typically caused by an inappropriate record delimiter, which results in a large number of fields exceeding the maximum allowed.

Solution

Follow these steps to import data again:

  1. Click CLOSE in the modal window to go to the Dataset Preview window.

    The Dataset Preview window shows error messages similar to the following:

    Importing data source failed. Error: max num fields exceeded, max: 4096, position 18397, Error File: /datasets/airlines.csv

    Please try to import data with different parameters(for example, a larger dataset size)

  2. Click Retry in the Dataset Preview window.
  3. In the Import Data Source window, enter the appropriate character string for the record delimiter.
  4. Create the dataset.

Batch dataflow scheduler

Batch dataflow does not run even after a schedule has been successfully added.

This problem might have been caused by the lack of parameterization of the export file name. Xcalar Design cannot export the result of the batch dataflow if you have not parameterized the export file.

Solution

Create a parameterized operation from the last icon in the batch dataflow graph. Then parameterize the export file name before re-running the schedule.

Out of memory

When you try to export a table or execute an operation, the following error appears:

Out of resources(XdbPages)

Exporting a table or executing a database operation requires the use of memory. This error happens when the cluster does not have sufficient memory to perform the export or execute the operation.

Solution

Drop tables or remove datasets that you no longer need. Pay attention to the status icon (the one next to the user icon in the upper right corner of the Xcalar Design screen). If the icon is yellow or red, hover over the icon to determine whether the low memory is due to too many tables or datasets. If the icon is green, meaning the memory level is normal, you reduce the chances of seeing this error.

Xcalar Design crash

Under rare circumstances, Xcalar Design might crash. The following REFRESH BROWSER message is displayed:

Solution

You can take one of the following actions:

  • Click Refresh to return to Xcalar Design.
  • Generate a ticket for Xcalar technical support. This has the same effect as generating a support ticket through Monitor > System in the Xcalar Design window. For more information about support tickets, see Understanding the support ticket.
  • Copy log to save log messages to the clipboard.
  • If you no longer need to use Xcalar at this moment, click Log Out. You can log in again at a later time.

Evaluation string

When you try to execute an operation such as filtering multiple values in a column, the following error message appears:

Eval string too long

This error happens when the evaluation string created by the operation exceeds the 4,096-character limit. (The evaluation string is the same as the one that you would enter in the function bar to run the operation. The total number of characters entered in the function bar must be under this limit.)

Solution

To shorten the evaluation string, split the operation into several operations. For example, you can filter one set of values first. Then in the resultant table, filter another set of values.

Dataset

When you try to delete a dataset, an error informs you that the dataset is locked by another user.

You cannot delete a dataset if at least one lock exists for the dataset, regardless of whether you created the dataset.

Solution

Wait until all locks are removed from the dataset. Then try to delete the dataset again.
