Glossary

The following table is a Xcalar glossary of terms.

Term Meaning
Batch dataflow You can save a dataflow under a name. A named dataflow is a batch dataflow, which can be scheduled to run at pre-set times. Running a batch dataflow lets you run operations against the entire data source unattended as opposed to executing individual operations interactively against sampled data. It also enables you to create a table based on the dataflow with selected columns. You can then export the table to another application or the active worksheet in Xcalar Insight. A batch dataflow goes through optimization in a similar way as a SQL query going through a query optimization. However, neither dataflow nor query optimization can guarantee optimal performance or meet Service Level Objectives (SLO) for real-world workload. To achieve that, you need to operationalize the batch dataflow.
Dataflow A set of elementary operations that shows the data lineage for the active table. Xcalar Insight shows a visual representation of the dataflow by using ovals and rectangles to mean tables and operations.
Dataset The representation of raw data and metadata that Xcalar creates (in memory when there is no memory pressure or in pageable memory when there is memory contingency) when a user points to a data source. The dataset can contain all data in the data source or a sample of the data in the data source, depending on the dataset size limit that the user specifies when pointing to a data source.
Dataset reference When you point to a data source, Xcalar creates a dataset. Xcalar Insight also creates an icon that provides you with a mechanism to access the dataset. The icon is called a dataset reference because it references a dataset, enabling you to preview the dataset and create tables from the dataset. It is possible to have a dataset reference without a dataset. For example, when you try to point to a data source but the pointing fails due to various reasons such as the dataset size limit being exceeded, the dataset reference is created nevertheless. This dataset reference simply gives you a convenient way to re-point to the data source. You can delete and move a dataset reference without affecting the dataset that is being referenced.
Data source The file containing raw data or the directory containing files with raw data that is being pointed to by XCE. NOTE: Do not use "dataset" to refer to the raw data on customer's disks because we reserve "dataset" to mean the collection of data and metadata created as a result of pointing to customer data.
Drop a table Dropping a table means completely removing the table (not just its contents) from a worksheet (for example, to conserve memory). This action is not reversible. It is not equivalent to "delete a table" in SQL.
Export This is the process for copying a table in Xcalar Insight (for example, a resultant table created by an operationalized dataflow) to a location for another application to use.
Field

It is not equivalent to "column." Usually there is a one-to-one relationship between field and column but not always. An operation (for example, a join) might cause the number of fields to grow to a large number. If the number of fields (not columns) exceeds 96 and you try to export the table, the export will fail. The number of columns in a table, however, can exceed 96.

XXX This definition needs refining.
Function A function defined by Xcalar is an XDF (Xcalar defined function) and a function defined by the user is a UDF (user defined function). You execute a function to either transform or extract data (for example, to find the average, to filter, and so on).
Hide a table To reduce clutter in the Xcalar Insight worksheet, you can hide a table, which simply means that the table is stashed. You can unhide it any time to view it.
Modeling

One or all of these tasks:

  • synthesize a schema
  • design an algorithm
  • design a query
  • design a linear regression analysis
  • design a new machine learning algorithm

XDP is a modeling tool, and this list of tasks encompassed by "modeling" is not all inclusive and might be broadened.
Node The physical machine that is a member of a cluster.
Operation A database operation, which is an overall process to accomplish a transformation. For example, "group by" is an operation. An operation can include one or multiple functions. For example, a map operation can contain a function that means, "If a value in column X is greater than 100, create a new column called Bad-percent, which is True, else set value in new column to False."
Operationalizing batch dataflows For Xcalar to meet SLOs (Service Level Objectives), customers need to contact Xcalar professional services who will do performance tuning on their batch dataflows. These batch dataflows, after being tuned, are called operationalized batch dataflows. Do not confuse the operationalized dataflows with batch dataflows that customers can run and schedule on their own. Without Xcalar's help in performance tuning, batch dataflows, while optimized, may not run at the highest performance level for real-world workloads.
Point to data It is the first thing you do when you use XCE to demystify your data. It means using Xcalar Insight to create the dataset reference, which lets you preview the dataset and create tables from the dataset.
Query A request for information from a database. You need the information to answer your particular question.
Sample Collection of records from the data source on which you can interactively model your dataflow.
Session A session is the same as a workbook in Xcalar Insight. It does not refer to a user's login. Sessions are visible only to the cluster administrator when the administrator views information about workbooks on the XCE nodes.
Xcalar Compute Engine (XCE) It is the engine that does the computing. Users interact with XCE through Xcalar Insight.

Xcalar Design (XD) It is the graphical user interface that users use to do interactive data modeling.
Xcalar Insight (XI) This is the visual analytics studio that allows thousands of users to work in a pure visual fashion with our Tera-Row technology.
