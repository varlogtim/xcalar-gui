Performing dataflow tasks and creating batch dataflows

You can display the dataflow for each active table and perform various tasks on the dataflow. This topic describes the tasks except how to edit a dataflow or how to perform tasks associated with a batch dataflow.

For information about editing a dataflow, see Editing a dataflow.

For information about batch dataflow tasks, see Using batch dataflows.

Displaying dataflows for all active tables

To display the dataflows for all active tables in a worksheet, click in the lower right corner of the Xcalar Design window. For more information about each element in the dataflow graph, see Dataflow graph.

After the dataflows are displayed, you might want to view the dataflow for a particular table. Follow one of these steps to display a particular dataflow graph:

  • In a worksheet, locate the table for which you want to view the dataflow graph. Click the table title to bring the table to focus. The corresponding dataflow graph is displayed.
  • Display the tables list to see all the active table names. Clicking an active table name brings the table to focus and displays its dataflow graph. For information about viewing the tables list, see Viewing the tables list.

Displaying a dataflow that contains a particular temporary table

A temporary table might appear in multiple dataflow graphs. If you want to see a dataflow that contains a particular temporary table, display the tables list to see all the temporary table names. Clicking a temporary table name displays a dataflow graph that contains the table. There might be multiple dataflow graphs with the selected temporary table. Xcalar Design displays the first one found.

    Adding a comment about an operation

    You can annotate an operation in the dataflow graph. For example, you can provide a reason for the arguments specified in the operation.

    Follow these steps to add a comment:

    1. Click an operation icon in the dataflow graph to display a pop-up menu.
    2. Select Add a comment.
    3. In the modal window displayed, type a comment.

    4. Click SAVE. The following screenshot shows a dataflow graph containing an annotated operation.

    To edit a comment, follow the preceding steps, but select View / edit Comment in the menu.

    Creating a batch dataflow from a complete dataflow

    To create a batch dataflow that starts at the dataset and ends at the active table, follow these steps:

    1. Follow these steps to display the DATAFLOW panel:

      1. Click the title bar of an active table to display the table options.
      2. Select Create batch dataflow….

      Alternatively, click to view the dataflows for all active tables. After you locate the dataflow that you want to save as a batch dataflow, click in the upper right corner of the dataflow graph.

    2. (Optional) Double click any row in the table's DATA column to display the Data Browser. If there are fields that you want to export from the batch dataflow but are currently hidden in the DATA column, pull the fields from the Data Browser. Only columns pulled into the table can be exported from the batch dataflow.

    3. In the panel, type the name of the dataflow.

      The following list describes the naming conventions:
      • The name is case sensitive.
      • Length must not exceed 255 characters.
      • Characters must be A-Z, a-z, 0-9, hyphen (-), and underscore (_).
    4. (Optional) If you want the batch dataflow to contain multiple dataflows, click Additional table to export and then select an additional table. The dataflow that created this table will be saved as one of the dataflows in the new batch dataflow.

    5. For each table selected, select the columns to be included in the batch dataflow result. 

    6. For each table selected, if the dataflow result contains columns that have duplicate names, you are prompted to rename the columns.

      EXAMPLE: All columns created by a batch dataflow are derived columns; no column names contain a prefix in the exported table. Suppose the table from which you create a batch dataflow has columns named airlines::DayOfWeek and DayOfWeek. Without renaming the columns when creating the batch dataflow, you would have an exported table containing two columns named DayOfWeek. You must rename these columns to prevent the duplicate names.

      The following partial screenshot of the DATAFLOW panel illustrates how to rename a duplicate column name when creating a batch dataflow. You can either type a new column name or accept the suggested new column name.link

      The following list describes the naming conventions:
      • The name is case sensitive.

      • The name must consist of alphanumeric characters, spaces, and special characters except the following characters:

        • caret (^)
        • single quote mark (')
        • double quote mark (")
        • left or right parenthesis (( ))
        • left or right square bracket ([ ])
        • left or right brace ({ })
        • backslash (\)
        • colons (:)
        • comma (,)
        • period (.)
      • The name must not begin or end with a space.
      • The name must not begin with a number.
      • Length must not exceed 255 characters.
      • The name must not be DATA. Because the name is case sensitive, data or Data is acceptable.

    7. Click CREATE.

    Creating a batch dataflow from a portion of a dataflow

    You can create a batch dataflow that starts and ends at any table in the dataflow graph. In a batch dataflow that starts at a table, you cannot trace the data lineage prior to this table. The batch dataflow uses the data in this table in the same way as it would use the data from a dataset.

    The procedure is similar to creating a batch dataflow from a complete dataflow. The following list provides only the steps that are different:

    1. In the DATAFLOW panel, click the Down arrow for the Table field. The drop-down menu displays a list of temporary tables in the current dataflow graph and all active tables in the worksheet. Click the name of the temporary table for which you want to create a batch dataflow. The following screenshot shows an example of creating a batch dataflow for a temporary table.

    2. If you want the batch dataflow to start from a table instead of the dataset, select the source table in the DATAFLOW panel. The following screenshot shows an example of creating a batch dataflow for a temporary table, which starts from another temporary table.

    3. (Optional) If the dataflow graph includes multiple dataflows connected to one another, you can specify additional source tables in the DATAFLOW panel. The resultant batch dataflow will obtain the source data from the specified tables.
