Working with Parquet Datasets and Files

Parquet datasets are stored in nested data structures in a flat columnar format. The parquet data structure includes schema and metadata files that describe each file, including its location.

Creating a Xcalar dataset from a parquet format

You can create a Xcalar dataset from a single parquet file or from a parquet dataset:

  • To create a Xcalar dataset from a single parquet file, see Creating a Xcalar dataset from a single parquet file.
  • To create a Xcalar dataset from a parquet dataset, see Creating a Xcalar dataset from a single parquet file.

Creating a Xcalar dataset from a single parquet file

To create a Xcalar dataset from a single parquet file:

  1. Verify that you are logged in to Xcalar Design.
  2. From the sidebar, click the Datasets icon and then click the Create Datasets icon.

    3. The Datasets panel appears.

  3. Click the Import Data Source icon,

    4. The Import Data Source page appears.

  4. From the Data Target list, select either a data target that was created for the parquet data source or Default Shared Root.
  5. From the Data Source Path field, click the BROWSE button.

    6. The Browse Data Source page appears.

  6. Locate and select in your parquet data source the required file for your Xcalar dataset by selecting the checkbox in the upper left corner.
  7. (Optional) To examine the parquet file's raw data, click VIEW RAW DATA. For more information, see Viewing raw data.
  8. Click NEXT.

    9. The Import Data Source page appears.

  9. In the Dataset Name field, enter a name for the dataset. Alternatively, use the default name provided.
  10. Verify that the Format list shows Parquet File.
  11. (Optional) To configure advanced Xcalar dataset options, such as error handling during import, expand the Advance Options section and follow the instructions in Setting advanced options.
  12. Click CREATE DATASET.

    13. The Dataset page appears, which displays a tabular view of the new Xcalar dataset.

  13. (Optional) To display a view of the Xcalar dataset's nested structure, click CLICK TO VIEW.

    14. The Files / Folders window appears, displaying a directory tree of all the files and folders located in the Xcalar dataset. To view files and folders of interest, expand or collapse folders .

Creating a Xcalar dataset from a parquet dataset

Creating a Xcalar dataset from a parquet dataset requires creating a second-order data target that is used to import your data. The instructions also describe how to select specific partition keys and columns of interest for your Xcalar dataset.

The following tasks describe how to create a Xcalar dataset from a parquet dataset:

  1. Create a data target that can access your parquet data source. For more information, see Creating a data target.
  2. Create a second-order data target, which works with the first data target to read your parquet data sources. For more information, see Creating a second-order data target for parquet data sources.
  3. Create a parquet dataset. For more information, see Creating a parquet dataset.

Creating a second-order data target for parquet data sources

This section discusses how to create a second-order data target for reading parquet data sources.

To create a second-order data target for Parquet data sources:

  1. Log in to Xcalar Design as a user with administrative privileges and from the sidebar, click the Datasets icon and then click the Data Targets icon.

    2. The Data Targets panel appears.

  2. Click the Create data target icon.

    3. The Create Data Target page appears.

  3. Do the following:
    1. In the Name field, enter a name for the second-order data target.

      2. The Target Configuration section appears.

    2. From the Target Type list, select Parquet Dataset.
    3. In the backingTargetName field, enter the name of the data target that contains the location for accessing the parquet files. Alternatively, use the Default Shared Root data target. For more information on the Default Shared Root data target, see Creating a data target.
  4. Click ADD.

    5. A new data target is created that can be used as a second-order target for parquet data.

Creating a parquet dataset

  1. Verify that you are logged in to Xcalar Design and from the sidebar, click the Datasets icon and then click the Create Datasets icon.

    2. The Datasets panel appears

  2. Click the Import Data Source icon,

    3. The Import Data Source page appears.

  3. From the Data Target list, select the second-order data target that was created using the Parquet Dataset target type. For more information, see Creating a second-order data target for parquet data sources.
  4. From the Data Source Path field, click the BROWSE button.

    5. The Browse Data Source page appears.

  5. Locate your parquet data source and select its checkbox, which is located to the left of the folder’s name.
  6. (Optional) To examine the parquet data source's raw data, click VIEW RAW DATA. For more information, see Viewing raw data.
  7. Click NEXT.

    8. The Import Data Source page appears.

  8. In the Dataset Name field, enter a name for the dataset. Alternatively, use the default name provided.
  9. Verify that the Format list shows Parquet Dataset.
  10. (Optional) If your parquet data source contains partitioning and you require specific partitioning values in the dataset, expand the Partitions (Advance) section.

    11. The list of parquet partitions appears. Do one or more of the following:

    • To specify a specific value for a partition, in the partition’s field, enter the specific value. For multiple values, separate each value with a comma.
    • To include all the partition's values, enter an asterisk (*).
  11. The Columns section configures which columns and their data values are imported into the Xcalar dataset from the parquet data source.
    12. IMPORTANT: Partitioning columns are automatically displayed in the Selected Columns list. They cannot be moved to the Available Columns list, which lists all available column names in the parquet data source.

    From the Columns section, do one or more of the following:

    • To create a dataset with specific columns of interest, from the Available Columns list, click on the column name of interest.

      • The column name is moved into the Selected Columns list.

    • To create a dataset from all the columns, from the Available Columns list, click Add All.

      • All column names are moved into the Selected Columns list.

    • To move an unwanted column name back into the Available Columns list, from the Selected Columns list, click on the column name.

      • The column name is moved back into the Available Columns list.

    • To search for specific columns to move into the Selected Column list, in the Search field enter a column name.
      • TIP: If you notice that the amount of column names in the Available Columns list contains fewer than expected names, verify that the Search bar does not contain a value.
  12. Click CREATE DATASET.

    13. The Dataset page appears, which displays a tabular view of the new Xcalar dataset, including your selected columns and partition keys.

  13. (Optional) To display a view of the Xcalar dataset's nested structure, click CLICK TO VIEW.

    14. The Files / Folders window appears, displaying a directory tree of all the files and folders located in the Xcalar dataset. To view files and folders of interest, expand or collapse the dataset folders.
