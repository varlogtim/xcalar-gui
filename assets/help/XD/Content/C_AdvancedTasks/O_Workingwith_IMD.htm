Working with temporal data

This section provides information about using the Xcalar Design IMD feature to analyze data that is continuously changing.

Working with mutable tables

IMD stands for insert, modify (update), and delete, which are commonly used data manipulation operations for making changes to data. The IMD feature in Xcalar Design enables you to process, track changes, and view the data changes from these operations on your most current and historical temporal data for comparison and analysis.

The IMD feature tracks data changes through base tables, known as Published Tables, which support the insert, update, and delete data manipulation operations and enables you to revert to any data state in a Published Table at any given time.

NOTE: Though Xcalar Published Tables are generally updated as part of a Xcalar batch dataflow pipeline that triggers data changes in real-time, you can also update Published tables from standalone datasets or multiple datasets that contain data timestamps. For more information about configuring a Xcalar batch dataflow pipeline for your real-time data changes, contact Xcalar at support@xcalar.com.

Your active Published tables are listed under the Published Tables section in the IMD screen. From the IMD screen, you can instantly see when changes are applied and compare and analyze changes on all your data or from a specific point in time.

When manually working with Xcalar Published tables, you can create and update a Published Table from the IMD extension in Xcalar Design.

The IMD extension enables the following:

  • Creates a base table from your Xcalar table, known as a Published Table.
  • Applies your latest data updates from a delta Xcalar table to the Published Table and generates a Xcalar table for review and analysis of the latest changes.

Combined, these functions store your mutable data for later viewing and analysis.

Working with the IMD extension

The IMD extension enables you to create a Published table and apply data updates to the Published table. This topic describes the IMD table pre-requisite tasks, the Xcalar tracking parameters, how to create a Published Table, and how to apply data updates to the Published Table:

  • Xcalar IMD table pre-requisite tasks
  • Understanding the Xcalar data manipulation action type column
  • Creating a base table
  • Applying data updates manually to the Published Table

Xcalar IMD table pre-requisite tasks

To ensure that your data includes parameters that track data manipulation actions, the following tasks, if applicable, must be completed before generating your Published Table or applying updates from your delta table.

  • Verify that the Xcalar table contains a column that can be used as a unique primary key identifier. This column identifies your data records and is used to track changes. If your table does not contain a primary key column, see Creating a primary key column.
  • (Optional) If your Xcalar table contains a column that is used to sort the data in ascending or descending order, known as an order by column, run a Xcalar Design Sort operation.
  • Verify that the Xcalar table contains an opcode column. For more information, see Understanding the Xcalar data manipulation action type column.
    • IMPORTANT: If no opcode column name is entered in the IMD operator field of the IMD extension's Update Table function, Xcalar assumes that all the data changes are inserts and populates all rows in the XcalarOpCode tracking column with a value of 1 (insert).
    NOTE: When creating a Published table, this task is optional, since no update actions are part of the creation process.

Creating a primary key column

This topic describes how to create a primary key column in your Xcalar table, which is used as your primary key identifier.

To create a primary key column:

  1. From the Xcalar Design sidebar, click the Extensions icon, select the GenRowNum extension, and then click Generate Row Num.

    2. The GenRowNum: Generate Row Num section appears.

  2. From the Table list, select the Xcalar table which will be used to create a base table.
    3. TIP: For a Table list with multiple Xcalar tables, you can bring the selected table into focus for verifying your selection by clicking .
  3. In the New Column Name field, enter a name for the primary key column.
  4. Click APPLY.

Understanding the Xcalar data manipulation action type column

The base table and each delta table must contain a data manipulation action type column, known as the opcode column, that identifies and tracks which IMD operations are performed during a data update.

You can either, create an opcode column from a user-defined function (UDF) or use the opcode column created by your change data capture (CDC) software. Whichever method you choose, you must verify that the data manipulation action types and values are identical to the Xcalar data manipulation action types and values below:

  • 0, which represents a row that has been deleted.
  • 1, which represents a row that has been inserted.
  • 2, which represents a row that has been updated.
TIP: To change incorrect values in your opcode column, use the Map function in Xcalar Design.

Creating a base table

This topic describes how to create a Published Table from a Xcalar table, which is your base table for storing changed data values.

To create a base table:

  1. From the Xcalar Design sidebar, click the Extensions icon, select the IMD extension, and then click Publish Table.

    2. The IMD: Publish Table section appears.

  2. From the Table list, select the Xcalar table to use for base table creation.
    3. TIP: For a Table list with multiple Xcalar tables, you can bring the selected table into focus for verifying your selection by clicking .
  3. In the Published Table Name field, enter a name for the base table.
  4. In the Primary Key field, enter the name of the column that contains your unique primary key values. For example, the column name you gave when creating a primary key column with the GenRowNum extension. For more information, see Creating a primary key column.
    5. TIP: To populate a column name directly from a Xcalar table into the Primary Key or IMD Operator entry field, click inside the entry field, verify that the Xcalar table is visible, and then in the Xcalar table, click the column name.
  5. In the IMD Operator field, enter the name of the opcode column.
    6. IMPORTANT: You must verify that the opcode column values are identical to those of the Xcalar data manipulation action type. For more information, see Understanding the Xcalar data manipulation action type column.
    NOTE: When creating a Published table, this task is optional, since no update actions are part of the creation process.
  6. Click APPLY, which does the following:
    • Creates a Published Table from the Xcalar table.
    • In the IMD screen, adds the Published Table to the Published Tables section and adds a generated time identifier line on the Published table's Timeline .
    • Generates a resultant Xcalar table in your active worksheet, which contains derived columns, including two Xcalar metadata columns named XcalarRankOver and XcalarOpCode for tracking.

The Published Table is now prepared for receiving data updates.

Applying data updates manually to the Published Table

This topic describes how to manually apply data updates to the Published table.

IMPORTANT: The delta table that is used to apply data updates must not contain columns that were not in the original Published Table. If the delta table contains a columns not in the original Published Table, before you apply the Update Table function of the IMD extension, in Xcalar Design hide the column with the Hide column option.
IMPORTANT: Your delta table must contain an opcode column. If no opcode name is entered in the IMD operator field, Xcalar assumes that all the delta table data changes are inserts.

To apply data updates to the Published Table:

  1. From the Xcalar Design sidebar, click the Extensions icon, select the IMD extension, and then click Update Table.

    2. The IMD:Update Table section appears.

  2. From the Table list, select the name of the delta table that contains the new data updates.
    3. TIP: For a Table list with multiple Xcalar tables, you can bring the selected table into focus for verifying your selection by clicking .
  3. In the Name of Published Table field, enter the Published table name that will receive the updates.
  4. In the Primary Key field, enter the name of the column that contains your unique primary key values. For example, the column name you gave when creating a primary key column with the GenRowNum extension. For more information, see Creating a primary key column.
    5. TIP: To populate a column name directly from a Xcalar table into the Primary Key or IMD Operator entry field, click inside the entry field, verify that the Xcalar table is visible, and then in the Xcalar table, click the column name.
  5. In the IMD operator field, enter the name of the opcode column.
  6. Click APPLY, which does the following:
    • Applies the delta table data updates to the Published table.
    • In the IMD screen, adds a generated time identifier line on the Published table's Timeline and adds a new data update entry in the Published table's Details about section.
    • Generates resultant Xcalar tables for both the Published table and delta Xcalar tables in your active worksheet.

Working with the IMD screen

This topic describes how to use the IMD screen to view and analyze your mutable data.

Displaying the IMD screen

From the Xcalar Design sidebar, click the IMD Panel icon, which displays the IMD screen as shown in the following image.

Understanding the IMD screen

The Xcalar Design IMD screen contains the following sections for viewing and working with your Published Tables:

  • Published Tables, which lists the Published tables that you are currently working with.

  • Inactive Tables, which lists the Published tables that you are currently not working with.
    • NOTE: You can apply data changes to Inactive Published tables.
    RECOMMENDATION: Inactive Published tables use very little memory. Therefore, move all Published tables that you are not working with into the Inactive Tables section.
  • Details about section, which provides the Published table's characteristics, including entries for each data update. For more information, see Understanding the Details section.
  • Published Tables Timeline section, which displays vertical time identifier lines for the Published table's creation date and subsequent data updates.

The following table lists the IMD screen UI elements:

Element

Description

Date-picker

This field, enables you to display within the IMD screen a specific date range.

Either enter a date using the MM/DD/YYYY format or select a date from the calendar element.

Time-picker

This field, which is to the right of the date-picker, enables you to display the specific time based on a 12 hour clock.

Either enter a time using the HH:MM:SS format or from the Time-picker element, select the plus or minus buttons to increase or decrease the time.

Timeline bar

 

This element displays the date in increments of hours, days, weeks, and months.

To zoom in and out, use the scroll wheel on your mouse.

Hovering your mouse over the Timeline bar displays a tooltip with the exact date and time.

Time-bar

This element controls actions across all visible Published tables, such as displaying all data changes up to that time.

A vertical time identifier line is displayed when the Published table is first created and for each time a data update occurs.

Generate Tables pop-up

This element opens when you click on the row or a Time-bar associated with a Published table in the Published Tables section of the IMD timeline screen.

The Generate Tables pop-up contains the following action buttons:

  • Point in time
  • Latest

For more information, see Generating a Xcalar table to analyze mutable data.
Scroll bar

This element enables you to scroll along the Published table's Timeline.

Check all check box

This check box, which is displayed inside the Published Tables and Inactive Tables header, enables you to quickly select or deselect all Published or Inactive tables.

Table Check box

This check box enables you to run actions on specific Published or Inactive tables.

For example, you can disable specific Published tables when they are not required, such as when they are not required to be part of a Point in time or Latest action.

  • Selecting this check box in the Published Tables section, keeps the Published table active and available for an operation or action.
  • Deselecting this check box in the Published Tables section, keeps the Published table active but removes the Published table from an operation or action.
NOTE: Data changes are still applied to deselected tables even though they are not viewable on the IMD Timeline.

Coalesce icon

This icon appears when you select the check box of a Published table.

Clicking this icon combines all the delta table data changes in the Published table. This action saves memory and improves refresh times.

IMPORTANT: Though this action saves memory and significantly reduces the amount of time to refresh your tables, it also removes all the delta table data updates from the Published table and prevents you from analyzing your data from a specific Point in time.

Drag icon

This element enables you to manually organize your Published and Inactive tables. Select and drag this icon to the required Published table hierarchy level.

Deactivate icon

This icon appears when you select the check box of a Published table.

Clicking this icon changes the state of the Published table to inactive, and moves it to the Inactive Tables section.

Activate icon

This icon appears when you select the check box of an Inactive Published table.

Clicking this icon changes the state of the Published table to active and moves it to the Published Tables section.

Sort icon

This icon sorts the Published or Inactive tables list.

Clicking this icon toggles between ascending or descending order.

Filter field

This field enables you to search for a specific Published table or filter out Published tables that are not of interest.

Refresh icon

Clicking this icon refreshes the IMD screen.

Trash can icon

This icon appears when you select the check box of either a Published table or an Inactive Published table.

Clicking this icon deletes a Published Table.

WARNING: When a Published Table is deleted, all delta table updates and any descendant tables generated from a delta table or the Published table are no longer viewable from the IMD screen.

To ensure that the data in the Published Table is available at a later time, do one of the following:
- Save memory by changing its state to Inactive.
- Generate a Xcalar table with either the Latest or Coalesce function before deletion, which ensures that a Xcalar table is available for creating a new Published table.
IMPORTANT: Applying any new updates from the time you deleted the Published table will fail.

Understanding the Details section

The Details section contains information about your Published tables. It includes entries for each data update and lists the Published table's column names and data types.

Clicking on a Published table in the Published Tables section populates the entry fields in the Details about section, which are described in the following table:

Field

Description
Details about

Displays the name of the Published Table.
Source

Displays the name of the Xcalar table that provided the source data.
Batch ID

Displays the identification number for each update of the Published Table. The first identification number, 0, denotes when the Published Table was first generated.

Clicking on a specific identification number opens the Generate Tables pop-up to the precise moment in the IMD timeline when the Published Table was first generated or when an update was applied.
Timestamp

Displays the date and time that the data entry was created.
Number of Rows

Displays the total number of rows contained in the Published Table.

View columns

Clicking this link lists the column names and their data types in the Published table.

View table details

Clicking this link lists the data source location, identification number, creation date, and the row count of each data entry.

Understanding the Xcalar Generate Tables popup

The Xcalar Generate tables popup enables you to generate a Xcalar table from the Published table using one of the following output types:

  • Point in time, which generates a Xcalar table for the selected active Published Table comprising all data changes up to that point.
  • Latest, which generates a Xcalar table for the selected active Published Table comprising all data changes up to the current time.

Generating a Xcalar table to analyze mutable data

This topic describes how to generate a Xcalar table that displays current or historical results of the data updates in your Published table.

To generate a Xcalar table to analyze mutable data:

  1. From the Xcalar Design sidebar, click the IMD Panel icon.

    2. The IMD screen appears.

  2. In the Published Tables section, select the row of the active Published table that is required to produce a Xcalar table.
  3. Depending on which output type you require, do one of the following:
    • To generate Point in time output, click on the Time-bar of the Published table's batch ID that contains the ending time, and then from the Generate Tables pop-up, click Point in time.
    • To generate Latest output, verify that the row of the Published table that is required to produce a Xcalar table is highlighted, click on the row, which opens the Generate Tables pop-up element, and then click Latest.

    A Refresh Tables Completed tooltip appears.

  4. (Optional) To generate a Xcalar table containing specific columns and/or rows, from the Generate Tables pop-up, select the ellipses menu by the side of the Xcalar table button option and do one the following:
    • To generate a Xcalar table containing rows and columns based on a condition:
      1. In the Filter String field, enter the condition as filter text.

        2. For example, to generate a Latest Xcalar table that contains only rows of data that was inserted, enter eq(XcalarOpCode, 1), where the filter syntax for the equal function is function(operand1, operand2).

      2. Click the Xcalar table option.

        3. A Refresh Tables Completed tooltip appears.

      For more information on working with filtering in Xcalar Design, see Filtering in a table column.

    • To generate a Xcalar table containing specific columns:
      1. In the Columns (comma separated) field, enter each column name required in the output followed by a comma.
      2. Click the Xcalar table option.

        3. A Refresh Tables Completed tooltip appears.

  5. From the Xcalar Design sidebar, select the Worksheets icon.
  6. In the Active tab of the Tables panel, open the newly generated Xcalar table for analysis.

Generating a worksheet of Xcalar tables from multiple Published tables

This topic describes how to generate a worksheet containing Xcalar tables from multiple Published tables that display current or historical results from the data updates. For more information on the output types, see Understanding the Xcalar Generate Tables popup.

To generate a worksheet of Xcalar tables from multiple Published tables:

  1. From the Xcalar Design sidebar, click the IMD Panel  icon.

    2. The IMD screen appears.

  2. For each Published table that is required to produce a Xcalar table in the worksheet, in the Published Tables section, select the Published table's check box.
  3. Depending on which output type you require, do one of the following:
    • To generate Point in time output, in the Timeline bar, click on the date and time required for the ending time, and then from the Generate Tables pop-up, click Point in time.
    • To generate Latest output tables, click anywhere in the Timeline bar, and then from the Generate Tables pop-up, click Latest.

    A Refresh Tables Completed tooltip appears.

  4. From the Xcalar Design sidebar, select the Worksheet Panel icon.
  5. In the Active tab of the Tables panel, locate the newly generated worksheet to view the Xcalar tables for analysis and comparison.

Conserving memory by compressing updates

The IMD Coalesce function is similar to the Latest function, in that all data changes are combined within the Published table. But unlike the Latest function, the Coalesce function removes all the delta table updates from the Published table's Timeline, which reduces the Published table's memory consumption. If you have a Published table that contains delta data that is no longer required for analysis, Xcalar recommends applying the IMD Coalesce function on the Published table.

NOTE: As all previous delta table updates are removed from the Published table after applying the Coalesce function, temporal data is no longer available for analysis.

To conserve memory by compressing updates:

  1. From the Xcalar Design sidebar, click the IMD Panel icon.

    2. The IMD screen appears.

  2. In the Published Tables section, select the Published Table's check box.
  3. (Optional) To generate a worksheet that contains multiple Xcalar tables from multiple Published tables. For each Published table that is required to produce a Xcalar table in the worksheet, in the Published Tables section, select the Published table's check box.
  4. Click the Coalesce icon.

    5. A COALESCE warning message appears, which states that the operation cannot be undone and asks you to confirm the action.

  5. Click CONFIRM, which:
    • Combines all the delta table updates in the Published table.
    • Removes all the delta table updates from the Published table and the Published table's Timeline.
    • Generates a Xcalar table comprising all the data changes in a new worksheet.
RECOMMENDATION: Inactive Published tables use very little memory. Therefore, move all Published tables that you are not working with into the Inactive Tables section.

Conserving memory by removing temporary tables

You can conserve memory by dropping temporary tables that you no longer require.

To conserve memory by removing temporary tables:

  1. From the Xcalar Design sidebar, click the Worksheet Panel icon.

    2. The Tables panel appears.

  2. Click the Temporary tab.
  3. From the Temporary tables list, select the check box of each table you no longer require.
    4. TIP: When you hover your mouse over the Temporary table the Table icon becomes a check box.
  4. From the Temporary tables header, click the Drop Tables icon.

For more information on conserving memory, see Understanding memory usage.

Conserving memory by displaying less update time-bars

Xcalar conserves memory by limiting the number of viewable update time-bars to 128. When the Published table exceeds this limit, a Load More Updates tooltip is displayed next to the last updated time-bar:

  • To display more instances of updated Published tables, click the Load More Updates element.

    • Repeat until the Load More Updates tooltip disappears.

  • To revert to the update time-bar limit and conserve memory, click the refresh icon.

IMD Published table naming conventions

The following list describes the naming conventions:
  • The name is case sensitive.
  • Characters must be A-Z, a-z, 0-9, hyphen (-), underscore (_), or space.
  • First character must be a letter.
  • Length must not exceed 255 characters.
