Editing a dataflow

For each active table, you can edit the operations in its dataflow. For example, when you performed data cleansing, you used the Filter operation to exclude the value "2" from a column, but later discovered that the correct value to exclude should be "3." You can edit the Filter operation with the correct argument and re-run the dataflow. Without the ability to edit operations, you would have to adjust your data model by manually re-creating all the tables descending from the operation that requires correction.

NOTE: You can edit a dataflow, not a batch dataflow. To change the values used in operations that are part of a batch dataflow, use customization as described in Customization of dataflow operations.

Operations that can be edited

You can edit most operations shown in a dataflow. In the dataflow graph, the icon for an editable operation has a blue border. You cannot edit indexing operations and the operation for creating the first table from a dataset.

What editing a dataflow means

Editing a dataflow means putting the dataflow graph in editing mode and changing the specifications for an editable operation. It does not mean that you can insert, remove, or re-order the operation icons in the dataflow graph.

A single operation icon in the dataflow graph might represent multiple Map operations, which you can edit individually. For example, the operation for splitting a column into three fields is shown as one icon, but it actually consists of three Map operations, one for each resultant field. You can edit each operation to change the delimiter, the name of the resultant column, and so on. You can also add or remove a split in this operation.

IMPORTANT: Changing the specifications for an operation is similar to entering the specifications the first time. For example, for a Map operation, you can change the operands, specify the resultant column name, and so on. However, when you edit a dataflow, Xcalar Design does not check the operands' data types. For example, if you inadvertently specify two integer columns as operands for the concat function, no error messages are displayed, and the dataflow can run successfully, creating a resultant column with FNF.

Editing an operation in a dataflow creates a data lineage for only one active table. The changes you make in one dataflow do not affect other dataflows. For example, an active table named airlines#XN103 is created by joining two tables airlines#XN101 and airlines#XN102, which remain on the worksheet after the join. Rerunning the edited dataflow for airlines#XN103 has no effects on the dataflows for airlines#XN101 and airlines#XN102.

What running an edited dataflow does

When Xcalar runs an edited dataflow, the edited operation is performed on the table to the left of the operation icon in the dataflow graph. All operations following the edited operation will run again to create a new data lineage leading to a new active table.

IMPORTANT: If you change the data type of a column or delete a column when editing an operation, and If a subsequent operation depends on that column, running the edited dataflow might produce unexpected results. For example, suppose the original dataflow contains an operation that splits the MonthDayYear column into the Month, Day, and Year columns. If you edit the split operation to delete the Map function that creates the Year column, and later in the dataflow, another operation calls for the Year column, running the edited dataflow results in an empty table.

If the table preceding the editable operation has been dropped, running the edited dataflow recreates the table with a different name. Dropped tables that are descendents of the editable operation have no effects on dataflow editing, because new tables will descend from the edited operation.

Locked tables also have no effects on dataflow editing. However, by default, all tables created by the edited dataflow are unlocked. If a new table is created to replace a locked table, apply the lock again.

Editing an operation

Follow these steps to edit an operation in an active table's dataflow:

  1. Display the dataflow graph for the table and click the Edit icon. After the dataflow enters editing mode, all other dataflows are no longer displayed. Also, the title bar of the active table in the dataflow changes to green.

    The following screenshot shows the location of the Edit icon.

    NOTE: When a dataflow is in editing mode, options normally accessible through the Worksheets icon () are disabled. For example, you cannot use this icon to display lists of worksheets, tables, or aggregates.

  2. Move the cursor over an operation icon with a blue border to display a tooltip, which provides detailed information about the operation. After you verify that the operation is the one you want to edit, click the operation icon to display a menu.

    The following partial screenshot shows the icons displayed in the upper right corner of the dataflow graph when the dataflow is in editing mode. You can use the icons to close the dataflow graph or exit editing mode.

    NOTE: Even after you close the dataflow graph, the dataflow continues to be in editing mode.

    NOTE: Several operation icons might coalesce into one to save space. Before editing one of the operations, reveal the operations that are represented by the single operation icon.

  3. From the menu, click Edit operation. If the operation selected is a single operation, a panel for editing is displayed immediately.

    If the operation consists of several operations, a pop-up window is displayed, in which you can edit or delete the individual operations, or add operations. In the following screenshot, you can edit the operation for splitting a column to change how each split is performed. After you click the Edit icon in the pop-up window, a panel for editing that operation is displayed.

    In the pop-up window, you can delete an operation. However, if there is only one operation in the pop-up window, you cannot delete it.

    When the edit panel is displayed, Xcalar Design highlights the icon for the table on which the edited operation will be performed. Faded table icons represent the tables that will be replaced as a result of the edited dataflow.

  4. In the panel for editing, enter the desired values in the fields. The description in the panel shows the syntax of the function. For information about Map functions, see Map functions.

  5. Click SAVE.

    NOTE: The edited dataflow is saved in place. The old dataflow is not retained.
  6. (Optional) If the editable operation consists of multiple Map functions, click the edited operation again. Then repeat steps 3 through 5 for each Map function in this editable operation, if needed.

  7. (Optional) After the panel for editing is closed, click the icon for the edited operation, which has a green border, to display a menu. The following list describes the menu options:

    • Revert to original operation removes all your edits. The original operation is restored.
    • Edit operation displays the panel for editing so that you can continue editing.
    • Exit editing mode discards all your changes and returns the dataflow graph to read-only mode. This has the same effect as clicking the Exit editing mode icon in the upper right corner of the dataflow graph.

Running an edited dataflow

If a dataflow contains an operation with a green border, you can click the Run icon to run the edited dataflow. The following screenshot shows the location of the icon and an example of an edited dataflow.

After the confirmation message is displayed, click CONFIRM. In this example, the original active table before the edited dataflow runs is airlines4#XN149. After the edited dataflow runs, this table becomes a temporary table, and a new active table named airlines4#XN150 is placed in the worksheet. The dataflow automatically exits editing mode.

You can undo the rerun of the dataflow by clicking the Undo icon, in the same way as you undo any operation in Xcalar Design. Undoing the rerun also removes all your changes to the dataflow.
