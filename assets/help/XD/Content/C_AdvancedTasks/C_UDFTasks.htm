Using UDFs

This section describes the tasks related to UDFs (user-defined functions). To see examples of UDFs, see User-defined function (UDF) examples.

Creating a UDF in Xcalar Design

UDFs are local to a workbook or to a batch dataflow. UDFs created by one Xcalar Design user can be read and copied by all other Xcalar Design users, so they can accomplish similar tasks.

You define one or multiple UDFs in a Python module. You can create a module in these ways:

  • Type the code directly in the Editor tab of the User Defined Function panel.
  • Upload from your computer to the Editor.
  • Use the JUPYTER NOTEBOOK window to display a UDF template and then use the Jupyter notebook editor to modify the code in the template. For more information about using the Jupyter Notebook, see Using Xcalar Design with Jupyter Notebook.

Typing the code directly

Follow these steps to write the UDF in the panel and then save it:

  1. Select the Editor tab if it is not selected already.
  2. Click the drop-down arrow to display a list of existing modules. Select one to use it as a template. Alternatively, select New Module if you want to type in an empty file.
  3. Type the code in the panel.

  4. In the module name field, type the module name.

    NOTE: Module names are case insensitive. If you type letters in uppercase, Xcalar Design displays them in lowercase when listing the modules. See UDF naming conventions for information about name requirements.

  5. Click ADD UDF. If a module with the same name already exists, a dialog box asks whether you want to overwrite the existing module. If so, click CONFIRM. Otherwise, click CANCEL and then rename the module.

Note: Overwriting a functioning UDF with a non-functional one can cause workbook activation to fail. Best practice is to apply disciplined versioning to your UDFs. For more information, see Effect of changing a UDF on workbook re-activation. If you are unable to open a workbook, file a support ticket with Xcalar support to assist with replacing the UDF.
TIP: When editing a UDF, you can use the keyboard shortcut appropriate for your operating system to search for or replace a string. For example, if you run Xcalar Design in a browser on Microsoft Windows, press the Ctrl and F keys to search for a string, and the Ctrl, Shift, and F keys to replace a string.

Uploading files

Follow these steps to upload a file as a UDF module from your computer to your cluster:

  1. Select the Editor tab if it is not selected already.
  2. Click Browse File, which enables you to navigate to the folder on your computer that contains the file. Then select the file. The contents of the selected file are displayed in the User Defined Function panel.
  3. In the module name field, the name of the file is automatically displayed. Change the name if necessary.
    4. NOTE: If the original filename does not follow the naming conventions, an error message notifies you that the name is invalid. See UDF naming conventions for name requirements.

  4. Click ADD UDF. If a module with the same name already exists, a dialog box asks whether you want to overwrite the existing module. If so, click CONFIRM. Otherwise, click CANCEL and then rename the module.

About Python code parsing

When you try to add a UDF, it is parsed by the Python interpreter. If there are parse errors, they are reported to you and the UDF is not added.

Downloading a module

Follow these steps to download a UDF module from your cluster to your computer:

  1. Select the Manager tab in the User Defined Function panel. A list of modules is displayed.
  2. Click to download the desired module to your computer.

Editing a UDF

IMPORTANT: Because UDFs are local to a Xcalar Design workbook, remember to notify other users of the changes so that they are aware of the revision. Also, changing the code in a UDF might have significant impact on both workbook re-activation and batch dataflows. Make sure you understand the effects of changing a particular UDF before proceeding, and consider aggressive use of versioning on your UDFs.

Follow these steps to edit an existing UDF:

  1. Select the Manager tab in the User Defined Function panel. A list of modules is displayed.
  2. Edit the module in one of three ways:
    • Click to edit the desired module. Then edit the code in the User Defined Function panel. Add the edited module to Xcalar without changing its name.
    • Click to download the desired module to your computer. Then modify the module by using an editor on your computer. Upload the file after you finish editing.
    • Click to view the desired module. Copy the desired module code text to Jupyter Notebook or external UDF editor. Paste new module code back into the UDF editor.
  3. Optionally, add a version number string to the module name.
  4. Click ADD UDF.

Effect of changing a UDF on workbook re-activation

When you re-activate a workbook, Xcalar must re-create each table in the workbook according to the information contained in the table's dataflow. If a dataflow contains a UDF that has been changed since the table was created, the table displayed in the re-activated workbook might contain values different from the original ones.

Effects of changing a UDF on a batch dataflow

When you create a batch dataflow, it receives a current local copy of all relevant UDFs. Even if the UDF is changed after batch dataflow creation, during execution Xcalar will use the current version as-of when the batch dataflow was created.

 

Deleting a module

IMPORTANT: Be sure you understand the effects of removing a UDF from your UDF module or deleting a UDF module before you proceed. You cannot undo UDF module deletion. For more information, see Effects of UDF deletion on workbooks and batch dataflows.

Follow these steps to delete a UDF module:

  1. Select the Manager tab in the User Defined Function panel. A list of modules is displayed.
  2. Click to delete the module.

Effects of UDF deletion on workbooks and batch dataflows

Because UDFs are local to your workbook and one or more modeling operations rely on your UDFs, removing a function from a UDF module or deleting a UDF module could have the following impacts:

  • When Xcalar Design tries to re-activate a workbook (for example, after Xcalar cluster is restarted):

    If the workbook contains a temporary or active table that is dependent on the UDF, Xcalar Design displays an error message reporting that the UDF cannot be found. Download a copy of your workbook and submit it with a support ticket for Xcalar support.

    A table is considered to be dependent on the UDF if one of these conditions applies:

    • The table has a column created by the UDF.
    • The table is a descendent of a dropped table or a temporary table that contains a column created by the UDF.

  • When you try to create a batch dataflow:

    If the dataflow contains the deleted UDF, Xcalar Design displays an error message. You cannot create a batch dataflow from a dataflow with a missing UDF.

Replacing a UDF in an inactive workbook

There are two circumstances when a UDF state could prevent workbook reactivation:

  • A UDF in-use is missing.
  • A UDF in-use fails to execute.

When this happens, add a working UDF into the workbook manually, as follows:

  1. Take note of the workbook and module names. The details of the activation error message contain both names in UDF path:

    /workbook/userName/workbookName/udf/moduleName

    Where:
    workbookName is the name of the workbook.
    moduleName is the name of the UDF module.
  2. Find a version of the module, and if necessary, repair it.
    • Common locations to find your UDF module:
      • Another workbook modeling with the same or similar data
      • A Jupyter notebook on your Xcalar Cluster
      • A local work directory
      • Collaboration emails with coworkers
    • Assorted techniques to repair your UDF:
      • Create a new workbook.
      • Add identical or similar data source files.
      • Apply found UDF code to the data.
      • Debug using visual tools, like Jupyter Notebook.
      • If an import UDF, accommodate recent data source file format or field changes.

      At the completion of this step, save your UDF module in a text file named moduleName.py.

  3. Download the workbook:
    1. Click to view the Workbook Browser.
    2. On the card for the workbook, click and select Download Workbook.

    Xcalar downloads the file workbookName.tar.gz to your browser downloads directory, where workbookName is the name of the workbook.

  4. Decompress and extract the workbookName.tar.gzworkbook archive:
    • If using MacOS, double-click on the file.
    • If using Linux, open a shell to your downloads directory and execute the command:
      tar -xvzf workbookName.tar.gz
    • If using Windows, choose and use an appropriate archiving and compression tool to extract the workbook.

    A folder named workbook is extracted.

  5. Move the original workbookName.tar.gz to another directory for safe keeping.
  6. Copy your new version of the UDF module to the /workbook/udfs directory.
  7. Create a new workbook archive file.
    1. If using MacOS or Linux, open a shell to the directory containing the workbook directory and enter the following command:
      tar cvf workbookName.tar.gz ./workbook/
    2. If using Windows, apply third party software or use Windows 10's Windows Linux Subsystem to use the above tar command to create a new workbookNametar.gzarchive file.
  8. Upload the new workbook to the cluster by clicking the BROWSE button under Upload Workbook in the Workbook Browser.
