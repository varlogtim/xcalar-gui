Data Browser

Each table contains a default column named DATA. You can move your cursor to any row in this column and double click to display the Data Browser.

NOTE: The DATA column might be minimized and not completely visible in the worksheet due to a setting in the General Settings window of the Monitor or the Minimize column option in the column drop-down menu. For information about the Monitor, see Changing the environment settings for Xcalar Design. To view the DATA column, click the minimized DATA column header to display the column drop-down menu and select Unminimize column, as shown in the following screenshot:

Contents of the Data Browser

The browser displays two kinds of fields:

  • Prefixed fields, which are fields from the data source. If the table is a result of one or multiple Join operations, multiple sets of prefixed fields from multiple data sources are displayed. Each set has its own prefix.
  • Derived fields, which are created through various operations. They do not have a prefix.
NOTE:  A table resulting from a batch dataflow contains only derived fields. Its Data Browser does not contain any fields from a data source. For more information about tables created from batch dataflows, see Using batch dataflows.

For each field, the value is also displayed. For example, in the following screen, the Data Browser for row 1 is displayed.

NOTE: If the key for a field does not exist, Xcalar Design displays FNF in the table cell, and the field does not appear in the Data Browser.

Searching in the Data Browser

If the Data Browser contains a long list of fields, you can easily locate a particular field by searching for it. Click the Search icon to display the search field. Then type the name of the field, as shown in the following screenshot. The search is not case-sensitive.

Tasks you can perform in the Data Browser

Specifying the select mode

You can add fields in the Data Browser to the active table as described in Adding columns from a dataset to a table. Specify the select mode, which determines how your select fields when adding them to the table:

  • Single select mode: If you select this mode, clicking a field name immediately adds the field to the table.
  • Multi select mode: If you select this mode, you can select multiple fields and then submit your selection.

Sorting fields

By default, the fields in the Data Browser are arranged in the same order as they are in the dataset. You can sort the fields in ascending or descending order. The following screenshot shows the location of the icon for sorting.

Duplicating the list of fields in the browser

Duplicating the list of fields is useful when you have a long list of fields. Because each copy of the list has its own scroll bar, you can scroll each copy independently, such that the fields that interest you appear side-by-side. In each panel, you can choose to view all fields, only the derived fields, or only fields with a particular prefix.

Comparing rows

The Data Browser enables you to compare values in different rows from the same table or from different tables. Suppose the Data Browser for row 5 is displayed, you can double click another row in the DATA column (for example, row 10) to bring its data into the Data Browser. Now you can view the values from two rows side-by-side.

NOTE: You might need to move the Data Browser to make the DATA column visible. Also, use the scroll bar to locate the table that interests you. When the Data Browser is displayed, you can only select rows visible in your window. You cannot scroll vertically to show rows that are currently out of the Xcalar Design window.

To see how values differ, click the Compare icon for each list of fields that you want to compare. The values that are different between the rows are highlighted. The following screen shows the comparison between rows 1 and 2 of the same table.

NOTE: You can compare data from more than two rows.
