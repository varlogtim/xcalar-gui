Glossary

The following table is a Xcalar glossary of terms.

Term Meaning
Batch dataflow You can save a dataflow under a name. A named dataflow is a batch dataflow, which can be scheduled to run at pre-set times. Running a batch dataflow lets you run operations against the entire data source unattended as opposed to executing individual operations interactively against sampled data. It also enables you to create a table based on the dataflow with selected columns. You can then export the table to another application or the active worksheet in Xcalar Design. A batch dataflow goes through optimization in a way similar to how a SQL query goes through query optimization. However, neither dataflow nor query optimization can guarantee optimal performance, nor necessarily meet Service Level Objectives (SLO) for real-world workloads. To achieve that, you need to operationalize the batch dataflow. For information about operationalizing dataflows, see Operationalizing batch dataflows
Dataflow A set of elementary operations that shows the data lineage for the active table. Xcalar Design displays a dataflow by using various icons to represent datasets, tables, and operations.
Dataset The representation of raw data and metadata that Xcalar creates when you import a data source. The dataset may contain all data or a sample of the data in the data source, depending on the limit specified by the Xcalar administrator. You can preview a dataset and create tables from it.
Data source The file containing raw data or the directory containing files with raw data that you import into Xcalar.
Drop a table Dropping a table means completely removing the table (not just its contents) from a worksheet (for example, to conserve memory). This action is not reversible. Dropping a table is not equivalent to deleting a table in SQL.
Export This is the process for copying a table in Xcalar Design (for example, a resultant table created by an operationalized dataflow) to a location for another application to use or to a Xcalar worksheet.
Import The first step you take when you use Xcalar to demystify your data. It means processing data from a data source and creating a dataset. Through Xcalar Design, you can preview the data in the dataset and create tables from it.
Modeling

One or all of these tasks:

  • synthesizing a schema
  • designing an algorithm
  • designing a query
  • designing a linear regression analysis
  • designing a new machine learning algorithm

Xcalar will expand this list of supported modeling tasks in future releases.
Node The physical or virtual execution environment that is a member of a cluster. For example, it can be a server or a virtual machine running Linux.
Operation A database operation, which is an overall process to accomplish a transformation. For example, "group by" is an operation. An operation can include one or multiple functions. For example, a map operation can contain a function that means, "If a value in column X is greater than 100, create a new column called Bad-percent, which is True, else set value in new column to False."
Operationalizing batch dataflows Batch dataflows can be run by the scheduler or on demand. For improving the performance of batch dataflows, Xcalar offers professional services for performance tuning and hardware planning to meet Service Level Objectives (SLOs). Batch dataflows, after being tuned, are called operationalized batch dataflows.
Query A request for information from a database. You need the information to answer your particular question.
Sample Collection of records from the data source on which you can interactively model your dataflow.
Session A session is the same as a workbook in Xcalar Design. It does not refer to a user's login. Sessions are visible only to the cluster administrator when the administrator views information about workbooks on the cluster nodes.
Xcalar Compute Engine The engine that does the computing. Users interact with Xcalar Compute Engine through Xcalar Design.
User-defined function (UDF)

A UDF is a program that accepts parameters and performs an operation to produce a value. Some built-in UDFs are included in Xcalar default modules for you to enhance your operations. You can create additional UDFs to further extend the functions of Xcalar.

An import UDF is a UDF for parsing data when Xcalar imports the data.

An export UDF is a UDF for processing data when Xcalar writes the data from Xcalar to an export target.
Xcalar Design The graphical user interface that users use to do interactive data modeling.
