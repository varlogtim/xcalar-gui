Naming conventions in Xcalar Design

This topic lists the naming conventions for all entities that you can name in Xcalar Design.

Batch dataflow, dataset folder, export target, export file

The following list describes the naming conventions:
  • The name is case sensitive.
  • Length must not exceed 255 characters.
  • Characters must be A-Z, a-z, 0-9, hyphen (-), and underscore (_).

Column

The following list describes the naming conventions:
  • The name is case sensitive.

  • The name must consist of alphanumeric characters, spaces, and special characters except the following characters:

    • caret (^)
    • single quote mark (')
    • double quote mark (")
    • left or right parenthesis (( ))
    • left or right square bracket ([ ])
    • left or right brace ({ })
    • backslash (\)
    • colons (:)
    • comma (,)
    • period (.)
  • The name must not begin or end with a space.
  • The name must not begin with a number.
  • Length must not exceed 255 characters.
  • The name must not be DATA. Because the name is case sensitive, data or Data is acceptable.

Column prefix

The following list describes the naming conventions:
  • The name is case sensitive.
  • Length must not exceed 31 characters.
  • Characters must be A-Z, a-z, 0-9, hyphen (-), and underscore (_).
  • First character must be a letter.

Dataset

The following list describes the naming conventions:
  • The name is case-sensitive.
  • Length must not exceed 255 characters.
  • Characters must be A-Z, a-z, 0-9, hyphen (-), and underscore (_).
  • First character must be a letter.

Parameter

The following list describes the naming conventions:
  • The name is case sensitive.
  • Length must not exceed 255 characters.
  • Characters must be A-Z, a-z, and 0-9.
  • No special characters (for example, underscore, parenthesis, hyphen, or space) are permitted.

Table

The following list describes the naming conventions:
  • The name is case sensitive.
  • Characters must be A-Z, a-z, 0-9, hyphen (-), underscore (_), and space.
  • First character must be a letter.
  • Length must not exceed 255 characters.
  • The name must not be a duplicate of an existing table name in the same workbook.

UDF

The following list describes the naming conventions:
  • Characters must be a-z, 0-9, hyphen (-), and underscore (_).
  • Letters must be lowercase.
  • First character must be a letter or an underscore.

Usernames

The following list describes the conventions for user names:
  • The name is case sensitive.
  • Characters must be A-Z, a-z, 0-9, period (.), hyphen (-), and underscore (_).
  • The first character must be a letter.
  • The last character must be a letter or number.
  • Minimum length is 3 characters.

Workbook

The following list describes the naming conventions:
  • The name is case-sensitive.
  • Length must not exceed 255 characters.
  • Characters must be A-Z, a-z, 0-9, hyphen (-), underscore (_), and space.
  • First character must be a letter.

Worksheet

Worksheet names are case-sensitive. There are no restrictions on the characters, but Xcalar Design removes the space if you type it as the first character.
