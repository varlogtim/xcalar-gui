Visualizing a Xcalar result set in Jupyter Notebook

After you have processed your data in Xcalar, many times you want to quickly visualize it using standard Python libraries. Xcalar provides an easy way for visualizing the result sets through integrated Jupyter Notebook environment. Jupyter Notebook is an open-source web application that allows you to create and share documents that may contain live Python code, equations, visualizations, and narrative text. This tutorial shows how you can visualize your Xcalar data in Jupyter Notebook using pandas dataframe.

The first step in creating an application in Xcalar is to create a project. A project is a container where you create, design, troubleshoot, and execute your application. Open Xcalar Design in a browser and log in. Create a new project called VisualizeDataInJupyter and open it by clicking on its project card.

Xcalar has two modes - the SQL Mode and the Developer Mode. In SQL Mode, you develop your queries using the SQL query statements and user-defined functions (UDFs) written in Python. In Developer Mode, you develop your applications using visual components, SQL query statements, and UDFs written in Python.

Since the Jupyter Notebook environment is only available in Developer Mode, click on the mode switch in the top right corner to go to Developer Mode.

Next, we will import the wine reviews file (winemag-data.json) into Xcalar. If you have already imported this file into a dataset, skip to the second step.

1. Importing the wine reviews file

To import the wine reviews file into Xcalar, click on the Sources icon in the sidebar, and then click Import Data Source in the Datasets panel. Since our file resides on Xcalar’s public S3 bucket, select Public S3 as the Connector, and enter /xcmarketplace-us-east-1/tutorial-datasets/ as the Data Source Path. Click Browse to browse the directory structure and select winemag-data.json. Click Next to preview the file.

Xcalar automatically detects the format of the file and displays its preview. Since this is a simple json file, we don't need to pull out any nested fields. Accept all default options and click Create Dataset. A dataset called winemag-data is created with 129,971 rows and 13 columns. The next step is to read the dataset.

2. Creating a module and reading the dataset

To read the dataset, we need to create a module. In Xcalar, a module is a graphical representation of a series of operations performed on data. Each operation is represented by a graphical component called Node. To create a module, click the Application Builder icon in the sidebar and then click either the Create new module or the New module tab. A new module called Module 1 is created.

Every module starts with a source Node, which brings the data into the module. Since we imported our data file into a dataset, let’s add a Dataset Node to the module and configure it to read the dataset we imported.

From the In category of Nodes, add a Dataset Node to the canvas. To configure the Node, click it, and select Configure. In the configuration panel, select winemag-data dataset and click Save. By default, the Node gets executed, and the result set is displayed:

Next, we will group the wines by their ratings, stored in the points column.

3. Grouping rows

To group the wines by ratings, right click the points column header and select Group By. A Group By Node is added to the canvas and its configuration panel opens up. Configure the Node as follows:

Fields to group on: $winemag_data::points

Function to apply to group: count

Field name to count occurrences of: $winemag_data::title

New column name for the group by resultant column: wine_count

Save and execute the Node. A new result set is displayed with 21 rows:

4. Changing the data type

If you look carefully at the data type of each column in the column header, you will notice that the points column is of type string (). To visualize this result set in Python, we may need to convert this column to integer. Right click on the points column header and select Change data type -> Integer. A Cast Node is added to the canvas and it's configuration panel is displayed. The points column is at the top of the panel showing a New Name and New Type:

Click Save and execute the Node.

A new result set is displayed with points column converted to integer.

5. Converting a result set to pandas dataframe

To convert the result set to a pandas dataframe for Jupyter Notebook, from the Out category of Nodes, drag and drop the Jupyter Node to the canvas and connect it to the output of the last Node. Click the Jupyter Node and select Configure. Accept all defaults in the configuration panel and click Save. The Node is executed by default.

NOTE: Xcalar restricts the number of rows sent to Jupyter Notebook to 1000.

Click on the Jupyter Node and select View Result. A new Jupyter Notebook is created and the Rename Notebook modal is displayed. Enter a name for the Notebook - VisualizeDataInJupyter and click Rename.

6. Visualizing data in Jupyter Notebook

To visualize the wine ratings data in Jupyter Notebook, let's insert a new cell by clicking the + icon in the Jupyter Notebook toolbar. Enter the following sample code in the cell:

df = table_DF2_5DE02FAF152B9FB8_1574973688844_0_dag_5DE02FAF152B9FB8_1574979949264_38_t_1574980028554_3_pd
fig = plt.figure()
ax = fig.add_axes([0,0,1,1])
ax.set_ylabel('Wine Count')
ax.set_xlabel('Wine Rating')
ax.bar(df.points, df.wine_count)
plt.show()

Run the cell and a bar graph is displayed for the wine ratings data:

This sample code is using the standard matplotlib library to plot the graph. You can play with the code and change the graph as per your needs.

As you can see, Xcalar provides you a very simple way for converting a result set to a pandas dataframe and then visualizing it in Jupyter Notebook.
