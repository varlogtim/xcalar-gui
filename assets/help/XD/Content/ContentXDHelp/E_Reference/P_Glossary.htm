Xcalar concepts

The following table describes the common Xcalar terms.

Term Meaning

Dataflow Mode

Dataflow Mode gives you a powerful visual design framework to prepare your data models. It augments the SQL Mode by providing features and functions for monitoring and troubleshooting your queries. You do not need to be an expert in SQL to work in the Dataflow Mode.

Dataflow A dataflow is a series of operations linked together to create data models in Dataflow Mode.
Dataflow editor A visual editor to develop and debug dataflows in Dataflow Mode.
Dataset To read your raw data into Xcalar, you need to import it in a table or in a dataset. When you import it in a dataset, Xcalar reads the raw data, derives its metadata and keeps it in the memory. This prevents the raw data to be stored on the cluster unnecessarily.
Data source The file containing raw data or the directory containing files with raw data that you import into Xcalar.
connector Xcalar can import data from various storage platforms, in cloud or on-premises. When you import data, you must provide information about the storage location and the method used by your Xcalar instance to access the location. A connector contains the specifications of the storage location and the access method.
Dropping table/result set Dropping a table/result set means completely removing it (not just its contents) from your workbook (for example, to conserve memory). This action is not reversible.
Export This is the process of saving a result set in Xcalar Design to a file so that other applications can use it.
Import The process of reading a source data, deriving its metadata, and building a table or a dataset is called importing a data source in Xcalar Design.
Modeling

One or all of these tasks:

  • synthesizing a schema
  • designing an algorithm
  • designing a query
  • designing a linear regression analysis
  • designing a new machine learning algorithm

Xcalar will expand this list of supported modeling tasks in future releases.
node The physical or virtual machine running Linux that is a member of a cluster.

Node

The visual representation of an operation in the dataflow editor.
Operation A function to transform the data. An operation can include one or multiple functions. For example, "group by" is an operation.
Query A request for information from a database generally written in SQL (Structured Query Language).
Result set The result of the execution of a dataflow or a Node or a SQL query in Xcalar Design, which you can view in a table format.
Skew Measure of how imbalanced the data is across the nodes of a cluster. If data is uniformly distributed across a cluster, data has no skew. The value ranges from 0 to 100. If it is 0, the data is perfectly distributed across all cluster nodes. If it is 100, all the data is on one cluster node.
SQL Workspace This is the workspace where you develop your data model using SQL queries.
SQL Function The SQL function is a series of operations that manipulate and retrieve data from one or more tables (Input Nodes) and returns the results in another table (Output Node). The SQL function can replace any table within a SQL query statement. The purpose of a SQL Function is to increase reusability and easier debugging of the data algorithm.
SQL Mode This user interface contains features and functions that enable you to import your data into tables, apply SQL queries to those tables for analysis, create tables from result sets for more analysis, and export your results outside of Xcalar.
Table

To read your raw data into Xcalar, you need to import it in a table or in a dataset. When you import it in a table, Xcalar reads the raw data and its metadata and saves it on the cluster. The data stored as tables is available both in the SQL Mode as well as in the Dataflow Mode. If you want to develop your data algorithms in the SQL Mode, you must import your data in the tables.

User-defined function (UDF)

A UDF is a function that accepts parameters and performs an operation to produce a value. It helps you extend the functionality of Xcalar. Some built-in UDFs are included in Xcalar's default module and you can create more.
Workbook Creating a workbook is the first step in designing your data algorithm in Xcalar Design. Workbook can be regarded as a file where you design your data algorithms and execute them.
Xcalar Compute Engine The engine that does the computing in Xcalar. Users interact with Xcalar Compute Engine through Xcalar Design.
Xcalar Design Xcalar Design is an HTML5 based browser application to develop and execute your data models on Xcalar platform.
