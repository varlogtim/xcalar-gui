Data types supported

This topic describes the data types supported in Xcalar. For information about specifying the data type for a column, see Changing the data type of a column.

Understanding data types

Xcalar supports the following data types in a table or dataset:

  • String
  • Integer
  • Float
  • Money
  • Boolean
  • Timestamp

Xcalar supports the following data types in a dataset but not in a table:

  • Object
  • Array
  • Mixed
  • Unknown

The icon in the column header indicates the data type of the column. The following screenshot explains the data type icons.

The array or object type is assigned by Xcalar for data imported from a JSON file to a dataset. You cannot cast a column to the array or object type. Also, you cannot change an array or object column to another type.

Understanding the mixed and unknown types

The mixed and unknown data types are less common than the other data types. These data types are assigned by Xcalar Design; you cannot cast a column to one of these types.

The following list describes the mixed and unknown types:

  • Mixed: Xcalar Design detects different types of data in the column.

    NOTE: Xcalar Design never assigns this type if the data source file format is CSV. When data is imported, If at least a non-numeric value is detected in a column, the column is cast as a string column. Otherwise, the column is cast as an integer or float column.

  • Unknown: If you pull a field from an array or object in the Data Browser, and the field in that array or object does not have a type assigned to it in the dataset, the newly created column has an unknown data type. For information about type casting a field in an array or object during dataset configuration, see Changing the data type of a column.

    NOTE: The terms unknown and undefined are used interchangeably in Xcalar Design.

Understanding the timestamp type

The timestamp type accepts the following formats:

  • yyyy-mm-dd (for example, 2019-03-13)
  • yyyy-mm-dd followed by the Unix epoch time (for example, 2019-03-11T00:00:00.000Z)

The following values are not valid timestamp values:

  • 03/13/2019
  • 03-13-2019
  • 2019/03/13

Xcalar does not support timezones.

Understanding the float and money types

The following list provides information about the float and money data types that you must know to avoid unexpected operation results:

  • Float: Xcalar uses the double-precision binary floating point representation defined in the IEEE 754-2008 standard. For rounding numbers, Xcalar uses the half even rounding rule.

    • Integer values that can be accurately represented must be in the range between -(253 - 1) and 253 - 1. For example, if a result set contains a value greater than 9007199254740991, Xcalar might fail to filter the value.

  • Money: Xcalar uses the 128-bit decimal floating point representation defined in the IEEE754-2008 standard. Internally, Xcalar loads, stores and manipulates a money value with 34 digits of precision. When a result set is created, the value is rounded to two decimal places using the half-up rounding rule.

    String values enclosed in quotation marks are supported. The quotation marks are discarded when an input value is converted to the money type. For example, the string "100.00" is converted to the money value 100.00.

    For a value to be cast to the money type, it must be either an integer or a well-formed decimal string. The following list shows examples of these strings:

    • 14000
    • "14000"
    • -14000.34221
    • "+14000.34221"

    Strings in scientific notation are also supported. The following list shows examples of these strings:

    • "1.4e4"
    • -1.400034221e4
    • +1.400034221e4

    The following special string values are supported and converted to NaN:

    • Inf (infinity)
    • -Inf (negative infinity)
    • NaN (not a number)
    NOTE: Extraneous leading or trailing characters are not supported. For example, leading or trailing dollar ($) signs or percent (%) signs are not supported. Comma separators are also not supported.

Results of casting float values to the integer, string, or money type

If a float value with a decimal point is cast to integer, all numerals after the decimal point are removed. No rounding takes place. For example, if the float value is 1.9, after you cast it to integer, the integer value is 1.

If a float value is cast to string, the characters corresponding to the value's floating point representation are displayed. Because a displayed float value can result from multiple representations, you might see different strings after casting the same displayed float value. For example, if the float value is displayed as 4.2, after you cast it to string, the string value might be 4.2 or 4.2000000000000002.

If a float value that is a whole number is cast to string, the resultant string contains a decimal point followed by 0. For example, if the float value is 1, the resultant string is 1.0.

IMPORTANT: Avoid converting values from float to money or vice versa because the conversion might cause loss of precision. For example, the float value 31.10 is not equal to the money value 31.10.

Results of casting a column to or from the timestamp type

The following list describes the results of casting a column to or from the timestamp type:

  • If you cast a string in the format of yyyy-mm-dd to the timestamp type, the Unix epoch time is automatically attach to the result. For example, if the string value is 2019-03-12, the result is 2019-03-11T00:00:00.000Z.
  • The cast operation automatically adds preceding zeros if necessary. For example, casting the string value 2019-3-2 to timestamp results in 2019-03-02T00:00:00.000Z
  • If you cast an integer to a timestamp, the integer is considered the number of milliseconds since the Unix epoch. For example, if the integer is 1, the timestamp is 1970-01-01T00:00:00.001Z.
