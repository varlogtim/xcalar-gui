Naming conventions in Xcalar Design

This topic lists the naming conventions for all entities that you can name in Xcalar Design.

Dataset folder, export target, export file

The following list describes the naming conventions:
  • The name is case sensitive.
  • Length must not exceed 255 characters.
  • Characters must be A-Z, a-z, 0-9, hyphen (-), or underscore (_).

Column

The following list describes the naming conventions:
  • The name is case sensitive if the column is in a dataset; it is case-insensitive if the column is in a table, in which case all letters are displayed in uppercase.

  • The name must consist of alphanumeric characters, spaces, and special characters except the following characters:

    • caret (^)
    • single quote mark (')
    • double quote mark (")
    • left or right parenthesis (( ))
    • left or right square bracket ([ ])
    • left or right brace ({ })
    • backslash (\)
    • colons (:)
    • comma (,)
    • period (.)
  • The name must not begin or end with a space.
  • The name must begin with an underscore or a letter.
  • Length must not exceed 255 characters.
  • The name must not be DATA, but the same string capitalized differently, such as data or Data, are acceptable.
  • The name must not be one of the following strings, regardless of capitalization:
    • None
    • False
    • True
  • The name must not contain two consecutive hyphens (--).

Column prefix

The following list describes the naming conventions:
  • The name is case sensitive.
  • Length must not exceed 31 characters.
  • Characters must be A-Z, a-z, 0-9, hyphen (-), or underscore (_).
  • First character must be a letter.

Dataset

The following list describes the naming conventions:
  • The name is case-sensitive.
  • Length must not exceed 255 characters.
  • Characters must be A-Z, a-z, 0-9, hyphen (-), or underscore (_).
  • First character must be a letter.

Parameter

The following list describes the naming conventions:
  • The name is case sensitive.
  • Length must not exceed 255 characters.
  • Characters must be A-Z, a-z, or 0-9.
  • No special characters are permitted (for example, underscore, parenthesis, hyphen, or space).

Table

The following list describes the naming conventions:
  • Length must not exceed 255 characters.
  • Characters must be A-Z, a-z, 0-9, or underscore (_).
  • First character must be a letter.
  • Letters are displayed in uppercase, but you can enter them in uppercase or lowercase.

SQL code snippet

The following list describes the naming conventions:
  • Characters must be a-z, 0-9, or underscore (_).
  • First character must be a letter.
  • Length must not exceed 255 characters.

SQL function

The following list describes the naming conventions:
  • Characters must be A-Z, a-z, 0-9, or underscore (_).
  • The name is case-sensitive.
  • Length must not exceed 255 characters.

UDF

The following list describes the naming conventions:
  • Characters must be a-z, 0-9, hyphen (-), or underscore (_).
  • Letters must be lowercase.
  • First character must be a letter or an underscore.
  • Character length:
    • UDF module name, the length must not exceed 1023 characters.
      NOTE: The UDF module name default is reserved by Xcalar.
    • UDF function name, the length must not exceed 255 characters.
    • UDF module source code, the length must not exceed 10485760 characters.

Usernames

The following list describes the conventions for user names:
  • The name is case sensitive.
  • Characters must be A-Z, a-z, 0-9, period (.), hyphen (-), or underscore (_).
  • The first character must be a letter.
  • The last character must be a letter or number.
  • Minimum length is 3 characters.

Workbook

The following list describes the naming conventions:
  • The name is case-sensitive.
  • Length must not exceed 255 characters.
  • Characters must be A-Z, a-z, 0-9, hyphen (-), underscore (_), or space.
  • First character must be a letter.
  • The name must not be one of the following strings:
    • udf
    • udfs
  • For the admin user, the name must not be PreUpgrade.
