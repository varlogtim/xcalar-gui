Naming conventions in Xcalar Design

This topic lists the naming rules for the components that you can name or rename in Xcalar Design.

RECOMMENDATION: As well as adhering to the Xcalar naming rules, Xcalar recommends avoiding any characters that can cause misinterpretation and conflicts outside of Xcalar Design.
For example, limit the use of special and punctuation characters that may not conform to the naming standards or conventions of languages such as SQL and Python, third-party business intelligence (BI) reporting software, or external data repositories, databases, and platforms.

Column

The following list describes the naming conventions:

To avoid any issues from the case sensitivity column naming differences between Xcalar and Xcalar SQL, column names are automatically processed into uppercase characters.

  • Length must not exceed 255 characters.
  • First character must be a letter of the alphabet or an underscore.
  • First character must not start or end with a space.
  • Characters must be A-Z, a-z, 0-9, or underscore (_).
  • Cannot contain special or punctuation characters, such as:
    • caret (^)
    • single (') or double (") quote marks
    • left (() or right ()) parenthesis
    • left ({) or right (}) brace
    • forward slash (/) or backslash (\)
    • colon (:) or semi-colon (;)
    • comma (,)
    • period (.)
    • single (-) or double hyphen (--)
  • Must not be one of the following reserved names:
    • Data
    • None
    • False
    • True

Column prefix

The following list describes the naming conventions:
  • Length must not exceed 31 characters.
  • First character must be a letter of the alphabet.
  • Characters must be A-Z, a-z, 0-9, or underscore (_).

Dataflow

The following list describes the naming conventions:
  • The name is case sensitive.
  • Length must not exceed 255 characters.
  • Characters must be A-Z, a-z, 0-9, hyphen (-), or underscore (_).

Dataset

The following list describes the naming conventions:
  • The name is case-sensitive.
  • Length must not exceed 255 characters.
  • First character must be a letter of the alphabet.
  • Characters must be A-Z, a-z, 0-9, hyphen (-), or underscore (_).

Datasets folder

The following list describes the naming conventions:
  • The name is case sensitive.
  • Length must not exceed 255 characters.
  • Characters must be A-Z, a-z, 0-9, hyphen (-), or underscore (_).

Export

The following list describes the naming conventions:
  • The name is case sensitive.
  • Length must not exceed 255 characters.
  • Characters must be A-Z, a-z, 0-9, hyphen (-), or underscore (_).

Parameter

The following list describes the naming conventions:
  • The name is case sensitive.
  • Length must not exceed 255 characters.
  • Characters must be A-Z, a-z, or 0-9.
  • Cannot contain special characters, such as:
    • hyphen (-)
    • left (() or right ()) parenthesis
    • space
    • underscore (_)

SQL code snippet

The following list describes the naming conventions:
  • Length must not exceed 255 characters.
  • First character must be a letter of the alphabet.
  • Characters must be a-z, 0-9, or underscore (_).

SQL function

The following list describes the naming conventions:
  • The name is case-sensitive.
  • Length must not exceed 255 characters.
  • Characters must be A-Z, a-z, 0-9, or underscore (_).

Table

The following list describes the naming conventions:

To avoid any issues from the case sensitivity column naming differences between Xcalar and Xcalar SQL, column names are automatically processed into uppercase characters.

  • Length must not exceed 255 characters.
  • First character must be a letter of the alphabet.
  • Characters must be A-Z, a-z, 0-9, or underscore (_).

User-defined function

The following list describes the naming conventions:
  • Character length:
    • A UDF module name must not exceed 1023 characters.
      IMPORTANT: Must not use the reserved name default.
    • A UDF function name must not exceed 255 characters.
    • A UDF module source code file must not exceed 10485760 characters.
  • First character must be a letter of the alphabet or an underscore.
  • Characters must be a-z, 0-9, or underscore (_).
    • IMPORTANT: Must not use uppercase letters of the alphabet.

User names

The following list describes the conventions for user names:
  • The name is case sensitive.
  • The minimum length is 3 characters.
  • First character must be a letter of the alphabet.
  • Last character must be a letter of the alphabet or a number.
  • Characters must be A-Z, a-z, 0-9, period (.), hyphen (-), or underscore (_).

Workbook

The following list describes the naming conventions:
  • The name is case-sensitive.
  • Length must not exceed 255 characters.
  • First character must be a letter of the alphabet.
  • Characters must be A-Z, a-z, 0-9, hyphen (-), underscore (_), or space.
  • Must not be one of the following reserved names:
    • udf
    • udfs
    • preupgrade
