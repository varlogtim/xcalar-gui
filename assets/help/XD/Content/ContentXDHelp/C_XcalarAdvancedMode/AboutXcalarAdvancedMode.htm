Working in Dataflow Mode

Xcalar Design has two user interface modes - the Dataflow Mode and the SQL Mode. This section describes the difference between the two modes and provides instructions on how to use the Dataflow Mode features. For more information about SQL Mode, see Working in SQL Mode.

Difference between the Dataflow and the SQL Modes

Dataflow Mode SQL Mode
Enables you to work with datasets as well as tables. Enables you to work with tables only.

Provides a visual design interface to design your data model.

Provides a SQL editor to write SQL queries and develop your data model.
Enables you to debug your data model at every step. Enables you to convert your SQL queries into dataflows and debug them in the Dataflow Mode.

The Dataflow Mode is intended for all those users who work with data for analysis. You do not need to be an expert in SQL to work in the Dataflow Mode. The SQL Mode is intended for those users who are experienced with SQL concepts, the SQL language, and applying and running SQL queries on their data for analysis.

Accessing the Dataflow Mode

The SQL Mode is the default mode in Xcalar Design. If after you log in to Xcalar Design the user interface is displaying the SQL Mode, do the following:

From the Xcalar Design menu bar, click the mode switch , which changes the Xcalar Design user interface from SQL Mode to Dataflow Mode and displays the features and functions that enable you to work in Dataflow Mode.

TIP: Xcalar Design retains the last mode that you were working with when you log out. After logging back into Xcalar Design, verify that you are in the correct mode by looking at the mode switch .

To learn more about the Dataflow Mode, see Understanding the Dataflow Mode workspace.
