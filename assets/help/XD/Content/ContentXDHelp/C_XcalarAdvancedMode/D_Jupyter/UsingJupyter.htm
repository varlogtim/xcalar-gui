Using Xcalar Design with Jupyter Notebook

Xcalar Design includes the Jupyter Notebook interactive Python development environment. Jupyter facilitates the creation of user-defined functions (UDFs) by enabling you to write code directly in the browser, to access and interactively work with samples of your modeling data, to run and debug code in a comparable Python environment to your cluster, and to annotate your work.

NOTE: This topic assumes that you are familiar with Jupyter Notebook. For more information about the Jupyter Notebook user interface, such as for running code and generating documents, see https://jupyter-notebook.readthedocs.io.

"The term notebook used in the Xcalar Design online help refers to Jupyter Notebook."

Programming language supported

While Jupyter Notebook can run code that is written in various programming languages, the Jupyter Notebook accessed from Xcalar Design supports only Python 3.6 for helping you write Xcalar UDFs.

Relationship between a Jupyter Notebook and a Xcalar Design workbook

You can create more than one Jupyter Notebook for each workbook. Each notebook is stored in a directory named for your workbook. You can use Xcalar Design to create well-commented Code Snippets that help you write UDFs and work with your data. Remember that the purpose of opening a notebook is to use Python to analyze data from a data source file or a Xcalar result set.

IMPORTANT: Xcalar recommends not running the Connection Code Snippet cell from another user's Jupyter notebook as this may end your Xcalar Design session without an opportunity to save your work.
EXAMPLE: Your current active workbook is named Workbook1. Suppose you created a notebook earlier named MyNotebook2 when the workbook named Workbook2 was active. Using MyNotebook2 for data from Workbook1 might cause errors. To work on data from Workbook1, create a new notebook specifically for Workbook1.

Naming a Jupyter Notebook

After you create a Juypyter Notebook, follow these Jupyter Notebook naming conventions to name the notebook:

  • The name must contain at least one character.
  • The name is case-sensitive.
  • The name does not contain any of the following:
    • colon (:)
    • forward slash (/)
    • backslash (\).

You can publish a result set to a notebook and then you can use Jupyter Notebook for data analysis or visualization for the published rows. For more information, see Publishing a result set to a Jupyter Notebook.

In addition to managing or editing your notebook, you can add the following Code Snippets using the JUPYTER NOTEBOOK window:

  • Connect to Xcalar Workbook
  • Create Map UDF
  • Create Import UDF

Opening a Jupyter Notebook not connected to the active workbook

Opening a Jupyter Notebook not associated with the current workbook results in read-only access. If you download and then upload a Jupyter Notebook, the notebook is editable and Xcalar Design automatically replaces the Connection Code Snippet cell with one appropriate for your workbook.

IMPORTANT: Xcalar recommends not running the Connection Code Snippet cell from another user's Jupyter notebook as this may end your Xcalar Design session without an opportunity to save your work.

Connecting a Jupyter Notebook to an active workbook

After uploading a Jupyter Notebook, you can connect it to an active workbook, regardless of which workbook you used when the notebook was first created.

You can also use the following procedure to re-establish the connection to the active workbook. For example, if you accidentally modified the Xcalar connection code snippet and you want to restore it to its original form, you can complete the procedure for Xcalar Design to re-enter the code in the notebook.

NOTE: You can establish a connection between a Jupyter Notebook and the active workbook. You cannot connect a Jupyter Notebook to any other workbook.

To add a new connection code snippet, follow these steps:

  1. At the top of the JUPYTER NOTEBOOK window, click CODE SNIPPETS, which displays a drop-down menu.

  2. Select Connect to Xcalar Workbook, which creates a new cell beneath the current cell in the notebook. The new cell contains the snippet to connect the notebook to the current workbook.

    IMPORTANT: A Jupyter Notebook with connections to multiple workbooks cause errors. When a Jupyter Notebook has a cell connecting to a workbook other than the current workbook, delete that cell.
  3. Move the new connection snippet cell to the top of your workbook.

Creating a UDF with the Xcalar UDF template

Xcalar Design's integrated Jupyter Notebook provides you with the following user-defined function (UDF) templates in the code snippets list:

  • Create Map UDF, which contains a function definition to modify and instructions on writing and testing a map UDF.
  • Create Import UDF, which contains a function definition to modify and instructions on writing and testing an import UDF for importing a file with custom data source format.

The following section describes how to display a UDF template and modify the code.

NOTE: If you want to create an import UDF, start Jupyter Notebook from the Import Data Source window.

The procedure requires that you assign a name to the module containing the function and a name to the function.

The following list describes the naming conventions:
  • Character length:
    • A UDF module name must not exceed 1023 characters.
      IMPORTANT: Must not use the reserved name default.
    • A UDF function name must not exceed 255 characters.
    • A UDF module source code file must not exceed 10485760 characters.
  • First character must be a letter of the alphabet or an underscore.
  • Characters must be a-z, 0-9, or underscore (_).
    • IMPORTANT: Must not use uppercase letters of the alphabet.

Writing a Map user-defined function

Follow these steps to display a Xcalar UDF template:

  1. Open a notebook and place the insertion point in a notebook cell.

  2. At the top of the JUPYTER NOTEBOOK window, click CODE SNIPPETS, which displays a drop-down menu:

Follow these steps to create a Map UDF:

  1. Click Create Map UDF to display the UDF Template modal window.
  2. In the UDF Template modal window, type the module name and function name.
  3. Select the table/result set and columns on which the UDF will run.
  4. Click Confirm. Jupyter Notebook adds a notebook cell containing specially-generated code:
    • A function definition for you to modify by entering custom code.
    • Test code, which will execute on 100 rows of data from the selected table/result set and columns when you run the cell.

  5. When you are satisfied with the result of your UDF, click Use UDF on table_name. The following list describes the results:

    • Xcalar Design displays the Map panel with the newly created UDF as the selected function.
    • Xcalar Design displays the Result set for which the UDF is written. The columns referenced in the UDF are highlighted.
    • The UDF is saved on the cluster.

    You are ready to run the UDF from the Map panel.

Writing an import parser as a user-defined function

Follow these steps to write an import parser as a user-defined function (UDF):

  1. Start the import process for importing your data for which you want to write the import parser. For more information, see Importing data into Xcalar Design.
  2. Once the Format window is displayed, verify that Custom Format is selected as the data source format. For information about the Format window, see Selecting format options during an import.
  3. In the Format window, click WRITE PARSER USING JUPYTER to display the Create Import UDF modal window.

  4. In the Create Import UDF modal window, follow these steps:

    1. Type the module name.
    2. Type the function name.

    The connector and data source path relative to the connector are automatically populated from your selected data source files.

  5. Click Confirm. Jupyter Notebook displays a notebook cell containing specially-generated code:

    • A function definition for you to modify by entering custom code to convert your data source content and yield Python dictionary format rows.

    • Test code, which will execute on 100 rows of data from the selected data source file(s) when you run the cell.

  6. When you are satisfied with the code, click APPLY UDF in the Jupyter Notebook cell containing the code. The following list describes the results:

    • The Format window is displayed with the UDF module name and function name, and the preview section is updated to show the result of parsing the data source with the newly created UDF.
    • The UDF is saved on the cluster.

    If the following error message is displayed, you did not start Jupyter Notebook from the Format window:

    Please select a dataset to import in the Dataset Panel and select 'Custom Format' as the current format.

    Because the UDF has already been saved, open the Import Data Source window to import the data source for which the UDF was written. In this Format window, select Custom Format as the format, and select and apply the UDF.

    NOTE: If you do not see the expected results in the preview section of the Format window, click WRITE PARSER USING JUPYTER to edit the UDF. After editing the UDF, click APPLY UDF in the Format window.
