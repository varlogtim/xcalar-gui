Using Xcalar Design with Jupyter Notebook

Xcalar Design includes the Jupyter Notebook interactive Python development environment. Jupyter facilitates your creation of user-defined functions (UDFs) by enabling you to write code directly in the browser, to access and interactively work with samples of your modeling data, to run and debug code in a comparable Python environment to your cluster, and to annotate your work.

NOTE: This topic assumes that you are familiar with Jupyter Notebook. For more information about the Jupyter Notebook user interface, such as for running code and generating documents, see https://jupyter-notebook.readthedocs.io.

"The term notebook used in the Xcalar Design online help refers to Jupyter Notebook."

Programming language supported

While Jupyter Notebook can run code that is written in various programming languages, the Jupyter Notebook accessed from Xcalar Design supports only Python 3.6 for helping you write Xcalar user-defined functions (UDF).

Relationship between a notebook and a Xcalar Design workbook

You can create more than one notebook for each workbook. Each notebook is stored in a directory named for your workbook. You can use Xcalar Design to create well-commented Code Snippets that help you write UDFs and work with your data. Remember that the purpose of opening a notebook is to use Python to analyze data from a data source file or a Xcalar result set.

NOTE: Xcalar recommends that you do not run the Connection Code Snippet cell from another user's workbook, as this may end your modeling session without opportunity to save work.
EXAMPLE: Your current active workbook is named Workbook1. Suppose you created a notebook earlier named MyNotebook2 when the workbook named Workbook2 was active. Using MyNotebook2 for data from Workbook1 might cause errors. To work on data from Workbook1, create a new notebook specifically for Workbook1.

Naming a notebook

After you create a notebook, follow these Jupyter Notebook naming conventions to name the notebook:

  • The name must contain at least one character.
  • The name is case-sensitive.
  • The name does not contain any of the following:
    • colon (:)
    • forward slash (/)
    • backslash (\).

Publishing data from a result set to a notebook

You can publish a result set to a notebook and then you can use Jupyter Notebook for data analysis or visualization for the published rows.

RECOMMENDATION: Xcalar recommends that you do not publish more than 1000 rows to a notebook at one time. Publishing more than 100 rows might take a long time.

  1. Click on the Out category of operations in the Advanced Mode toolbar to display the Jupyter Node.

  2. Drag the Jupyter Node to the dataflow canvas.
  3. Connect the result set, you want to publish, to the Jupyter Node.
  4. Right click the Jupyter Node and choose Configure from the drop-down menu. The Jupyter panel is displayed.
  5. Enter the Number of rows you want to publish. The default and the maximum value is 1000.
  6. Rename any column names by clicking on the column name field in Rename columns section.
  7. Click the Save button. The configuration is saved and the result set is generated on execution.
  8. Right click on the Jupyter Node and select View Result from the drop-down to view the notebook.
  9. A Rename modal window is displayed for you to rename the Jupyter notebook name. Enter a name and click the Rename button.
  10. The result set is saved to the Jupyter notebook name you chose.
  11. Depending on whether you have an open notebook, the JUPYTER NOTEBOOK window shows one of the following:
    1. The Jupyter Notebook dashboard with a file tree, which lists the notebooks created for all workbooks. You can manage your notebooks by opening a notebook for editing or uploading a notebook.

    2. The notebook editor with the contents of the open notebook. You can edit the contents as you normally would in Jupyter Notebook.

      IMPORTANT: Do not edit the Xcalar connection code snippet, which is necessary for Xcalar Design to work with the notebook. The code is in the first cell of the notebook. If you accidentally change the code, you can restore the code by following the instructions in Connecting a notebook to an active workbook.

Opening a notebook not connected to the active workbook

Opening a notebook not associated with the current workbook will result in viewing as read-only. If you download and then upload a notebook, the notebook is editable and Xcalar Design will automatically replace the connection code snippet with one appropriate to your workbook.

Important: Do not play the connection code snippet of another user's Jupyter notebook. Such an action may end your Xcalar Design session and log you out, without an opportunity to save your work.

Connecting a notebook to an active workbook

After uploading a notebook, you can connect it to an active workbook, regardless of which workbook you used when the notebook was first created.

You can also use the following procedure to re-establish the connection to the active workbook. For example, if you accidentally modified the Xcalar connection code snippet and you want to restore it to its original form, you can complete the procedure for Xcalar Design to re-enter the code in the notebook.

NOTE: You can establish a connection between a notebook and the active workbook. You cannot connect a notebook to any other workbook.

To add a new connection code snippet, follow these steps:

  1. At the top of the JUPYTER NOTEBOOK window, click CODE SNIPPETS, which displays a drop-down menu.

  2. Select Connect to Xcalar Workbook, which creates a new cell beneath the current cell in the notebook. The new cell contains the snippet to connect the notebook to the current workbook.

    IMPORTANT: If the notebook has another cell with the connection code snippet for a workbook other than the current workbook, best practice is to delete that cell. A notebook with connections to multiple workbooks causes errors.
  3. Move the new connection snippet cell to the top of your workbook.

Creating a UDF with a Xcalar UDF template

Xcalar Design's integrated Jupyter Notebook provides you with the following UDF templates in the code snippets list:

  • Create Map UDF, which contains a function definition to modify and instructions on writing and testing a map UDF.
  • Create Import UDF, which contains a function definition to modify and instructions on writing and testing an import UDF for importing a file with custom data source format.

The following section describes how to display a UDF template and modify the code.

NOTE: If you want to create an import UDF, start Jupyter Notebook from the Import Data Source window.

The procedure requires that you assign a name to the module containing the function and a name to the function.

The following list describes the naming conventions:
  • Characters must be a-z, 0-9, hyphen (-), or underscore (_).
  • Letters must be lowercase.
  • First character must be a letter or an underscore.
  • Character length:
    • UDF module name, the length must not exceed 1023 characters.
      NOTE: The UDF module name default is reserved by Xcalar.
    • UDF function name, the length must not exceed 255 characters.
    • UDF module source code, the length must not exceed 10485760 characters.

Follow these steps to display a Xcalar UDF template:

  1. Open a notebook and place the insertion point in a notebook cell.

  2. At the top of the JUPYTER NOTEBOOK window, click CODE SNIPPETS, which displays a drop-down menu:

Follow these steps to create a Map UDF:

  1. Click Create Map UDF to display the UDF Template modal window.
  2. In the UDF Template modal window, type the module name and function name.
  3. Select the table/result set and columns on which the UDF will run.
  4. Click Confirm. Jupyter Notebook adds a notebook cell containing specially-generated code:
    • A function definition for you to modify by entering custom code.
    • Test code, which will execute on 100 rows of data from the selected table/result set and columns when you run the cell.

  5. When you are satisfied with the result of your UDF, click Use UDF on table_name. The following list describes the results:

    • Xcalar Design displays the Map panel with the newly created UDF as the selected function.
    • Xcalar Design displays the Result set for which the UDF is written. The columns referenced in the UDF are highlighted.
    • The UDF is saved on the cluster.

    You are ready to run the UDF from the Map panel.

Follow these steps to create an import UDF:

  1. In the Import Data Source window, verify that Custom Format is selected as the data source format. For information about the Import Data Source window, see Importing a data source.
  2. In the Import Data Source window, click WRITE PARSER USING JUPYTER to display the Create Import UDF modal window.

  3. In the Create Import UDF modal window, follow these steps:

    1. Type the module name.
    2. Type the function name.

    Data target and data source path relative to the data target are automatically populated from your selected data source files.

  4. Click Confirm. Jupyter Notebook displays a notebook cell containing specially-generated code:

    • A function definition for you to modify by entering custom code to convert your data source content and yield Python dictionary format rows.

    • Test code, which will execute on 100 rows of data from the selected data source file(s) when you run the cell.

  5. When you are satisfied with the code, click APPLY UDF in the Jupyter Notebook cell containing the code. The following list describes the results:

    • The Import Data Source window is displayed with the UDF module name and function name, and the preview section is updated to show the result of parsing the data source with the newly created UDF.
    • The UDF is saved on the cluster.

    If the following error message is displayed, you did not start Jupyter Notebook from the Import Data Source window:

    Please select a dataset to import in the Dataset Panel and select 'Custom Format' as the current format.

    Because the UDF has already been saved, open the Import Data Source window to import the data source for which the UDF was written. In this window, select Custom Format as the format, and select and apply the UDF.

    NOTE: If you do not see the expected results in the preview section of the Import Data Source window, click WRITE PARSER USING JUPYTER to edit the UDF. After editing the UDF, click APPLY UDF in the Import Data Source window.
