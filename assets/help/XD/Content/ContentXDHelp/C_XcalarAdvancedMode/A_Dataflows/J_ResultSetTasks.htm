Performing result set tasks

Xcalar creates a result set after it executes a Node. If you execute an entire dataflow or a group of Nodes, Xcalar creates a result set for each executed Node.

NOTE: When you execute a dataflow with optimization, the execution does not create a result set for each Node. The result of such execution is an optimized dataflow. For information about optimized dataflows, see Understanding dataflow execution.

Understanding the Node status indicator

You can determine whether you can create a result set or view a Node's result set by checking the status of a Node. The following table explains how to interpret the Node status indicator:

Status indicator color

Description

Clear

The Node is not configured. For example, immediately after you created a Node, the indicator is clear. You must configure the Node before it is executable. To execute the entire dataflow, all Nodes must be configured.

Yellow

The Node is configured, but it does not have a result set. After you execute the Node, a result set is created.

Green

The Node was successfully executed. You can view the result set. However, if the result set has been dropped, the indicator remains green, and you must generate the result set to view it again.

Red

The Node is incorrectly configured. For example, if a Node in the middle of a dataflow does not have a link to a parent Node, or if you entered invalid information during Node configuration, the Node's status indicator is red. You might be able to execute a Node with a red status indicator, but the result set is meaningful. For example, the result set might contain no rows.

Viewing a result set

To view a result set, follow these steps:

  1. Right click the Node to display a menu.

  2. Click View Result. The Result Set view appears above the canvas.

    You can view one result set at a time. To select another result set, repeat the steps in this procedure for the desired Node. The result set of the newly selected Node will be displayed.

NOTE: You can configure a dataflow setting to automatically display the result set automatically after execution.

Understanding the Result Set view

The Result Set view consists of data generated from the execution of a Node, which is arranged in a tabular format. The result set does not have a title, but Xcalar Design indicates from which Node the result set originated. The following list describes how you can navigate the Result Set view:

  • If you have a large number of columns and you need to search for a particular one, enter a partial or complete column name in the search bar. The search is case-insensitive. If there is a match, the column header is highlighted. Press Enter to see the next match.
  • You can enter the row number to go to a particular row.
  • You can scroll horizontally and vertically.

By default, the Result Set view contains a special column called DATA. This column enables you to display a modal window named Data Browser. For more information about how to use the Data Browser, see Performing tasks in the Data Browser.

Changing the appearance of the Result Set view

You can change how data is displayed in the result set from the column drop-down list, which appears when you right click the column header. This section describes how you select an option in the list to change the appearance of the result set. Making these changes does not affect the dataflow.

IMPORTANT: The changes are temporary. if you re-execute a Node (for example, after a reconfiguration or reset), the change you made to the result set is not retained. For example, you can hide columns in a result set to reduce clutter; but if you configure and execute the Node again, the columns previously hidden re-appear in the result set.
NOTE: The column drop-down list contains other options that you can use as a shortcut to creating an operation in the dataflow. For example, you can add an operation to split a column. Information about these column operations is provided in other topics.

Minimizing and unminimizing a column

Select Minimize column to make it narrow. To view the contents of the column, mouse over the minimized columh header and then right click to display the drop-down list. Select Unminimize column to revert the column to its original width.

Hiding a column

Select Hide a column to remove the column from the Result Set view. Hiding a column reduces clutter in the display; it does not delete the column. You can always add a hidden column back to the Result Set view. For information about adding a column, see Adding a field to the Result Set view.

Aligning text in a column

Select Text align. Then select Left Align, Right Align, Center Align, or Wrap Text. The Wrap Text option automatically aligns text on the left.

Resizing a column

Use one of these methods to resize a column:

  • Rest the cursor on the boundary of the column header until the cursor changes to a resize cursor. Then drag left or right.

  • Follow these steps to resize a column:

    1. In the column drop-down menu, select Resize.
    2. Select one of the following options:
      • Size to header:  The column width is changed to accommodate the header.
      • Size to contents: The column width is changed to accommodate the widest value in the column
      • Size to fit all: The column width is changed to accommodate both the header and the widest value in the column.
NOTE: You can resize all columns at the same time through the result set menu. For more information about the menu, see Understanding the result set menu.

Sorting a column

Select Sort to sort the values in ascending or descending order. If the column is a string column with numerals in some cells, you can choose to sort the column as numerically.

Using a cell option in the Result Set view

In the Result Set view, you can right click any cell to perform these tasks:

  • Start a filter or exclude operation in the dataflow. For more information about these operations, see Filtering in a result set column.
  • Copy the value in the cell to the clipboard.

Understanding the result set menu

In the Result Set view, Xcalar Design provides a menu with options for these purposes:

  • Exporting data from the result set. For more information about exporting data, see Exporting the result of a dataflow.
  • Creating a cast operation to change the data types of desired columns. For information about changing data types, see Changing the data type of a column.
  • Displaying correlation between numeric columns and displaying common aggregates. For more information about viewing correlation and common aggregates, see Counting occurrences of unique values in a column.
  • Creating a Jupyter Notebook from the result set. For information about Jupyter Notebook, see Using Xcalar Design with Jupyter Notebook.
  • Sorting columns by column name, prefix, or data type.
  • Resizing all columns.
  • Resetting all columns to the original width.
  • Creating an erroneous rows result set or a complement result set.

Dropping a result set

Dropping a result set enables you to free up cluster memory. The following list describes the scenarios where a result set is dropped:

  • When Xcalar detects that memory is low, it automatically drops result sets.
  • You can go to the System > Monitor panel to manually drop a result set to free up memory. For more information about the Monitor, see Viewing the System status.
NOTE: You can re-create the result set after you drop it.

Generating a result set

After a result set is dropped, you can re-create it. To generate a result set, follow these steps:

  1. Right click a Node in the dataflow for which the result set has been dropped.
  2. In the drop-down list, select Generate Result. The result set, which was dropped, is available again.

Locking and unlocking a result set

If you do not want a result set to be dropped, either by Xcalar due to memory pressure or by you, you can lock it. When you remove a Node from the dataflow, however, the result set is dropped even if it is locked.

NOTE: You can still perform tasks described in this topic to manipulate the locked result set. For example, you can use the result set menu to sort or resize columns in a locked result set.

To lock and unlock a result set, whether or not you are currently viewing it, follow these steps:

  1. Right click a Node in the dataflow that has a result set.
  2. In the drop-down list, select Lock Result. A lock icon is applied to the Node.
  3. To unlock a result set, select Unlock Result in the drop-down list.
