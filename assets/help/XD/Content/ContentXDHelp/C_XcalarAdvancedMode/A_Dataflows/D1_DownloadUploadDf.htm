Downloading and uploading a dataflow

You might want to download a dataflow so that you or another user can upload it, either to the same or another Xcalar instance. The following list describes some common scenarios where you want to download a dataflow:

  • Handing off a dataflow to other users so that they can continue building the dataflow or creating a SQL function from the dataflow.
  • Archiving or backing up a dataflow so that you can revert to it if needed.

What you can download

The following list describes what you can download and the results from the download:

  • A dataflow or a SQL function: The dataflow file includes all UDFs used in the dataflow. The file name of the dataflow file ends with .xlrdf.tar.gz.
  • A dataflow image file: A .png file is created, which matches what is displayed on the canvas.
  • Operation statistics: A .json file is created, which contains statistics about all operations in the dataflow.
NOTE: For an optimized dataflow, you can download the image file or operation statistics.

Downloading a dataflow

To download a dataflow:

  1. Click the dataflow icon to display the Dataflows panel.

  2. In the Dataflows panel, locate the name for the dataflow to be downloaded. Then click the download button for the dataflow.

    Alternatively, display the dataflow to be downloaded and right click anywhere in the canvas. When a list is displayed, click Duplicate Dataflow, as shown in the following screenshot:

  3. In the Download Dataflow modal window, follow these steps:

    1. In the Download as field, select the item to download. You can create and download a dataflow file, an image file, or an operation statistics file.
    2. In the Destination file name field, enter the file name for the downloaded file, without the file name extension.
    3. Click Download.

To download an optimized dataflow, follow these steps:

  1. Display the optimized dataflow by viewing the result set of the Node that produces the optimized dataflow.
  2. In the tab for the optimized dataflow, right click anywhere in the canvas to display a list.
  3. From the list, click Download.
  4. In the Download Dataflow modal window, enter the file name for the downloaded file, without the file name extension.
  5. Click Download.

At the location on your computer where your browser saves downloaded files, a file is created.

Uploading a dataflow

Follow these guidelines when you upload a dataflow:

  • If you downloaded a SQL function, the file must be uploaded as a SQL function
  • If you downloaded a dataflow, the file must be uploaded as a dataflow.
  • After you upload a dataflow, it is unshared. To share the uploaded dataflow, place the uploaded dataflow in the Published folder as you normally would when publishing a dataflow.
NOTE: Only dataflow files with the .xlrdf.tar.gz file name extension can be uploaded.

Naming a dataflow

The following list describes the naming conventions:
  • The name is case sensitive.
  • Length must not exceed 255 characters.
  • Characters must be A-Z, a-z, 0-9, hyphen (-), underscore (_), or left or right parenthesis.

Naming a SQL function

The following list describes the naming conventions:
  • Characters must be A-Z, a-z, 0-9, or underscore (_).
  • The name is case-sensitive.
  • Length must not exceed 255 characters.

To upload a dataflow from your computer, follow these steps:

  1. Open the workbook to which you want to upload the dataflow.
  2. Click the Dataflow icon to display the Dataflows panel.

  3. Click the upload dataflow button, as shown in the following screenshot:

  4. In the Upload Dataflow modal window, for the Source File field, click Browse to navigate to the location of the dataflow file on your computer. Then select the dataflow file.

  5. In the Destination dataflow name field, enter the dataflow or SQL function name if you accept the default name, which is the downloaded dataflow file's name.

    NOTE: If the name you entered includes a disallowed character for the dataflow or SQL function, Xcalar automatically removes the character.

  6. If your dataflow uses user-defined fuctions (UDFs), specify whether the UDFs in the dataflow file will overwrite the UDFs with the same name on the destination cluster, whether or not the UDFs are shared. Click the Overwrite UDFs check box if you want overwriting to occur.

    EXAMPLE: Suppose the dataflow downloaded uses a UDF module named test.py in a Map operation. If the cluster to which you upload the dataflow also has a UDF module named test.py, clicking Overwrite UDFs will replace the UDF on the cluster with the one in the uploaded dataflow.

  7. Specify whether datasets will be restored if they do not exist on the cluster. Click the Restore datasets check box if you want the datasets used in the dataflow to be created on the cluster. If you do not click the Restore datasets check box, after the dataflow upload, you must re-create each dataset for the dataflow to be executed.

  8. Click Upload. The dataflow is displayed in the Dataflows panel. If you uploaded a SQL function, it is displayed in the Home / SQL Functions folder.
