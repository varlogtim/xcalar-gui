Performing tasks in the Data Browser

Each result set in the Dataflow Mode contains a default column named DATA. You can move your cursor to any row in this column and double click to display the Data Browser modal. The purpose of the Data Browser is to:

  • Display all the fields of a result set even if they are hidden.
  • Provide the capability to unhide a column of the result set.
  • Provide a search option to search in all the fields of the result set, including the hidden fields.
NOTE: The DATA column might be minimized due to the Minimize column option in the column drop-down list or may not be completely visible due to a setting in General Settings. To view the minimized DATA column, click the minimized DATA column header to display the column drop-down list and select Unminimize column. For more information about general settings, see Configuring general settings.

Contents of the Data Browser

The Data Browser displays all columns in a result set. A column is called a field in the Data Browser. The columns are organized in the following categories:

  • Prefixed fields: If the source Node of your dataflow is a dataset, the fields from the data source have a prefix in their names, which is the dataset name. If the result set is from one or multiple Join operations, multiple sets of prefixed fields from multiple data sources are displayed in the Data Browser. Each set has its own prefix.
  • Derived fields: If the source Node of your dataflow is a table, all fields are the derived fields. A field created by an operation is also a derived field. Derived fields do not have a prefix.

For each field, the value in the row, from which you display the Data Browser, is displayed. For example, in the following screen, the Data Browser for row 1 is displayed.

In addition to the prefixed and the derived fields, the Data Browser also displays the information about the hidden fields. For example, suppose the Carrier field is hidden in the result set, the following information is displayed in the Data Browser:

Fields not present in current row

"Carrier": integer

Clicking on the field name unhides it adds it to the result set.

Searching in the Data Browser

If the Data Browser contains a long list of fields, you can easily locate a particular field by searching for it. Click the Search icon to display the search field and then enter the name of the field. The search is case-insensitive.

Displaying Data Browser fields side-by-side

If the Data Browser contains a long list of fields, it may be hard to see them simultaneously. In such cases, you can display different sections of the list in side-by-side panels and scroll each panel independently. This enables you to view fields simultaneously regardless of the length of the field list. In each panel, you can select to view all fields, only the derived fields, or only the fields with prefixed names. Comparing rows

The Data Browser enables you to compare values in different rows from the same result set. Suppose the Data Browser for row 5 is displayed, you can double click another row in the DATA column (for example, row 10) to bring its data into the Data Browser. Now you can view the values from two rows side-by-side.

To compare rows:

  1. Double click a row in the DATA column to display the values of that row in the Data Browser.
  2. Locate another row that you want to compare. You can scroll to find the row. Double click to select it. The values of the selected row are displayed in the browser.
  3. To see how values differ between the selected rows, click the Compare check box for each field list. The values that are different between the rows are highlighted. The following screen shows the comparison between rows 1 and 2.
NOTE: You can compare data from more than two rows.

Adding a field to the result set view

To add a field currently not displayed in the result set view:

  1. Locate the field in the Data Browser that you want to pull from the browser and add to the result set view.
  2. To add multiple selected fields, go to the next step. To add one or all fields, do one of the following:
    • Click the field name, and the field is immediately added to the result set view.
    • Click the icon for pulling all fields. All missing fields are added to the result set view.

  3. To add multiple selected fields, select Multi Select Mode from the Data Browser's drop-down menu. Then select the fields individually and submit the selection.
