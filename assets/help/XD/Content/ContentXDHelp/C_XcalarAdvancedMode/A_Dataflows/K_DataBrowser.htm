Performing tasks in the Data Browser

Each result set contains a default column named DATA. You can move your cursor to any row in this column and double click to display the Data Browser.

NOTE: The DATA column might be minimized and not completely visible due to a setting in the General Settings window or the Minimize column option in the column drop-down list. For information about the general settings, see XXX. To view the DATA column, click the minimized DATA column header to display the column drop-down list and select Unminimize column.

Contents of the Data Browser

The browser displays all columns in the result set. A column is called a field in the Data Browser. The columns are organized in the following categories:

  • Prefixed fields: If the source node of your dataflow is a dataset, the fields from the data source have a prefix in the name, which is the dataset name. If the result set is from one or multiple Join operations, multiple sets of prefixed fields from multiple data sources are displayed in the Data Browser. Each set has its own prefix.
  • Derived fields: If the source node of your dataflow is a table, all fields are derived fields. If the source node is a dataset, a field created by an operation is a derived field. Derived fields do not have a prefix.

Searching in the Data Browser

If the Data Browser contains a long list of fields, you can easily locate a particular field by searching for it. Click the Search icon to display the search field. Then type the name of the field, as shown in the following screenshot. The search is not case-sensitive.

Specifying the select mode

You can add fields in the Data Browser to the Result Set view as described in Adding a field to the Result Set view. Specify the select mode, which determines how your select fields:

  • Single select mode: If you select this mode, clicking a field name immediately adds the field to the Result Set view.
  • Multi select mode: If you select this mode, you can select multiple fields and then submit your selection.

Sorting fields

By default, the fields in the Data Browser are arranged in the same order as they are in the dataset. You can sort the fields in ascending or descending order. The following screenshot shows the location of the icon for sorting.

Duplicating the list of fields in the browser

Duplicating the list of fields is useful when you have a long list of fields. Because each copy of the list has its own scroll bar, you can scroll each copy independently, such that the fields that interest you appear side-by-side. In each panel, you can choose to view all fields, only the derived fields, or only fields with a particular prefix.

Comparing rows

The Data Browser enables you to compare values in different rows from the same result set. Suppose the Data Browser for row 5 is displayed, you can double click another row in the DATA column (for example, row 10) to bring its data into the Data Browser. Now you can view the values from two rows side-by-side.

To compare rows, follow these steps:

  1. Double click a row in the DATA column to display the values of that row in the Data Browser.
  2. Locate another row that you want to compare. You can scroll to find the row. Double click to select it. The values of the selected row are displayed in the browser.
  3. To see how values differ between the selected rows, click the Compare check box for each field list. The values that are different between the rows are highlighted. The following screen shows the comparison between rows 1 and 2.
NOTE: You can compare data from more than two rows.

Adding a field to the Result Set view

To add a field currently not displayed in the Result Set view, follow these steps:

  1. Locate the field in the Data Browser that you want to pull from the browser and add to the Result Set view.
  2. To add multiple selected fields, go to the next step. To add one or all fields, do one of the following:
    • Click the field name, and the field is immediately added to the Result Set view.
    • Click the icon for pulling all fields. All fields are added to the Result Set view and the existing fields are not duplicated.

  3. To add multiple selected fields, select Multi Select Mode from the Data Browser's drop-down menu. Then select the fields individually and submit the selection. The following screenshot illustrates the steps for adding multiple fields.

  4. If the data type of a field is Array or Object, you can add an element of the array or object to the Result Set view as a column. The following partial screenshot shows the Data Browser containing an array. In this example, a column named votes.useful of the type Integer is created.

    Alternatively, you can select multiple elements and add them to the Result Set view when the Data Browser is in Multi Select mode.

    NOTE: Pulling elements of an array or object to form columns of the result set does not change the dataflow.
