Uploading a dataflow

This topic describes the steps to upload an existing dataflow from your computer to the Xcalar instance. You may need to upload a dataflow in the following scenarios:

  • Another user handed off his dataflow to you:
    • to use it as a template
    • to debug
    • to build additional algorithm on top of it
  • You need to revert back to a previously downloaded dataflow

Follow these guidelines when you upload a dataflow:

  • If you downloaded a SQL function, the file must be uploaded as a SQL function

  • If you downloaded a dataflow, the file must be uploaded as a dataflow

    After you upload a dataflow, it is unshared. To share the uploaded dataflow, place the uploaded dataflow in the Published folder as you normally would when publishing a dataflow.

NOTE: Dataflow files with only the .xlrdf.tar.gz file name extension can be uploaded.

Naming a dataflow

The following list describes the naming conventions:
  • The name is case sensitive.
  • Length must not exceed 255 characters.
  • Characters must be A-Z, a-z, 0-9, hyphen (-), or underscore (_).

Naming a SQL function

The following list describes the naming conventions:
  • The name is case-sensitive.
  • Length must not exceed 255 characters.
  • Characters must be A-Z, a-z, 0-9, or underscore (_).

To upload a dataflow from your computer:

  1. Verify that you are in Dataflow Mode. If not, from the top menu bar of Xcalar Design, click the mode switch to display the Dataflow Mode workspace.
  2. Open the workbook to which you want to upload the dataflow.
  3. From the sidebar, click the Dataflow Panel icon to display the Dataflows panel.

  4. Click the upload dataflow button, as shown in the following screenshot:

  5. In the Upload Dataflow modal window, for the Source File field, click Browse to navigate to the location of the dataflow file on your computer. Then select the dataflow file.

  6. In the Destination dataflow name field, enter the dataflow or the SQL function name. Alternatively, accept the default name, which is the file name of the selected dataflow.

    NOTE: If the name you entered includes a disallowed character for the dataflow or the SQL function, Xcalar automatically removes that character.

  7. If your dataflow uses user-defined fuctions (UDFs), specify whether the UDFs in the dataflow file will overwrite the UDFs with the same name on the destination cluster and whether or not the UDFs are shared. Check the Overwrite UDFs check box if you want overwriting to occur.

    EXAMPLE: Suppose the downloaded dataflow uses a UDF module named test.py in a Map operation. If the cluster to which you upload the dataflow also has a UDF module named test.py, clicking Overwrite UDFs will replace the UDF on the cluster with the one in the uploaded dataflow.

  8. Specify whether datasets will be restored if they do not exist on the cluster. Check the Restore datasets check box if you want the datasets used in the dataflow to be created on the cluster. If you do not check the Restore datasets check box, after the dataflow upload, you must re-create each dataset for the dataflow to be executed.

  9. Click Upload. The dataflow is uploaded and displayed in the Dataflows panel. If you uploaded a SQL function, it is displayed in the Home / SQL Functions folder.
