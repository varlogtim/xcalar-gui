Executing a dataflow

You can execute a partial dataflow or an entire dataflow. In addition, if the dataflow is built for optimized execution, you can execute the dataflow with or without optimization. For the meaning of optimized execution, see Understanding dataflow execution.

Requirements for dataflow execution

For a dataflow to be executed, all Nodes in the dataflow or the selected part of the dataflow must be executable. The following conditions must be met for a Node to be executable:

  • The Node must be configured.

  • The Node must not depend on a Single Value Node that descends from that Node. For example, you cannot execute the dataflow in the following screenshot because the Single Value Node calculates the aggregate value from the result set of the Map Node, which in turn uses the aggregate value in the Map function. If the Single Value Node is linked to Node 4 (Cast Node) instead of the Map Node, you can execute the dataflow.

  • Unless the Node is the source Node (the first Node of a dataflow or sub-dataflow), the Node must have a parent Node. A Node might not have a parent Node, for example, due to a removed link. In addition, all the parent Nodes must be executable.

Executing a partial dataflow

To execute a partial dataflow, follow these steps:

  1. Display the dataflow.

  2. Select multiple Nodes by using the multi-selection method appropriate to your computer or using the selection rectangle to include the Nodes. For more information about making selections, see Multi-selecting Nodes.

    TIP: To execute multiple Nodes on the same branch of a dataflow, you can select only the last Node because all its parent Nodes will also be executed.
  3. Right click anywhere in the canvas (but not on an unselected Node) to display a list.
  4. Click Execute.

Results of executing a dataflow

The following list describes what happens after you successfully execute a dataflow:

  • When the execution is in progress, the percentage of completion is displayed for each Node.
  • After the execution of a Node is finished, a statistics box is displayed above the Node with the following information:
    • Number of rows in the result set (if applicable to the Node)
    • Time taken to complete the execution
    • Skew of the result set (if applicable to the Node).
    NOTE: Executing a Link out Node does not produce a statistics box.

  • If applicable, a result set for each executed Node is available for you to view. If you execute a Single Value Node, you can view the resultant value.

  • All nodes' status indicator turns green. For more information about the status indicator, see Understanding the Node status indicator.

Results of executing a dataflow with optimization

If you execute a Link out Optimized or Export Optimized Node, a new dataflow tab is displayed, showing the optimized dataflow. This dataflow looks similar to the one you executed but might contain different Nodes. The title for this dataflow tab is in italics, indicating that you cannot rename or edit the dataflow in this tab.

In the dataflow from which you start the execution, except for the terminal Node, each Node has a yellow status indicator. This is because an optimized version of the dataflow was executed, and the execution did not step through each Node. There is not a result set associated with each Node. In the optimized dataflow, however, each Node has a green status indicator.

The following list describes what you can do in this dataflow tab:

  • Download an image file for this dataflow. For more information about downloading a dataflow, see Downloading a dataflow.

  • Copy Nodes from this dataflow and paste them to another dataflow.

    Recommendation: If you copy a Dataset Node from an optimized dataflow, you cannot execute it with its configuration or re-configure it in the dataflow where you paste it. Copy and paste only Nodes that are not the Dataset Node from an optimized dataflow.
