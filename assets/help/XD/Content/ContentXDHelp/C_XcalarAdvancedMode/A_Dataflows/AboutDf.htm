Understanding a dataflow

A dataflow is a series of actions performed on data to develop a data model in Dataflow Mode in Xcalar Design. You control the actions in a dataflow by configuring the operations and specifying the order in which Xcalar executes them. After Xcalar executes an operation in the dataflow, you can view the result set produced by that operation. You can edit the dataflow by reconfiguring and reordering the operations, until you arrive at the desired data model.

You can configure dataflow settings that affect how dataflows behave. For more information about dataflow settings, see Configuring dataflow settings.

Understanding the dataflow components

A dataflow consists of operations and links that connect the operations. In Xcalar Design, each operation is shown as a named Node. You create your dataflow by dragging and dropping these Nodes on to the dataflow canvas and linking them. Typically, each dataflow consists of the following three operations:

  1. Operations for getting the data into a dataflow: Each dataflow starts with one of these Nodes that gets the data into the dataflow:
    • A Dataset Node representing a dataset that exists on the Xcalar cluster.
    • A Link In Node representing the result of another dataflow in the same workbook.
    • A Table Node representing a SELECT statement on a published table that exists on the Xcalar cluster.
  2. Operations for creating a data model from imported data: For example, in a dataflow, you can have operations for data preparation, such as type casting and filtering, or aggregate operations, such as Group By.
  3. Operations for outputting the data: The terminating Node of a dataflow can be one of the following:
    • An Export Node writes the dataflow result set to files or objects residing at a destination specified in the export configuration.
    • A Link Out Node saves the dataflow result set under a reference name. You can refer to this Node in a Link In Node in another dataflow and continue designing your data model further.
    • A Publish Table Node publishes the dataflow result to a published table on the Xcalar cluster.
    • A Jupyter Node saves the dataflow result to a DataFrame usable by Jupyter Notebook.

Typical steps for creating and using a dataflow

The following list outlines the typical steps for creating and using a dataflow:

  1. Create a dataflow in the dataflow editor by dragging the operation Nodes to the dataflow canvas and linking them, as described in Creating and editing a dataflow.
  2. Configure each Node in the dataflow, as described in Configuring an operation Node.
  3. Execute selected Nodes or the entire dataflow, as described in Executing a dataflow.
  4. View the result set, as described in Viewing the result set of a Node.
  5. Edit the dataflow if necessary, as described in Editing a dataflow.
  6. Publish the result set, as described in Publishing a result set in Dataflow Mode.

Understanding sub-dataflows

In a dataflow, you can create multiple sub-dataflows. A sub-dataflow looks like a dataflow, but a sub-dataflow does not have a name. The following list provides some examples of when to use sub-dataflows in a dataflow:

  • You create sub-dataflows when you experiment with different operations for developing the data analysis model.
  • You use multiple source Nodes and you plan to join the result sets that originate from the source Nodes.
