Performing tasks on a Node

After you create a dataflow, it is display by links. The first step after creating this dataflow is to configure each Node so that it can be executed. This topic provides the general information required for configuring a Node. For specific information pertaining to various operations, see the topics about the operations. This topic assumes that you understand the steps for editing a dataflow, as described in Creating, editing, and deleting a dataflow.

In addition to configuring a Node, this topic describes the following tasks that you perform on a Node:

  • Execute
  • Reset
  • Copy
  • Cut
  • Add or edit a description
  • View schema
  • Remove

For information about viewing and performing various tasks on result sets, see Performing result set tasks .

For information about creating a custom Node from an existing Node, see Creating and using a Custom Node.

Configuring a Node

You can configure a Node whether or not it is already configured.

To configure a Node:

  1. Skip this step if the Node to be configured is the source Node, which is the first Node of a dataflow.

    Create a link from the Node to the appropriate parent Node. The result set of the parent Node will become the input to the Node.

  2. Right click the Node to display a menu.
  3. Click Configure in the menu to display a panel.
  4. Follow the instructions in the panel to enter the appropriate information. The information required depends on the operation.
  5. Click Save in the panel to save the Node configuration. Depending on your dataflow settings, the Node might be executed automatically. If the Node is not executed automatically, its status indicator becomes yellow, which means that the Node has been configured but not executed.

Executing a Node

After a Node is configured, you can execute it. Similar to executing an entire dataflow, executing a Node is successful only if the following requirements are met:

  • The Node and its parent Nodes are properly configured.
  • Aggregate values that are used in the Node or its parent Nodes can be calculated.

To execute a Node, follow these steps:

  1. Right click the Node to display a menu.

  2. Click Execute in the menu to execute the Node. The following list describes what happens after you successfully execute a Node:

    • The status indicator on the Node and on each parent Node become green.
    • The result sets produced by the Node and its parent Nodes are available for viewing. For information about result set tasks, see Performing result set tasks .
    • If an aggregate value is used in the execution, the Single Value Node's status indicator becomes green.
    • Operation information is displayed above the Node, its parent Nodes, and any Single Value Nodes (if applicable.
    • If the Node is a Link out Optimized Node or Export Optimized Node, an optimized dataflow is created and displayed in a new tab.
NOTE: The dataflow settings controls what you see after a Node or a dataflow is executed. For information about configuring dataflow settings, see Configuring settings for dataflows.
NOTE: If a Node's status indicator is green, you cannot re-execute the Node. You must reset the Node if you want to execute the Node again.

Resetting a Node

Resetting a Node undoes the previous execution, removing the result set. The configuration for the Node remains unchanged.

Because the result set for the reset Node is removed, the subsequent Nodes receive no input from the reset Node. Therefore, when you reset a Node, all subsequent Nodes are also automatically reset.

To reset a Node, follow these steps:

  1. Right click the Node to display a menu.

  2. Click Reset in the menu to reset the Node. The following list describes what happens after you reset the Node:

    • The status indicators for the Node and subsequent Nodes become yellow.
    • The result sets of the Node and subsequent Nodes are removed.
    • The operation information above the Node and each subsequent Node is removed.
    • If the reset Node is a Link out Optimized Node or Export Optimized Node, the optimized dataflow created by the previous execution is removed.

Copying or cutting a Node

You can copy or cut a Node so that you can later paste it, for example, in a different part of the dataflow.

To copy or cut a Node, follow these steps:

  1. Right click the Node to display a menu.

  2. Click Copy or "Cut in the menu Alternatively, you can use the key combination appropriate to your computer for the copy or cut function. For example, you can use Ctrl-C for copy.

    If you cut a Node, the Node and all its links are removed.

The Node you copied or cut is on the clipboard, available for you to paste to the desired location.

NOTE: On the canvas, you can right click anywhere to display a menu where Paste is an option. This option indicates that the clipboard has contents that can be pasted. It does not perform the paste function. To paste the Node, use the keyboard combination appropriate to your computer, such as Ctrl-V.
NOTE: If you copy and paste a Link out Node to the same dataflow, remember to configure either the pasted Node or the original Node so that they have different reference names.

Adding or editing a description

To add or edit a Node description, follow these steps:

  1. Right click the Node to display a menu.
  2. Click Add Description. The Description modal window is displayed.

  3. In the Description modal window , enter a new description or modify an existing description, and then click Save.

    An information icon is displayed on the Node, as shown in the following screenshot. If you hover over the icon, the description is displayed in a tooltip.

Viewing schema

You can view the schema if a result set exists for a Node.

To view the schema:

  1. Right click the Node to display a menu.

  2. Click View Schema. A pop-up is displayed, showing the schema of the result set. Fields that exist are displayed in green; fields that have been removed during the execution are displayed in red.

    You can click a field to see its lineage. All Nodes in the dataflow that contain the field are highlighted. In addition, a tooltip is displayed above the highlighted Node to explain what happened to the field. For example, you can see in which Node the field was created, removed, and so on.

Removing a Node

Removing a Node is similar to cutting a Node. However, the removed Node is not saved to the clipboard.

To remove a Node, follow these steps:

  1. Right click the Node to display a menu.
  2. Click Remove. The Node and its links are removed. The status indicator of each subsequent Node becomes red, because each subsequent Node no longer has a valid configuration.
