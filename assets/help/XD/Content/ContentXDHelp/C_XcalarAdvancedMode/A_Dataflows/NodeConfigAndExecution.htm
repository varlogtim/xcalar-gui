Performing tasks on a Node

This topic provides information for the following general Node tasks:

  • Execute
  • Reset
  • Copy
  • Cut
  • View schema
  • Add or edit a description
  • Remove
  • Exit Configuration

To understand the following Node tasks, check their respective links:

  • Configure - Configuring an operation Node
  • View Result - Understanding the result set view in Dataflow Mode
  • Create Optimized Dataflow - Understanding an optimized dataflow tab
  • Create Custom Node - Modularizing dataflows using a Custom Node
  • Lock Result - Locking and unlocking a result set

For specific information about various Nodes, see Understanding the operation Nodes.

Executing a Node

After a Node is configured, you can execute it. Similar to executing an entire dataflow, executing a Node is successful only if the following requirements are met:

  • The Node and its parent Nodes are properly configured.
  • Aggregate values that are used in the Node or its parent Nodes can be calculated.

To execute a Node:

  1. Right click the Node to display a drop-down.

  2. Click Execute or Create Optimized Dataflow in the drop-down to execute the Node. Create Optimized Dataflow creates and executes an optimized version of the dataflow. For more information about an optimized dataflow, see Understanding a dataflow with optimization. The following list describes what happens after you successfully execute a Node:

    • The status indicator on the Node and on each parent Node becomes green.
    • In case of the execution of an optimized dataflow, the final result is available for viewing. In other cases, the result sets produced by the Node and its parent Nodes are available for viewing. For information about result set window, see Understanding the result set view in Dataflow Mode.
    • If an aggregate value is used in the execution, the Single Value Node's status indicator becomes green.
    • The operation statistics is displayed above the Node, its parent Nodes, and any Single Value Nodes (if applicable).
    • If a Link Out Node or an Export Node is executed as optimized, an optimized dataflow is created and displayed in a new tab.
NOTE: The dataflow settings controls what you see after a Node or a dataflow is executed. For information about configuring dataflow settings, see Configuring dataflow settings.
NOTE: If a Node's status indicator is green, you cannot re-execute the Node. You must reset the Node if you want to execute the Node again.

Resetting a Node

Resetting a Node undoes the previous execution, removing the result set. The configuration of the Node remains unchanged.

Because the result set for the reset Node is removed, the subsequent Nodes receive no input from the reset Node. Therefore, when you reset a Node, all subsequent Nodes are also automatically reset.

To reset a Node:

  1. Right click the Node to display a drop-down.

  2. Click Reset in the drop-down. The following list describes what happens after you reset the Node:

    • The status indicators for the Node and subsequent Nodes become yellow.
    • The result sets of the Node and subsequent Nodes are removed.
    • The operation information above the Node and each subsequent Node is removed.

Copying or cutting a Node

You can copy or cut a Node so that you can paste it in the same or a different dataflow.

To copy or cut a Node:

  1. Right click the Node to display a drop-down.

  2. Click Copy or Cut in the drop-down Alternatively, you can use the key combination appropriate to your computer for the copy or cut function. For example, you can use Ctrl-C for copy.

    If you cut a Node, the Node and all its links are removed.

The Node you copied or cut is on the clipboard, available for you to paste to the desired location using the keyboard combination appropriate to your computer, such as Ctrl-V.

Viewing the schema of a Node

You can view the schema if a result set exists for a Node.

To view the schema:

  1. Right click the Node to display a drop-down.

  2. Click View Schema. A pop-up is displayed, showing the schema of the result set. Fields that are added in this Node, are displayed in green; fields that have been removed during the execution are displayed in red; other fields are displayed in gray.

    You can click a field to see its lineage. All Nodes in the dataflow that contain the field are highlighted. In addition, a tooltip is displayed above the highlighted Node to explain what happened to the field. For example, you can see in which Node the field was created, removed, hidden, and so on.

Adding or editing a Node description

To add or edit a Node description:

  1. Right click the Node to display a drop-down.
  2. Click Add Description.
    The Description modal window is displayed.

  3. In the Description modal window, enter a new description or modify an existing description, and then click Save.

    An information icon is displayed on the Node, as shown in the following screenshot. If you hover over the icon, the description is displayed in a tooltip.

Removing a Node

Removing a Node is similar to cutting a Node. However, the removed Node is not saved to the clipboard.

To remove a Node:

  1. Right click the Node to display a drop-down.
  2. Click Remove.
    The Node and its links are removed. The status indicator of each subsequent Node becomes red, because each subsequent Node no longer has a valid configuration.

Exiting the Node configuration

The Exit configuration option is available when you are configuring a Node. Right click on a Node and select Exit configuration to exit the configuration without saving it. It is equivalent to closing the configuration panel from the x icon on the top right.
