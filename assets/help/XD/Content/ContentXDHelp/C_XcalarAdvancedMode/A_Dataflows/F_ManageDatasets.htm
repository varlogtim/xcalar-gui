Managing datasets

This topic describes the tasks you can perform to manage your datasets.

To manage your datasets, you must be in the Dataflow Mode. If not, from the top menu bar of Xcalar Design, click the mode switch to display the Dataflow Mode user interface.

Understanding dataset names

Xcalar Design provides a default name for each dataset when you import data. The name is based on the name of the source file or directory. You can enter a different name or accept the default. You cannot rename the dataset at a later time. The Dataset names are unique for each user. If you import the same data source twice, the system will add a running number to the dataset name.

Understanding the Datasets panel

From the sidebar, click the Sources icon to display the Datasets panel. The Datasets panel displays all the datasets that you can use. They are:

  • datasets created by you, which are not shared
  • datasets in the folder named Shared, which are created by you or other users and then shared

Acknowledging a dataset

Suppose you are previewing one dataset while another one is being created. After Xcalar finishes creating the other dataset, it lists the newly created dataset in the Datasets panel and displays an icon for the new dataset. The icon has a check mark associated with it, which indicates the addition of the dataset to the cluster. To acknowledge it, click the dataset icon, and the check mark is removed.

Organizing datasets in folders

In the Datasets panel, you can organize datasets by folders. For example, you can create a folder named Finance for datasets that import data sources from the Finance department, another folder named Engineering for datasets that import data sources from the Engineering department, and so on.

Follow these steps to place datasets in folders:

  1. Right click anywhere in the Datasets panel.
  2. Select New Folder in the pop-up menu. A New Folder is created. Alternatively, click the Add New Folder icon on top of the dataset panel and a new folder is created.

  3. Right click on the folder name and select Rename menu to rename the folder name.

    The following list describes the naming conventions:
    • The name is case sensitive.
    • Length must not exceed 255 characters.
    • Characters must be A-Z, a-z, 0-9, hyphen (-), or underscore (_).

    4. Do not specify Shared as the folder name because it is reserved for the folder containing shared datasets.

  4. If you enter a name that already exists, an error message is displayed.

  5. Drag and drop any datasets to the newly created folder.

    NOTE: You can use the drag-and-drop method to move a dataset into a user-created folder but you cannot move a dataset to the Shared folder by dragging and dropping. Also, to move a dataset out of a user-created folder, you must right click the dataset name and then select Move up a level.

Sorting datasets

The Datasets panel displays a SORT BY menu for you to sort the datasets by Name, Type or Size. By default datasets and dataset folders are not sorted in any order.

  • Clicking NAME sorts the datasets and folders alphabetically in the ascending order.
  • Clicking TYPE sorts the datasets alphabetically in the ascending order but all the folders are listed first.
  • Clicking on SIZE sorts the datasets in the increasing order of size.

Alternatively, right click anywhere in the Datasets panel to display a drop-down menu. Select Sort by on the menu to sort by Name, Type, Size, or None.

Understanding shared datasets

By default, all datasets created by a user are private to that user. If dataset sharing is enabled, users can make their datasets available to other users, who are on the same cluster. Shared dataset are automatically moved to the Shared folder.

The dataset sharing function is managed by the Xcalar Administrator from the Administration panel.

  • When enabled, all users can share and unshare their datasets.
  • When disabled, users can no longer share their datasets. Existing shared datasets will remain shared until they are unshared by the user who shared them.
NOTE: Datasets created before Version 1.3.1 are automatically placed in the Shared folder. Xcalar Admin can see all the datasets created by all users whether they are shared or not.

The advantages of sharing datasets:

  • Eliminates the number of datasets created from the same data source connector. For example, after a user creates a dataset from a file containing airlines data , this user can share the dataset so that other users can perform various analytics tasks on the same data.
  • Increases memory. Each dataset consumes memory, therefore sharing datasets increases available memory.
  • Increases productivity. Users do not have to go through the steps of importing if they can use a dataset that has already been created by another user.

Tasks you can perform on a shared dataset

In the Shared folder, whether or not you are the owner of a dataset, you can perform these tasks:

  • Preview a dataset.
  • Create result sets from a dataset.
  • Activate a dataset.
  • Deactivate and Delete a dataset if it is not being used by any other user.
  • Get more information on the dataset.

You can create sub-folders in the Shared folder and move datasets in and out of sub-folders. You can also rename the sub-folders.

Sharing a dataset

To share a dataset:

  1. Open the Datasets panel.
  2. Locate the dataset in the Datasets panel and right click on it.
  3. Select the Share menu from the drop-down. A confirmation message is displayed.
  4. Click Confirm and the dataset is moved into the Shared folder with your user name in parenthesis. This enables other users to know the owner of that dataset in the Shared folder.

Stop sharing a dataset

You can remove a currently shared dataset from the Shared folder if no other user is using it. To unshare a dataset in the Shared folder:

  1. Open the Datasets panel.
  2. Locate the dataset in the Datasets panel and right click on it.
  3. Select the Unshare menu from the drop-down. A confirmation message is displayed.
  4. Click Confirm. The dataset is unshared and moved out of the Shared folder into the Home folder.

You cannot stop sharing a dataset:

  • unless you are the owner of the dataset.
  • if some other user is using it.

Effects of sharing and unsharing a dataset

Suppose you share a dataset and another user (User1) creates a result set from it. After User1 deactivates the workbook containing the result set, the shared dataset is no longer being used by User1. Provided that no other users use that dataset, you can unshare that folder. As a result, no other user but you can use that dataset.

When User1 reactivates the workbook and tries to configure the dataset, an error message is displayed that the dataset does not exist:

User1 has a couple of options to restore the dataset:

  • Right click on the erroneous dataset Node, and select the Restore Dataset menu item from the drop-down to restore the unshared dataset. This creates a new copy of that dataset and only User1 can see it.

  • Alternatively, close this Dataset panel by clicking on the (x) in the top right corner. This displays another Dataset panel with a link to restore the dataset:

    Click on the Restore Dataset field and a new copy of the original dataset is created and only User1 can see it.

Understanding dataset deletion

To delete a dataset, you need to deactivate it first. You can deactivate a dataset (shared or unshared) if it is not being used by any of the dataflows and any of the users. If you try to deactivate a dataset which is in use, you get an error message and the deactivation fails.

Deleting a dataset

To delete a dataset:

  1. In the Datasets panel, locate the dataset.
  2. Right click on the dataset name and select Deactivate dataset menu item from the drop-down. If the dataset was not being used by any workbook and any user, a confirmation message is displayed. Click Confirm and the dataset is deactivated.
  3. Right click on the dataset name again and select the Delete menu item from the drop-down.
  4. When a confirmation message is displayed, click Confirm if you are sure that you no longer need the dataset. The dataset is deleted.

Effects of deleting a dataset

Suppose you share a dataset and another user (User1) creates a result set from it. After User1 deactivates the workbook containing the result set, the shared dataset is no longer being used by User1. Provided that no other users use that dataset, you can delete the dataset. As a result, it is no longer visible to any user.

When User1 reactivates the workbook, an error message is displayed that the dataset does not exist. User1 can right click on the erroneous dataset Node, and select the Restore Dataset menu item from the drop-down to restore the deleted dataset. This creates a new copy of that dataset and only User1 can see it.

Effect of a cluster restart on datasets

After the cluster is restarted, some datasets previously in your list of datasets might not be displayed. Only the datasets used by an active workbook on the cluster are shown immediately. This is true of datasets inside or outside the Shared folder.

A dataset appears in the list again in one of these situations:

  • A user activates a workbook containing a result set or aggregate that depends on the dataset.
  • You activate the workbook that was active when you created the dataset.
EXAMPLE: You create a dataset called airlines, which is used in workbook1, and a dataset called carrier, which is used in workbook2. Both datasets are visible in the list of datasets, regardless of which workbook is active. After a cluster restart, if workbook1 is active, only the dataset named airlines is displayed. To also display the dataset named carrier, activate workbook2.

Activating a dataset

To activate an inactive dataset:

  1. Click on the Datasets icon under Sources icon in the sidebar. The Datasets panel is displayed.
  2. In the Datasets panel, locate the dataset you want to activate.
  3. Right click on the dataset name and select Activate dataset menu item from the drop-down. A confirmation message is displayed. Click Confirm and the dataset is activated.
