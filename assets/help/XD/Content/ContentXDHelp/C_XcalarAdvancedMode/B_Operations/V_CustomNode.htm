Creating and using a Custom Node

Custom Node is a convenient way of modularizing your dataflow. If you have a set of operations in your dataflow that is used at many places, you can create a Custom Node for those operations and share it. Once a Custom Node is shared, it is available in the Advanced Mode toolbar as a compound operator for you to use across all your workbooks. This new Node can be used like any other Node in a dataflow.

NOTE: The shared Custom Node is only visible to the user who created it.

Creating and sharing a Custom Node

Follow these steps to create and share a Custom Node:

  1. Create a dataflow and test it.
  2. Once you are satisfied with the dataflow, select all the Nodes of the dataflow. See Limitations of creating a Custom Node for more information on what can constitute a Custom Node.
  3. Right click on any of the selected Nodes and choose the Create Custom Node menu item from the drop-down.
  4. The selected dataflow Nodes are compacted in a single Custom Node.
  5. Right click on the Custom Node and select Share Custom Node menu item from the drop-down. A new Custom Node appears in the Custom category of the Advanced Mode toolbar.
  6. Drag this Custom Node from the toolbar to any dataflow to use it.

Limitations of creating a Custom Node

There are certain restrictions on what a Custom Node can include:

  1. It cannot include Link in, Export Optimized, Link out, Link out Optimized Nodes.
  2. It cannot have more than one output Nodes.

  3. It cannot include disconnected dataflows. For example, in the following dataflow, you cannot create a Custom Node by selecting all the Nodes except Node3 because that is equivalent to having two disconnected dataflows, one with Node1 and Node2 and the other with Node4 and Node5.

Expanding a Custom Node

If you are using a Custom Node in a dataflow and need to see all the Nodes that constitute that Custom Node:

  1. Right click on the Custom Node in the dataflow.
  2. Select Expand menu item from the drop-down and the Node is expanded in place.
  3. You can click the Undo icon in the top left corner of Xcalar Design to go back to the previous state.

Editing a Custom Node

Follow these steps to edit the configuration of the individual Nodes of a Custom Node in a dataflow:

  1. Right click the Custom Node in a dataflow and select Edit menu item from the drop-down.

  2. A new tab opens up with the expanded Custom Node. Right click on any of the expanded Nodes (except the Nodes named Input and the Output) and configure it again.

    NOTE: You cannot add or remove any Node from the expanded Custom Node.

Renaming a shared Custom Node

When you create or share a Custom Node, system generates a default name for it. You cannot rename a Custom Node in the dataflow but you can rename a shared Custom Node in the toolbar. Follow these steps to rename a shared Custom Node:

  1. Select the Custom Node you want to rename in the Advanced Mode toolbar.

  2. Click on the pencil icon on the left.

  3. A Rename modal window is displayed. Enter the new name and click Save button. The Node is renamed.
  4. Renaming the Custom Node in the toolbar has no effect on already used Custom Nodes in the dataflow.

Deleting a shared Custom Node

Follow these steps to delete a shared Custom Node from the toolbar:

  1. Select the Custom Node you want to delete in the toolbar.

  2. Click on the trash icon on the left.

  3. A DELETE OPERATOR modal window is displayed. Click Confirm and the Node is deleted.
  4. Deleting a Custom Node from the toolbar has no effect on already used Custom Nodes in the dataflow.

Editing a shared Custom Node

A shared Custom Node cannot be edited. You can drag a shared Custom Node to the dataflow canvas and edit it but the editing is not saved back to the shared Custom Node. It stays in the dataflow where you edit it, the shared Custom Node in the toolbar stays unchanged.
