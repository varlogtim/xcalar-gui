Joining result sets

The Join function works in the same way as the SQL join clause. You can join two result sets or join a result set to itself (perform a self join). You cannot use one operation to join more than two result sets.

Joining by one or more columns

By default, the Join function uses one key from each result set for joining. You can select one column from each result set by which the result sets are joined. You can, however, perform a multi-clause join, which is a Join function that uses pairs of join keys.

Follow these steps to join the result sets:

  1. There are two ways of starting a Join operation - starting from a result set or starting from a Join Node.

    1. Starting from a result set: To join by one column, right click the header of a column in one of the result sets. Select Join… from the drop-down menu. The Join panel is displayed and a highlighted Join Node appears in the dataflow canvas. Connect one more result set Node to the Join Node. The Join panel is displayed.

      NOTE: To join by multiple columns, select all the columns in a result set that you want to join. Then right click the header of one of the selected columns to display the column drop-down menu.

    2. Starting from a Join Node:
      • Click on the Join category of operations in the Advanced Mode toolbar to display the 3 types of join Nodes.
        • Join Node allows you to join on any of the 7 supported join types. See Definition of join types for more details.
        • Lookupjoin Node is equivalent to a Left Outer Join.

        • Filterjoin Node is equivalent to a Left Semi Join.

      • For our example, drag the Join Node to the dataflow canvas.
      • Connect two result set Nodes to the Join Node.
      • Right click the Join Node and choose Configure from the drop-down menu. The Join panel is displayed.

  2. Select the type of Join function to perform in the Join panel. By default, Inner Join is selected as the type of Join operation. The following types of Join operations are supported:

    • Inner Join
    • Left Outer Join
    • Right Outer Join
    • Full Outer Join
    • Left Semi Join
    • Left Anti Semi Join

    • Cross Join

      See Definition of join types for more details on joins.

  3. If you started from a result set, the Left Columns field already displays the column name but you can select a different column. Otherwise click on the Left Columns field and select a column name from the drop-down. This is the left result set.

  4. Click on the Right Columns field to select a column from the right result set.

    (Optional) For a multi-clause join, click Add Another Clause to add another pair of columns to join.

    (Optional) Enter a Filter Eval String if you want to filter the result set. You can use any Map function that returns a Boolean value but you cannot use prefixed columns in this eval string. For example, if you only want to see the results where Carrier is UA, enter: eq($derived_carrier_col_name, "UA") where $derived_carrier_col_name is a derived column for Carriers.

    (Advanced Option) Click on Join Options and check Null Safe checkbox if you consider the Join condition satisfied when both values in the left and the right columns are NULL.

  5. Click Next.

    NOTE: For columns to be joined, they must be of the same data type. If the columns in a clause have mismatched data types, an error message is displayed, and you can change one or both columns. After you finish changing the columns, click Next.

  6. In the Column Selector section, select the columns from the left and the right result sets to be included in the resultant result set. Click on a column name in Columns To Keep to remove it from the resultant result set. Click on a column in Columns To Drop to add it back to the resultant result set.

  7. Click Save. The Node configuration is saved and a new result set is displayed when the Node executes.

Improving multi-column join performance

If a result set's skew value is high, you might experience performance degradation when running an operation, especially one that uses multiple keys, such as group by, join, or sort. You can improve performance by selecting the column with the most evenly distributed values as the first key. To see how values are distributed within a column, use Profile... in the column drop-down menu.

Definition of join types

Xcalar supports the following types of joins:

  • Inner Join - The result set of an Inner Join contains all rows from both result sets that have matching values in the joining columns.
  • Left Outer Join - Selecting Left Outer Join is equivalent to starting from a Lookupjoin Node in the Advanced Mode toolbar. The Left Outer Join returns all rows from the left result set and the matched rows from the right result set. Where there is no match in the right result set, the columns return Null values.
  • Right Outer Join - The Right Outer Join returns all rows from the right result set and the matched rows from the left result set. Where there is no match in the left result set, the columns return Null values.
  • Full Outer Join - The Full Outer Join combines the results of both left and right outer joins and returns all (matched or unmatched) rows from the result sets on both sides of the join clause.
  • Left Semi Join - Selecting Left Semi Join is equivalent to starting from a Filterjoin Node in the Advanced Mode toolbar. A Left Semi Join returns only those rows in the left result set that have a matching row in the right result set.
  • Left Anti Semi Join - A Left Anti Semi Join returns only those rows in the left result set that have no matching row in the right result set.
  • Cross Join - The Cross Join returns a result set which is the number of rows in the first result set multiplied by the number of rows in the second result set. This kind of result is called the Cartesian Product.

Using Lookupjoin Node

The Lookupjoin Node is used exactly the same way as the Join Node, described in Joining by one or more columns except that you cannot choose the join type, it is preselected as Left Outer Join.

Using Filterjoin Node

Follow these steps to join the result sets using Filterjoin:

  • Click on the Join category of operations in the Advanced Mode toolbar to display the 3 types of join Nodes.
  • Drag the Filterjoin Node to the dataflow canvas.
  • Connect two result sets to the Filterjoin Node.
  • Right click the Filterjoin Node and choose Configure from the drop-down menu. The Join panel is displayed.
  • Click on the Left Columns field and select a column name from the left result set to join on.

  • Click on the Right Columns field to select a column from the right result set.

    (Optional) For a multi-clause join, click Add Another Clause to add another pair of columns to join.

    (Optional) Enter a Filter Eval String if you want to filter the result set. You can use any Map function that returns a Boolean value but you cannot use prefixed columns in this eval string. For example, if you only want to see the results where Carrier is UA, enter: eq($derived_carrier_col_name, "UA") where $derived_carrier_col_name is a derived column for Carriers.

    (Advanced Option) Click on Join Options and check Null Safe checkbox if you consider the Join condition satisfied when both values in the left and the right columns are NULL.

  • Click Next.
  • Click on a column name in Columns To Keep to remove it from the resultant result set. Click on a column in Columns To Drop to add it back to the resultant result set.
  • Click Save. A new result set is created as a result of joining the result sets.

Renaming columns to avoid column name duplication

Because columns in a result set created by a Join operation come from two result sets, it is possible for these columns to have the same name and hence cause a conflict. If there is potential for a conflict, the Join panel asks you to rename columns by filling out the New Name field. This is true both for prefixed columns and derived columns.

IMPORTANT: Even if you do not rename a column, you must copy the original name to the New Name field. Do not leave the New Name field blank.

Renaming prefixed columns

Fields that exist in different data sources are distinguished from one another by column prefixes. If the result sets being joined have the same prefix (for example, in the case of a self join), and if prefixed columns are in the resultant result set, change the prefix for one result set or both result sets to eliminate the naming conflict.

For example, you can change the prefix for the left result set from airlines1 to Left_airlines1 and the prefix for the right result set from airlines1 to Right_airlines1. The resultant result set will have columns named Left_airlines1::Carrier and Right_airlines::Carrier, which no longer conflict with each other.

The following list describes the naming conventions:
  • The name is case sensitive.
  • Length must not exceed 31 characters.
  • Characters must be A-Z, a-z, 0-9, hyphen (-), or underscore (_).
  • First character must be a letter.

Renaming derived columns

Derived columns do not have a prefix. Therefore, you must change the column name to avoid name duplication:

The following list describes the naming conventions:
  • The name is case sensitive if the column is in a dataset; it is case-insensitive if the column is in a table, in which case all letters are displayed in uppercase.

  • The name must consist of alphanumeric characters, spaces, and special characters except the following characters:

    • caret (^)
    • single quote mark (')
    • double quote mark (")
    • left or right parenthesis (( ))
    • left or right square bracket ([ ])
    • left or right brace ({ })
    • backslash (\)
    • colons (:)
    • comma (,)
    • period (.)
  • The name must not begin or end with a space.
  • The name must begin with an underscore or a letter.
  • Length must not exceed 255 characters.
  • The name must not be DATA, but the same string capitalized differently, such as data or Data, are acceptable.
  • The name must not be one of the following strings, regardless of capitalization:
    • None
    • False
    • True
  • The name must not contain two consecutive hyphens (--).
TIP: If a lot of derived fields need renaming, follow the instructions in Renaming in bulk to rename them quickly.

Renaming in bulk

Xcalar Design provides a convenient way for you to specify multiple new names. The following steps are the same for both prefixes and derived column names:

  1. Click the Rename in bulk icon as shown in the following screenshot to display a pop-up menu:

  2. In the menu, you can click Copy All to copy existing names to the New Name field.

    Alternatively, type a string under Copy & Append. The string is appended to each existing name before the name is copied to the New Name field. For example, you can append -left to all the fields.

    You can enter another string to overwrite what is currently in the New Name field. For example, if you want another string instead of -left to be appended, simply type the new string in the pop-up menu. The change is reflected in the New Name column immediately.
