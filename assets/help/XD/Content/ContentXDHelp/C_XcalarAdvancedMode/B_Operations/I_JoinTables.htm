Joining result sets

The Join function works in the same way as the SQL join clause. You can join two result sets or join a result set to itself (perform a self join). You cannot use one operation to join more than two result sets.

Joining by one or more columns

By default, the Join function uses one key from each result set for joining. You can, however, perform a multi-clause join, which is a Join function that uses multiple pairs of join keys.

To join the result sets:

  1. There are two ways of starting a Join operation - starting from a result set or starting from a Join Node.

    • Starting from a result set: Display the result set that you want to use in a Join operation. Right click the header of a column and select Join. The Join panel is displayed and a Join Node is added in the dataflow canvas. Connect one more result set Node to the Join Node.

      NOTE: To join by multiple columns, select all the columns in a result set that you want to join. Then, right click the header of one of the selected columns and select Join.

    • Starting from a Join Node:
      1. Click on the Join category of operations in the Dataflow Mode toolbar to display the three types of join Nodes:
        • The Join Node enables you to join on any of the seven supported join types. See Definition of join types for more details.
        • The Lookup Join Node is equivalent to a Left Outer Join.

        • The Filter Join Node is equivalent to a Left Semi Join.

      2. For our example, drag the Join Node to the dataflow canvas.
      3. Locate the two Nodes with the result sets that you want to join and connect their outputs to the Join Node.

      4. Right click the Join Node and select Configure.

    The Join panel is displayed.

  2. Select the type of Join function to perform in the Join panel. By default, Inner Join is selected as the type of Join operation. The following Join operations are supported:

    • Inner Join
    • Left Outer Join
    • Right Outer Join
    • Full Outer Join
    • Left Semi Join
    • Left Anti Semi Join

    • Cross Join

    See Definition of join types for more details on joins.

  3. Click on the #1 Input field and select a column name from the drop-down. This is the left input.

  4. Click on the #2 Input field and select a column name from the drop-down. This is the right input.

    (Optional) For a multi-clause join, click Add Another Column to add another pair of columns to join on.

    (Optional) Enter a Filter Eval String if you want to filter the result. You can use any Map function that returns a Boolean value but you cannot use prefixed columns in this eval string. For example, if you only want to see the results where Carrier is UA, enter: eq($derived_carrier_col_name, "UA") where $derived_carrier_col_name is a derived column for Carriers.

    (Advanced Option) Click on Join Options and check Null Safe check box if you consider the Join condition satisfied when both values in the left and the right columns are NULL.

  5. Click Next.

    NOTE: For columns to be joined, they must be of the same data type. If the columns in a clause have mismatched data types, an error message is displayed, and you can change one or both columns. After you finish changing the columns, click Next.
  6. In the Column Selector section, select the columns from the left and the right inputs to be included in the joined result. Click on a column name in Columns To Keep to remove it from the joined result.
  7. Click on a column name in Columns To Drop section to add the dropped columns back to the joined result.

  8. Click Save.

    The Node configuration is saved and a joined result is displayed when the Node executes.

Improving multi-column join performance

If a result set's skew value is high, you might experience performance degradation when running an operation, especially one that uses multiple keys, such as group by, join, or sort. You can improve performance by selecting the column with the most evenly distributed values as the first key. To see how values are distributed within a column, in a result set in Dataflow Mode, right click the column header and select Profile.

Definition of join types

Xcalar supports the following types of joins:

  • Inner Join - The result of an Inner Join contains all rows from both the inputs that have matching values in the joining columns.
  • Left Outer Join - Selecting Left Outer Join is equivalent to starting from a Lookup Join Node in the Dataflow Mode toolbar. The Left Outer Join returns all rows from the left input and the matched rows from the right input. Where there is no match in the right input, the columns return Null values.
  • Right Outer Join - The Right Outer Join returns all rows from the right input and the matched rows from the left input. Where there is no match in the left input, the columns return Null values.
  • Full Outer Join - The Full Outer Join combines the results of both the left and the right outer joins and returns all (matched or unmatched) rows from the inputs on both sides of the join clause.
  • Left Semi Join - Selecting Left Semi Join is equivalent to starting from a Filter Join Node in the Dataflow Mode toolbar. A Left Semi Join returns only those rows in the left input that have a matching row in the right input.
  • Left Anti Semi Join - A Left Anti Semi Join returns only those rows in the left input that have no matching row in the right input.
  • Cross Join - The Cross Join returns a result which is the number of rows in the first input multiplied by the number of rows in the second input. This kind of result is called the Cartesian Product.

Using a Lookup Join Node

The Lookup Join Node is used exactly the same way as the Join Node, described in Joining by one or more columns except that you cannot select the join type, it is preselected as Left Outer Join.

Using a Filter Join Node

To join result sets using the Filter Join Node:

  1. Click on the Join category of operations in the Dataflow Mode toolbar to display the three types of Join Nodes.
  2. Drag the Filter Join Node to the dataflow canvas.
  3. Locate the two Nodes with the result sets that you want to join, and connect their outputs to the Filter Join Node.

  4. Right click the Filter Join Node and select Configure.

    The Join panel is displayed.

  5. Click on the #1 Input field and select a column name from the left input to join on.

  6. Click on the #2 Input field and select a column name from the right input to join on.

    (Optional) For a multi-clause join, click Add Another Column to add another pair of columns to join.

    (Optional) Enter a Filter Eval String if you want to filter the result. You can use any Map function that returns a Boolean value but you cannot use prefixed columns in this eval string. For example, if you only want to see the results where Carrier is UA, enter: eq($derived_carrier_col_name, "UA") where $derived_carrier_col_name is a derived column for Carriers.

    (Advanced Option) Click on Join Options and check Null Safe check box if you consider the Join condition satisfied when both values in the left and the right columns are NULL.

  7. Click Next.
  8. Click on a column name in Columns To Keep section to remove it from the joined result. Click on a column name in Columns To Drop section to add it back to the joined result.

  9. Click Save.

    The Node configuration is saved and generates a joined result when the Node executes.

Renaming columns to avoid the column name duplication

Because columns in a joined result created by a Join operation come from two inputs, it is possible for these columns to have the same names and cause a conflict. If there is a potential for conflict, the Join panel renames the columns of one of the inputs in the New Name fields. This is true both for prefixed columns and derived columns.

Renaming prefixed columns

Fields that exist in different datasets are distinguished from one another by column prefixes. If the inputs being joined have the same prefix (for example, in the case of a self join), and if the prefixed columns are going to be in the joined result, the Join panel renames the prefix for the second input. You can accept the system generated new prefix or change one or both prefixes to eliminate the naming conflict.

For example, you can change the prefix for the left input from airlines1 to Left_airlines1 and the prefix for the right input from airlines1 to Right_airlines1. The joined result will have columns named Left_airlines1::Carrier and Right_airlines::Carrier, which no longer conflict with each other.

The following list describes the naming conventions:
  • Length must not exceed 31 characters.
  • First character must be a letter of the alphabet.
  • Characters must be A-Z, a-z, 0-9, or underscore (_).

Renaming derived columns

If there is a potential for conflict in the derived columns, the Join panel generates new names for the columns of the second input. You can accept the system generated new names or enter new names in the New Name fields to avoid the name duplication:

The following list describes the naming conventions:

To avoid any issues from the case sensitivity column naming differences between Xcalar and Xcalar SQL, column names are automatically processed into uppercase characters.

  • Length must not exceed 255 characters.
  • First character must be a letter of the alphabet or an underscore.
  • First character must not start or end with a space.
  • Characters must be A-Z, a-z, 0-9, or underscore (_).
  • Cannot contain special or punctuation characters, such as:
    • caret (^)
    • single (') or double (") quote marks
    • left (() or right ()) parenthesis
    • left ({) or right (}) brace
    • forward slash (/) or backslash (\)
    • colon (:) or semi-colon (;)
    • comma (,)
    • period (.)
    • single (-) or double hyphen (--)
  • Must not be one of the following reserved names:
    • Data
    • None
    • False
    • True
TIP: If a lot of derived fields need renaming, follow the instructions in Renaming in bulk to rename them quickly.

Renaming in bulk

Xcalar Design provides a convenient way for you to rename multiple columns in bulk. The following steps are same for both the prefixed and the derived column names:

  1. Click the Rename in bulk icon as shown in the following screenshot to display a pop-up:

  2. You have two options in the pop-up:

    • To copy existing names to the New Name fields and then update them one by one, click Copy All.

    • Alternatively, enter a string under Copy & Append. The new string is appended to each existing name and copied to the New Name.

      For example, you can append -left to all the existing names. You can also enter another string to overwrite what is currently in the New Name field. For example, if you want a different string instead of -left to be appended, enter the new string in the Rename in bulk pop-up. The change is reflected in the New Name column immediately.
