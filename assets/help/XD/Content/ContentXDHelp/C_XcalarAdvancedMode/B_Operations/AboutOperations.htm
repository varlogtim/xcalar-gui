Understanding the operation Nodes

This section describes the Dataflow Mode operation Nodes by category. A dataflow consists of operations and links that connect the operations. In Xcalar Design, each operation is shown as a named Node. The operation Nodes are divided into several categories and each category has one or more operation Nodes, except the Custom category in which the Nodes are created by users.

Category In: The Nodes under this category are used for connecting data to a dataflow. It contains three Nodes:

Table 1 In Nodes
Operation Node Description
Dataset

A Dataset Node is one of the starting Nodes of a dataflow. It reads the data from a data source and displays it in a result set. For more information, see Using a dataset in a dataflow.

NOTE: A dataset cannot be used in the SQL Mode.

Table A Table Node is one of the starting Nodes of a dataflow. It reads the data from an already published table and displays it in a result set. For more information, see Using a table in a dataflow.
Link In

A Link In Node is used as the first Node in a dataflow to get the result set from a Link Out Node. The Link In and the Link Out nodes together help in modularizing the dataflows. For more information, see Linking dataflows.

Input

An Input Node is a source Node used in SQL function dataflows. For more information see Understanding Input and Output Nodes in SQL Functions.

Category Out: The Nodes under this category are used to output data from a dataflow. It contains six Nodes:

Table 2 Out Nodes
Operation Node Description
Export

An Export Node is used to save your result set in a file. For more information, see Exporting the result of a dataflow.
Export Optimized An Export Optimized Node is used to save the result set of an optimized execution of the dataflow to a file. You execute a dataflow with optimization if you have finalized your dataflow and you want to execute it in an optimized fashion. For more information, see Exporting the result of a dataflow.
Link Out

A Link Out Node is used as the last Node in a dataflow to feed a Link In Node in another dataflow. The Link In and the Link Out nodes together help in modularizing the dataflows. For more information, see Linking dataflows.
Link Out Optimized A Link Out Optimized Node is used as the last Node in a dataflow to feed a Link In Node in another dataflow. Executing a Link Out Optimized Node executes the dataflow in an optimized fashion. The Link In and the Link Out Optimized nodes together help in modularizing the dataflows. For more information, see Linking dataflows.
Output

An Output Node is a terminal Node used in two places:

  • in SQL function dataflows
  • in previewing a SQL query as a dataflow

For more information see Understanding Input and Output Nodes in SQL Functions.
Jupyter A Jupyter Node is used to publish a result set to a Jupyter notebook. For more information, see Using Xcalar Design with Jupyter Notebook.
Publish Table A Publish Table Node is used to export a result set to a published table. For more information, see Publishing a result set to a table.

Category SQL: This category has only one operation Node.

Table 3 SQL Node

Operation Node

 Description

SQL

A SQL Node is used to execute a SQL query in a dataflow. For more information, see Running a SQL snippet using a SQL Node.

Category Column Ops: The Nodes in this category are used for performing operations on columns. It contains the following six Nodes:

Table 4 Column Ops Nodes
Operation Node Description
Cast

A Cast Node is used for changing the data type of a column. For more information, see Changing the data type of a column.

Split

A Split Node is used for splitting the column strings. For more information, see Splitting a column.
Map

A Map Node is used for performing various operations on the data. For more information, see Using a Map function to create new values .

Round

A Round Node is used for rounding Floats to the specified number of decimal places. For more information, see Rounding a column value.
Row Num

A Row Num Node is used for adding a new column to the result set with increasing unique row numbers. For more information, see Adding row numbers using the Row Num Node.

Project

A Project Node is used for removing columns from a result set. For more information, see Using the Project operation to reduce memory usage.

Category Row Ops: The Nodes in this category are used for performing operations on rows. It contains the following three Nodes:

Table 5 Row Ops Nodes
Operation Node Description
Sort

A Sort Node is used for sorting values in a column. For more information, see Sorting a result set and finding quartiles.

Filter

A Filter Node is used for filtering a result set. For more information, see Filtering in a result set column.
Explode

An Explode Node is used for dividing a column string into multiple substrings. Each substring gets a new row in a new column. For more information, see Using an Explode operation.

Deskew

A Deskew Node is used for redistributing the data evenly across all cluster nodes and potentially reducing the data skew. For more information, see Understanding skew.

Category Join: The Nodes in this category are used for performing join operations on result sets. It contains the following three types of join Nodes:

Table 6 Join Nodes
Operation Node Description

Join

A Join Node works in the same way as the SQL Join clause. For more information, see Joining result sets.
Lookup Join

A Lookup Join Node works in the same way as the SQL Left Outer Join clause. For more information, see Joining result sets.

Filter Join

A Filter Join Node works in the same way as the SQL Left Semi Join clause. For more information, see Joining result sets.

Category Set: The Nodes in this category are used for performing Set related operations on result sets. It contains the following three Nodes:

Table 7 Set Nodes
Operation Node Description
Intersect

An Intersect Node is used to perform the Intersect operation on two or more result sets similar to the Set operator INTERSECT in SQL. For more information, see Performing a Set operation on multiple result sets.

Union

A Union Node is used to perform the Union operation on two or more result sets similar to the Set operator UNION in SQL. For more information, see Performing a Set operation on multiple result sets.
Except

An Except Node is used to perform the Except operation on two or more result sets similar to the Set operator EXCEPT in SQL. For more information, see Performing a Set operation on multiple result sets.

Category Aggregates: The Nodes in this category are used for aggregating functions like average, sum, count etc. It contains the following two Nodes:

Table 8 Aggregate Nodes
Operation Node Description
Single Value

A Single Value Node is used to compute aggregate of a column. For more information, see Determining aggregate values.

Group By

A Group By Node is used for grouping the result sets similar to the Group By clause in SQL. For more information, see Grouping data in a result set.

Category Extensions: The Node in this category is used to apply the Extension code to the result set. It contains only one Node:

Table 9 Extension Nodes
Operation Node Description
Extension

An Extension Node enables you to run the Extension code in your dataflow. For more information, see Using an extension in a dataflow.

Category Custom: This category contains compound-operation Nodes created by users. For more information, see Modularizing dataflows using a Custom Node.

Category System Generated Nodes: This category contains only one Node:

Table 10 System generated Nodes
Operation Node Description
Synthesize

A Synthesize Node is a system generated Node which is created and inserted into a dataflow in the following scenarios:

  • while creating an optimized dataflow
  • while previewing a SQL query as a dataflow

For more information, see Understanding a Synthesize Node.
