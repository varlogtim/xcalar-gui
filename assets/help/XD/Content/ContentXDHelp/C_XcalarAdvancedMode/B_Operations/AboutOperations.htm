Understanding the Node operators

This section describes the Advanced Mode Node operators by category.

A dataflow consists of operations and links that connect the operations. In Xcalar Design, each operation is shown as a named Node. The Node operators are divided into several categories and each category has one or more Node operators, except Custom category in which the Nodes are created by users.

Category In: The operators under this category are used as input to a dataflow. It contains three Node operators:

Table 2 In Nodes
Node Operator Description
Dataset A Dataset Node is one of the starting Nodes of a dataflow. It reads the data from a data source and displays it in a result set. For more information, see Using a Dataset Node.
Table A Table Node is one of the starting Nodes of a dataflow. It reads the data from an already published table and displays it in a result set. For more information, see Using a Table Node.
Link in

A Link in Node is used as the first Node in a dataflow to get a result set from a Link out Node. For more information, see Linking dataflows.

Category Out: The operators under this category output data from a dataflow. It contains six Node operators:

Table 3 Out Nodes
Node Operator Description
Export

An Export Node is used to save your result set outside of Xcalar. For more information, see Exporting the result of a dataflow.
Export Optimized An Export Optimized Node is used to save your result set with optimized dataflow. For more information, see Exporting the result of a dataflow.
Link out

A Link out Node is used as the last Node in a dataflow to feed a Link in Node in another dataflow. For more information, see Linking dataflows.
Link out Optimized A Link out Optimized Node is used as the last Node in a dataflow to feed a Link in Node in another dataflow. For more information, see Linking dataflows.
Jupyter A Jupyter Node is used to publish a result set to a Jupyter notebook. For more information, see Using Xcalar Design with Jupyter Notebook.
Publish IMD A PublishIMD Node is used to export a result set to a Published Table. For more information, see Publishing a result set to a table.

Category SQL: This category has only one Node operator.

Table 4 SQL Node

Node Operator

 Description

SQL

A SQL Node is used to execute SQL queries in a dataflow. For more information, see Working with a SQL Node.

Category Column Ops: The operators under this category perform operations on columns. It contains the following six Node operators:

Table 5 Column Operator Nodes
Node Operator Description
Cast

A Cast Node lets you change the data type of a column. For more information, see Changing the data type of a column.

Split

A Split Node lets you split the column strings. For more information, see Using a Split operation.
Map

A Map Node creates new values to help you derive meaning from your dataset. For more information, see Using the Map function to create new values .

Round

A Round Node rounds a Float number to specified number of decimal places. For more information, see Using a Round operation.
RowNum

A RowNum Node adds a derived column to the result set with unique increasing row numbers. For more information, see Using a RowNum operation.

Project

A Project Node lets you remove columns from a result set. For more information, see Using a Project operation to reduce memory usage.

Category Row Ops: The operators in this category perform row operations. It contains the following three Node operators:

Table 6 Row Operator Nodes
Node Operator Description
Sort

A Sort Node lets you sort values in a column so that values are displayed in ascending or descending order. For more information, see Sorting a column and finding quartiles.

Filter

A Filter Node lets you filter a result set column. For more information, see Filtering in a result set column.
Explode

An Explode Node divides a column string into multiple substrings and each substring gets a new row and a derived column. For more information, see Using an Explode operation.

Category Join: The operators in this category perform join operations on the result sets. It contains the following three types of join Node operators:

Table 7 Join Operator Nodes
Node Operator Description

Join

A Join Node works in the same way as the SQL Join clause. For more information, see Joining result sets.
Lookupjoin

A Lookupjoin Node works in the same way as the SQL Left Outer Join clause. For more information, see Joining result sets.

Filterjoin

A Filterjoin Node works in the same way as the SQL Left Semi Join clause. For more information, see Joining result sets.

Category Set: The operators in this category perform Set related operations on result sets. It contains the following three Node operators:

Table 8 Set Nodes
Node Operator Description
Intersect

An Intersect Node performs the Intersect operation on two or more result sets similar to the Set operator INTERSECT in SQL. For more information, see Performing a Set operation on multiple result sets.

Union

A Union Node performs the Union operation on two or more result sets similar to the Set operator UNION in SQL. For more information, see Performing a Set operation on multiple result sets.
Except

An Except Node performs the Except operation on two or more result sets similar to the Set operator EXCEPT in SQL. For more information, see Performing a Set operation on multiple result sets.

Category Aggregates: The operators in this category use aggregate functions like average, sum, count etc. to compute results. It contains the following two Node operators:

Table 9 Aggregate Nodes
Node Operator Description
Single Value

A Single Value Node lets you compute aggregate value using column values as input. For more information, see Determining aggregate values.

Group By

A Group By Node lets you group result sets similar to the Group By clause in SQL. For more information, see Grouping data in a result set.

Category Extensions: The operator in this category applies Extension code to the data. It contains only one Node operator:

Table 10 Extension Nodes
Node Operator Description
Extension

An Extension Node allows you to run code in your dataflow. For more information, see Managing extensions.

Category Custom: These are user created compound operators. For more information, see Creating and using a Custom Node.
