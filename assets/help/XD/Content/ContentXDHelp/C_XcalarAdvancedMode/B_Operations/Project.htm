Reducing memory usage using a Project operation

The Project operation enables you to keep only the columns of interest in a result set, thus removing the unnecessary columns from the memory. You can also perform a Project operation on any column of a displayed result set to reduce the visual clutter. However, the main purpose of the Project operation is to reduce the memory usage.

Performing Project operation on selected columns

To perform Project operation on selected columns:

  1. Verify that you are in Dataflow Mode. If not, from the top menu bar of Xcalar Design, click the mode switch to display the Dataflow Mode workspace.

  2. From the sidebar, click the Dataflow Panel icon.

    The Dataflow Mode workspace opens.

  3. There are two ways of starting a Project operation:
    • Staring from a result set: Display the result set in which you want to perform the Project operation. Right click a column header in the result set and select Project. A Project Node is added to the dataflow canvas.

    • Starting from a Project Node: Click on the Column Ops category of operations in the Dataflow Mode toolbar. Drag the Project Node to the dataflow canvas. Locate the Node with the result set in which you want to perform the Project operation, and connect its output to the Project Node. Right click the Project Node and select Configure.

    The Project configuration panel is displayed with a list of Derived columns and Prefixed columns. You may not see both the lists if you did not have both column types in your result set.

    NOTE: Prefixed columns are the columns with their data source name prefixed to their names during an import into a dataset. Derived columns are generally produced as a result of an operation.

  4. Select the columns you want to keep in each list. Prefixed columns are considered one single unit and can be selected or deselected as one unit only. You cannot selectively remove some fields in that unit. Derived columns, on the other hand, can be selected individually.

  5. Click Save.

    The configuration is saved and displays a result set when the Node is executed. The columns not selected in the Project panel are no longer displayed in the result set. They are also removed from the Data Browser.

Example of a Project operation on derived fields

Suppose the Data Browser shows 19 fields. 4 of the fields are derived fields (DayOfWeek_integer, DepTime_integer, ArrTime_integer, and DepDelay_integer), which are created as a result of a Map operation. The rest of the 15 fields (airlines::MonthDayYear, airlines::Distance, airlines::DepTime, and so on) are the prefixed fields from the source dataset.

If you perform a Project operation on DayOfWeek_integer and DepTime_integer, the new result set contains only these 2 columns. All fields except for DayOfWeek_integer and DepTime_integer are eliminated. The new result set now consumes much less memory than the result set before the Project operation. For the new result set, the Xcalar cluster no longer needs memory to track all 4 derived fields and all prefixed fields.

Example of a Project operation on prefixed fields

For a Project operation, you cannot select individual entries from a list of prefixed fields. The set of prefixed fields is eliminated or retained as one unit through the Project operation. The prefixed fields consume a fixed amount of memory, which is about the same as that consumed by a single derived field.

Suppose you are only interested in airlines::Distance and airlines::MonthDayYear, you cannot perform Project on these two fields only and discard the other prefixed fields such as airlines::Carrier and airlines::FlightNum etc. Xcalar must keep the entire collection of prefixed fields. After the Project operation, the unselected derived fields no longer exist in the result set or in the Data Browser. The Project operation frees up memory that otherwise would be needed to track the 4 derived fields.
