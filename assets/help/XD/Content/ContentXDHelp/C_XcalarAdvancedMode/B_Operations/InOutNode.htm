Understanding Input and Output Nodes in SQL Functions

The Input and the Output Nodes are special Nodes which are available in the Dataflow Mode in special circumstances. The Input Node is a source Node and the Output Node is a terminal Node.

The Input and the Output Nodes, are used together when you create a SQL function. A SQL function is a series of operations, developed as a dataflow in the Dataflow Mode between one or more Input Nodes and an Output Node. This whole dataflow is plugged into a SQL snippet when you call the dataflow as a function with table names as inputs. For more information on SQL functions, see Understanding Xcalar SQL functions.

The Output Node alone is generated by the system when you preview your SQL query as a dataflow in the SQL Mode.

Using an Input Node in a SQL function dataflow

To use an Input Node in a SQL function dataflow:

  1. Verify that you are in Dataflow Mode. If not, from the top menu bar of Xcalar Design, click the mode switch to display the Dataflow Mode workspace.

  2. From the sidebar, click the Dataflow Panel icon.

    The Dataflow Mode workspace opens.

  3. If you already have the required number of Input Nodes in the dataflow canvas, skip to the next step. Otherwise, click the In category of operations in the Dataflow Mode toolbar to display the Input Node.

    NOTE: The Input Node is visible only if you are in a SQL function dataflow.
  4. Drag the Input Node to the dataflow canvas.

  5. Right click on the Input Node and select Configure.

    The SQL Function Input panel is displayed.

  6. Select a table in the Source Table drop-down.

    The Column Schema of the selected table is displayed in the panel.

    NOTE: The selected table is used for testing your SQL function dataflow. When you call this SQL function from your SQL query, the query will be providing the input to the SQL function and the table selected in this panel will not be used.
  7. To remove a column from the result set, click the X before the column name.

  8. To add a column to the result set, click the Add Column field.

    This adds an empty Column Name and an empty Column Type field to the column schema.

  9. Click the empty Column Name field and select a column from the drop-down.

    The Column Type is automatically filled in based on the selected column name.

  10. (Optional) To remove all columns from the result set, click CLEAR ALL.
  11. (Optional) To display all columns of the selected table, click DETECT.

  12. Click Save.

    The Input Node configuration is saved and displays the new result set on execution.

Using an Output Node in a SQL function dataflow

To use an Output Node in a SQL function dataflow:

  1. Verify that you are in Dataflow Mode. If not, from the top menu bar of Xcalar Design, click the mode switch to display the Dataflow Mode workspace.

  2. From the sidebar, click the Dataflow Panel icon.

    The Dataflow Mode workspace opens.

  3. If you already have an Output Node in the dataflow canvas, skip to the next step. Otherwise, click on the Out category of operations in the Dataflow Mode toolbar to display the Output Node.

    NOTE: The Output Node is visible only if you are in a SQL function dataflow.
  4. Drag the Output Node to the dataflow canvas.
  5. Connect the final result set Node of your SQL function dataflow to the Output Node.

  6. Right click the Output Node and select Configure.

    The SQL Function Output panel is displayed with the column schema of the parent Node's result set.

  7. To remove a column from the result set, click the X before the column name.

  8. To add a column to the result set, click the Add Column field.

    This adds an empty Column Name field and an empty Column Type field to the column schema.

  9. Click on the empty Column Name field and select a column from the drop-down.

    The Column Type is automatically filled in based on the selected column name.

  10. (Optional) To remove all the columns from the result set, click CLEAR ALL.
  11. (Optional) To display all the columns of the selected table, click DETECT.

  12. Click Save.

    The Output Node configuration is saved and displays the new result set on execution.
