Sorting a result set and finding quartiles

Xcalar Design enables you to sort columns in a result set so that the values are displayed in an ascending or descending order.

To sort columns in a result set, verify that you are in Dataflow Mode. If not, from the top menu bar of Xcalar Design, click the mode switch to display the Dataflow Mode workspace. From the sidebar, click the Dataflow Panel icon. The Dataflow Mode workspace opens.

Sorting a result set using the Sort Node

To sort a result set using a Sort Node:

  1. Select the Row Ops category in the Dataflow Mode Toolbar.
  2. Drag the Sort Node to the dataflow canvas.
  3. Locate the Node with the result set in which you want to sort the columns, and connect its output to the Sort Node. See Creating and editing a dataflow for more information on connecting Nodes and creating dataflows.

  4. Right click the Sort Node and select Configure to display the Sort configuration panel.

  5. In the Columns to sort section, click on the Column name field and select a column to sort on.
  6. Select the sort order for the selected column - Ascending or Descending.
  7. (Optional) To add more columns to sort on, click Add Column and select another column and the sort order.
  8. (Optional) To remove a column from the Columns to sort section, hover over the column name and click on the cancel icon (x).

  9. Click Save.

    The Node configuration is saved and a sorted result set is displayed when the Node executes.

    IMPORTANT: Sorting by this method preserves the result of sort operation in the subsequent operations but a Sort Node in the middle of a dataflow is not recommended for performance reasons.

Sorting a result set from the column header

To sort a column in a result set:

  1. Right click the result set column header and select Sort. A pop-up is displayed for you to specify whether the values are sorted in ascending or descending order. Alternatively, click one of the stacked triangles in the column header, which indicates the direction of the sort, as shown in the following screenshot:

  2. If the data type of the column is String and Xcalar Design determines that most of the values in the column are numeric, it displays the Sort Suggestion dialog box. Depending on the desired method of sorting, click Alphabetically or Numerically.

    NOTE: If you select Numerically, a derived column is created based on the original column, with its data type set to Integer or Float, depending on whether Xcalar Design detects a significant number of values as Floats or Integers.

  3. The column header for a sorted column shows the direction of the sort. The following screenshot shows the column header of a column sorted in descending order:

    IMPORTANT: Sorting by this method only affects how the rows are displayed in the user interface. Subsequent operations do not preserve the result of the sort. For example, you can sort on a column named Age to display rows ordered by age. Later if you perform an arithmetic function to sum two other columns, the new result set displayed after the function is in the original, unsorted order. The rows are no longer sorted by Age.

Sorting a result set on multiple columns

To sort a result set on multiple columns, repeat the steps in Sorting a result set from the column header for each column. Alternatively, use the method described in this section, in which you select all the columns before the sort takes place. If you use this method, you are not given the option to sort a String column numerically.

To sort a result set on multiple columns:

  1. Select multiple columns in the result set.

  2. Right click a selected column's header and select Sort -> Advanced.

    The Sort configuration panel is displayed and the selected columns are added in the Columns to sort section automatically in the same order as you selected the columns.

  3. For each column in the Columns to sort section, select the sort order.

  4. Click Save.

    The Node configuration is saved and a new result set is displayed when the Node executes.

The following screenshot is an example showing the column headers after a multi-column sort:

The sort triangle in the header displays a number (for example, 1, 2, and so on) to indicate the sort key order.

NOTE: The column you sort on most recently becomes the primary sort key. For example, after sorting on the DayOfWeek and AirTime columns, if you sort on the Distance column, Xcalar updates the numbers in the column headers: The sort key number for Distance is 1, the sort key number for DayOfWeek is 2, and the sort key number for AirTime is 3.

Improving multi-column sort performance

If a result set's skew value is high, you might experience performance degradation when running an operation, especially one that uses multiple keys, such as group by, join, or sort. You can improve performance by selecting the column with the most evenly distributed values as the first key. To see how values are distributed within a column, in a result set in Dataflow Mode, right click the column header and select Profile.

Finding quartiles in a column

To determine the quartiles in a column of a result set, right click the column header and select Profile. The Profile modal containing the quartile summary section is displayed. If the column is already sorted, the quartiles are displayed automatically. Otherwise, click Generate in the bottom right corner to create the quartile summary.
