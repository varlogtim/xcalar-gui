Understanding skew

The skew is a measure of how imbalanced the data is across the nodes of a cluster. If the data is uniformly distributed across a cluster, the data has no skew.

Xcalar Design shows the distribution of data in terms of a skew value after the execution of each operation. The skew value ranges from 0 to 100. If the skew is 0, the data is perfectly distributed across all cluster nodes. If the skew is 100, all the data is on one cluster node.

Clicking the skew value displays the distribution of data among all cluster nodes.

Although Xcalar automatically distributes the data evenly across all cluster nodes, sometimes, depending on the operation, the results may still show a high skew value. A high skew value slows down the next operation in the dataflow and should be avoided. However, there are scenarios where it is safe to ignore a high skew value. For example, if a table is extremely small, the impact of high skew will also be extremely small.

There are several techniques for reducing the data skew. One of the techniques is to change the key values to redistribute the data in an even manner. This technique is called salting. In Xcalar Design you can use a Map Node to create a salted version of a column by adding a random number to the skewed column values and then use the Deskew Node to reshard the data on the newly created column.

Understanding a Deskew Node

The purpose of the Deskew Node is to redistribute the data evenly across all Xcalar cluster nodes and reduce the data skew. The more evenly the data is distributed, the faster the operations run on a cluster.

Using a Deskew Node

To use a Deskew Node:

  1. Verify that you are in Dataflow Mode. If not, from the top menu bar of Xcalar Design, click the mode switch to display the Dataflow Mode workspace.

  2. From the sidebar, click the Dataflow Panel icon.

    The Dataflow Mode workspace opens.

  3. Click on the Row Ops category of operations in the Dataflow Mode toolbar.
  4. Drag the Deskew Node to the dataflow canvas.

  5. Connect the output of an operation Node with a high skew value to the input of the Deskew Node. See Creating and editing a dataflow for more information on connecting Nodes and creating dataflows.

  6. Right click the Deskew Node and select Configure.

    The Deskew configuration panel is displayed.

  7. In the configuration panel, select a column from the drop-down for resharding the result set. For best results, select a column which has many distinct values (this can be the salted version of the original column as explained in ).

  8. Click Save.

    The Node configuration is saved and a new result set is displayed after the Node executes.
