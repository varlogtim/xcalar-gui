Using a table in a dataflow

A dataflow cannot be executed successfully if it does not start with a source Node. The Table Node is a source Node. If your data is stored in a table, use a Table Node to start building your dataflow.

To use a Table Node in a dataflow:

  1. Verify that you are in Dataflow Mode. If not, from the top menu bar of Xcalar Design, click the mode switch to display the Dataflow Mode workspace.
  2. Click the In category of operations in the Dataflow Mode toolbar to display the Table Node.
  3. Drag the Table Node to the dataflow canvas.

  4. Right click the Table Node and select Configure.

    The Table configuration panel is displayed.

  5. Select a table from the Published Table drop-down.
  6. Select table version section enables you to select the version of the table you want to use. The first version of a table is always 0. The version of the table is incremented by 1 each time there is an update (Insert, Modify, Delete) on the table. Follow these steps to select a version:
    1. Use latest version check box is automatically checked for you to use the latest version of the selected table. Alternatively, you can uncheck this check box and use -1 in the drop-down below it. The value -1 always points to the latest version.
    2. If you want to use an older version of the table, you can uncheck the Use latest version check box and select a version in the drop-down below it.

  7. (Optional) If you want to filter the table data, enter a Filter function in the Filter String. For example, to fetch only those rows that have carrier as UA from the airlines table, enter:

    eq(airlines::Carrier, "UA")

    For more information on a list of functions, see Conditional functions.

  8. Click Next.

    The Column Schema of the selected table is displayed.

  9. (Optional) If you do not want to read all the columns from the table, click X before a column name to remove it from the Column Schema.
  10. (Optional) If you want to add a column back to the Column Schema, click Add Column. This adds an empty Column Name field and a Column Type field to the Column Schema. Click on the empty column name field and select a column from the drop-down. Column Type is automatically filled in based on the column name you select.

  11. Click Save.

    The Table Node configuration is saved and displays the new result set when the Node executes. If you added a column but did not select the column name, an error message is displayed. You can either select a column name for the empty column or remove the column and click Save.
