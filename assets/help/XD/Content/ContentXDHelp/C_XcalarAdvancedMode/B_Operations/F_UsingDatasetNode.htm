Using a dataset in a dataflow

To read your raw data into Xcalar, you can import it into a table or into a dataset. If you do not know the format of the raw data, you must import it into a dataset and then extract the parts you want for your data modeling.

Once you have a dataset, start building a dataflow with a Dataset Node:

  1. Verify that you are in Dataflow Mode. If not, from the top menu bar of Xcalar Design, click the mode switch to display the Dataflow Mode workspace.
  2. Click the In category of operations in the Dataflow Mode toolbar to display the Dataset Node.
  3. Drag the Dataset Node to the dataflow canvas.

  4. Right click the Dataset Node and select Configure.

    The Dataset configuration panel is displayed with a list of datasets. You can sort the datasets by Name, Size and Type. By default the datasets are not sorted in any order. You can also preview an active dataset by clicking the eye icon in front of the dataset name.

  5. Select a dataset in the list to use in your dataflow.

    NOTE: If the desired dataset does not appear in the list, import the dataset first. For more information on importing a dataset, see Importing a data source. If the dataset appears inactive, activate the dataset by clicking the Activate link.
  6. (Optional) The Prefix field is filled in based on the dataset name you select. If you prefer a different prefix for the columns in the result set, enter a different text in this field.

  7. Click Save.

    The configuration is saved and generates a result set on execution.

Changing the source of a Dataset Node

If you want to change the dataset used in your Dataset Node:

  1. Right click on the Dataset Node in the dataflow and select Configure.

    The Dataset configuration panel is displayed with a list of datasets.

  2. Select a different dataset from the list.

  3. Click Save.

    If the Dataset Node is not connected to any Node, the new configuration is saved and generates a result set on execution.

    If the Dataset Node is connected to another Node, another Dataset panel is displayed for you to map the previous dataset columns to the new dataset columns. This mapping is needed so that the connected Node understands the new dataset. The connected Node's status may turn red if you don't do this mapping. Follow these steps to do the mapping:

    1. Click on each of the Previous Columns field and select a column from the drop-down to map to the New Name. Leave it blank if there is no column in the Previous Columns to map to the New Name.

    2. Click Save.

      The configuration is saved in the Dataset Node and generates a result set on execution.
