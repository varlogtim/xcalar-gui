Sorting a column and finding quartiles

You can sort values in a column so that values are displayed in ascending or descending order.

Sorting values in a column

Follow these steps to sort a column:

  1. In the column drop-down menu, select Sort. A pop-up is displayed for you to specify whether the values are sorted in ascending or descending order. Alternatively, click one of the triangles, in the column header, which indicates the direction of the sort, as shown in the following screenshot:

  2. If the data type of the column is String and Xcalar Design determines that most of the values in the column are numeric, it displays the Sort Suggestion dialog box. Depending on the desired method of sorting, click Alphabetically or Numerically.

    NOTE: If you select Numerically, a derived column is created based on the original column, with its data type set to Integer or Float, depending on whether Xcalar Design detects a significant number of values in floating point format.

  3. The column header for a sorted column shows the direction of the sort. The following screenshot shows the column header of a column sorted in descending order:

    IMPORTANT: Sorting by this method only affects how the rows are displayed. Subsequent operations do not preserve the result of the sort. For example, you can sort on a column named Age to display rows ordered by age. Later if you perform an arithmetic function to sum two other columns, the resultant result set displayed after the function is in the original, unsorted order. The rows are no longer sorted by Age.

Sorting a result set starting from a dataflow Node

You can sort a result set starting from a dataflow Node. Follow these steps to sort a result set starting from a Sort Node:

  1. Select the Row Ops category in the Advanced Mode Toolbar, to display the Sort Node.
  2. Drag the Sort Node to the dataflow canvas.
  3. Connect an existing result set Node to the Sort Node. See Creating, editing, and deleting a dataflow for more information on connecting Nodes and creating dataflows.

  4. Right click on the Sort Node and click on the Configure menu to display the Sort panel.

  5. Click on the Column name field in the Columns to sort section to display the columns available for sort in a drop-down. Select a column.
  6. Choose the sort order for the column - Ascending or Descending.
  7. Click on the Add Column button to add more columns to sort on. Select the sort order for each column.
  8. Click on the cancel icon in the column name to remove the column from the Columns to sort section.

  9. Click Save. The Node configuration is saved and a new result set is displayed when the Node executes.

    IMPORTANT: Sorting by this method preserves the result of sort operation in the subsequent operations but a Sort Node in the middle of a dataflow it is not recommended for performance reasons.

Sorting a result set on multiple columns

To sort a result set on multiple columns, repeat the steps in Sorting values in a column for each column. Alternatively, use the method described in this section, in which you select all the columns before the sort takes place. If you use this method, you are not given the option to sort a String column numerically.

Follow these steps to sort a result set on multiple columns:

  1. To sort a particular column first, display the column options from that column. Then select Sort to display the pop-up.
  2. Click Advanced … in the pop-up to display the Sort panel. The column name is displayed in the Sort panel.

  3. Select another column (by clicking anywhere in the column) as the second column to be sorted. Repeat this step for each additional column that you want to sort.

    To deselect a column, click the column again.

    The column names are listed in the Sort panel in the same order as you select the columns. Also, each column header is associated with a number (for example, 1, 2, and so on), indicating the order in which the columns will be sorted.

  4. In the Sort panel, for each selected column, specify whether the sort is in ascending or descending order.
  5. Click Save. The Node configuration is saved and a new result set is displayed when the Node executes.

The following screenshot is an example showing the column headers after a multi-column sort:

NOTE: The column you sort on most recently becomes the primary sort key. For example, after sorting on the DayOfWeek and AirTime columns, if you sort on the Distance column, Xcalar updates the numbers in the column headers: The sort key number for Distance is 1, the sort key number for DayOfWeek is 2, and the sort key number for AirTime is 3.

Improving multi-column sort performance

If a result set's skew value is high, you might experience performance degradation when running an operation, especially one that uses multiple keys, such as group by, join, or sort. You can improve performance by selecting the column with the most evenly distributed values as the first key. To see how values are distributed within a column, use Profile... in the column drop-down menu.

Using a single sort key again after sorting with multiple keys

Suppose your result set is currently sorted on two columns. If you want to sort the result set only on one of these columns instead, sort the column again in the same direction as you did.

EXAMPLE: if the result set is currently sorted on the DayOfWeek and AirTime columns, but you want to change the result set such that it is sorted on DayOfWeek only, you can sort DayOfWeek in the same direction (for example, in ascending order) as before. Doing so keeps the DayOfWeek column sorted but removes AirTime as a sort key. The values in AirTime are no longer unsorted.

Finding quartiles in a column

Follow these steps to determine the quartiles in a column:

  1. Right click the column header to display the column drop-down menu.

  2. Click Profile… in the drop-down menu. The Profile modal window containing the quartile summary section is displayed. If the column is already sorted, the quartiles are displayed automatically. Otherwise, click Generate in the bottom right corner to create the quartile summary. The following screenshot shows an example of the quartile summary.
