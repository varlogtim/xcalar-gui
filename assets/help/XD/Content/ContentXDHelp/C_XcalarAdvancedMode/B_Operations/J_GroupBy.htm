Grouping data in a table

You can use the Group By function in a similar way as you would with the Group By clause in SQL.

Understanding the Group By operation

Before starting the Group By operation, determine what aggregate functions to use for the calculation. For example, when you use a table containing airlines information, you might be interested in the average arrival delay time and the sum of distance. You can select more than one function on which Xcalar performs the Group By operation.

You also select the columns to group on. For example, from the airlines table, you might want to see the arrival delay time and the sum of distance, which are grouped by the day of the week and the carrier name. In the resultant table, you can see the aggregate function results (average arrival delay and total distance) for each column that is grouped on. For example, one row of the resultant table might show that on day 1, the total distance flown by the carrier named XE is 37845 with an average arrival delay of 58.2.

Aggregate functions supported

The following aggregate functions are supported:

  • Average (avg)
  • Count
  • List aggregate (listAgg)
  • Maximum (max)
  • Maximum float (maxFloat)
  • Maximum integer (maxInteger)
  • Maximum string (maxString)
  • Minimum (min)
  • Minimum float (minFloat)
  • Minimum integer (minInteger)
  • Minimum string (minString)
  • Sum
  • Sum integer (sumInteger)

Using aggregate functions on mixed columns

If a column's data type is mixed, the result of an aggregate function is undefined. Cast a mixed column to the desired data type before you run an aggregate function on it.

Understanding the listAgg (list aggregate) function

The listAgg function is a string function. It concatenates all strings in a column. The maximum number of characters in a result is 16,000. If the result exceeds this limit, the following error message is displayed:

Error: Value too large for defined data type

This function ignores null strings. For example, if a null string is concatenated with the string ABC, the result is ABC.

Grouping data by columns

Follow these steps to group data by one or multiple columns:

  1. Right click the header of the column that you want to group by to display the drop-down menu. Then select Group By….

    The following lists describe the rules and tips for entering information in a Xcalar Design panel.

    For selecting columns:

    • By default, the name of the column from which you start the function is displayed as an argument. You can select another column.
    • If necessary, click in the panel to bring the specified table into focus. You can then scroll the table to select the desired column.
    • If the column you want to select is not in the table, go to the Data Browser to pull the column. Then you can select the column as input to the function.

    For typing a column name:

    • Always start with the dollar sign ($).
    • Always include the prefix if one exists. (A derived column does not have a column name prefix.)
    • To specify the resultant column name, type the name without preceding it with the dollar sign ($).
    • To avoid mistakes when typing a column name, place the insertion point in the argument field, move your cursor to the active table and click anywhere in the desired column. The column's name is automatically entered for you.
    • If an operand accepts a column name, it also accepts the aggregate name that you created when you calculated the aggregate value for a column.

    For typing an operand value:

    • In most cases, do not surround the value with quote marks, even if it is a string with a space. In cases where the function can evaluate the value as a string or numeric value, use the quote marks. You can check the syntax displayed in the panel. The syntax displays the quote marks for an operand if the operand is to be interpreted as a string.
    • If the string contains the backslash (\) or double quote mark ("), escape the character with a backslash (\). Not doing so causes an error message, indicating that the input string is a malformed evaluation string.
    • The $ character has special meaning in Xcalar Design: It denotes a column name. If your input string starts with $, escape it with a backslash (\).
    • The caret (^) has special meaning in Xcalar Design: It denotes a named aggregate value. If your input string starts with a caret, escape it with a backslash (\).

    In the panel, you can undo any typing with the undo key or key combination that you normally use in your browser.

  2. If the column displayed for Fields to group on is not the desired column, select another column.

    NOTE: You can leave Fields to group on empty and click the None check box instead. The purpose of not grouping on a specified field is to create a table with aggregate values, as described in Creating a table for listing aggregate values.
  3. (Optional) In the GROUP BY panel, click ADD ANOTHER COLUMN to group on an additional column. For example, in the table for airlines, you can start the Group By operation from the Carrier column and add the DayOfWeek column as a second column to group with. In the resultant table, data is grouped by both the Carrier and DayOfWeek columns.

  4. In the GROUP BY panel, follow these steps to create a column based on the selected aggregate function:

    1. Select the aggregate function.

      The following functions are supported:
      • avg
      • count
      • listAgg
      • max
      • maxInteger
      • min
      • mininteger
      • sum
      • sumInteger

    2. (Optional) If you want to apply the aggregate function to distinct values only, click the Distinct check box. The operation will eliminate duplicate values in the column when calculating the aggregate value.

      EXAMPLE: With Distinct selected, if you apply the sum function to a column with two rows, each containing the number 3. The result is 3 because only one occurrence of 3 is used when the sum is calculated.
    3. Enter the field name to which the function applies. For example, in the table for airlines information, specify the Distance field to which you apply the avg function. This function calculates the average distance.
  5. Accept the default name for the new column or enter a new name.

  6. (Optional) If you want to apply another aggregate function to the Group By operation, follow these steps:

    1. Click that precedes Apply Additional Functions.
    2. Specify the function name and the column to which the function applies.
    3. Accept the default name for the new column or enter a new name.

    Repeat this step for each additional function.

  7. In the New table name field, enter the name of the resultant table.

  8. (Optional) Click Advanced Options to further customize the result of the Group By operation. The following table describes the options:

    Advanced option Description
    Include only erroneous rows This option filters only rows where the function cannot produce a value.
    Keep original table This option keeps the original table as an active table. By default, the original table is sent to the temporary tables list.
    Join table back to original This option joins the resultant table to the original table. The operation is a left outer join, with the original table being the left table. Joining back the table provides a convenient way to compare the values in the column that results from the Group By operation to values in the original column.
    Keep some columns

    By default, the resultant table contains only the new column created from the Group By operation and the columns on which you group. All columns in the resultant table, including the columns on which you group, become derived columns. The Data Browser for the resultant table contains only the derived fields.

    This option enables you to select additional columns to include in the resultant table. The column created by the Group By function is a derived column. All prefixed columns retain the column prefix.

    NOTE: This option is not applicable if you select Join table back to original because the table created by the Join operation automatically includes all the columns from the original table.
  9. Click GROUP BY.

Creating a table for listing aggregate values

You can use the GROUP BY panel to create a table listing aggregate values. For example, from a table containing airlines data, you might want to create another table listing the average air time, the maximum departure delay, and the count of unique carriers. To create a table containing aggregate values calculated on selected columns, follow the procedure described in Grouping data by columns. Instead of specifying a field to group on, click the None check box in the GROUP BY pane.

Improving multi-column group by performance

If a table's skew value is high, you might experience performance degradation when running an operation, especially one that uses multiple keys, such as group by, join, or sort. You can improve performance by selecting the column with the most evenly distributed values as the first key. To see how values are distributed within a column, use Profile... in the column drop-down menu.
