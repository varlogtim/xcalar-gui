Filtering in a result set column

In Dataflow Mode, you can filter a result set column in one of the two ways:

  • Select values in a result set column and use an option from the drop-down to include or exclude them from the result set.
  • Use a column drop-down on a column header to specify conditions for filtering values in the column.

You can filter any value, including NULL.

TIP: Before filtering a result set, you might want to find the distribution of values through the Profile operation. See Using Profile to find the distribution of values in a column for information about counting the number of times a value appears in a column. You can also use the Profile modal for filtering values as described in Filtering by using the Profile window.

Verify that you are in Dataflow Mode. If not, from the top menu bar of Xcalar Design, click the mode switch to display the Dataflow Mode workspace.

Filtering based on a value selectable in a column

If you want to filter a result set based on one or multiple values in a column:

  1. Right click a cell containing a value that you want to filter or exclude.

    The cell options are displayed. To select multiple values, use your computer's key combination (for example, Ctrl-Shift) when clicking additional cells. The value you select can be in any row. For example, to filter the value 10, click any cell containing 10 in the column. The result is same regardless of the row number.

  2. To include the selected value, click Filter this value; to exclude the selected values, click Exclude this value. If multiple values are selected, the options are Filter these values and Exclude these values.

Filtering based on a condition

You can filter a result set based on a condition. For example, for a selected column, you can filter depending on whether the value string contains a particular substring. If the result of the filter function is true, the value is kept in the column. If the result of the filter function is false, the value is excluded from the column.

NOTE: If you select a filter function not applicable to the data type of the column, an error message is displayed. Use the Cast field to change the type so that the function can run. This data type change is temporary. The column retains the original data type after the filter function.

You can also filter based on multiple conditions. For information about filtering on multiple conditions, see Filtering based on multiple conditions.

To filter data based on a condition:

  1. There are two ways of starting the filtering process - starting from a column header and starting from a Filter Node.
    • Starting from a column header: Display the result set on which you want to perform the Filter operation. Right click the header of the column containing the value you want to filter, and select Filter.

    • Starting from a Filter Node: Click the Row Ops category in the Dataflow Mode Toolbar, to display the Filter Node. Drag the Filter Node to the dataflow canvas. Locate the Node with the result set on which you want to perform the Filter operation, and connect its output to the Filter Node. Right click the Filter Node and select Configure.

      NOTE: See Creating and editing a dataflow for more information on connecting Nodes and creating dataflows.

    The Filter configuration panel is displayed.

  2. Select a Filter Function. For example, for an Integer or Float column, you can select the between function to filter the values that exist between two values.

    The following functions are supported:
    • and
    • between
    • contains
    • endsWith
    • eq
    • eqNonNull
    • exists
    • ge
    • gt
    • isBoolean
    • isFloat
    • isInf
    • isInteger
    • isNull
    • isNumeric
    • isString
    • le
    • like
    • lt
    • neq
    • not
    • or
    • regex
    • startsWith

  3. Based on the description of the function, fill out the other fields.

    NOTE: These functions work in the same way as the Map functions in the conditional category. For information about these Map functions, see Conditional functions.
    4. The following lists describe the rules and tips for entering information in a Xcalar Design panel.

    For selecting columns:

    • By default, the name of the column from which you start the function is displayed as an argument. You can select another column.
    • If the column you want to select is not in the result set, go to the Data Browser to pull the column. Then you can select the column as input to the function.

    For typing a column name:

    • Always start with the dollar sign ($).
    • Always include the prefix if one exists. A derived column does not have a column name prefix.
    • To specify the resultant column name, type the name without preceding it with the dollar sign ($).
    • To avoid mistakes when typing a column name, click on the argument field, move your cursor to the active result set and click anywhere in the desired column. The column's name is automatically entered for you.
    • If an operand accepts a column name, it also accepts the aggregate name that you created when you calculated the aggregate value for a column.

    For typing an operand value:

    • In most cases, do not surround the value with quote marks, even if it is a string with a space. In cases where the function can evaluate the value as a string or numeric value, use the quote marks. You can check the syntax displayed in the panel. The syntax displays the quote marks for an operand if the operand is to be interpreted as a string.
    • If the string contains the backslash (\) or double quote mark ("), escape the character with a backslash (\). Not doing so causes an error message, indicating that the input string is a malformed evaluation string.
    • The $ character has special meaning in Xcalar Design. It denotes a column name. If your input string starts with $, escape it with a backslash (\).
    • The caret (^) has special meaning in Xcalar Design. It denotes a named aggregate value. If your input string starts with a caret, escape it with a backslash (\).

    • Boolean values are case-insensitive. For example, in an input field that accepts a Boolean value, you can type true, True, false, or False.

    In the panel, you can undo any typing with the undo key or key combination that you normally use in your browser.

  4. If the function is for comparing strings, you can specify whether the string comparison is case sensitive. By default, the comparison is case sensitive.

  5. Click Save.

    The Node configuration is saved and a new result set is displayed when the Node executes.

Filtering based on multiple conditions

You can specify multiple conditions for filtering and select how to combine the results as described below:

  • The and operator means that all specified conditions must be true for a row to be included in the new result set.
  • The or operator means that at least one of the specified conditions must be true for a row to be included in the new result set.

To filter based on multiple conditions:

  1. Follow the instructions in Filtering based on a condition to specify the first condition.
  2. Click Additional Conditions.
  3. Select a filter function and fill out the fields as you do for the first condition.
  4. Repeat steps 2 and 3 for each additional condition.
  5. Select the operator (or, and) to connect the conditions.

  6. Click Save.

    The Node configuration is saved and a new result set is displayed when the Node executes.
