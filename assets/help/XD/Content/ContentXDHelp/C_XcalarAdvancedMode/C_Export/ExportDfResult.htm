Exporting the result of a dataflow

The Export feature of Xcalar Design lets you export your result set to a file so that it can be used outside Xcalar.

NOTE: Columns of Arrays or Objects cannot be exported.

To export a result set from Dataflow Mode:

  1. Verify that you are in Dataflow Mode. If not, from the top menu bar of Xcalar Design, click the mode switch to display the Dataflow Mode workspace.

  2. From the sidebar, click the Dataflow Panel icon.

    The Dataflow Mode workspace opens.

  3. There are 2 ways to start the export process:

    1. Starting from a result set:
      1. Click on the Result Set view menu and select Export.
      2. An Export Node is added and connected to the result set Node in the dataflow editor.
    2.  Starting from the Dataflow Mode toolbar:
      1. Click on the Out category of operations in the Dataflow Mode toolbar.
      2. Drag the Export Node to the dataflow canvas.
      3. Connect the result set you want to export to the Export Node.
      4. Right click on the Export Node and select Configure from the drop-down.
  4. The Export panel is displayed.
  5. From the Columns to export list:
    • To select specific columns from the result set, do one of the following:
      • For each column to include in the export, select the check box by the side of the column name.
      • Select the Select All check box and then de-select those columns not required.
      • Enter a full or partial column name in Search Columns to search for matching column names and select the check box by the side of the column name. Clear the search text to display all the columns.
    • To include all the columns from the result set, select the Select All check box.
  6. From the Select an export method list, do one of the following:

    • To export the result set as multiple files with ASCII delimiters, select Multiple CSV files using only ASCII delimiters. This export method uses parallel processing to generate and transmit the files to the destination storage repository.
      Enter the configuration values as follows:

      1. From the Connector list, select the Connector whose meta data contains the storage repository location and access authentication credentials of the exported files destination.
      2. In the Destination directory path field, enter, as a relative path, the path and destination directory.
      3. (Optional) In the File Name field, enter the base name for the files, without the extension. The name of each new file will contain the base name and a sequentially incremented number.
      4. (Optional) If you do not require the column header row as the first line in each exported file, deselect the Export column names check box.
      5. (Optional) In the Field delimiter field, enter the ASCII character that will be used to separate the data column fields in the exported files.
      6. (Optional) In the Record delimiter field, enter the ASCII character that will be used to separate the data row records in the exported files.
      7. (Optional) In the Quote character field, enter the character that will be used to surround the existing quote characters and the field and record separators. The existing quote characters are also escaped with a backslash (\) character. For example, ab "c will become "ab\"c".
        8. RECOMMENDED:Xcalar recommends this option.

    • To export the result set as multiple files using any character as a delimiter, select Multiple CSV files. This export method uses parallel processing to generate and transmit the files to the destination storage repository.
      Enter the configuration values as follows:

      1. From the Connector list, select the Connector whose meta data contains the storage repository and access authentication credentials of the exported files destination.
      2. In the Destination directory path field, enter, as a relative path, the path and destination directory.
      3. (Optional) In the File name field, enter the base name for the files, without the extension. The name of each new file will contain the base name and a sequentially incremented number.
      4. (Optional) If you do not require the column header row as the first line in each exported file, deselect the Export column names check box.
      5. (Optional) In the Field delimiter field, enter the character that will be used to separate the data column fields in the exported files.
      6. (Optional) In the Record delimiter field, enter the character that will be used to separate the data row records in the exported files.
      7. (Optional) In the Quote character field, enter the character that will be used to surround the existing quote characters and the field and record separators. The existing quote characters are also escaped with a backslash (\) character. For example, ab "c will become "ab\"c".

    • To export the result set as a single file using any character as a delimiter, select Single CSV file.
      Enter the configuration values as follows:

      1. From the Connector list, select the Connector whose meta data contains the storage repository and access authentication credentials of the exported files destination.
      2. In the File Path field, enter, as a relative path, the path, the destination directory, and the file name for the new exported file.
      3. (Optional) If you do not require the column header row as the first line in each exported file, deselect the Export column names check box.
      4. (Optional) In the Field delimiter field, enter the character that will be used to separate the data column fields in the exported files.
      5. (Optional) In the Record delimiter field, enter the character that will be used to separate the data row records in the exported files.
      6. (Optional) In the Quote character field, enter the character that will be used to surround the existing quote characters and the field and record separators. The existing quote characters are also escaped with a backslash (\) character. For example, ab "c will become "ab\"c".
    • To export the result set using an export user-defined function (UDF), select Custom export using a UDF (depreciated).
      IMPORTANT: This option is no longer supported by Xcalar and is provided for existing customers who have existing export UDFs.

      Enter the configuration values as follows:
      1. In the Destination directory path field, enter, as an absolute path, the path and destination directory.
      2. In the Export UDF name field, enter the name of the UDF that will be used to export the result set.
        IMPORTANT: The UDF must be available in the UDF Manager's sharedUDFs folder.
      3. In the File Name field, enter the base name for the files, without the extension. The name of each new file will contain the base name and a sequentially incremented number.
      4. In the Header options field, enter one of the following:
        • If you require the column header row as the first line in each exported file, enter every.
        • If you do not require the column header row as the first line in each exported file, enter none.
        • If you require a file that contains only the column header row, enter separate.
          • NOTE: This option is generally used to improve the processing time of each export file. For information on how to apply the header row back to the exported files, see the Xcalar Installation and Administration Guide.
      5. (Optional) In the Field delimiter field, enter the character that will be used to separate the data column fields in the exported files.
      6. (Optional) In the Record delimiter field, enter the character that will be used to separate the data row records in the exported files.
      7. (Optional) In the Quote character field, enter the character that will be used to surround the existing quote characters and the field and record separators. The existing quote characters are also escaped with a backslash (\) character. For example, ab "c will become "ab\"c".

  7. Click Save.

    The Node configuration is saved and the file is generated when the Node executes.

  8. On a configured Export Node, you can right click and select Create Optimized Dataflow to create and execute an optimized version of the dataflow. For more information on optimized dataflows, see Understanding a dataflow with optimization.

Learning about export drivers

To learn more about using export drivers, view the Exports (Tutorial5-ExportDrivers) tutorial included with Xcalar Design:

  1. Click the Tutorial Workbooks icon under System icon in the sidebar. This displays the tutorials provided by Xcalar to help you learn various topics.
  2. Find the Exports (Tutorial5-ExportDrivers) tutorial in the list.
  3. Click the Download button to download it and follow the instructions in the tutorial to learn about using export drivers.
