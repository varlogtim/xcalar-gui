Exporting the result of a dataflow

You can save your result set outside of Xcalar with the Xcalar Design Export feature.

To export a result set from Advanced Mode:

  1. Verify that you are in Advanced Mode. If not, from the top menu bar of Xcalar Design, click the mode switch to ADVANCED MODE.

  2. Click on the Out category of operations in the Advanced Mode toolbar.

  3. If you want to export the result set with optimized dataflow, use Export Optimized Node otherwise use Export Node. The process of configuring both the Nodes is same. For more information on optimized dataflows, see Understanding dataflow execution.
  4. For the purpose of our example, drag the Export Node to the dataflow canvas.
  5. Connect a result set you want to export to the Export Node.
  6. Right click on the Export Node and select Configure from the drop-down menu. The Export panel opens up.
  7. From the Columns to export list:
    • To choose specific columns from the result set, do one of the following:
      • For each column to include in the export, select the check box by the side of the column name.
      • Select the Select All check box and then de-select those columns not required.
    • To include all the columns from the result set, select the Select All check box.
  8. From the Export Driver list, select the format for the result set. The options are as follows:
    • fast_csv, which splits the result set into smaller files during the export.
    • legacy_udf, which exports your result set using a user-defined function (UDF). This is for legacy support only.
    • multiple_csv, which splits the result set into smaller files during the export. This is similar to fast_csv except that multiple_csv lets you add a header line as the first line of the file. Xcalar recommends this option.
    • single_csv, which exports the result set as one file.
  9. For the selected format, follow the entry field descriptions and enter the required values.
  10. Click Save. The Node configuration is saved and the file is generated when the Node executes.
