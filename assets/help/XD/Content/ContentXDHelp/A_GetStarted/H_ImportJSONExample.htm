Example of importing a JSON data source

A JSON data source can be imported into a dataset as well as into a table. However, if the JSON contains arrays and objects, the process of importing into datasets and tables is slightly different.

Importing a JSON file into a dataset

To import a JSON file into a dataset:

  1. Verify that you are in Dataflow Mode. If not, from the top menu bar of Xcalar Design, click the mode switch to display the Dataflow Mode workspace.
  2. From the sidebar, click on the Sources icon.
  3. Click on the Import Data Source icon in the Datasets panel.
  4. Browse and select a JSON file in the Select Data Source Location window and click Next.
  5. The Import Data Source window is displayed with the preview of the file. Xcalar detects the format of the file as JSON.
  6. (Optional) If you wish to use a JMESPath expression, enter it in the JMESPath text box. Otherwise, leave it empty.
  7. (Optional) Use the Advanced Options:
    1. To add the File Name or the Record Number to each record.
    2. To select an error tolerance.

  8. Click Create Dataset.

    The dataset is generated and the preview is displayed.

  9. The columns containing arrays and objects are imported as is.

Importing a JSON file into a table

To import a JSON file into a table:

  1. From the sidebar, click on the Sources icon and then click the Tables icon.
  2. Click on the Import Data Source icon in the Tables panel.
  3. Browse and select a JSON file in the Select Data Source Location window and click Next.
  4. The Import Data Source window is displayed with the preview of the file. Xcalar detects the file format as JSON. The schema of the JSON is displayed in the Schema text box.
  5. The columns with arrays and objects are grayed out because these columns cannot be imported into Xcalar tables directly. You need to edit the schema and pull out the specific elements of arrays and objects into tables.
    1. Example of importing the values of the name-value pairs in an array:
      If a column named votes, contains name value pairs, such as:
      {"funny":0,"useful":2,"cool":1}
      And you want to separate out the funny, useful and cool votes in separate columns, enter the following text in the Schema text box:
      [{"name":"votes.funny", "newName": "votes_funny", "type":"integer"},{"name":"votes.useful", "newName": "votes_useful", "type":"integer"},{"name":"votes.cool", "newName": "votes_cool", "type":"integer"}]
      It specifies the data to extract as name, the column to be created as newName and the type of the column as type. After the import, it will generate the following 3 columns - votes_funny, votes_useful and votes_cool with the corresponding data.
      You can reference nested arrays the same way.
    2. Example of importing the elements of an object:
      If a column named categories, contains a list of categories, such as:
      ["Art","Music","Science"]
      You can refer to each element using a subscript. In the Schema text box, enter the following text:
      [{"name":"categories[0]", "newName": "category0", "type":"string"},{"name":"categories[1]", "newName": "category1", "type":"string"},{"name":"categories[2]", "newName": "category2", "type":"string"}]
      It specifies the data to extract as name, the column to be created as newName and the type of the column as type. After the import it will generate the following 3 columns - category0, category1 and category2 with the corresponding data.
  6. (Optional) If you wish to use JMESPath expression, enter it in the JMESPath text box. Otherwise, leave it empty.
  7. (Optional) Use the Advanced Options:
    1. To add the File Name or the Record Number to each record.
    2. To select an error tolerance.
  8. Click Create Table. The table is generated and the preview is displayed.
