Selecting a data source

This topic describes the steps required to select a data source for importing data in Xcalar Design.

IMPORTANT: You must select a connector, as described in Selecting a connector, before you select a data source.

After selecting a connector in the Select Data Source Location window, select a Data Source Path. You have a choice of entering the complete path or navigating and selecting the files and folders through the file browser.

NOTE: If you have chosen a Database or a Generated connector, you do not need to enter a path. For more information on these connectors, see Understanding the connector types.

Entering the complete path to select a data source

Use this method if all of the following conditions are true:

  • You know the exact location of the file or the folder to be used as the data source.
  • You know the format and the content of the data source.
  • You are creating only one dataset or one table.

To import data from a directory or a file at the specified path:

  1. In the Data Source Path field, enter the path to a file or a folder.

    NOTE: The path is relative to the connector.

  2. Click Next.

    The Import Data Source window is displayed.

    The next step is to import data through the Import Data Source window as described in Importing a data source.

Using the file browser to select the data source

Use this method if any of the following conditions is true:

  • You want to preview the raw data in the data source file before the import.
  • You want to recursively import data from a data source folder.
  • You want to create multiple datasets at one time.
  • You do not know the exact location of the file or the folder to be used as the data source.

In the Select Data Source Location window, click BROWSE to browse the directory structure and navigate the folders by double clicking on them:

Once you have selected a file in the file browser, you can view the raw data (ASCII or hex) inside the file by right clicking the file name and selecting VIEW RAW DATA. The Offset field displays the position of the character where you place the cursor. To go to a character at a particular offset, enter a number in the Skip to field and press Enter.

Selecting a single item as a data source

If you want to use a single file as a data source without viewing its information, do one of the following:

  • Double click the file in the file browser to go to the Import Data Source window. This method works only if the file icon does not have a check mark.
  • Select the check mark on the file and then click NEXT to go to the Import Data Source window.

The next step is to import data through the Import Data Source window as described in Importing a data source.

Selecting multiple items as a data source

To specify multiple items in the file browser as data source:

  1. Use the key combination appropriate for your computer to select multiple items in the file browser. Each selected item is highlighted.

  2. Click the check box for an item. A check mark is placed in the box for each selected item. All checked items are added to the list of data sources:

  3. (Optional) To select additional items in another folder, you can navigate to that folder in the file browser and check the check mark on the item to make selection.

  4. (Optional) If you selected a folder as a data source, you can specify that Xcalar traverses the folder recursively for importing all data in the folder. Click the box preceding the folder path as shown in the following partial screenshot:

  5. If you want to create datasets, and if you have selected multiple items, use the One Dataset/Multiple Datasets selector to create one dataset for all items or one dataset per item.

    NOTE: If you want to create a table and you have added multiple items to the list, the contents of each item are concatenated to form a single table. You do not have the option of creating multiple tables with one import.
    EXAMPLE: Suppose you have selected the folder named /data/flightData and the files named /data/airlines2017.csv and /data/airlines2018.csv. If you select Multiple Datasets, you create three datasets, one from each item. If you select One Dataset, you create one dataset comprising of all the three items. The number of datasets that can be created is determined by the number of items on the list. Selecting Multiple Datasets does not create separate datasets for files inside the /data/flightData folder because the folder is considered one item.

  6. Click Next.

    The Import Data Source window is displayed.

    The next step is to import data through the Import Data Source window as described in Importing a data source.

Advanced search options

The following section describes the advanced search options available in the file browser. If you don't need to search files, you can go to the next step of importing data through the Import Data Source window as described in Importing a data source.

Searching for items in the file browser

If a folder contains a large number of files or sub-folders, use the Search field to find the relevant files. Entering text or clicking in the Search field displays a drop-down list showing different search methods. If you do not select a search method, the search is a plain text search. Regardless of the method, the search is always case sensitive and recursive.

The following table explains the different kinds of search:

Search method

Description

regex(match)

A match is found if a path name relative to the current folder is an exact match for the regular expression.

regex(contains)

A match is found if a path name relative to the current folder contains the regular expression.

glob(match)

A match is found if a file name is an exact match for the glob pattern.

glob(contains)

A match is found if a file name contains the glob pattern.

Plain text search

A match is found if a path name relative to the current folder contains the search string.

To search for files or folders:

  1. In the Search field, enter a search string.
  2. If you want to do a regex or glob search, select a search method in the drop-down list from the Search field. If you want to do a plain text search, press Enter.
NOTE: You can switch to another search method after the search results are displayed. If a search method is selected, and you want to switch to plain text search, place the cursor in the Search field and then press Enter.
TIP: To discard the search result and to display the contents of the current folder again, remove all characters from the Search field and then press Enter. Alternatively, restore the contents of the file browser by clicking the Refresh icon next to the Search field.

Selecting all results of a regular expression search

To specify all files matching a regular expression as data sources:

  1. Use the Browse Data Source window to select the lowest-level folder containing all data source files.
  2. In the search field, enter a regular expression, which selects all the appropriate data source files in the current folder, and optionally, its subdirectories.
  3. Select regex (match) from the search pull down menu.
  4. Modify the regular expression until you are satisfied that only the appropriate data source files are displayed in the file browser.
  5. In the Selected Files / Folders list, click the Click to Add Regex link. An item is added to the list containing the current folder followed by an editable regular expression, .*$.
  6. Copy the regular expression from the search field, and paste it to replace the .*$ in the Selected Files / Folders list.
  7. To apply the regex to all subdirectories, click the Recursive check box next to your expression.

  8. Click Next.

    The Import Data Source window is displayed where you can preview the data and provide specifications for the creation of a table or a dataset, as described in Importing a data source.

NOTE: By default, the search field performs a recursive search. If some data source files are in subdirectories, your results will only be identical if you click the Recursive check box.
NOTE: This topic assumes you are familiar with writing regular expressions (regex). For more information about writing regex, see https://docs.python.org/3/howto/regex.html#regex-howto.
