Selecting a data source

This topic provides the steps up to the point where you provide the specifications for Xcalar to create a table or dataset by importing data from a data source. The exact steps depend on the following factors, which are described in this topic:

  • Types of data source (for example, whether the data source contains a directory or a file)
  • Whether you want to examine the raw data in a file being considered as a data source
  • Whether you want to import data recursively in a folder
  • (Advanced Mode only) Whether you want to create a single dataset or multiple datasets from multiple files or folders
IMPORTANT: You must follow the instructions in Preparing the data source before you use Xcalar Design to import data.

Understanding a data source

A data source can be a folder, a file, or a combination of folders and files.

Xcalar allows a data source to include files in different formats. For example, Xcalar accepts your selection of a CSV file and a JSON file as a single data source. Having files of different formats in a data source, however, requires an import user defined function (UDF) to prepare the data when the data is being imported.

IMPORTANT: If your data source is a parquet file or dataset, follow additional instructions in Importing from a Parquet file or dataset.

Methods to select a data source

The following list describes two methods to select the data source:

  • Display the file browser and then navigate to the data source. Use this method if any of these conditions are true:
    • You do not know the exact location of the file or folder to be used as the data source.
    • You need to preview the raw data in the data source file before the import.
    • You need to recursively import data from a data source folder.
    • (Advanced Mode only) You want to create multiple datasets at one time, based on one specified path.
  • Type the data source path to select it immediately. Use this method if all of the following conditions are true:
    • You know the exact location of the file or folder to be used as the data source.
    • You know the format and contents of the data source.
    • (Advanced Mode only) You are creating one dataset based on the file or folder.

Searching for items in the file browser

If the folder contains a large number of files or sub-folders, use the Search field to find relevant files.

Typing or clicking in the Search field displays a drop-down list showing different search methods. If you do not select a search method, the search is a plain text search.

Regardless of the method, the search is case sensitive. Also, the search is recursive.

The following table explains the different kinds of search.

Search method

A match is found if…

regex(match)

A path name relative to the current folder is an exact match for the regular expression. For example, if the current folder is /home/datasets, which contains myfiles/file1.txt and myfiles/file2.txt, the regular expression is compared to myfiles/file1.txt and myfiles/file2.txt to determine if there is an exact match.

regex(contains)

A path name relative to the current folder contains the regular expression. For example, if the current folder is /home/datasets, which contains myfiles/file1.txt and myfiles/file2.txt, the regular expression is compared to myfiles/file1.txt and myfiles/file2.txt to determine if the relative path name contains the regular expression.

glob(match)

The file name is an exact match for the glob pattern. For example, if the current folder is /home/datasets, which contains myfiles/file1.txt and myfiles/file2.txt, the glob pattern is compared to file1.txt and file2.txt to determine if there is an exact match.

glob(contains)

The file name contains the glob pattern. For example, if the current folder is /home/datasets, which contains myfiles/file1.txt and myfiles/file2.txt, the glob pattern is compared to file1.txt and file2.txt to determine if the file name contains the glob pattern.
Plain text search

A path name relative to the current folder contains the search string. For example, if the current folder is /home/datasets, which contains myfiles/file1.txt and myfiles/file2.txt, the search string is compared to myfiles/file1.txt and myfiles/file2.txt to determine if the path name contains the search string.

Searching for files or folders

To search for files or folders:

  1. In the Search field, enter a search string .
  2. If you want to do a regex or glob search, click a search method in the drop-down list from the Search field. If you want to do a plain text search, press Enter.
NOTE: You can switch to another search method after the search results are displayed. If a search method is selected, and you want to switch to plain text search, place the cursor in the Search field and then press Enter.
TIP: To discard the search result and to display the contents of the current folder again, remove all characters from the Search field and then press Enter. Alternatively, restore the contents of the file browser by clicking the Refresh icon next to the Search field.

Selecting multiple items as a data source

Follow these steps to specify multiple items in the file browser as data sources:

  1. Use the key combination appropriate for your computer to select multiple items in the file browser. Each selected item is highlighted in blue.

  2. Click the check box for an item. A check mark is placed in the box for each selected item. All checked items are added to the list of data sources. The following screenshot shows how to select two files as data sources:

    NOTE: You can uncheck an item by clicking the check mark again. An unchecked item is removed from the list of data sources.

  3. (Optional) To select additional items in another folder, you can navigate to that folder in the file browser and then make the selections.

    IMPORTANT: Add the desired items to the list of data sources before navigating away from the current folder. Otherwise, you lose the selections in the current folder as soon as you go to another folder. Checked items are not affected when you leave the current folder.

  4. (Optional) If an item selected as a data source is a folder, you can specify that Xcalar traverses the folder recursively to import all data in the folder. Click the box preceding the folder path as shown in the following partial screenshot:

    NOTE: If the folder contains only sub-folders but no files, you must select Recursive. Otherwise, the folder cannot be used as a data source, and an error message is displayed.

  5. (Advanced Mode only) If you want to create datasets, and if the list of selected files and folders contains multiple items, use the One Dataset/Multiple Datasets selector to create one dataset for all items or one dataset per item.

    NOTE: If you want to create a table and you put multiple items on the list, the contents of each item are concatenated to form a single table. You do not have the option of creating multiple tables with one import.
    EXAMPLE: Suppose you have selected the folder named /data/flightData and the files named /data/airlines2017.csv and /data/airlines2018.csv. If you select Multiple Datasets, you create three datasets, one from each item. If you select One Dataset, you create one dataset comprising all three items. The number of datasets that can be created is determined by the number of items on the list. Selecting Multiple Datasets does not create separate datasets for files inside the /data/flightData folder because the folder is considered one item.
  6. Click NEXT. The top section of the resultant window shows a preview of the data to be imported. The bottom section displays fields for you to provide specifications for the table or dataset, as described in Importing a data source.

Selecting all results of a regular expression as a dataset

To specify all files matching a regular expression as data sources:

  1. Use the Browse Data Source window to select the lowest-level folder containing all data source files.
  2. In the search field, enter a regular expression, which selects all the appropriate data source files in the current folder, and optionally, its sub-directories.
  3. Select regex(match) from the search pull down menu.
  4. Modify the regular expression until you confirm that only appropriate data source files display in the file browser.
  5. In the Selected Files / Folders list, click the Click to Add Regex link. An item is added to the list containing the current folder followed by an editable regular expression, .*$.
  6. Copy the regular expression from the search field, and paste it to replace the .*$ in the Selected Files / Folders list.
  7. To apply the regex to all sub-directories, click the Recursive check box next to your expression.
  8. Click NEXT.
NOTE: The search field performs a recursive search, by default. If some data source files are in sub-directories, then your results will only be identical if you click the Recursive check box.
NOTE: This topic assumes you are familiar with writing regular expressions (regex). For more information about writing regex, see https://docs.python.org/3/howto/regex.html#regex-howto.

Viewing raw data

This section is applicable if you want to select a file as a data source and you want to examine the raw data in the file.

NOTE: You cannot view raw data in a data source if the data source is a folder.

You can view the raw data inside a file to make sure that the file is the desired one. Follow these steps to view the raw data of a file:

  1. Click a file once to select it for viewing.

    NOTE: Do not double click because doing so selects an item as a data source.
  2. Click VIEW RAW DATA. You can view the data in ASCII format or as a hex dump.

    3. The following screenshots show how you view the raw data in airlines.csv, in ASCII and in hex code.

  3. (Optional) The Offset field displays the character (ASCII or hex) at the location where your place the cursor. In the Skip to field, type a number and press Enter if you want to go to a character at a particular offset.
NOTE: If you want to switch to another file in the same folder for raw data viewing, click the desired file in the file browser. The raw data section is updated with data from the newly selected file.

Entering a data source path to select it immediately

To import data from a folder or file at the specified path:

  1. In the Data Source Path field, enter the path to a file or a folder.

    NOTE: The path is relative to the data target. For example, if the data target points to /flightData/2017, and the complete path to the data source file is /flightData/2017/airlines.csv, enter /airlines.csv or airlines.csv in Data Source Path. Xcalar Design accepts a path with or without the preceding forward slash (/).
    EXAMPLE: Enter the folder path, /flightData/2017 or /flightData/2017/, to import data in all files at the top level of the folder.
    EXAMPLE: Enter the file path, /flightData/2017/airlines.csv to import the data in the file.

  2. Click NEXT. To see a preview of the table or dataset and specify how data should be imported, follow the instructions in Importing a data source.

Each data source path that Xcalar Design uses to create a table or dataset preview is recorded in the data path history. The next time you click in the Data Source Path field, a drop-down list shows up to 5 previously used data source paths.
