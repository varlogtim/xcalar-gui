Importing a data source

This topic assumes that you have selected a data source for Xcalar Design to import, as described in Selecting a data source.

If you import multiple items at once in the Import Data Source window, be sure to read Using multiple items in one import for additional information.

IMPORTANT: If your data target type is Custom Database Connector, see the Xcalar Administration Guide for additional information about importing data.

Understanding the Import Data Source window

The Import Data Source window is divided into two portions:

  • The top portion displays a preview of the table or dataset. You can see the sample records, data types, and so on.
  • The bottom portion is a form for you to specify how data is imported. The preview responds to your specifications.

How to view records in the preview

You can change how records are displayed in the following ways:

If the format of your data source is CSV, Xcalar Design performs a smart cast when importing the data. Smart cast means that Xcalar Design tries to automatically assign the appropriate data type to each field.

You can change the data type and column name in the preview. For a non-CSV data source, you can change the data type only after creating the table or dataset. You cannot change the column names.

NOTE: Smart casting and changing the column name and data type are possible only for a CSV data source with up to 127 fields.

Changing the data type and column name in a CSV data source

To change the data type in the preview for a column:

  1. Click the data type icon in the column header. A pop-up list appears showing the possible data types.
  2. Select the appropriate data type.
NOTE: If Xcalar encounters a value in the column that cannot be cast, the data type of the column is set to Mixed. For example, if the Age column contains both numeric values and the string NA, you cannot change the data type from String to Integer because NA is not a numeric value. Instead, the column becomes a Mixed column. Similarly, if the column contains both Integer and Float numbers, the column is set to Number. After you create the dataset, you can change this column's data type. Any value that cannot be cast as an integer is replaced with FNF. For more information about changing data types, see Changing the data type of a column.

Follow these steps to change the column name:

  1. Click the column header.

  2. Type the new column name.

    The following list describes the naming conventions:
    • The name is case sensitive if the column is in a dataset; it is case-insensitive if the column is in a table, in which case all letters are displayed in uppercase.

    • The name must consist of alphanumeric characters, spaces, and special characters except the following characters:

      • caret (^)
      • single quote mark (')
      • double quote mark (")
      • left or right parenthesis (( ))
      • left or right square bracket ([ ])
      • left or right brace ({ })
      • backslash (\)
      • colons (:)
      • comma (,)
      • period (.)
    • The name must not begin or end with a space.
    • The name must begin with an underscore or a letter.
    • Length must not exceed 255 characters.
    • The name must not be DATA, but the same string capitalized differently, such as data or Data, are acceptable.
    • The name must not be one of the following strings, regardless of capitalization:
      • None
      • False
      • True
    • The name must not contain two consecutive hyphens (--).
  3. Press Enter.

Specifying how data is imported

Follow these steps in the Import Data Source window to specify how data is imported:

  1. In the Table Name or Dataset Name field, accept the suggested name or enter the desired name. The name cannot be changed later.

    NOTE: Table names are automatically set to uppercase.
    The following list describes the naming conventions:
    • Length must not exceed 255 characters.
    • Characters must be A-Z, a-z, 0-9, or underscore (_).
    • First character must be a letter.
    • Letters are displayed in uppercase, but you can enter them in uppercase or lowercase.
    TIP: If the path to the data source is too long to be displayed completely, click the path to copy it to the clipboard. You can paste it in the Table Name or Dataset Name field if you want to use a portion of the path as the name. However, you must remove all the slashes in the path in the name.

  2. In the Format field, accept the data format suggested by Xcalar Design or select an appropriate format from the drop-down list.

    NOTE: If the data source is an Excel file, you can import data from one worksheet, which can be any worksheet from the Excel workbook.

    If Xcalar Design fails to parse the data source as a file in your specified format, an error message is displayed for you to recommend another format. For example, if a file cannot be parsed as a JSON file, an error message is displayed, and you can change your format selection.

    IMPORTANT: If your data source requires a UDF during the import, you must select Custom Format in the drop-down list, even though the original format of the data source is one of the listed formats, such as CSV and JSON. For example, if your data source is a CSV file, but you need a UDF to remove illegal characters from it when it is being parsed, select Custom Format, not CSV. Also, because you can change data types and column names in the preview only for CSV files, after you select Custom Format, you cannot edit it in the preview.

  3. Skip this step if you do not select Custom Format.

    1. If the UDF already exists, select the module containing the function and the function.

      Otherwise, click WRITE PARSER USING JUPYTER to create a UDF or edit an existing UDF with Jupyter. For information about Jupyter, see Using Xcalar Design with Jupyter Notebook. After saving the UDF, you can select the UDF in the Import Data Source window.

    2. Click APPLY UDF.

      Xcalar Design generates a preview of the dataset after parsing the data with the import UDF. In this process, Xcalar Design displays the percent completed. To terminate the process, click the Cancel icon, as shown in the following screenshot:

    If you see unexpected results in the dataset preview, you can select another UDF. For more information about UDFs, see Understanding UDFs and Using UDFs.

  4. Specify the values for the other fields as described in the following table. The exact fields available depend on the data format.

    Field Description
    Promote First Row as Header

    Depending on the data format, Xcalar Design automatically selects or deselects this option. For example, if the data source is a CSV file, Xcalar Design recommends that you promote the first row as header.

    When importing multiple files with the same first row, Xcalar Design promotes the first row of the first file being processed and omits the first row of subsequent files.

    If you use the Skip Rows field to skip rows that do not require analysis, Xcalar Design promotes the first row after the row-skipping. 

    EXAMPLE: The file contains comments in the first row, and the strings Name and Salary in cells on the second row. You can specify 1 in the Skip Rows field to eliminate the first row and promote the row containing Name and Salary as header.
    IMPORTANT: You can promote the first row only at this point. You will not be able to change the first row into the header row at a later time.
    Record Delimiter The UTF-8 character for separating records (for example, \n or a null character). It must be a single character, unless it is CRLF (\r\n). If your data uses a multiple-character delimiter, you must use an import UDF when importing the data.
    Field Delimiter The UTF-8 character separating fields (for example, a tab or a comma). The delimiter can consist of multiple characters. The maximum length is 255 bytes.
    Quoting Character Character used as a quote mark. Delimiter characters between quote marks are interpreted literally, not as delimiters. Leave this field empty if you want to treat delimiter characters as delimiters regardless of where these characters appear.
    Skip Rows

    The number of rows to omit at the beginning of each file when Xcalar imports the data source. Typically you omit rows that do not contain data requiring analysis (for example, rows containing comments). If you import a folder with multiple files, the same number of rows are skipped in each file.
    Worksheet Number

    For an Excel data source, you can specify the worksheet number in the Excel workbook. You can import only one worksheet at a time.

    The default worksheet number is 0, which specifies the first worksheet in the Excel workbook.
    JMESPath

    For JSON data sources, you can import either all top-level JSON elements or certain JSON elements using JMESPath query syntax. Enter one or more JMESPath expressions into the JMESPath field and click the reload button to preview the data source. For example:

    [?searchkey==`value`].{key1:key1, key2:key2, ..., keyN:keyN}

    where:

    • searchkey: a key to search for specific values
    • value: the value to find within the key named searchkey
    • key1, key2, ..., keyN: keys to add to the table or dataset

    For details on JMESPath expressions, see http://jmespath.org.

    XPath

    For an XML data source, you must specify at least one XPath. Each XPath is a path expression selecting an XML node or node-set in your data source. To specify multiple XPaths, click the + New XPath button, and enter in another XPath string.

    Click the refresh button to preview the import using your XPath. In the data source preview, there will be one field for each XPath. Each element that matches the XPath will become one record. If an element matches multiple XPaths, it will become multiple records.

    Optionally, you can add columns to the table or dataset as described in the following list:

    • To add a field to the table or dataset from within the current XPath:
      1. Click the + New Key button. This will add an Extra Key input field and a Key Name input field.
      2. In Extra Key, enter the relative XPath for the additional element to add.
      3. In Key Name, enter the name of the field to contain this element.
    • If you select Matched XPath, Xcalar Design creates a field that contains the matched XPath in each record.

    • If you select Element XPath, Xcalar Design creates a field that contains the XPath to the element in each record.

      NOTE: Dataset creation might take a long time if you select Element XPath.

    For additional information, see an XPath resource, such as https://www.w3schools.com/xml/xml_xpath.asp.

    To better understand how to import an XML data source, see Example of importing an XML data source.

  5. If you are creating a dataset, skip this step. If you are creating a table, specify the table schema, as described in Configuring the schema for the table.

  6. (Optional) To add columns for identifying and managing data in your table or dataset, and to choose how Xcalar will handle import errors, expand the Advanced Options section. Then follow the procedures in Setting advanced options.

  7. Click Create Table or Create Dataset. A spinner is displayed, indicating that table or dataset creation is in progress. You can click the spinner to cancel the process if desired.

    NOTE: If you import data to a table, you cannot cancel the final steps of the creation process.

    After the table or dataset is successfully created, you see a preview. You also see the table in the Tables panel or the dataset in the Datasets panel.

Example of importing an XML data source

The following example illustrates how to import an XML data source file named books.xml.

Some of the source file's contents are shown below:

<?xml version="1.0"?>

<catalog>

<author>Gambardella, Matthew</author>

<description>An in-depth look at creating applications

with XML.</description>

</book>

<book id="bk102">

<author>Ralls, Kim</author>

<genre>Fantasy</genre>

an evil sorceress, and her own childhood to become queen

of the world.</description>

</book>

For XPaths, enter the following list, and select both Matched XPaths and Element XPath:

/catalog/book,/catalog/book/author

Setting advanced options

The advanced options section, contains import data source settings that add columns for identifying and managing data in your table or dataset and preference options for how you would like Xcalar to handle errors during your data source import.

Add Columns option

By default, columns are created only from the fields in the data source. Before you import your data source, you can include the following columns that will help you identify and work more effectively with the data:

  • A column that contains the original data source's file name for each record.

    To add a column, where each field contains the data source's file name for each record, click the File Name check box and enter a name for the column (for example, fileName).

    This column displays the data source file name. It is useful if the dataset or table contains records from multiple files. For example, if a table or dataset consists of records from /datasets/flights/airlines1.csv and /datasets/flights/airlines2.csv, adding this column enables you to see which records are from airlines1.csv and which records are from airlines2.csv.

  • A column that contains an identifier number for each record.

    • To add a column, where each field contains an identifier number for each record, click the Record Number check box and enter a name for the column (for example, recordNumber). Numbering starts at 0 for the first record in a file.

    IMPORTANT: For each new file imported, the record number begins at 0. Therefore, when creating a table or dataset that imports from multiple files, Xcalar recommends that you also create a File Name column. For example, if one dataset is created from the /flights folder that contains the files airlines1.csv and airlines2.csv, the table or dataset will contain duplicate identifier numbers for the second set of records from airlines2.csv. Creating this column distinguishes records that contain the same identifier number.

Error Tolerance option

While importing your source files, you can choose how Xcalar Design handles errors:

  • Low - Abort import on error 

    • When Xcalar Design detects an error, log the error and stop the import. This is the default setting.

  • Medium - Skip to next file on error, continue

    • When Xcalar Design detects an error, log the error, skip to the next file, and continue reading records.

  • High - Skip to next file on unrecoverable error, continue

    • When Xcalar Design detects an error, log the error, and continue reading records from the file. For unrecoverable errors, log the error, skip to the next file, and continue reading records.

If you log errors, you can view the errors as described in Methods for viewing logs.

Using multiple items in one import

If you specify multiple items to import, the Import Data Source window enables you to preview one item at a time, as shown in the following screenshot:

The values you specify in the Import Data Source window apply to records from all items. For example, in Advanced Mode, if you create two datasets from two files, the datasets will use the same line delimiter and record delimiter. Therefore, make sure that the import specifications are applicable to all items in one import.
