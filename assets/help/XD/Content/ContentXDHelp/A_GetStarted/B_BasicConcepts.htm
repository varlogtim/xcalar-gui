Basic Xcalar concepts

This section explains the basic concepts you must understand to use Xcalar Design efficiently.

Understanding the purpose of Xcalar Design

You can use Xcalar Design, an HTML5-based visual tool, to construct and execute SQL queries. The SQL queries are run in Xcalar Design's SQL Mode, which is the default mode. To augment the features of SQL Mode, Xcalar Design provides additional data analytic tools in its Dataflow Mode. Use Dataflow Mode if you want to monitor or troubleshoot your SQL queries.

Data imported to Xcalar Design can be unstructured or organized in arbitrary formats, and can reside on a local or cloud storage system. After you finish executing a SQL query, you can export the result set to a file system for other applications to use. If you use Dataflow Mode, you can export a result set after executing a dataflow, as well as publish data to a Jupyter Notebook for further data analysis or visualization.

Starting Xcalar Design

To start Xcalar Design, point the browser using the HTTPS protocol to the location where Xcalar Design is installed, as in the following example:

https://xcalar.mycompany.com/index.html

Understanding workbooks

You create workbooks in Xcalar Design to perform your work in SQL Mode or Dataflow Mode. A workbook can be regarded as a file, in which you run SQL queries or manipulate a dataflow.

Multiple workbooks can be active at the same time. You can deactivate or activate a workbook as desired.

Understanding the Xcalar Design login

Log in to Xcalar Design with the login name and password obtained from the Xcalar administrator or by using the Microsoft Azure account credentials.

NOTE: Login names are case-insensitive.

You can have multiple logins at any given time. For example, after logging in from one browser tab, you can log in again from another browser tab. However, you cannot open the same workbook simultaneously from multiple browser tabs.

Suppose you are logged in to Xcalar Design from two browser tabs. After you open a workbook in one browser tab, your attempt to open the same workbook in another browser tab generates a warning message. You can either go back to the previous screen or continue opening the workbook in the second browser tab. If you continue, the workbook in the first browser tab is closed.

Understanding data sources and data imports

The raw data you want to query or analyze can be in one file, or multiple files in the same directory or different directories. The term data source refers to the file, files, or directory containing the raw data. For example, your data source can be an HDFS file system that your Xcalar nodes can access or an NFS file system mounted on each node of the cluster.

By reading the data at its source, Xcalar Design obtains the metadata to build a table (if you use SQL Mode) or a dataset (if you use Dataflow Mode). This process of reading the data and building the table or dataset is called importing a data source.

Data formats supported

Xcalar Design natively supports CSV, JSON, Excel, XML, raw text, and Parquet files. For other formats, you must create a user-defined function (UDF) for importing the data and instruct Xcalar Design to use the UDF during the import.

NOTE: The CSV format includes not only comma-separated values but also other values separated by alternate delimiters. The default field delimiter is tab and the default record delimiter is newline (\n). You can select the delimiter for separating fields and for separating records.

Understanding autosave in Xcalar Design

Xcalar Design saves all your work automatically. You do not need to manually save your changes.

Entities shared among users

Because the Xcalar cluster supports multiple users, be aware that your actions on shared entities might affect others. The following list describes the entities shared by all users:

  • Published table.
  • Connector.
  • UDF, if creator chooses to share.
  • Published dataflow (in Dataflow Mode).
  • Shared dataset (in Dataflow Mode).
NOTE: The creator of a dataset can decide whether to share the dataset with other users, provided that the cluster administrator allows dataset sharing. By default, however, datasets are accessible only to the dataset owner.

In addition, memory is shared. The amount of memory used by one user affects the amount of memory available for other users.

Entities not shared among users

The following list describes the entities that are not shared among Xcalar users:

  • Result set.
  • Aggregate (in Dataflow Mode).
  • Dataset (in Dataflow Mode).

In addition, the Xcalar Design general settings are user-specific. They are not shared.

Understanding the license key

For all Xcalar Design features to be available, the Xcalar administrator must enter a valid license key during installation. Without a valid license, you cannot perform SQL queries, execute dataflows, and so on. You can, however, perform a limited number of tasks, such as downloading workbooks and exporting data from Xcalar Design.

If you cannot use Xcalar Design due to an invalid license, contact your Xcalar administrator as soon as possible. To ensure uninterrupted services, make sure that you have a valid license at all times.

If you are an administrator, you can update the license in the Setup panel of the Monitor. For more information about the Monitor, see Configuring Xcalar (admin only).

Displaying product and license information

To display the Xcalar product version and the license expiration date, follow these steps:

  1. Click in the upper right corner of the Xcalar Design window.

  2. In the drop-down menu, click About to display a modal window that lists product version, license expiration date, and copyright information.

    If the License Key Expiration is Unlicensed, the cluster does not have a valid license key.
