Example of importing an XML data source

An XML data source can be imported into a dataset as well as into a table. However, the process of importing into datasets and tables is slightly different. This topic describes the procedure for both.

The partial content of the XML file (books.xml) used in the example:

<?xml version="1.0"?>

<catalog>

<book id="bk101">

<author>Gambardella, Matthew</author>

<title>XML Developer's Guide</title>

<genre>Computer</genre>

<price>44.95</price>

<publish_date>2000-10-01</publish_date>

<description>An in-depth look at creating applications

with XML.</description>

</book>

<book id="bk102">

<author>Ralls, Kim</author>

<title>Midnight Rain</title>

<genre>Fantasy</genre>

<price>5.95</price>

<publish_date>2000-12-16</publish_date>

<description>A former architect battles corporate zombies,

an evil sorceress, and her own childhood to become queen

of the world.</description>

</book>

Importing an XML into a dataset

To import an XML data source file named books.xml into a dataset:

  1. Verify that you are in Dataflow Mode. If not, from the top menu bar of Xcalar Design, click the mode switch to display the Dataflow Mode workspace.
  2. From the sidebar, click on the Sources icon.
  3. Click on the Import Data Source icon in the Datasets panel.

  4. Browse and select books.xml in the Select Data Source Location window and click Next.

    The Import Data Source window is displayed with the preview of the file. Xcalar detects the format of the file as XML.

  5. If you wish to extract all the books and the authors, enter the following Xpaths in two separate XPath fields:

    /catalog/book, /catalog/book/author

  6. (Optional) Select both Matched XPaths and Element XPath:.

  7. Click Create Dataset.

    The dataset is generated and the preview is displayed.

Importing an XML into a table

To import the XML file into a table:

  1. From the sidebar, click on the Sources icon and then click the Tables icon.
  2. In the Tables panel, click on the Import Data Source icon.

  3. Browse and select the books.xml file in the Select Data Source Location window and click Next.

    The Import Data Source window is displayed with the preview of the file. Xcalar detects the file format as XML.

  4. If you want to extract all the authors from the file, enter the following in the XPath field:
    /catalog/book/author
  5. Click the Refresh icon to see the table preview and the updated Schema text box.
  6. Click on Create Table, which generates the table with the author column.
  7. If you want to extract the titles and the authors of the books, enter the following in the XPath field:
    /catalog/book
  8. Click on the Refresh icon to see the table preview and the updated Schema text box. You will notice that the Schema text box is still empty because the book column is an array and it cannot be imported into the table directly. You have to specify the schema to extract authors and titles.

  9. Enter the following in the Schema text box:
    [{"name":"book.author", "newName": "author", "type":"string"},{"name":"book.title", "newName": "title", "type":"string"}]

    It specifies the data to extract as name, the column to be created as newName and the type of the column as type.

  10. Click Create Table.

    A table is created with 2 columns - author and title.
