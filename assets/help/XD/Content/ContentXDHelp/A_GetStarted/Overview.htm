Xcalar Design overview

Xcalar Design is an HTML5 based browser application for developing and executing data models on Xcalar platform. Xcalar Design enables users to access raw data hosted in cloud or on-premises, develop and debug complex data models using SQL, visual programming, and Python, and export the results for other applications to consume.

The Xcalar Design integrated development environment (IDE) has two modes - the SQL Mode and the Dataflow Mode. In SQL Mode, you develop your data models using SQL and Python. In Dataflow Mode, you develop your data models using visual components, SQL and Python.

Understanding a workbook

Workbook can be visualized as a container where you design your data models and execute them. Creating a workbook is the first step in designing your data model in Xcalar Design. You can create different workbooks for different data models and easily switch between them. You can also download a workbook to share it with another user or to upload it on a different node in the Xcalar cluster.

Understanding a dataset

To read your raw data into Xcalar, you can import it into a table or into a dataset. During import, Xcalar reads the raw data, derives its metadata, and keeps it in memory. If you do not know the format of the raw data, you must import it into a dataset and then extract the parts you want for your data modeling. Datasets can be used in Dataflow Mode only. A dataset may contain all data or a sample of data from the data source, depending on the limit specified by the Xcalar administrator.

Understanding a table

To read your raw data into Xcalar, you can import it into a table or into a dataset. During import, Xcalar reads the raw data, derives its metadata, and keeps it in memory. Tables can be used both in SQL Mode as well as in Dataflow Mode.

Understanding a dataflow

A dataflow is a series of operations linked together to do data modeling in the Dataflow Mode. The visual IDE of Xcalar Design provides you the visual representation of common operations as Nodes which you can link and configure to generate the dataflow you need. You can build a dataflow in the dataflow editor by adding one operation Node at a time, configuring and executing it and viewing the results. You can reconfigure and reorder the operations at any time until you arrive at the desired data model. You can also download a dataflow to share it with other users or to upload it on a different node in the Xcalar cluster.

NOTE: Manipulating data in Xcalar Design does not modify the source data.

Learning about Xcalar Design

Xcalar provides the following resources for you to learn more about Xcalar Design:

  • Xcalar Design Online Help: To access, from the Xcalar Design sidebar, click the Help icon.
  • Tutorials: To access, in the Xcalar Design menu, from the user icon list, select Tutorials.
  • Walk-throughs: To access, in the Xcalar Design menu, from the user icon list, select Walkthroughs.
  • Xcalar Discourse community forum: To access, in the Xcalar Design menu, from the user icon list, select Discourse.

Xcalar Design tutorials

The Xcalar Design tutorials help you to learn how to use Xcalar Design to manipulate your data and create data models. To access the tutorials, in the Xcalar Design menu, from the user icon list, select Tutorials. In the Tutorial window, from the list of Xcalar Design subjects, select a tutorial.

The tutorial download includes a workbook that contains dataflows, datasets, tables, and SQL code that are required for you to learn the tutorial's subject matter. Follow the instructions, to explore and learn more about Xcalar Design.

NOTE: You need an Internet connection to download the tutorials.
