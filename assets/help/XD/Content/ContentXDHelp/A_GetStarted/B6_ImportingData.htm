Importing data

The process of reading a data file, deriving its metadata, and building a table or a dataset is called importing data in Xcalar Design. This topic describes the steps for importing data in Xcalar Design.

IMPORTANT: Access and prepare your data source as described in Accessing and preparing a data source before you begin the import process.

Choosing between a dataset and a table

Before importing your data into Xcalar Design, you must select whether you want to import the data into a dataset or into a table. If you are going to work in the SQL Mode, you must import your raw data into tables. If you do not know the format of the raw data, you must import it into a dataset and then extract the parts you want for your data model.

NOTE: By default, tables are shared among all users on a Xcalar cluster and datasets are private to the user who creates them unless he shares them explicitly.

Steps for importing data

Here are the steps for importing data in Xcalar Design:

  1. Select a connector as described in Selecting a connector.
  2. Select a data source as described in Selecting a data source.
  3. Import the data as described in Importing a data source.

If your data source is a parquet file or parquet dataset, follow the additional instructions in Importing from a Parquet file or dataset.

NOTE: Xcalar does not make a copy of source data files on the Xcalar cluster.
