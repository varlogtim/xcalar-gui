Importing from a Parquet file or dataset

Parquet datasets are columnar-oriented datasets stored in a nested file structure. The parquet data structure includes schema and metadata files that describe each file, including its location. You can create a table or a Xcalar dataset by importing from a single parquet file or from a parquet dataset.

Importing from a single parquet file

Importing from a single parquet file requires the same steps as importing from other data sources. In the Format screen, provide the following information specific to importing a parquet file:

  • In the Format field, select Parquet File.

  • In the Parquet Parser field, select your preferred parser.

    NOTE: As the parquet format has many variants, Xcalar provides open source options for parsing, including PyArrow and ParquetTools. If your preferred parser fails on a particular parquet file, consider trying again with another option.

Importing from a set of parquet files

Creating a table or a Xcalar dataset from a set of parquet data files requires a parquet connector.

To create a Xcalar dataset from a parquet dataset:

  1. Create a connector to access the location where your parquet data source is stored. For more information about creating a connector, see Creating connectors.
  2. Create a parquet connector, which works with the first connector to read your parquet data sources, as described in Creating a parquet connector.
  3. Create a table or a Xcalar dataset, as described in Importing from a parquet dataset.

Creating a parquet connector

Because a parquet data source is columnar and it can be accessed either in traditional row format or as a columnar database, Xcalar requires two connectors:

  • standard connector
  • parquet connector

A parquet connector, is a second-order connector, which is a data-connector that uses another connector, to access data.

To create a second-order connector, for parquet data sources:

  1. Log in to Xcalar Design as an administrator. From the sidebar, click the Sources icon and then the Connectors icon.

    2. The Connectors panel appears.

  2. Click the Create connector icon.

    3. The Create Connector page appears.

  3. Do the following:
    1. In the Name field, enter a name for the second-order connector.

      2. The Connector Configuration section appears.

    2. From the Connector Type list, select Parquet Dataset.

    3. In the backingTargetName field, enter the name of the connector, that contains the location for accessing the parquet files.

      Alternatively, use the Default Shared Root connector. For more information on the Default Shared Root connector, see Understanding the predefined connector.

  4. Click Add.

    5. A new connector is created that can be used as a second-order connector for parquet data.

  5. Share the Name of this connector with users who need to work with the parquet datasets on the storage location represented by backingTargetName.

Importing from a parquet dataset

To create a table or a Xcalar dataset by importing from an entire or partial parquet dataset:

  1. From the Xcalar Design sidebar, click the Tables or Datasets icon to display the Tables or Datasets panel.
  2. Click the Import Data Source icon.

    3. The Source window appears.

  3. Select the parquet connector that was created using the Parquet Dataset connector type.

    Ask your Xcalar administrator for help if you do not see the connector on the list.

  4. From the Data Source Path section, click the BROWSE button.

    5. The Browse Data Source window appears.

  5. Locate the folder containing your parquet data source and check its check box, which is located to the left of the folder’s name.
    6. NOTE: To create a Xcalar dataset from multiple parquet datasets of a common schema, click the check box for each folder containing a single parquet data source.
  6. (Optional) To examine the parquet data source's raw data, right click and select VIEW RAW DATA.
  7. Click NEXT.

    8. The Format window appears.

  8. In the Table Name or the Dataset Name field, enter a name for the table or the dataset. Alternatively, use the default name provided.
  9. Verify that the Format list shows Parquet Dataset.
  10. (Optional) If your parquet data source contains partitioning and you require specific partitioning values in the dataset, expand the Partitions (Advance) section.

    11. The list of parquet partitions appears. Do one or more of the following:

    • Enter a specific value for the partition, in the partition’s field. For multiple values, separate each value with a comma.
    • To include all the partition's values, enter an asterisk (*).
  11. The Columns section configures which columns and their data values are imported into the Xcalar.
    12. NOTE: Partitioning columns are automatically selected, since they contain schema content necessary to import the parquet dataset. These columns are displayed in the Selected Columns list, and cannot be moved to the Available Columns list, which lists the unselected column names in the parquet data source.

    From the Columns section, do one or more of the following:

    • To create a table or a dataset with specific columns of interest, from the Available Columns list, click the column name of interest.

      • The column name is moved to the Selected Columns list.

    • To create a table or a dataset from all the columns, from the Available Columns list, click Add All.

      • All column names are moved to the Selected Columns list.

    • To move an unwanted column name back to the Available Columns list, from the Selected Columns list, click the column name.

      • The column name is moved back to the Available Columns list.

    • To search for specific columns to move to the Selected Column list, in the Search field, enter a column name.
      • TIP: If the Available Columns list contains fewer column names than you expect, verify that the Search bar does not contain a value.
  12. Click Create Table or Create Dataset.
