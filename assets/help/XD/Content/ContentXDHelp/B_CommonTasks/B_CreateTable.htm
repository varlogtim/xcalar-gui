Creating a table

This section assumes that you have created one or multiple datasets. Also, this section assumes that you are viewing the worksheet where you want to create the table. See Understanding workbooks for information about how to switch between worksheets.

You can create as many tables as you want from one dataset.

NOTE: Tables can be created as a result of table operations. For example, joining two tables results in a new table. Running a batch dataflow, which is an advanced feature, can also create a table. This section is about creating a table by selecting fields in the data source.

Understanding the Dataset Preview window

The Dataset Preview window provides you with the following information:

  • Data source path relative to the data target
  • Date of the dataset creation
  • Number of records in the dataset
  • Size of the dataset
  • Data format
  • UDF used for parsing the data when the dataset was created

You can perform the following tasks in this window:

  • View the file system structure containing the data source.
  • View the data target name and type.
  • Copy the data source path.
  • View the errors that occurred during dataset creation.
  • Create a table from the columns shown in the preview.

To copy the data source path to your computer's clipboard, follow these steps:

  2. Click anywhere on the data source path. After a check mark is displayed briefly to replace the copy icon, the path is copied to your computer's clipboard.

Viewing the errors that occurred during dataset creation

A yellow or red icon displayed in the Dataset Preview window indicates that there were errors during the import process, even though the dataset was successfully created. The following partial screenshot shows the location of the error icon.

The color of the error icon depends on how you specify the Stop Import option in the Import Data Source window, as explained in the following list:

  • If you selected Import every file, logging all errors, the error icon is yellow, alerting you to the errors encountered when Xcalar imported all records except the erroneous ones.
  • If you selected Import every file, stopping each file on first erroneous record, the error icon is red, alerting you to the errors encountered when Xcalar imported records in all files up to the first error in each file. A maximum of 1 error per file is in this modal because Xcalar stops importing as soon as it detects the first error in each file.

To view the error messages, click the error icon. The Dataset Import Error modal window is displayed. For each error message, the following information is also displayed:

  • data source path
  • the record number in the data source file that caused the error

Create a table from the columns in the Dataset Preview window

In the Dataset Preview window, follow these steps to create a table:

  1. Follow one of these steps:

    • Click one or multiple column headings to select the columns that form the table. Clicking a selected column heading deselects the column.
    • Click Select All if you want all columns to be in the table.
    • Create a table without any columns. You can select fields from the dataset and add them to the table after the table is created.
    NOTE: 

  2. In the Create New Table panel, select the worksheet to which the selected columns will be sent.

  3. (Optional) In the Create New Table panel, edit the table name. A default table name, based on the dataset name, is provided. You can rename the table at a later time. For information about changing a table name, see Renaming a table.

    The following list describes the naming conventions:
    • Length must not exceed 255 characters.
    • Characters must be A-Z, a-z, 0-9, or underscore (_).
    • First character must be a letter.
    • Letters are displayed in uppercase, but you can enter them in uppercase or lowercase.

  4. (Optional) Display the advanced options, as shown in the following screenshot, to enter the column prefix. If you do not specify a prefix, Xcalar Design provides a name that is based on the table name.

    A column prefix is used to distinguish columns with the same name in the same table. Typically, column name conflicts occur due to a Join operation. If the left and right tables have the same name for a column, the prefixes (if different) help you identify the origin of each column in the resultant table.

    NOTE: If the prefixes of the tables being joined happen to be the same, you are prompted to change the prefix before starting the Join operation.

    You can specify a prefix at table creation time or when preparing for a Join operation. You cannot change the prefix at other times. Changing the table name after table creation does not change the prefix.

    Column prefixes are displayed in column headers with a user-selectable background color.

    The following list describes the naming conventions:
    • The name is case sensitive.
    • Length must not exceed 31 characters.
    • Characters must be A-Z, a-z, 0-9, hyphen (-), or underscore (_).
    • First character must be a letter.
  5. If you want to remove columns from the Create New Table panel, click the Delete icon to remove a column, or click REMOVE ALL COLUMNS to remove all columns listed in the panel.
  6. Click CREATE TABLE. All columns shown in the Create New Table panel are sent to the same worksheet.
