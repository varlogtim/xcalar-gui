Using Administration (Xcalar admin only)

As an Xcalar administrator, you can perform all functions described in Viewing the System status, by clicking the System icon () in the sidebar. In addition, you can click the Administration icon () in the sidebar to perform the following tasks:

  • Configuring Xcalar Compute Engine parameters
  • Managing the cluster, which includes the following tasks:
    • starting a cluster
    • stopping a cluster
    • restarting a cluster
    • checking the cluster status
    • updating the cluster license
    • sending an alert message to all logged in users
  • Managing users, which includes the following tasks:

    • specifying login configuration information

      NOTE: This task is described in Understanding and changing login configuration (Xcalar admin only).
    • viewing the amount of memory used by each user
    • logging in as another user
  • Viewing Xcalar log files

You can perform these tasks without having an active workbook.

Configuring parameters

The Xcalar Compute Engine parameters affect how your cluster works. In certain circumstances, Xcalar technical support might instruct you to fine-tune some parameters to improve performance or to customize your cluster for your particular environment. The list of parameters shown in the window might vary from one cluster to another.

IMPORTANT: Leave the parameters at their defaults unless you are told otherwise by Xcalar technical support or by the instructions in this topic.

If a Config Parameter Name field does not appear in the list, you can add it by clicking on the plus button () under the parameter list but do so under the guidance of Xcalar technical support.

When changes take effect

A check mark in the Restart Required field indicates that the change to the parameter value takes effect after you restart the cluster. Otherwise, the change takes effect immediately after you save it.

Parameter descriptions

The following table explains the parameters:

Parameter Description
MaxInteractiveDataSize Sets the maximum amount of source data that can be imported to create a dataset for modeling. The default is 10 GB, and the possible value can range from 1 MB to 1 TB. Both the current value and the new value are displayed in bytes.
XdbSerDesMode

(Only for Xcalar in operational mode) Determines whether Xcalar serializes data to a lower tier of the memory hierarchy when the cluster is under memory pressure. The following list describes the meaning of its setting:

  • If the parameter is set to 1, serialization is disabled.
  • If the parameter is set to 2, data is serialized to local disks. The path to the serialization destination is determined by the XdbLocalSerDesPath parameter.
XdbLocalSerDesPath

(Only for Xcalar in operational mode) Full path to which Xcalar serializes data. By default, data is serialized to the node_id directory under the Xcalar root directory. For example, if the Xcalar root directory is /var/opt/xcalar and node 0 is under memory pressure, data is serialized to /var/opt/xcalar/0/.

Recommended: To maximize performance, make sure that the path specified by this parameter is backed by SSDs (solid state drives).

XcalarRootCompletePath

The XcalarRootCompletePath configuration parameter controls the location of the shared storage mount used by all Xcalar cluster nodes.

Its default value is /mnt/xcalar.

SendSupportBundle

Boolean value specifying whether support bundles are automatically sent to Xcalar when you generate a support ticket. For information about support bundles, see Generating a support ticket.

The default value is false, which means that support bundles are saved only in XcalarRootCompletePath/support, where XcalarRootCompletePath is the full path where Xcalar is installed. If Xcalar needs diagnostic information about your cluster for troubleshooting, you have to manually send it.

 

Steps for changing parameter values

If you are instructed by Xcalar technical support to configure the parameters, follow these steps:

  1. Click Administration, which is one of the icons listed under .

  2. In the Configure Parameters window, enter the new value of a parameter in the New Value field and then click Save.

    If you want to restore the parameter to its default value, click .

  3. If required, restart the cluster to make the changes take effect.

Stopping and starting the cluster

After you install Xcalar, the cluster is started automatically. You need to start the cluster manually in the following situations:

  • after the cluster crashes
  • after you stop the cluster for maintenance or troubleshooting
  • after you change a parameter value that requires a restart

Stopping the cluster

To stop the cluster, click Stop Cluster. Xcalar Design opens a STOP CLUSTER message window for you to confirm this action.

Results of stopping a cluster

All nodes are stopped at the same time. The following list describes the effects of stopping the cluster:

  • Xcalar Design displays the following message to all the logged in users:

  • Operations in progress are canceled.

Starting or restarting the cluster

To start or restart the cluster, do one of the following:

  • If the cluster is currently stopped (that is, if all the nodes are stopped), click Start Cluster.
  • If the cluster is running or if some nodes are running, and you want to start the cluster again, click Restart Cluster.

Checking the cluster status

From the Administration panel, you can click Cluster Status to display a modal window with status information. Report the information to Xcalar technical support if needed.

Updating the license

To update the license, click Update License in the Administration panel. Paste the license key given to you by Xcalar in the License Key field and then click Confirm.

Sending a message to all users

Follow these steps to send an alert message to all users who are using Xcalar Design:

  1. From the Administration panel, click Admin Alert.
  2. In the Admin Alert text field, enter your message.
  3. Click Send. The message is displayed in each user's Xcalar Design window.

Managing users

From the Administration panel, you can expand the Users list to display a list of users that have logged in to the cluster. The user names are listed regardless of whether the users are currently logged in or whether the users have any workbooks.

You cannot edit the user list. You can, however, perform the following tasks:

  • Sorting users by names or by their memory usage.
  • Clicking a user's memory usage icon to display the amount of memory used by the user's workbooks.
  • Hovering over a user's memory usage icon to display the amount of memory consumed by that user in a tooltip. This works only if you have already sorted the memory usage or clicked on the memory usage icon to see the details.
  • Logging in as another user.

Clicking a user's memory usage icon shows a modal window that displays the memory used by the user's workbooks. It includes a breakdown of memory consumed by the result sets. The information helps you determine which result sets to delete for the purpose of releasing memory.

Logging in as another user

You can switch to another user's account to work on that user's behalf and perform all the operations which that user can perform. If the user is already logged in, switching to that user's account logs that user out of Xcalar Design. After you finish working in Xcalar Design as that user, you can return back to the administrator's account. Follow these steps to use Xcalar Design as another user:

  1. From the Administration panel, expand the Users list and click the Use Xcalar as this user icon () next to a user's name.

  2. A USE XCALAR AS message window opens. Click Confirm.

    After you switch to the other user (for example, a user named a123), a message similar to the following is displayed in the lower right corner of the Xcalar Design window:

To return to the administrator's account, click the Cancel icon (), next to that line.

Understanding log messages

Xcalar Compute Engine generates log messages and saves them to log files. By default, the log files are saved in the directory named /var/log/xcalar.

You can use Xcalar Design to view log messages generated by each node, which are in a file named /var/log/xcalar/node.X.log, where X is the node number. For example, on a two-node cluster, the log directory contains /var/log/xcalar/node.0.log and /var/log/xcalar/node.1.log.

NOTE: If you want Xcalar logs to be saved at a different location, contact Xcalar for instructions on changing the path to the log directory.

Methods for viewing logs

You can view log messages through Xcalar Design in one of the following two ways:

  • View a specified number of recent messages
  • Stream messages as they are generated

Specifying the nodes from which you retrieve the logs

You can view messages from all nodes or from particular nodes. Follow these steps if you want Xcalar to retrieve the logs from particular nodes:

  1. Display the Administration panel under System. The Logs window is displayed below the Configure Parameters window.
  2. In the Hostname Pattern field, enter a regular expression to match the hostnames. If a node has a hostname that matches the regular expression, its log will be displayed. If you leave this field blank, logs from all nodes will be displayed.

Viewing recent log messages

Follow these steps to view recent log messages:

  1. Display the Administration panel under System. The Logs window is displayed below the Configure Parameters window.

  2. In the Logs window, enter a log file name in the Log Name field. To display messages for all nodes, you can type the following:

    node.*.log

  3. Enter the number of messages from each log file to display, ranging from 1 to 500. For example, if you enter 10, then 10 messages from each log file are displayed.
  4. Click the check mark icon () to submit the values. The specified number of messages for each log file appear in the window immediately.

Streaming log messages

Follow these steps to view log messages as they are generated:

  1. Display the Administration panel under System. The Logs window is displayed below the Configure Parameters window.

  2. In the Logs window, type the Log Name. To display messages for all nodes, you can type the following:

    node.*.log

  3. Click Start Monitor Stream. The log messages from the specified log files are displayed in real time.
  4. If you want to stop the streaming, click Stop Stream.
NOTE: If you previously set a number of messages to display, that value has no effect on message streaming.

Clearing log messages

To remove all log messages from the Logs window, click Clear Logs. This button clears the messages from Xcalar Design only. The messages remain in the log files on the nodes.
