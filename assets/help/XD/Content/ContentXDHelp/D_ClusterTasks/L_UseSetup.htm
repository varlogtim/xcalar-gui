Managing the cluster (admin only)

After you install Xcalar, the cluster is started automatically. You need to start the cluster manually in the following situations:

  • after the cluster crashes
  • after you stop the cluster for maintenance or troubleshooting
  • after you change a parameter value that requires a restart
NOTE: The following tasks assume that you are logged into Xcalar Design as a user with administration privileges.

Stopping the cluster

To stop the cluster:

  1. From the sidebar, click the System icon and then the Administration icon .

  2. Click Stop Cluster in the Administration panel.

    Xcalar Design opens a STOP CLUSTER message window for you to confirm this action.

Result of stopping a cluster

All nodes are stopped at the same time. The following list describes the effect of stopping the cluster:

  • Xcalar Design displays the following message to all logged in users:

  • All operations in progress are canceled.

Starting or restarting the cluster

To start or restart the cluster:

  1. From the sidebar, click the System icon and then the Administration icon.

  2. In the Administration panel, do one of the following:

    • If the cluster is currently stopped (that is, if all the nodes are stopped), click Start Cluster.
    • If the cluster is running or if some nodes are running and you want to start the cluster again, click Restart Cluster.

Checking the cluster status

From the Administration panel, you can click Cluster Status to display a modal window with status information. Report the information to Xcalar technical support if needed.

Updating the license

To update the license:

  1. From the sidebar, click the System icon and then the Administration icon.
  2. Click Update License in the Administration panel.

  3. Paste the license key given to you by Xcalar in the License Key field and then click Confirm.

Sending a message to all users

To send an alert message to all users who are using Xcalar Design:

  1. From the sidebar, click the System icon and then the Administration icon.
  2. In the Administration panel, click Admin Alert.
  3. In the Admin Alert text field, enter your message.

  4. Click Send.

    The message is displayed in each logged in user's Xcalar Design window.

Managing users

From the Administration panel, you can expand the Users list to display a list of users who have logged in to the cluster. The user names are listed regardless of whether the users are currently logged in or whether the users have any workbooks.

You cannot edit the user list. You can, however, perform the following tasks:

  • Sorting users by names or by their memory usage.
  • Locating the memory usage of a user.
  • Accessing a Xcalar Design user account as an administrator.
