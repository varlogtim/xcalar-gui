Understanding workbook activation

A workbook's state can be active or inactive. In the Workbook Browser, the card for an active workbook is blue, and the card for an inactive workbook is white.

Logging in and out of Xcalar Design does not affect the state of a workbook. However, if the Xcalar cluster is stopped and then started again, all workbooks become inactive, and you must activate them again before working on your data.

Multiple workbooks can be active at the same time. However, you can have only one browser tab for each active workbook. For example, you cannot display the same workbook in two browser tabs.

Recommendation: Deactivating a workbook releases memory. If the cluster is low on memory, deactivate workbooks that you are not currently using.

For the steps for activating a workbook, see Creating and activating a workbook:

Deactivating a workbook

NOTE: Xcalar Design cannot deactivate a workbook if an operation in the workbook is in progress. An error message is displayed.

To deactivate a workbook, follow these steps:

  1. In the Workbook Browser, locate the workbook card for the workbook you want to deactivate.
  2. Click the ellipsis icon on the workbook card and select Deactivate Workbook.
  3. When the confirmation modal window is displayed, click Confirm.

Switching between workbooks

Switching workbooks means displaying a workbook to replace the one that is currently displayed in a browser tab. The workbook you switched from remains active.

To switch from one workbook to another, follow these steps:

  1. Click the Workbook icon to display the Workbook Browser. This closes the workbook in the current browser tab.
  2. Locate the workbook card for the workbook to which you want to switch.

  3. Follow one of these steps:

    • Click anywhere (except the preview icon and ellipsis icon) on the workbook card. A confirmation modal window is displayed. To display the workbook in the current browser tab, click Confirm.
    • Click the ellipsis icon on the workbook card and select Open workbook in new tab. The workbook is displayed in a new browser tab.
    NOTE: If you try to switch from a workbook containing an operation in progress, a warning message is displayed. Wait until the execution is complete or cancel the operation before you switch out of the workbook.

Deleting a workbook

You can delete a workbook only if it is inactive.

Deleting a workbook has the following effects:

  • All aggregates and result sets in the workbook are deleted. They are not recoverable after the deletion.
  • UDFs in the workbook are deleted.
IMPORTANT: You cannot undo workbook deletions. The removal of the workbook and its associated items is permanent. Therefore, download a workbook as a backup before you delete the workbook. For information about downloading workbooks, see Understanding workbook downloads.

To delete a workbook, follow these steps:

  1. If the workbook is not inactive, deactivate it.
  2. Click the ellipsis icon and select Delete Workbook.
  3. When the confirmation modal window is displayed, click Confirm.

Duplicating a workbook

You can duplicate an active or inactive workbook. The following list describes what are duplicated:

  • All dataflows and aggregates.
  • SQL snippets.
  • All UDFs in the original workbook.
  • Jupyter Notebooks associated with the original workbook.

To duplicate a workbook, click the ellipsis icon on the workbook card and select Duplicate Workbook. A new workbook card is displayed, with a workbook name based on the original name. Initially, the new workbook is inactive.

Understanding workbook downloads

You might want to download a workbook so that you or another user can upload it, either to the same or another Xcalar instance. The following list describes possible scenarios where you want to download a workbook:

  • Handing off a workbook to other users: After executing SQL queries or performing interactive modeling on data from a data source, you can download the workbook and share it with other users, who can use it in one of the following ways:
    • As a template to build a similar data pipeline.
    • Continue building SQL queries in SQL Mode.
    • Continue modeling through the dataflows in Advanced Mode.
  • Archiving or backing up a workbook: After some complex SQL queries or operations, you might want to download the workbook before proceeding with further steps. Because the downloaded workbook contains all the relevant workbook information, such as UDFs and Jupyter Notebooks, it serves as a copy of the workbook to which you can revert if needed.
  • Diagnosing workbook activation problems: If for some reason a workbook cannot be activated, you can download it and send the workbook file to Xcalar for evaluation. In addition, if a fix for your particular workbook problem is available on another Xcalar instance, you can upload the workbook to that Xcalar instance.
NOTE: Any Jupyter Notebook that references the workbook is also downloaded.

Understanding the workbook file

The downloaded workbook is called a workbook file, which is a compressed archive, with a .tar.gz file name extension. Xcalar Design downloads the workbook to the download location on your computer, based on your browser's download setting.

The unpacked contents of the workbook file are in open and accessible formats, such as Python and JSON. The file includes all the assets and information required for Xcalar to re-create the workbook after you upload the workbook file.

IMPORTANT: To ensure that the workbook file can be unpacked and replayed by Xcalar, do not modify the workbook file in any way after the download, unless you are advised to do so by Xcalar technical support.

Understanding UDFs in a downloaded workbook

A UDF module named default is provided by Xcalar. This UDF module is unmodifiable by users and is shared by all workbooks.

All other UDFs are created by users. The creator of a UDF can choose to share the UDF. A shared UDF can be used by all workbooks on the cluster.

The following list describes which UDFs are included or excluded in the downloaded workbook:

  • All UDFs created in the workbook are included. After the workbook is uploaded and activated, the uploaded workbook uses the UDFs unpacked from the workbook file.
  • Shared UDFs created by another user but used in the workbook are not included.
  • The default UDF module is not included, whether or not the workbook uses any functions in the module.

Differences between downloading a workbook and downloading a dataflow

Downloading a workbook might seem very similar to downloading a dataflow. However, there are differences between the two procedures. The following list describes the differences, which helps you decide whether to download a workbook or a dataflow:

  • A workbook might contain multiple related dataflows. When you download a workbook, all dataflows in the workbook are downloaded. Therefore, if you need to transfer related dataflows from one workbook to another or from you to another user, download a workbook.
  • The downloaded workbook file contains more assets than a downloaded dataflow file. For example, all UDFs created in a workbook are included in the downloaded workbook file, but only the UDFs used in a dataflow are included in the downloaded dataflow file.

Downloading a workbook

To download a workbook, click the ellipsis icon on the workbook card and select Download Workbook. The workbook file with the .xlrwb.tar.gz filename extension is created in the download location of your computer.

IMPORTANT: Do not modify the contents of this file.

Uploading a workbook

You can upload a workbook file to a Xcalar instance if the workbook was created on a Xcalar instance running a compatible version. Upload might fail if you try to upload a workbook to a Xcalar instance with an earlier version than the version from which you downloaded the workbook.

NOTE: Any Jupyter Notebook that references the workbook is also uploaded.

To upload a workbook:

  1. In the Workbook Browser, locate the New Workbook card and click Browse on the card.
    A window is displayed for you to navigate to the location where the downloaded workbook file is saved.
  2. Select the workbook file and click Open.
    After the upload is finished, a workbook card is displayed in the Workbook Browser. The workbook is initially inactive. You are ready to perform workbook tasks on this workbook as usual.
