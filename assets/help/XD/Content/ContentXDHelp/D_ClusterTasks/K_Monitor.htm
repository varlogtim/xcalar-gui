Viewing the System status

The System icon in the sidebar enables you to view the status of Xcalar Compute Engine and specify the Xcalar Design environment. This section describes the various states you can check within the System.

Operations

Xcalar Design keeps a record of all operations since you logged in and displays information about them. Follow these steps to display the list of the operations and their status:

  1. Click System icon () in the sidebar to monitor the system status. By default, the Operations panel is displayed with the list of all operations. The following partial screenshot illustrates the icons in the Operations panel that control how the list of operations is displayed.

    1. Clicking the first icon displays all operations.
    2. Clicking the second icon displays the failed operations.
    3. Clicking the third icon displays the operations in progress.
    4. Clicking the fourth icon displays the completed operations.
    5. Clicking the fifth icon lets you remove the selected operations from the list.
    NOTE: The icon for removing an operation only affects the operation list in the Operations panel. It does not affect the output of the operation. For example, a result set created by a Filter function remains in the workbook even after you remove the Filter function from the Operations panel.

  2. Select an operation in the Operations panel to see detailed information about it. For example, you can see how long the operation took to complete.

    NOTE: Some operations are grayed out. They represent the ones automatically generated by Xcalar Compute Engine. You can configure Xcalar Design to hide the system generated operations from this list. See Settings for more information.

Canceling an operation

If for any reason you want to terminate an operation that is in progress, click on the System icon in the sidebar. By default, the Operations panel is displayed with the list of all operations. Select the one you want to terminate and then click the Cancel icon either in the Operations panel or the Operation Details window.

TIP: There is another method for canceling an operation. You can cancel it when Xcalar Design displays the clock icon on the workbook indicating the operation is in progress. If you mouse over the icon, it changes to a cancellation icon, which you can click to stop the operation.

Monitor

The Monitor icon () under the System icon in the sidebar displays the Xcalar Compute Engine statistics. The Monitor panel contains the following controls:

  • The Memory Usage, the Swap Usage and the CPU switches control whether the Memory, the Swap and the CPU usage graphs are displayed in the Statistics window or not.
  • The Graph Interval slider controls the granularity of data points in the graph. By default it is set to 3 seconds.
  • The Release Memory option lets you drop the result sets of your workbook to release the memory associated with it. For more information about memory usage and how to release memory by dropping result sets, see Understanding memory usage.
NOTE: Both the Graph Interval slider and the Release Memory options are grayed out if you are not in a workbook.

The Statistics window on the right displays the following statistics:

  • The first linear graph on the top displays the total Memory, the total Swap space and the average CPU usage by all nodes.
  • The Memory donut chart displays the total memory usage by all nodes in the following buckets:
    1. OS Free is the free memory on the cluster.
    2. OS Used is the memory used by non Xcalar applications on the cluster.
    3. Xcalar Free is the free memory that is still available to Xcalar Compute Engine.
    4. The next 4 items together make up the total memory used by Xcalar Compute Engine:
      • Other Users' Result Sets is the memory used by the result sets of other users on Xcalar Design.
      • Your Result Sets is the memory used by your result sets.
      • Published Tables is the memory used by all tables.

      • Datasets is the memory used by all datasets.

      Item number 3 and 4 together make up the total memory Xcalar Compute Engine can use.

      Item number 1, 2 and 4 together make up the total OS memory.

      You can toggle between the absolute numbers and the percentages by clicking on the node data below the donut chart.

  • The Swap donut chart displays the current swap space usage by all nodes. You can toggle between the absolute numbers and the percentages by clicking on the node data below the donut chart.
  • The CPU usage donut chart displays the average CPU usage of all nodes, as percentages.
  • The NETWORK donut chart displays the amount of data sent and received by the nodes.
NOTE: You can also view the high level statistics at any time by clicking the system status icon () next to the user icon in the top right corner of Xcalar Design.

Tutorial Workbooks

The Tutorial Workbooks icon () under the System icon in the sidebar, displays the tutorials provided by Xcalar to help you learn various topics. Downloading a tutorial opens up a new workbook with relevant dataflows, datasets, tables and sql snippets already setup for that particular tutorial. Follow the instructions in the tutorial to explore and learn various tasks.

NOTE: You need an Internet connection to download these tutorials.

File Manager

The File Manager icon () under the System icon in the sidebar displays the File Manager for user-defined function (UDF) files. It has two folders by default:

  • sharedUDFs: This folder contains the UDF files shared among all users. By default it contains one UDF file called default.py.
  • workbook: All the UDF files created by users are saved under this folder.

You can navigate these folders by double clicking on them. Once you select a UDF file (a file with py extension) in a folder, following actions are available to you:

  • Open: Selecting this action opens the selected UDF file in the User Defined Function panel.
  • Download: Selecting this action downloads the UDF file on your computer.
  • Delete: Selecting this action deletes the UDF file from the cluster after confirmation.
  • Duplicate: Selecting this action makes a copy of the UDF file with a running number added to the new file name.
  • Copy to...: Selecting this action allows you to create a copy of the UDF file that you can save with a different name in a different folder where you have write permission.
  • Share: Selecting this action copies the UDF file in the sharedUDFs folder, if it does not exist there already.

Open, Duplicate, Copy to and Share actions are not available, if you have selected multiple files.

Delete, Duplicate, Copy to and Share actions are not available, if you are not in a workbook.

Additionally, an Upload button is available to you when you are in a folder with write permission. The Upload button allows you to browse your computer and upload a text file containing valid Python code.

Settings

The Settings icon () under the System icon in the sidebar provides a way for you to customize the environment in which you run Xcalar Design.

Understanding Xcalar settings

Some settings are available both to administrators as well as to users. User settings take precedence over administrator settings. These settings determine the following Xcalar Design behavior:

  • Whether the DATA column is displayed when Xcalar Design creates a result set.
  • Whether the operations list displayed by the System includes operations started automatically by Xcalar Compute Engine. If you exclude these operations, the list only shows the operations started by you. For example, the ProfileSort operation is an automatic operation.
  • How often Xcalar Design saves your changes. The range is from 10 seconds to 600 seconds.

The adminstrators get two additional settings:

  • Whether the users can share a Dataset.
  • The duration of inactivity after which the user is automatically logged out. The range is from 10 minutes to 120 minutes.

For fields that accept a numeric value, you can type the value or drag the slider to make a change.

Administration

See Using Administration (Xcalar admin only) for information about how to set up the cluster, configure system parameters, manage users and view the log files through this icon.
