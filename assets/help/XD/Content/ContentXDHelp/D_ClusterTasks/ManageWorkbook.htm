Managing workbooks

This topic describes the tasks for managing existing workbooks. To learn about creating and activating a workbook, see Creating and activating a workbook:

You must be in the Workbook Browser to manage your workbooks. If not, click the Workbooks icon in the upper left corner of the Xcalar Design window. The Workbook Browser is displayed.

Switching between workbooks

Switching between workbooks means displaying a workbook to replace the one that is currently displayed in a browser tab. The workbook you switched from, still remains active.

To switch from one workbook to another:

  1. In the Workbook Browser, locate the workbook card for the workbook to which you want to switch.

  2. Do one of the following:

    • If you want to open the workbook in the current tab, click anywhere except the ellipsis icon on the workbook card.

      The workbook is opened in the current tab.

    • If you want to open the workbook in a new tab, click the ellipsis icon and then select the Open workbook in new tab option from the drop-down.

      The workbook is opened in a new tab.

      The option Open workbook in new tab is disabled if the workbook is already open in the current tab.

NOTE: The name of the current workbook is displayed at the end of the URL in each tab.

Duplicating a workbook

Duplicating a workbook is helpful when you want to try a different variation of your dataflow or query without losing the current one. You can duplicate a workbook whether it is active or inactive. When duplicating a workbook, the following items of the original workbook are duplicated:

  • All dataflows and aggregates
  • All SQL snippets
  • All workbook user-defined functions (UDFs)
  • Jupyter Notebooks associated with the workbook

To duplicate a workbook, click the ellipsis icon on the workbook card and select Duplicate Workbook. A new workbook is created with a new workbook name based on the original name and a corresponding workbook card is displayed in the Workbook Browser. Initially, this new workbook is inactive.

Understanding a workbook download

You might want to download a workbook so that you can share it with another user or upload it to another Xcalar cluster. The following list describes a few scenarios when you may want to download a workbook:

  • Handing off a workbook to other users: After building your dataflow or query, you can download the workbook and share it with other users, who can use it:
    • As a template to build a similar dataflow or query.
    • To continue building the existing dataflow or query.
  • Archiving or backing up a workbook: After building your dataflows or queries, you might want to download the workbook as a backup before proceeding further.
  • Diagnosing workbook activation problems: If for some reason a workbook cannot be activated, you can download it and send the workbook file to Xcalar for evaluation. In addition, if a fix for your particular workbook problem is available on another Xcalar instance, you can upload the workbook to that Xcalar instance.
  • Uploading the workbook to another instance: After you have completed your dataflow or query, you can download it and then upload it on another Xcalar instance to execute it.
NOTE: Any Jupyter Notebook that references the workbook is also downloaded.

Understanding the workbook file

The downloaded workbook is called a workbook file, which is a compressed archive, with a .tar.gz file name extension. Xcalar Design downloads the workbook to the download location on your computer, based on your browser's download setting.

The unpacked contents of the workbook file are in open and accessible formats, such as Python and JSON. The file includes all the assets and information required for Xcalar to re-create the workbook after you upload the workbook file on a Xcalar instance.

IMPORTANT: To ensure that the workbook file can be unpacked and used by Xcalar, do not modify the workbook file in any way after the download, unless you are advised to do so by Xcalar technical support.

Understanding user-defined functions in a downloaded workbook

The following list describes which user-defined functions (UDFs) are included or excluded in the downloaded workbook:

  • All UDFs created in the workbook are included. After the workbook is uploaded and activated, the uploaded workbook uses the UDFs unpacked from the workbook file.
  • Shared UDFs used in the workbook are not included.
  • The default UDF module is not included even if it is used in the workbook. This is an unmodifiable UDF module provided by Xcalar and it is shared by all the workbooks. This UDF module is present in every Xcalar instance.

Differences between downloading a workbook and downloading a dataflow

Downloading a workbook might seem very similar to downloading a dataflow. However, there are differences between the two procedures. The following list describes the differences, which helps you decide whether to download a workbook or a dataflow:

  • A workbook might contain multiple related dataflows. When you download a workbook, all dataflows in the workbook are downloaded. Therefore, if you need to transfer all related dataflows from one workbook to another or from you to another user, download a workbook.
  • The downloaded workbook file contains more assets than a downloaded dataflow file. For example, all user-defined functions (UDFs) created in a workbook are included in the downloaded workbook file, but only the UDFs used in a dataflow are included in the downloaded dataflow file.

Downloading a workbook

To download a workbook, click the ellipsis icon on the workbook card in the Workbook Browser and select Download Workbook. The workbook file with the .xlrwb.tar.gz extension is downloaded in the download location of your computer.

Uploading a workbook

You can upload a workbook file to a Xcalar instance if the workbook was created on a Xcalar instance running a compatible version. Upload might fail if you try to upload a workbook to a Xcalar instance with an earlier version than the version from which you downloaded the workbook.

NOTE: Any Jupyter Notebook that references the workbook is also uploaded.

To upload a workbook:

  1. In the Workbook Browser, locate the New Workbook card and click Browse on the card.
    A window is displayed for you to navigate and select a workbook file on your computer.
  2. Select the workbook file and click Open.
    After the upload is finished, a workbook card is displayed in the Workbook Browser. The workbook is initially inactive. You are ready to perform workbook tasks on this workbook as usual.

Deactivating a workbook

Deactivating a workbook releases the memory associated with that workbook. If the cluster is low on memory, deactivate a few workbooks that you are not currently using.

To deactivate a workbook:

  1. In the Workbook Browser, locate the workbook card for the workbook you want to deactivate.

  2. Click the ellipsis icon on the workbook card and select Deactivate Workbook.

    The DEACTIVATE WORKBOOK modal window is displayed.

  3. Click Confirm.

    The workbook is deactivated.

NOTE: You cannot deactivate a workbook if an operation in the workbook is in progress; Xcalar Design displays an error.

Deleting a workbook

Deleting a workbook has the following effects:

  • All aggregates and result sets in the workbook are deleted. They are not recoverable after the deletion.
  • All user-defined functions (UDFs) in the workbook are deleted.
IMPORTANT: You cannot undo workbook deletions. The removal of the workbook and its associated items is permanent. Therefore, download a workbook as a backup before you delete the workbook. For information about downloading workbooks, see Understanding a workbook download.

You can delete a workbook only if it is inactive. To delete a workbook:

  1. If the workbook is active, deactivate it.

  2. Click the ellipsis icon and select Delete Workbook.

    The DELETE WORKBOOK modal is displayed.

  3. Click Confirm.

    The workbook is permanently deleted.
