Getting support from Xcalar

You can contact Xcalar for technical support through Xcalar Design. This topic describes how to obtain technical support by filing a support ticket and by starting a live chat. You can follow the steps in this topic even when Xcalar is not up and running.

Understanding the support ticket

You can send a support ticket to report a problem or to send a comment to Xcalar. To send a ticket from the cluster, the cluster must have Internet connection.

Information in a support ticket

A support ticket includes the following types of information:

  • Type of issue
  • Comment
  • License key
  • Workbook name
  • Number of nodes
  • Status
  • Node information, which includes:
    • CPU usage in percentage
    • Memory used in percentage
    • Memory used in bytes
    • Number of bytes received from the network
    • Number of bytes sent to the network
    • Node ID
    • Memory available in bytes
  • User name
  • User unique ID
  • Xcalar version

Information in a support bundle

If you report a performance issue or a crash, you can include a support bundle, which contains additional information.

A support bundle includes the following types of information:

  • Any core dumps caused by a crash.
  • Xcalar configuration (/etc/xcalar/*) and Xcalar-generated data (/var/opt/xcalar/*).
  • System statistics and configuration, such as number of CPUs, memory, packages installed.
  • System and Xcalar logs if permissions allow (for example, /var/log/messages, /var/log/httpd/*, /var/log/Xcalar*.log, and /var/log/xcalar/*).
NOTE: The exact information included in the support bundle might vary, depending on the access rights given to Xcalar. Because by default, Xcalar does not run as root, the file permissions set by the system administrator can prevent some information from being sent to Xcalar. For example, by default, Xcalar cannot collect syslog information unless it has permission to read the syslog file.

Generating a support ticket

To generate a support ticket, follow these steps:

  1. Click the User icon in the top right corner to display a drop-down menu.

  2. Select File Support Ticket in the drop-down menu. The File Support Ticket modal window is displayed.

  3. In the Type of issue field, use the drop-down menu to specify whether you are generating a new ticket or modifying an existing ticket.

  4. If you want to file a new ticket, select the severity level of the issue.

    If you want to modify a ticket, for example, to add comments or to send a support bundle, follow either of these steps:

    • From the ticket list, which shows all tickets submitted by you, select the ticket to modify.
    • Enter an existing ticket ID, which is for a ticket in your organization, as shown on the Xcalar customer portal.

  6. Select or deselect Generate Support Bundle.

    NOTE: Generating a support bundle might take a long time.
  7. If the ticket is new, type a subject line and then comments describing your issue.

    8. If the ticket is existing, type a comment providing additional information about the issue.

  8. Click Generate Ticket to submit a new ticket or Update Ticket to modify an existing ticket. When the ticket generation is in progress, the File Support Ticket icon turns gray. After the ticket is generated, the icon resumes its original color (white).

What happens if a support ticket cannot be sent

The most common reason for failing to send a support ticket to Xcalar is that the Xcalar cluster is not connected to the Internet. If your ticket includes the support bundle, the bundle stays on the cluster until the Internet connection is available. Xcalar automatically sends the support bundle as soon as it can use the Internet.

The contents of the support ticket, excluding the support bundle, can be downloaded when Xcalar fails to send the ticket. When an error message indicates that the you can download the ticket, follow these steps to manually send the ticket to Xcalar:

  1. In the File Support Ticket modal window, click Download, which replaces Generate Ticket. A ticket file is created and downloaded to your computer.
  2. Upload the ticket file from your computer by using the Submit a request link on the portal.

Starting live chat with Xcalar technical support

To start a live chat with Xcalar technical support, follow these steps:

  1. Click the User icon to display a drop-down menu.
  2. Select Live Chat in the drop-down menu.
  3. In the Live Chat modal window, enter your name and email address. A chat log will be automatically sent to the email address after the chat is finished.
  4. Click START and wait for a Xcalar technical support engineer to respond. A support ticket is generated as soon as the Xcalar technical support engineer starts the chat with you. The ticket ID is saved to your customer portal, which you can refer to later if necessary.
TIP: At any time during the chat, you can click the icon in the left corner of the Live Chat modal window title bar to send the chat log to your email address.
