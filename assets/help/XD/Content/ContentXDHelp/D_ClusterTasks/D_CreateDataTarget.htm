Creating connectors (admin only)

This section describes how to create a Xcalar Connector, which enables you to access your source data.

IMPORTANT: Depending on the connector type, you may need to perform pre-tasks before you can create a connector. For information about the required tasks, see the Xcalar Installation and Administration Guide.
NOTE: The following procedure assumes that you are logged into Xcalar Design as a user with administrative privileges and that you are in a Workbook.

You can create a connector whether you are in a SQL Mode or Dataflow Mode workspace.

To create a connector:

  1. From the sidebar, click the Sources icon , then the Connectors icon , and then from the Connectors panel, click the Create connector icon , as shown in the following screenshot:

  2. From the Create Connector window, in the Name field, enter a name for the connector.

  3. From the Connector Type list, select the connector type that best suites your source data's storage repository and access authentication method. For more information, see Understanding the connector types.

  4. Based on the selected connector type, entry fields appear that require access credentials, connection input, and the location of the connector's source data storage.

    For example, if you select the Shared File System connector type, enter the path for the Xcalar cluster's shared storage mount area.

  5. (Optional) Verify each field's entry values.

    IMPORTANT: Once the connector is created, you cannot make changes, such as modifying a connector's name or a mount point's path. If the connector entry values change, a new connector must be created.

  6. Click Add.

    The SUCCESS message appears, informing you that the connector was created and added it into the Connectors lists.

IMPORTANT: Before deleting a connector, verify that there are no workbooks depending on the datasets that are imported from the connector. When a connector is deleted, workbooks containing data from the deleted connector can no longer be activated after a cluster restart.
