Creating and managing user-defined functions

This section describes the tasks related to user-defined function (UDF) modules.

Understanding UDF Manager

The UDF Manager helps you manage user-defined function (UDF) modules. To display the UDF Manager, from the sidebar, click the System icon and then the UDF Manager icon . The UDF Manager is displayed with the following two folders at the root level:

  • sharedUDFs: This folder contains the UDF modules shared among all users and all workbooks. By default, it contains one UDF file called default.py with a few built-in UDFs.
  • workbook: The UDFs stored in this folder are owned by individual workbooks. A workbook cannot use a UDF owned by another workbook unless it is a shared UDF.

You can navigate the UDF folders by double clicking the folder names. Once you select a UDF module in a folder, the following actions are available to you:

  • Open
  • Download
  • Delete
  • Duplicate
  • Copy to
  • Share
  • Upload
NOTE:  If you are not in a workbook, all the actions are disabled. If you are in a workbook and select multiple modules, the Open, Duplicate, Copy to and Share actions are disabled.

Creating a user-defined function in Xcalar Design

The Xcalar user-defined functions (UDFs) are written in Python language. One Python module can contain multiple UDFs.

The UDF created by one user cannot be used by other users unless the UDF is shared. However, all UDFs can be viewed and copied by all Xcalar Design users.

You can create a UDF module in any of the following ways:

  • Enter the code directly in the User Defined Function panel.
  • Upload the Python code from your computer to the UDF Manager.
  • Use the JUPYTER NOTEBOOK window to display a UDF template and then use the Jupyter notebook editor to modify the code in the template. For more information about using the Jupyter Notebook, see Using Xcalar Design with Jupyter Notebook.

Entering the user-defined function code in the UDF panel

To enter the user-defined function (UDF) code directly in the User Defined Function panel:

  1. Use one of the following methods to display the User Defined Function panel:
    • From the sidebar, click the UDF icon . 
    • From the sidebar, click the System icon and then the UDF Manager icon .
  2. In the User Defined Function panel, click the drop-down arrow to display the list of existing modules and select New Module.
  3. Enter the code in the panel.

  4. Click on the Save File icon on the top of the panel to save the code.

    A SAVE AS modal appears where you can enter a new module name with extension py. The default location for saving the module is:

    /workbook/<current user name>/<current workbook name>/

    You can save the UDF in the default location or navigate to save it in the sharedUDFs folder. The UDFs saved in the sharedUDFs folder are shared with all other users and workwooks.

    NOTE: Module names are case insensitive. If you type letters in uppercase, Xcalar Design displays them in lowercase when listing the modules. See User-defined function name requirements for information about name requirements.

  5. Click Save. If a module with the same name already exists, a modal asks whether you want to overwrite the existing module. If so, click Confirm. Otherwise, click Cancel and enter a different name.

Note: Overwriting a functioning UDF with a non-functional UDF can cause the execution of some operations (for example a Map operation invoking the UDF) to fail. The best practice is to apply disciplined versioning to your UDFs.
TIP: When editing a UDF, you can use the keyboard shortcut appropriate for your operating system to search for or to replace a string. For example, if you run Xcalar Design in a browser on Microsoft Windows, press the Ctrl and F keys to search for a string, and the Ctrl, Shift, and F keys to replace a string.

Uploading a user-defined function module

To upload a file as a user-defined function (UDF) module from your computer to your Xcalar cluster:

  1. From the sidebar, click the System icon and then the UDF Manager icon.
    The UDF Manager is displayed.

  2. Navigate to the folder where you want to upload the UDF module by double clicking the folder names in the UDF Manager.

    NOTE: The Upload button may be disabled if you navigate to a folder where you do not have the write-permission. For example, if you select a folder which does not belong to the current workbook. Every user has write-permission in the following folders: /workbook/<current user>/<current workbook name>/ and /sharedUDFs/

  3. Click the Upload button in the top right corner.
    The file browser modal is displayed.

  4. Navigate to the folder on your computer which contains the file you want to upload, select a file with extension py or txt and click Open.

    If the file has valid Python code, it is uploaded and the file name with extension py appears in the UDF Manager, otherwise an error is reported. If a file with the same name already exists, it is replaced with the new file.

NOTE: If the file name does not follow the naming conventions, an error message notifies you that the name is invalid. See User-defined function name requirements for naming requirements.

Downloading a user-defined function module

To download a user-defined function (UDF) module from your Xcalar cluster to your computer:

  1. From the sidebar, click the System icon and then the UDF Manager icon.
    The UDF Manager is displayed.
  2. Navigate to the desired folder by double clicking the folder name in the UDF Manager and select the desired UDF module.
  3. Click Download from the Action drop-down menu to download the selected module to your computer.

Editing a user-defined function module

You can only edit the UDFs in the following folders: /workbook/<current user>/<current workbook name>/ and /sharedUDFs. To edit an existing user-defined function (UDF) module:

  1. From the sidebar, click the System icon and then the UDF Manager icon.
    The UDF Manager is displayed.
  2. Navigate to the desired folder by double clicking the folder name in the UDF Manager and select the desired UDF module.
  3. Edit the module in one of the following ways:
    • Click Open from the Action drop-down menu to edit the code in the User Defined Function panel.
    • Click Download from the Action drop-down menu to download the selected module to your computer. Modify the module by using an editor on your computer. Upload the file after you finish editing.
    • Click Open from the Action drop-down menu to display the code in the User Defined Function panel. Copy the desired module code to the Jupyter Notebook or to an external UDF editor and edit it. Paste the new module code back into the User Defined Function panel.
  4. Click Save icon .

Deleting a user-defined function module

IMPORTANT: Be sure you understand the effects of deleting a user-defined function (UDF) module before you proceed. You cannot undo the deletion of a UDF module.

To delete a UDF module:

  1. From the sidebar, click the System icon and then the UDF Manager icon.
    The UDF Manager is displayed.
  2. Navigate to the desired folder by double clicking the folder name in the UDF Manager and select the UDF module you want to delete.

  3. Click Delete from the Action drop-down menu.

    A confirmation dialog box opens up.

    NOTE: The Delete button may be disabled if you navigate to a folder where you do not have the write-permission. For example, if you select a folder which does not belong to the current workbook. Every user has write-permission in the following folders: /workbook/<current user>/<current workbook name>/ and /sharedUDFs/

  4. Click Confirm and the module is deleted.

Copying a user-defined function module

To copy a user-defined function (UDF) module:

  1. From the sidebar, click the System icon and then the UDF Manager icon.
    The UDF Manager is displayed.
  2. Navigate to the desired folder by double clicking the folder name in the UDF Manager and select the UDF module you want to copy.
  3. Click Copy from the Action drop-down menu. A COPY FILE dialog box opens up.

  4. Select a folder where you have the write-permission. Every user has write-permission in the following folders:

    /workbook/<current user>/<current workbook name>/

    /sharedUDFs/

  5. Click Save.

    The module is saved if you have the write-permission in the selected folder.

Duplicating a user-defined function module

To duplicate a user-defined function (UDF) module:

  1. From the sidebar, click the System icon and then the UDF Manager icon.
    The UDF Manager is displayed.
  2. Navigate to the desired folder by double clicking the folder names in the UDF Manager and select the UDF module you want to duplicate.

  3. Click Duplicate from the Action drop-down menu.

    NOTE: The Duplicate action is disabled if you do not have the write-permission in the selected folder. For example, if you select a folder which does not belong to the current workbook. Every user has write-permission in the following folders: /workbook/<current user>/<current workbook name>/ and /sharedUDFs/

  4. A duplicate module with a running number at the end is created in the same folder.

Sharing a user-defined function module

The creator of a user-defined function (UDF) can choose to share a UDF module, which can then be used by all other users in any workbook on the Xcalar cluster. To share a UDF module:

  1. From the sidebar, click the System icon and then the UDF Manager icon.
    The UDF Manager is displayed.
  2. Navigate to the current workbook's folder (/workbook/<current user>/<current workbook name>/) by double clicking the folder names in the UDF Manager.
  3. Select the UDF module you want to share.

  4. Click Share from the Action drop-down menu.

    NOTE: You can only share the UDF modules from your current workbook. The Share action is disabled if you are not in your current workbook folder - /workbook/<current user>/<current workbook name>/

  5. The selected module is copied in the sharedUDFs folder. If a module with that name already exists, it is overwritten.

    NOTE: At load time, the UDF modules are first searched in the current workbook and then in the sharedUDFs folder. This enables users to customize a shared UDF by copying it to their workbook folder and then customizing it without changing the module name.
