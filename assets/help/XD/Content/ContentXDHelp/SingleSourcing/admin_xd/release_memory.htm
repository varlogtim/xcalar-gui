Releasing memory

This section describes the best practices for conserving memory and how to maintain and release memory when insufficient memory issues arise.

Best practices for conserving memory

Xcalar Design users can follow these best practices to conserve the memory:

  • Reduce the number of result sets as described in Effect of a result set on memory consumption.
  • Deactivate and delete datasets and tables as described in Deactivating and deleting datasets and tables.

  • Reduce the number of fields in the Data Browser as described in Reducing memory usage using a Project operation.

What happens when the cluster is low on memory

If the cluster is healthy without low-memory issues, a green icon is displayed next to the user icon in the top right corner of Xcalar Design. You can mouse over this icon to view the percentage of memory used. Click the icon to display the Statistics window, where you can see more details about resource usage.

If Xcalar Compute Engine is low on memory, Xcalar Design displays a warning icon next to the user icon. A yellow icon indicates that memory consumption is at 60% and a red icon indicates that memory consumption is at 80%. A warning tooltip is also displayed, recommending the action to take for releasing memory.

Noticing low memory as an administrator

If you are logged in as an administrator and if you notice low memory, check the following:

  • The Memory chart, to determine which memory component is consuming the largest amount of memory. For more information, see About the memory chart.
  • Users, to determine which workbooks are consuming the most memory.

    • Users consuming large quantities of memory should be advised on Best practices for conserving memory.

    When users are not logged in to Xcalar Design, you can access the user's account on their behalf and drop their unused result sets. For more information, see Accessing a Xcalar Design user account as an administrator.

    NOTE: You can view the amount of memory used by a user, even when they are not logged in to Xcalar Design.

Locating the memory usage of a user

To locate the memory usage of a user:

  1. From the sidebar, expand the System icon and then click the Administration icon .

    2. The Administration panel appears.

  2. Do one of the following:
    • If the Users section is not expanded, click Users.
    • If the Users section is expanded, do nothing.
  3. In the SORT BY field, click MEMORY USAGE, which arranges the users in descending order thereby displaying the users using the most memory first.
  4. Hover over the user's memory usage icon to display the exact amount of memory.
  5. (Optional) If the user has active tables, click the usage icon.

Deactivating and deleting datasets and tables

Memory can also be saved when datasets and tables are deactivated or deleted.

NOTE: The following procedure assumes that your are logged into Xcalar Design and that you are in a Workbook.

To deactivate or delete a dataset or table:

  1. From the sidebar, expand the Sources icon and do one of the following:
    • To delete or deactivate a dataset:
      1. Verify that you are in the Dataflow Mode workspace. if not, from the top menu bar of Xcalar Design, click the mode switch to Dataflow Mode.
      2. Click the Datasets icon.
      3. Verify that you are displaying the datasets as a list. If not, from the Datasets menu, click the Grid View icon to the List View icon .
      4. In the SORT BY field, click SIZE, which arranges the datasets in ascending order thereby displaying the largest dataset last.
      5. For each dataset that is to be deactivated or deleted, right click and select Deactivate or Delete.
        6. TIP: You can select multiple datasets by clicking and dragging a box around the datasets.
    • To delete or deactivate a table:
      1. Click the Tables icon.
      2. Verify that you are displaying the tables as a list. If not, from the Tables menu, click the Grid View icon to the List View icon .
      3. In the SORT BY field, click SIZE, which arranges the tables in ascending order thereby displaying the largest table last.
      4. For each table that is to be deactivated or deleted, right click and from the list select Deactivate or Delete.
        5. TIP: You can select multiple tables by clicking and dragging a box around the tables.
