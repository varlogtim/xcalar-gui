Monitoring the system

This section describes the Monitor feature in Xcalar Design for monitoring your system's statistics.

To display the Xcalar cluster system statistics, from the sidebar, expand the System icon , and then click the Monitor icon .

TIP: You can view the high level statistics at any time by clicking the system status icon in the Xcalar Design menu bar.

The Monitor panel contains:

  • Toggle switches that control whether the memory, swap, and CPU usage is visible in the Statistics graph on the Statistics page.
    When enabled, this graph displays the total amount of memory and swap space and the average CPU usage by all nodes of the cluster, which enables you to monitor the overall memory availability and cluster CPU health of your system.
  • A Graph Interval slider that controls the time interval granularity of each data point. By default, this is set to 3 seconds.
  • Support section containing a Release Memory option that increases the available memory by dropping unwanted result sets from your workbook.
NOTE: The Graph Interval slider and the Release Memory options are grayed out if you are not in a workbook.

The Statistics page displays to the right of the Monitor panel containing the following charts:

  • Memory
  • Swap
  • CPU
  • NETWORK

About the memory chart

The Memory chart displays the component types and the amount of memory consumed by each component from all the nodes on the Xcalar cluster, as follows:

  • OS Free, which displays the amount of operating system memory left on the cluster.
  • OS Used, which displays the amount of operating system memory used by non Xcalar services and applications on the cluster.
  • Xcalar Free, which displays the amount of memory available for Xcalar Compute Engine processing.
  • Other User's Results Sets, which displays the amount of memory used by results sets other than your own.
  • Your Result Sets, which displays the amount of memory used by your result sets.
  • Published Tables, which displays the amount of memory used by all the tables.
  • Datasets, which displays the amount of memory used by all the datasets.
NOTE: The Other User's Results Sets, Your Result Sets, Published Tables, and Datasets components total the amount of memory that is currently being used by Xcalar Compute Engine.

The amount of used memory is displayed inside the chart and node table as either an absolute number or as a percentage. You can toggle between displaying the amount as either a number or a percentage by clicking inside the chart or node table.

TIP: To expand the component's display width, in the memory chart's legend, hover over the component.

For information about conserving and releasing memory, see Releasing memory.

About the swap chart

The Swap chart displays the current swap space usage by all nodes in the Xcalar cluster.

TIP: When users experience slow running operations, check the Swap chart to see if the cluster is using the swap space. When the Xcalar cluster starts utilizing the swap space you should consider increasing the cluster's physical memory as the existing physical memory is full.

The amount of used swap space is displayed inside the chart and node table as either an absolute number or as a percentage. You can toggle between displaying the amount as either a number or a percentage by clicking inside the chart or node table.

About the CPU chart

The CPU chart displays as a percentage the average CPU usage of all nodes in the cluster.

About the network chart

The NETWORK chart displays the amount of data sent and received by all the nodes in the cluster.
