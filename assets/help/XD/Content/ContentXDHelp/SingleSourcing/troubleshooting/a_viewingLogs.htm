Displaying log entries in Xcalar Design (admin only)

This section discusses how to view log entries in Xcalar Design.

TIP: Only Xcalar Design administrators can view log file entries in Xcalar Design. Therefore, Xcalar recommends creating a log directory specifically for user-defined function (UDF) log files that Xcalar Design Python users can access directly to troubleshoot their UDF files.
NOTE: The following tasks assume that you are logged into Xcalar Design as a user with administration privileges. Workbook access for System related tasks is not required.

Displaying Xcalar Compute Engine and component log entries

To display Xcalar Compute Engine and component log entries:

  1. From the sidebar, expand the System icon and then click the Administration icon .

    2. The Administration panel appears.

  2. From the Logs section, do one of the following:
    • To display the log entries from one log file and from all nodes in the Xcalar cluster:
      1. In the Log Name field, enter the name of the log file. For more information about the Xcalar log file types, see the Xcalar Installation and Administration Guide.
      2. In the Get Recent Logs field, enter the number of log entries to display from the log file you entered in step a.
        3. NOTE: You can view between 1 and 500 log entries.
      3. Click the check mark icon .
    • To display log entries from multiple log files and from all nodes in the Xcalar cluster:
      1. In the Log Name field, enter node.*.log.
      2. In the Get Recent Logs field, enter the number of log entries to display from each log file.
        3. NOTE: You can view between 1 and 500 log entries.
      3. Click the check mark icon .
     
  3. (Optional) To display log entries from specific nodes within the Xcalar cluster, in the Hostname Pattern field, enter a regular expression that matches the node's hostname or IP address.
  4. (Optional) To update the list with new log entries as they are generated, click Start Log Stream. When you have finished viewing the log entries, click Stop Log Stream.
    5. NOTE: This does not delete the log entries from the log.
  5. (Optional) To clear the list of log entries from the text box, click CLEAR LOGS.
    6. NOTE: This does not delete the log entries from the log files.
  6. (Optional) To save the current log entries from the text box to a text file, click the Download Log icon .
  7. (Optional) To copy the current log entries from the text box to the clipboard, click the Copy Log icon .

Changing the Log file directory location

Xcalar monitors and maintains disk space usage by compressing and archiving log files when they reach 100 MB. Depending on the daily volume of your log entries, Xcalar recommends purging old log files, resizing your log file storage area, or changing the default location.

This section describes how to change the log file location, which is usually performed when debugging or troubleshooting issues.

To change the log file directory location:

  1. Schedule a service window with your users.
  2. Create a new partition for your log files.
    3. TIP: Xcalar recommends storing logs on a local disk.
  3. Do one of the following:
    • From the User menu, click the admin user icon and select Administration.
    • From the sidebar, expand the System icon and then click the Administration icon .

    The Administration panel opens.

  4. In the Configure Parameters section, from the Config Parameter Name list, locate the XcalarLogCompletePath parameter.
  5. In the New Value field, enter the path to the new directory location for the log files.
  6. Click Save.
  7. From the Administration panel, click Restart Cluster.

Understanding the Xcalar Design logs

Each Xcalar Design user's import and workbook actions are recorded and stored in Xcalar Design log files. The human and machine log entries are specific to each workbook, therefore you must be in an active workbook for the Logs icon to be visible under the System icon.

The human and machine log file entries for the active workbook are displayed in the Logs page, where:

  • The human Log, records information about actions performed in the Xcalar Design user interface (UI) by a Xcalar Design user. The UI actions include the commands sent to Xcalar, as well as the actions performed by the user that changed the UI, such as changing the width of a column or closing a panel. For example, the human log lists each command run by the user when importing data from a connector and the drag action they performed to expand the Import Data Source Previewing section when viewing the resultant table.

    • To display more or less information about the UI operation, click the Expand all or Collapse all icons.

  • The machine Log, lists the commands that are sent to Xcalar for processing. For example, the machine log displays details about a filter or sort operation, it does not display any UI actions that were performed by the Xcalar Design user as part of the filter or sort operation.

You can switch between displaying the human or machine log details by toggling between the human icon or the machine icon .

If a log is of interest, you can download the contents of the Logs page in a text format to your computer by clicking the Download log icon .
