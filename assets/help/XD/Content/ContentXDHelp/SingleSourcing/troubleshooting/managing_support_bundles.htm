Working with support bundles (admin only)

This section discusses the administrative tasks for sending support information to Xcalar technical support.

Understanding the Xcalar support bundle

Support bundles contain normal operational information from UDF sessions and dataflow directories, log and configuration files, HTTPs server logs and system data, CPU and memory data, and recovery files. The exact information included in your support bundles is dependent on the access permissions granted to the Xcalar user owner for your cluster. When working with the Xcalar support team or creating a support ticket, your most recent support bundles are often requested to help solve the issues that you are encountering.

NOTE: By default, Xcalar does not run as the root user, therefore the file permissions set by your system administrator may prevent specific information from being sent to Xcalar. For example, by default, Xcalar cannot collect system information unless it has permission to read the syslog file.

Enabling or disabling the transmission of support bundles

Generally a support bundle is created on demand. However, you can enable the daily delivery of your support bundle to Xcalar. The Xcalar cluster collects the information which is then automatically uploaded to cloud storage.

NOTE: The following task assumes that you are logged into Xcalar Design as a user with administration privileges. For all Xcalar Design System related tasks you do not have to be in a Workbook.

To enable or disable the transmission of support bundles:

  1. Do one of the following:
    • From the User menu, click the admin user icon and select Administration.
    • From the sidebar, expand the System icon and then click the Administration icon .

    The Administration panel opens.

  2. In the Configure Parameters section, locate the SendSupportBundle configuration parameter.
  3. In the SendSupportBundle New Value field, do one of the following:
    • To enable the daily transmission of support bundles, enter true.
    • To disable the daily transmission of support bundles, enter false. By default, the daily transmission of support bundles is disabled.
    IMPORTANT: Daily support bundles are automatically generated even when their transmission is disabled, .
  4. (Optional) If you do not require your core files to be included in the support bundle, in the ChildNodeCore New Value field, enter false.

For more information about setting the support bundle configuration parameter, see Configuring Xcalar Compute Engine parameters (admin only).

Support bundle location

By default, support bundles are stored at this location:

XCALAR_HOME/support

Where XCALAR_HOME is the directory in which Xcalar and its components are installed.

RECOMMENDATION: Xcalar recommends periodically purging older support bundles.
