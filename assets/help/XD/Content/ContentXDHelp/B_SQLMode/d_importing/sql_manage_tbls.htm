Managing tables

Besides working with tables from the Tables panel, you can also view and activate your tables from the SQL Workspace. This sub-section discusses how to view the Tables list for the first time in the SQL Workspace and how to remove a table when it is no longer required.

Displaying the table list in the SQL Workspace

The Preview section in the SQL Workspace enables you to view in detail a list of tables that can be used in your SQL query.

To display the Table list in the SQL Workspace:

  1. From the sidebar, click the SQL Workspace icon .

    2. The SQL Workspace opens.

  2. In the SQL Editor panel, do one of the following:
    • To view a list of tables in the Preview section, from the SQL Editor menu, click Show Tables.
    • To view a recently generated table from imported data, in the Preview section click Refresh, which applies the recently generated table to the Tables list.

      • If after applying Refresh your table does not appear in the Preview section Tables list, from the Sources Tables panel, check that the table has been created and is active.

Refreshing the table list in the Tables panel

Since a user can have multiple workbooks open in different browser tabs, a new table created by one workbook, may not get displayed in the other tab if the Tables panel was already displayed. If you do not see a table in the Tables panel, click the Refresh icon at the top of the Tables panel to refresh the table list.

Removing a table with dependencies

Xcalar does not allow the removal of tables with dependencies to other tables. Before deleting a table you must verify whether removing the table will affect other tables.

If other tables are affected do one of the following, where the dependent table is named tableB and the table with the dependency is named tableA:

  • If tableB is still required remove its dependency history by:
    1. Exporting tableB as a file by creating a result set from tableB and exporting the result set. For more information, see Exporting a result set in SQL Mode.
    2. Removing tableB and then removing tableA.
    3. Importing tableB by importing the exported result set file into a new table.
  • If tableB is no longer required, delete tableB and then delete tableA.

You can view, remove, activate and deactivate a table from the Sources Tables panel.

To remove a table:

  1. From the sidebar, click the Sources icon, which expands to reveal the Tables icon.
  2. Click the Tables icon.
  3. From the Tables panel, locate and right-click on the table you no longer require.
  4. Click Delete.

    5. A DELETE TABLES warning message appears explaining that the operation cannot be undone and asking you to confirm the action.

  5. Click Confirm.
IMPORTANT: Once removed you cannot undo the drop action by using the undo icon or any other undo action.
