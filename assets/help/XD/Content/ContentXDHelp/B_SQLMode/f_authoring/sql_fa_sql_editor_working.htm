Working with query statements

This sub-section describes the SQL Editor, entering SQL code, which includes creating and using SQL functions, UDF modules and Xcalar defined functions in your query statements, and storing and managing query statements for future use.

Understanding the SQL Editor

The Xcalar Design SQL Editor (code editor) panel contains the following features that help you author your SQL query statements:

  • Syntax highlighting for most of the SQL query statement keywords.

    • SQL query keywords are displayed in another color. This improves readability and aids authoring, editing, and locating issues.

  • Autocomplete suggestion and insertion assistance.

    • When entering your query statement in the code editor, autocomplete suggestions appear, such as SQL keywords, table names, and column names. Either disregard, or select and accept a suggestion by pressing the tab or enter key.

  • Case-insensitive keywords.

    • The Xcalar SQL query statement keywords are not case-sensitive. For example, SELECT is the same as select and FROM is the same as from.

  • SQL keyboard shortcuts.

    • Xcalar provides SQL keyboard shortcuts. For more information on the Xcalar SQL Keyboard shortcuts, see Xcalar SQL keyboard shortcuts.

  • Optional semicolon termination for a single SQL statement.

    • When applying a single SQL query statement, terminating the query with a semicolon is optional.

    However, for code snippets that contain multiple statements, as they are intended to be separately applied to your data, you must highlight the statements that are to be applied before clicking Execute or by pressing F5. If you do not highlight the statements, all queries are handled as one single statement.

  • Stores multiple SQL queries in a code snippet.

    • When applying multiple SQL query statements, terminating the statement with a semicolon is mandatory.

    For more information, see Executing code from snippets containing multiple statements.

  • Automatically saves query entries.

    • You can enter one or multiple SQL query statements in the code editor, which remain visible and executable until you remove them, even after you log out of Xcalar Design.
