Storing SQL query statements

Xcalar automatically saves any content you entered in the SQL Editor's code editor, even after you log out of Xcalar Design.

You can also store SQL queries for reuse in a code snippet.

NOTE: The following procedure assumes that you are logged into Xcalar Design and that you are in a Workbook.

To store SQL query statements in a code snippet:

  1. Verify that you are in SQL Mode. If not, from the top menu bar of Xcalar Design, click the mode switch to SQL MODE.
  2. From the sidebar, click the SQL Workspace icon.

    3. The SQL Workspace opens.

  3. In the code editor, enter your SQL query or queries.
  4. Save the code snippet by doing one of the following:
    • Save the code snippet from the Actions list:
      1. From the Actions list, click the down arrow.

      2. Select Save As.

        The Save Snippet modal window appears

      3. In the Save As field, enter a name for the code snippet.
        4. The following list describes the naming conventions:
        • Characters must be a-z, 0-9, or underscore (_).
        • First character must be a letter.
        • Length must not exceed 255 characters.
      4. Click Confirm, which saves the code snippet, adds it to the Snippets modal window list, and displays the code snippet name in the SQL Editor menu.

      This option enables you to view existing code snippet names before naming the snippet.

    • Save the code snippet from the Untitled snippet name:
      1. Double-click the Untitled name.
      2. Enter a name for the code snippet.
      3. Press Enter, which saves the code snippet, adds it to the Snippets modal window list, and renames the Untitled code snippet in the SQL Editor menu.

      This option enables you to quickly name a code snippet.

NOTE: Performing a New Snippet or Open action from the Actions list before saving an Untitled snippet results in a Leaving Draft Snippet warning message modal window. Either, discard the draft you are working on by clicking Discard Draft, or save the snippet by entering a snippet name in the Save As field and clicking Save Snippet.

Managing SQL code snippets

The following actions are performed from the Actions list:

  • New Snippet, which:
    • When the code editor contains draft code, opens the Leaving Draft Snippet warning message modal window for you to either discard the draft code or save the draft code after entering a name for the code snippet.
    • When the code editor is empty, automatically creates a new Untitled code snippet for you to enter new draft code .
  • Open, which opens the Snippets modal window for you to choose a saved code snippet.
  • Save As, which opens the Save Snippet modal window for you to save the code in a code snippet. For more information, see Storing SQL query statements.
  • Delete, which deletes a code snippet and removes it from the Snippets modal window list. When the DELETE SNIPPET message appears, confirm the deletion of the code snippet by clicking Confirm.
  • Download Snippet, which saves the SQL code in the snippet to your computer for sharing with co-workers or for using in other applications. From the Save As dialog box, click the down arrow and browse to the directory in which you want to save the SQL code and click Save. Optionally, in the Save As field, enter a new name for the SQL file.

Understanding the Untitled snippet

The Untitled code snippet enables you to quickly apply SQL queries on your data. Xcalar automatically saves any SQL code that you enter in the Untitled code snippet, even after you log out of Xcalar Design.

RECOMMENDATION: Xcalar recommends that you rename the Untitled code snippet, as soon a possible, using a name that uniquely identifies the statement for reuse.
NOTE: To ensure that there is always a code snippet available for you to create new SQL queries, the Untitled code snippet is always generated. Even if you delete all your code snippets because they are no longer required.
TIP: Xcalar Design enables you to easily identify draft code that has not been saved by italicizing the Untitled code snippet.
