Converting a dataflow into a SQL function

This topic discusses how to convert a dataflow from the Advanced Mode into a SQL function that can be used in your SQL statements.

After debugging Xcalar Design provides the following options for bringing your changes back into your SQL query:

  • Manually adding your changes as SQL code into the SQL query statement.
  • Creating a SQL function from the dataflow and incorporating it into the SQL query statement or as a standalone statement.

To convert a dataflow into a SQL function:

  1. Verify that you are in the Advanced Mode and that your dataflow is visible in its dataflow canvas.
  2. From the tab bar, click the + New SQL function button.

    3. The New SQL Function window appears.

  3. In the Number of parameters field, enter the number of Input Nodes you require, which is the equivalent to the number of tables that are in your SQL statement dataflow.
  4. Click Confirm.

    5. The SQL function canvas opens containing the number of Input Nodes you requested and one Output Node.

  5. Double-click the SQL function tab and enter a name for this SQL function. For more information on the SQL Function naming conventions, see Naming conventions in Xcalar Design.
  6. From the tab menu, click the tab that contains your dataflow.

    7. The dataflow canvas opens containing your SQL statement dataflow.

  7. In the dataflow canvas, right-click and from the floating menu, select Select All.
  8. Copy the Nodes by pressing Ctrl +c.
  9. From the tab menu, click the tab that contains the new SQL function.

    10. The SQL function canvas displays.

    TIP: The SQL function tab contains the icon
  10. Press Ctrl + v, which pastes the Nodes that you copied from the dataflow canvas into the SQL function canvas.
    11. NOTE: The SQL function feature of the Advanced Mode has its own set of Node components. Therefore, not all the Nodes from the SQL dataflow are pasted into the SQL function canvas, such as the Input and Output Nodes. For Nodes components that were not pasted into the SQL function canvas, from the SQL Function toolbar, drag and drop the Nodes you require into the SQL function canvas and configure. For more information on how to configure Node's, see LINK HERE.
  11. For each Input Node, do the following:
    1. Select the Input Node, right-click and select Configure.
    2. From the SQL Function Input panel, click in the Source Table (For testing only) field and select a table, in the same order as each table in the dataflow.
    3. From the Column Schema section, verify the schema columns and data types.
    4. Click Save.
      5. NOTE: If the Auto preview the results after execution setting is enabled, the table is displayed in the Advanced Mode's previewing section. You can enable the Auto preview the results after execution setting from the Dataflow Settings modal window by clicking the Dataflow settings icon in the Xcalar Design menu bar.
  12. Connect the Input and Output Nodes to the operator Nodes in the same order as the dataflow by clicking one of the Node's anchors and dragging your cursor to the connecting Node.
  13. (Optional) As you connect each Node, right-click and select Execute, which tests each phase of the SQL function.
  14. Configure the Output Node by right-clicking the Node and selecting Execute.

The next step is to use the SQL function in an SQL statement. For more information, see Invoking a SQL function in a SQL statement.
