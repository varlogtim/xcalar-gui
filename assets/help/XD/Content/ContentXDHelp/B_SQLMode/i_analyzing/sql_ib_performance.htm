Analyzing your SQL statement's performance

The SQL Workspace enables you to visually see the performance and query phases of your SQL statement as a dataflow diagram from the Previewing section.

Opening a SQL result set optimized dataflow in the Previewing section

NOTE: The following procedure assumes that you are logged into Xcalar Design and that you are in a Workbook.

To open a SQL result set optimized dataflow in the Previewing section:

  1. Verify that you are in SQL Mode. If not, from the top menu bar of Xcalar Design, click the mode switch to SQL MODE.
  2. From the sidebar, click the SQL Workspace icon.

    3. The SQL Workspace opens.

  3. From the SQL History section, locate the row that contains the SQL processing results.
  4. From the Action column, click the magnifying glass icon.

    5. The Previewing section displays a visual representation of the query as an optimized dataflow diagram.

  5. (Optional) Expand the Previewing section, by dragging the right edge of the SQL Editor to the left and/or the SQL History section down.

The dataflow diagram contains individual Node components that represent each phase of the query, such as the input, the operations executed from the query statement, and the final output.

This dataflow diagram can be used as the first step to help identify performance issues and identifying bottlenecks within your query statement. Each phase of the query contains information about the node components performance:

  • Rows, the number of rows contained in the table after processing.
  • Time, the time the node component’s operation took to process.
  • Skew, the skew value the operation created after processing.

From these values you can determine if the query is functioning as expected. For example:

  • Is the node’s run-time or the resultant number of rows consistent with your expectations?
  • Is the skew higher than expected for the operation?
  • Can improvements, such as filters, be added to the query to improve performance or skew?

Once you have made your assessment you can now debug.

IMPORTANT: You cannot make changes or debug your SQL query statement in the Previewing section.

For more information on how to debug and improve the performance of a SQL statement, see Debugging your SQL statement.
