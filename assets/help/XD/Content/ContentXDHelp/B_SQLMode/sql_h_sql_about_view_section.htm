Working with your results

This sub-section describes how to work with your results from the SQL Workspace after applying SQL to your data .

Understanding the Preview section

Most of the features for working with your results are in the Preview section.

The Preview section contains the following features:

  • Lists active and inactive tables.

    • To display a list of tables that can be used in your SQL query, from the SQL Editor menu, click Show Tables.  

    The list contains the following entries:

    • Create Time, which displays the exact time the table was created, in the HH:MM:SS MM/DD/YYYY date format .
    • Rows, which displays the number of rows contained in the table.
    • Size, which displays the size of the table in megabytes.
    • Status, which lists the state of the table as either Active or Inactive. Only Active tables can be used in your SQL query.

      • To activate a table, in the Status field, click Inactive, which automatically activates the table and changes Inactive to Active.

  • Autocomplete table searches.

    • When entering a table name in the Search field, Xcalar Design predicts and displays the tables that match the text.

  • Displays a table's schema.

    • Each table is associated with a schema that contains the table's column names and their data type formats.

    To display a table's schema, from the Tables list, select a table and then click the View Schema icon.

    TIP: To got back to the Tables list, click Back.
  • Displays the result set from a query operation.

    • To view the result set from a query operation, from the SQL History section, select the row that contains the SQL processing results and then from the Results column, click View. The result set displays in the Preview section.

  • Creates a table from the result set.

    • For more information, see Saving a result set to a table.

  • Exports a result set outside of Xcalar.

    • For more information, see Exporting a result set.

Saving a result set to a table

NOTE: The following procedure assumes that you are logged into Xcalar Design and that you are in a Workbook.

You can create a new table from the result set that was created from the operations applied with SQL.

To save a result set to a table:

  1. Verify that you are in the SQL Mode workspace. If not, from the top menu bar of Xcalar Design, click the mode switch to SQL Mode.
  2. From the sidebar, click the SQL Workspace icon .

    3. The SQL Workspace opens.

  3. (Optional) If the result set is not displaying in the Preview section, do the following:
    1. From the SQL History section, select the row that contains the SQL processing results of the result set.
    2. From the Results column, click View.

      3. The result set appears in the Preview section.

    NOTE: If the query statement successfully executed but is now displaying an error in the Results column, the statement must be re-executed.
    To re-execute, from the Query column, click the query and from the SQL Query modal window, copy and paste the query into the code editor. Click Execute or press F5.
  4. From the Preview section, click Save Result as Table.

    5. The Create Table modal window appears.

  5. In the Table Name field, enter a name for the table. For naming conventions, see Table.
  6. From the Primary Keys list, do one of the following:
    • For a primary key that uniquely identifies each record from a single column name, select the column name.
    • For a composite primary key that uniquely identifies each record from multiple column names, select the first column name and then for each subsequent name for the unique identifier, click the Add Primary Key icon and select the subsequent column names.
      • NOTE: If you select the No Primary Key check box, the table cannot be updated.
  7. From the Columns in Table section, do one of the following:
    • To choose specific columns from the result set, select the column check box for each column to include in the table.
    • To choose all columns in the result set, select the Select All check box.
      • NOTE: After assigning a column or columns for the primary key, Xcalar Design automatically ticks the primary key column check box.
  8. Click Confirm, which creates a table from the result set.

Exporting a result set

NOTE: The following procedure assumes that you are logged into Xcalar Design and that you are in a Workbook.

You can save your result set outside of Xcalar with the Xcalar Design Export feature.

To export a result set:

  1. Verify that you are in the SQL Mode workspace. If not, from the top menu bar of Xcalar Design, click the mode switch to SQL Mode.
  2. From the sidebar, click the SQL Workspace icon .

    3. The SQL Workspace opens.

  3. In the SQL History section, select the row that contains the SQL processing results that you wish to export.
  4. From the Results column, click View.

    5. The result set appears in the Preview section.

  5. From the View Result section, click Export Result.

    6. The Export Table modal window appears.

  6. From the Columns to export list, do one of the following:
    • To choose specific columns from the result set, do one of the following:
      • For each column to include in the export, select the check box by the side of the column name.
      • Select the Select All check box and then de-select those columns not required.
      • In the Search Columns field, search for column names and then select their check box.
    • To include all the columns from the result set, select the Select All check box.
  7. From the Select an export method list, do one of the following:
    • To export the result set as multiple files with ASCII delimiters, select Multiple CSV files using only ASCII delimiters. This export method uses parallel processing to generate and transmit the files to the destination storage repository.
      Enter the configuration values as follows:
      1. From the Connector list, select the Connector whose meta data contains the storage repository location and access authentication credentials of the exported files destination.
      2. In the Destination directory path field, enter, as a relative path, the path and destination directory.
      3. (Optional) In the File Name field, enter the base name for the files, without the extension. The name of each new file will contain the base name and a sequentially incremented number.
      4. (Optional) If you do not require the column header row as the first line in each exported file, deselect the Export column names check box.
      5. (Optional) In the Field delimiter field, enter the ASCII character that will be used to separate the data column fields in the exported files.
      6. (Optional) In the Record delimiter field, enter the ASCII character that will be used to separate the data row records in the exported files.
      7. (Optional) In the Quote character field, enter the character that will be used to surround the existing quote characters and the field and record separators. The existing quote characters are also escaped with a backslash (\) character. For example, ab "c will become "ab\"c".
        8. RECOMMENDED: Xcalar recommends this option.
    • To export the result set as multiple files using any character as a delimiter, select Multiple CSV files. This export method uses parallel processing to generate and transmit the files to the destination storage repository.
      Enter the configuration values as follows:
      1. From the Connector list, select the Connector whose meta data contains the storage repository and access authentication credentials of the exported files destination.
      2. In the Destination directory path field, enter, as a relative path, the path and destination directory.
      3. (Optional) In the File name field, enter the base name for the files, without the extension. The name of each new file will contain the base name and a sequentially incremented number.
      4. (Optional) If you do not require the column header row as the first line in each exported file, deselect the Export column names check box.
      5. (Optional) In the Field delimiter field, enter the character that will be used to separate the data column fields in the exported files.
      6. (Optional) In the Record delimiter field, enter the character that will be used to separate the data row records in the exported files.
      7. (Optional) In the Quote character field, enter the character that will be used to surround the existing quote characters and the field and record separators. The existing quote characters are also escaped with a backslash (\) character. For example, ab "c will become "ab\"c".
    • To export the result set as a single file using any character as a delimiter, select Single CSV file.
      Enter the configuration values as follows:
      1. From the Connector list, select the Connector whose meta data contains the storage repository and access authentication credentials of the exported files destination.
      2. In the File Path field, enter, as a relative path, the path, the destination directory, and the file name for the new exported file.
      3. (Optional) If you do not require the column header row as the first line in each exported file, deselect the Export column names check box.
      4. (Optional) In the Field delimiter field, enter the character that will be used to separate the data column fields in the exported files.
      5. (Optional) In the Record delimiter field, enter the character that will be used to separate the data row records in the exported files.
      6. (Optional) In the Quote character field, enter the character that will be used to surround the existing quote characters and the field and record separators. The existing quote characters are also escaped with a backslash (\) character. For example, ab "c will become "ab\"c".
    • To export the result set using an export user-defined function (UDF), select Custom export using a UDF (depreciated).
      IMPORTANT: This option is no longer supported by Xcalar. It is provided for customers who are currently using export UDFs and require no assistance.

      Enter the configuration values as follows:
      1. In the Destination directory path field, enter, as an absolute path, the path and destination directory.
      2. In the Export UDF name field, enter the name of the UDF that will be used to export the result set.
        IMPORTANT: The UDF must be available in the UDF Manager's sharedUDFs folder.
      3. In the File Name field, enter the base name for the files, without the extension. The name of each new file will contain the base name and a sequentially incremented number.
      4. In the Header options field, enter one of the following:
        • If you require the column header row as the first line in each exported file, enter every.
        • If you do not require the column header row as the first line in each exported file, enter none.
        • If you require a file that contains only the column header row, enter separate.
          • NOTE: This option is generally used to improve the processing time of each export file. For information on how to apply the header row back to the exported files, see the Xcalar Installation and Administration Guide.
      5. (Optional) In the Field delimiter field, enter the character that will be used to separate the data column fields in the exported files.
      6. (Optional) In the Record delimiter field, enter the character that will be used to separate the data row records in the exported files.
      7. (Optional) In the Quote character field, enter the character that will be used to surround the existing quote characters and the field and record separators. The existing quote characters are also escaped with a backslash (\) character. For example, ab "c will become "ab\"c".
  8. Click Confirm.
