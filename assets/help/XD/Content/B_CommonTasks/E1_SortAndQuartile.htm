Sorting a column and finding quartiles

You can sort values in a column so that values are displayed in ascending or descending order.

IMPORTANT: Sorting only affects how the rows are displayed. Subsequent operations do not preserve the result of the sort. For example, you can sort on a column named Age to display rows ordered by age. Later if you perform an arithmetic function to sum two other columns, the resultant table displayed after the function is in the original, unsorted order. The rows are no longer sorted by age.

Sorting values in a column

Follow these steps to sort a column:

  1. Follow one of these steps:

    • In the column drop-down menu, select Sort. A pop-up is displayed for you to specify whether the values are sorted in ascending or descending order.

    • In the column header, click one of the triangles, which indicates the direction of the sort, as shown in the following screenshot:

  2. If the data type of the column is String and Xcalar Design determines that most of the values in the column are numeric, it displays the Sort Suggestion dialog box. Depending on the desired method of sorting, click Alphabetically or Numerically.

    NOTE: If you select Numerically, a derived column is created based on the original column, with its data type set to Integer or Float, depending on whether Xcalar Design detects a significant number of values in floating point format.

The column header for a sorted column shows the direction of the sort. The following screenshot shows the column header of a column sorted in descending order.

Sorting a table on multiple columns

To sort a table on multiple columns, repeat the steps in Sorting values in a column for each column. Alternatively, use the method described in this section, in which you select all the columns before the sort takes place. If you use this method, you are not given the option to sort a String column numerically.

Follow these steps to sort a table on multiple columns:

  1. To sort a particular column first, display the column options from that column. Then select Sort to display the pop-up.
  2. Click Advanced … in the pop-up to display the Sort panel. The column name is displayed in the Sort panel.

  3. Select another column (by clicking anywhere in the column) as the second column to be sorted. Repeat this step for each additional column that you want to sort.

    To deselect a column, click the column again.

    The column names are listed in the Sort panel in the same order as you select the columns. Also, each column header is associated with a number (for example, 1, 2, and so on), indicating the order in which the columns will be sorted. In the following screenshot, the DayOfWeek column is selected first, followed by the AirTime column.

  4. In the Sort panel, for each selected column, specify whether the sort is in ascending or descending order.
  5. Click SORT.

The following screenshot is an example showing the column headers after a multi-column sort:

NOTE: The column you sort on most recently becomes the primary sort key. For example, after sorting on the DayOfWeek and AirTime columns, if you sort on the Distance column, Xcalar updates the numbers in the column headers: The sort key number for Distance is 1, the sort key number for DayOfWeek is 2, and the sort key number for AirTime is 3.

Using a single sort key again after sorting with multiple keys

Suppose your table is currently sorted on two column. If you want to sort the table only on one of these columns instead, sort the column again in the same direction as you did.

EXAMPLE: if the table is currently sorted on the DayOfWeek and AirTime columns, but you want to change the table such that it is sorted on DayOfWeek only, you can sort DayOfWeek in the same direction (for example, in ascending order) as before. Doing so keeps the DayOfWeek column sorted but removes AirTime as a sort key. The values in AirTime are no longer unsorted.
NOTE: You can re-sort a table on a column in the same direction as before if the column is one of the multiple sort keys. If the table is sorted only on one column, trying to re-sort the table on this column in the same direction generates a message, indicating that the table is already sorted.

Reverting a table to the unsorted state

If you no longer want the table to be sorted on any column, from the dataflow graph, select a table before the table was sorted and put the table back in the worksheet.

For information about how to revert to a particular table, see Using the dataflow graph.

Finding quartiles in a column

Follow these steps to determine the quartiles in a column:

  1. Right click the column header to display the column drop-down menu.

  2. Click Profile… in the drop-down menu. The Profile modal window containing the quartile summary section is displayed. If the column is already sorted, the quartiles are displayed automatically. Otherwise, click Generate to create the quartile summary. The following screenshot shows an example of the quartile summary.
