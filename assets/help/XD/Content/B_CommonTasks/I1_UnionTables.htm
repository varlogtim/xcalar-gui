Creating a union of multiple tables

You can create a union of two or more tables in a similar way as you use the UNION operator in SQL. You can create a union of multiple tables in one of two modes:

  • Union all, which combines all rows in the specified tables, including duplicates.
  • Union, which combines all rows in the specified tables but eliminates the duplicates.

Before you start the union operation

Understand and follow these guidelines when you enter the specifications for the union operation:

  • You can union as many tables as desired.
  • The input tables can have a different number of columns. For example, you can union Table 1, which has 2 columns, and Table 2, which has 5 columns.
  • You can specify which column from an input table will make up a resultant column in the output. For example, suppose you union Table 1 and Table 2, each containing columns named Date2016 and Date2017. For the union output, the resultant column named Date can comprise data from Date2016 of Table 1 and Date2017 of Table2.
  • If the columns of the input tables do not have matching data types, you must specify the data type of the resultant column. For example, if Table 1 has an integer column named Date and Table 2 has a string column named Date, you must specify the data type of the resultant Date column in the output. Suppose you specify Integer as the data type, Xcalar casts the Date column of Table 2 to Integer before the union.
  • You can specify the columns to be excluded in the output. For example, if Table 1 and Table 2 have 3 columns named Date, Distance, and Duration, you can specify that the Date column be dropped from the resultant table.

Understanding the output

The following list describes the characteristics of the union output:

  • All columns in the output table are derived columns.
  • The input tables become temporary tables. The output of the union is the active table.
  • As in SQL, creating a union in Xcalar does not preserve the row order. For example, if Table 1 and Table 2 are the input tables for the union, the resultant table might display some rows from Table 1, followed by some rows from Table 2 and then more rows from Table 1.
  • These columns are excluded from the output:
    • Array or object columns.
    • Blank, unpopulated columns.

Starting the union operation

Follow these steps to create a union table from two or more tables:

  1. From a table that you want to add to a union, click the header of a column that you want to include in the union output. The column drop-down menu is displayed.

  2. Select Union… to display the UNION panel, as shown in the following screenshot:

  3. In the Mode field, specify whether you want to use Union All or Union.
  4. Specify the resultant table name in the Union Table field.
  5. Select Table 2 (the second table in the union).
  6. (Optional) Click Add Table to include another table in the union. Repeat this step for each additional table.
  7. (Optional) Change the resultant column's name. By default, the resultant column's name is the same as the name of the column in the first input table.

  8. The To be Dropped section provides a list for each input table. The list shows the columns that can potentially be in the union table. Select the columns that will make up the resultant column by following these guidelines:

    • The column from which you display the panel is automatically selected as an input column from Table 1 (the first table in the union). You can, however, replace it with another column.

    • In the To be Dropped section, click a column name or the Plus icon preceding the name to create a resultant column that will contain the clicked column.

      TIP: To make sure that the selected column is the intended column, move the cursor over the column name in the list in the To be Dropped section. Click the focus icon to locate the column in the worksheet. This enables you to verify that the column is the appropriate one.
    • You can create a column in the union table that consists of fewer columns than input tables. For example, in the union table, a column named DayOfWeek can be made of the DayOfWeek column in Table1 and no column from Table 2. The rows from Table 2 in the DayOfWeek column of the union table will show FNF.

  9. The Columns in Union Table section shows the columns that already exist in the union table. In this section, you can select a column from each input table that will make up the column in the union table.

    For each input table, click a column name or the Down arrow next to it to display a drop-down list. From this list, select a column that will make up the resultant column.

    TIP: To make sure that the selected column is the intended column, move the cursor over the column name in the drop-down list. Click the focus icon to locate the column in the worksheet. This enables you to verify that the column is the appropriate one.

    Repeat this step for each column in the union table.

  10. Click UNION ALL or UNION. If a resultant column consists of columns of mismatched types, an error message is displayed. You must specify the data type of the resultant column, and then click UNION ALL or UNION again.
