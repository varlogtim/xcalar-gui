Managing workbooks

To manage workbooks, open the Workbook Browser, which you can display by clicking in the upper left corner of the Xcalar Design window. This topic describes the tasks for existing workbooks. To create a workbook, follow the instructions in Creating a workbook as your workspace.

Tasks you can perform in the Workbook Browser

You can perform the following tasks in the Workbook Browser:

  • Activating a workbook.
  • Deactivating a workbook.
  • Switching between workbooks.
  • Deleting an inactive workbook.
  • Previewing a workbook.
  • Editing a workbook name.
  • Creating or editing a workbook description.
  • Duplicating a workbook.
  • Downloading a workbook.
  • Uploading a workbook.

Understanding workbook activation

A workbook's state can be active or inactive. In the Workbook Browser, the card for an active workbook is blue, and the card for an inactive workbook is white.

Upon creation, a workbook is inactive. You must activate the workbook before you can use it as your workspace.

Logging in and out of Xcalar Design does not affect the state of a workbook. However, if the Xcalar cluster is stopped and then started again, all workbooks become inactive, and you must activate them again before working on your data.

You do not need to deactivate one workbook to activate another. Multiple workbooks can be active at the same time. However, you can have only one browser tab for each active workbook. For example, you cannot display the same workbook in two browser tabs.

You can activate multiple workbooks at the same time, provided that they are displayed in different browser tabs. You do not need to wait for the activation of one workbook to complete before activating another one.

Recommendation: Deactivating a workbook releases memory that is used by the workbook's datasets and tables. Therefore, if the cluster is low on memory, deactivate workbooks that you are not currently using. However, be aware that re-activating a workbook takes longer than switching from one active workbook to another. During activation, Xcalar might re-create the datasets and active tables, which might take a while, especially if the tables were created with complex operations.
TIP: To see how the activation or deactivation of a workbook affects memory consumption, you can preview a workbook to determine the total memory used by all tables in the workbook. For information about previewing a workbook, see Previewing a workbook.

During workbook activation, you can click the memory alert icon (the icon next to the user icon) to display Xcalar cluster statistics.

Activating a workbook

To activate a workbook, follow these steps:

  1. In the Workbook Browser, locate the card for the workbook you want to activate.

  2. Follow one of these steps:

    • Click anywhere (except the preview icon and ellipsis icon) on the workbook card.

      If another workbook is already active, a confirmation modal window is displayed. To display the workbook in the current browser tab, click CONFIRM.

    • Click the ellipsis icon on the workbook card and select Open workbook in new tab. The workbook is displayed in a new browser tab.

If you have other active workbooks, they remain active.

Deactivating a workbook

NOTE: Xcalar Design cannot deactivate a workbook if the workbook has an operation in progress. Wait until the operation is finished before you switch from this workbook.

To deactivate a workbook, follow these steps:

  1. In the Workbook Browser, locate the workbook card for the workbook you want to deactivate.
  2. Click the ellipsis icon on the workbook card and select Deactivate Workbook.
  3. When the confirmation modal window is displayed, click CONFIRM.
NOTE: Logging out of Xcalar Design or closing a browser tab does not deactivate a workbook.

Switching between active workbooks

If you have multiple active workbooks and you want to switch from one to another, follow these steps:

  1. In the Workbook Browser, locate the workbook card for the workbook to which you want to switch.

  2. Follow one of these steps:

    • Click anywhere (except the preview icon and ellipsis icon) on the workbook card. The workbook is displayed in the current browser tab. A confirmation modal window is displayed. To display the workbook in the current browser tab, click CONFIRM.
    • Click the ellipsis icon on the workbook card and select Open workbook in new tab. The workbook is displayed in a new browser tab, while the workbook from which you switch remains in its browser tab.
NOTE: The workbook you switch from remains active.

Deleting a workbook

You can delete a workbook only if it is inactive.

Deleting a workbook has the following effects:

  • All aggregates, tables, and worksheets in the workbook are deleted. They are not recoverable after the deletion.
  • A dataset created through the workbook might or might not remain, depending on whether the dataset is also being used by other workbooks. The dataset is removed only if the workbook being deleted is the only one using the dataset. For more information about dataset deletion, see Understanding dataset deletion.
  • UDFs in the workbook are deleted.
IMPORTANT: You cannot undo workbook deletions. The removal of the workbook and its associated items is permanent. Therefore, download a workbook as a backup before you delete the workbook. For information about downloading workbooks, see Understanding workbook downloads.

To delete a workbook, follow these steps:

  1. If the workbook is not inactive, deactivate it.
  2. Click the ellipsis icon and select Delete Workbook.
  3. When the confirmation modal window is displayed, click CONFIRM.

Previewing a workbook

Before you activate a workbook or switch to another active workbook, you might want to verify that the selected workbook is the desired one. Click on the workbook card to display the Workbook Preview modal window, which contains a list of tables organized by worksheet.

For each table, click its name to display the table's dataflow graph. The dataflow information might help you determine whether the selected workbook is the appropriate one.

Editing a workbook name and description

You can edit the name of an active or inactive workbook. You can also add or edit a workbook's description.

NOTE: If you change the workbook name, the Jupyter Notebook directory for the workbook is updated accordingly.

To edit a workbook name or description, follow these steps:

  1. Click the ellipsis icon on the workbook card and select Edit Name and Description.

  2. In the Edit Workbook modal window, edit the name or description.

    The following list describes the naming conventions:
    • The name is case-sensitive.
    • Length must not exceed 255 characters.
    • Characters must be A-Z, a-z, 0-9, hyphen (-), underscore (_), or space.
    • First character must be a letter.
    • The name must not be one of the following strings:
      • udf
      • udfs
    • For the admin user, the name must not be PreUpgrade.
  3. Click Confirm.
TIP: If the complete description is too long to fit on the workbook card, move the cursor over the description to display a tooltip with the complete description.

Duplicating a workbook

To duplicate an active or inactive workbook, click the ellipsis icon on the card for the workbook and select Duplicate Workbook. A new workbook card is displayed, with a workbook name based on the original name. Initially, the new workbook is inactive.

The following list describes what are duplicated:

  • All worksheets, temporary and active tables, and aggregates.
  • All UDFs in the original workbook.
  • Jupyter Notebooks associated with the original workbook.
NOTE: The workbook description is not duplicated.

Understanding workbook downloads

You might want to download a workbook so that you or another user can upload it, either to the same or another Xcalar instance. The following list describes possible scenarios where you want to download a workbook:

  • Handing off a workbook to other users: After performing interactive modeling on data from a data source, you can download the workbook and give it to other users, who can use it in one of the following ways:
    • Use the workbook as a template to build a similar data pipeline.
    • Continue modeling from the dataflows in the workbook.
  • Archiving or backing up a workbook: After some complex operations, you might want to download the workbook before proceeding with further modeling steps. Because the downloaded workbook contains all the relevant workbook information, such as UDFs and Jupyter Notebooks, it serves as a copy of the workbook to which you can revert if needed.
  • Diagnosing workbook activation problems: If for some reason a workbook cannot be activated, you can download it and send the workbook file to Xcalar for evaluation. In addition, if a fix for your particular workbook problem is available on another Xcalar instance, you can upload the workbook to that Xcalar instance.
  • Fixing UDF problems in a workbook: If UDF problems prevent a workbook from being activated, you can download the workbook, fix the UDF problems, and then upload the workbook to replace the original one. For more information about how to fix UDF problems in this scenario, see Replacing a UDF in an inactive workbook.
NOTE: Any Jupyter Notebook that references the workbook is also downloaded.

Understanding the workbook file

The downloaded workbook is called a workbook file, which is a compressed archive, with a .tar.gz file name extension. Xcalar Design downloads the workbook to the download location on your computer, based on your browser's download setting.

The unpacked contents of the workbook file are in open and accessible formats, such as Python and JSON. The following items are included in the workbook file:

  • UDF modules. The UDF module named default might not be included, depending on the factors described in Understanding UDFs used by a downloaded workbook.
  • Jupyter Notebooks for the workbook.
  • workbook metadata.

Batch dataflows created from dataflows in the workbook are not included in the workbook file. This is because each batch dataflow, upon creation, is put in a global namespace, which is accessible to all users on the Xcalar instance.

IMPORTANT: To ensure that the workbook file can be unpacked and replayed by Xcalar, do not modify the workbook file in any way after the download, unless you are advised to do so by Xcalar technical support.

Understanding UDFs used by a downloaded workbook

A UDF module named default is provided by Xcalar. This UDF module is unmodifiable by users and is shared by all workbooks.

All other UDFs are created by users and are not shared by workbooks. A UDF can be used by the workbook in which it is created or by the workbooks to which the UDF is copied. To see what UDFs are available for each workbook, use the Manager tab of the User Defined Function panel.

The following list describes how UDFs are used by a downloaded workbook:

  • During workbook download, Xcalar includes all UDFs created specifically for the workbook in the workbook file. After the workbook is uploaded and activated, the uploaded workbook uses the UDFs unpacked from the workbook file.

  • During workbook download, Xcalar includes the default UDF module in the workbook file only if the workbook being has used any functions in the module. The default module is not included if no functions in the module has been used.

    The default module, however, is not used after you upload and activate the workbook. The uploaded workbook always uses the default module on the Xcalar instance. The default module included in the workbook file is only for Xcalar technical support to diagnose problems that the uploaded workbook might have.

Downloading a workbook

To download a workbook, click the ellipsis icon on the workbook card and select Download Workbook.

Understanding workbook uploads

You can upload a workbook file to a Xcalar instance if the workbook was created on a Xcalar instance running a compatible version. Upload might fail if you try to upload a workbook to a Xcalar instance with an earlier version than the version from which you downloaded the workbook.

NOTE: Any Jupyter Notebook that references the workbook is also uploaded.

Understanding the datasets used by the uploaded workbook

When you activate an uploaded workbook for the first time, datasets on the cluster instance are affected in the following ways:

  • If a dataset used by the uploaded workbook does not exist on the Xcalar instance, Xcalar creates the dataset, and the user who uploads the workbook is the creator of the dataset.
  • If a dataset used by the uploaded workbook exists on the Xcalar instance, whether a dataset is created depends on the factors described in the following list:
    • If the user uploading the workbook is also the creator of the dataset on the Xcalar instance, the dataset is shared between the original workbook and the uploaded workbook. Xcalar does not create a dataset when activating the uploaded workbook.
    • If the user uploading the workbook is not the creator of the dataset on the Xcalar instance, Xcalar creates the dataset, and the user who uploads the workbook is the creator of the new dataset.

When you activate an uploaded workbook, all datasets used by the workbook become locked.

Before you begin

To ensure that Xcalar Design can activate your uploaded workbook, verify that each data source used in the workbook is accessible to the cluster where the upload takes place. When trying to activate a workbook, if Xcalar cannot import the data source, the activation process fails with an error message similar to the following:

  • No such target 'xc'

  • '/path' does not exist

Uploading a workbook

To upload a workbook, follow these steps:

  1. In the Workbook Browser, locate the New Workbook card and click Browse on the card. A window is displayed for you to navigate to the location where the downloaded workbook file is saved.
  2. Select the workbook file and click Open. After the upload is finished, a workbook card is displayed in the Workbook Browser. The workbook is initially inactive. You are ready to perform workbook tasks on this workbook as usual.
