Managing datasets

When you import data, Xcalar Design creates a dataset (typically in memory), which is the representation of raw data and metadata. You can preview the data in the dataset and create tables with all or selected fields in the dataset.

Understanding dataset names

Xcalar Design provides a default name for each dataset when you import data. The name is based on the name of the source file or directory. You can enter a different name or accept the default. You cannot rename the dataset at a later time.

Understanding shared datasets

By default, datasets are not shared. After creating a dataset, you can move it to a folder named Shared if you want other users to be able to use it.

NOTE: Datasets created before Version 1.3.1 are automatically placed in the Shared folder.

When you can stop sharing a dataset

You can remove a currently shared dataset from the Shared folder if these conditions are true:

  • You created the dataset.
  • No other users have a lock on it. If you are the only user who has a lock on it, you can move it out of the Shared folder. For the meaning of a dataset lock, see Understanding dataset locking.

What you can or cannot do to shared datasets

In the Shared folder, whether or not you are the owner of a dataset, you can perform these tasks:

  • Preview a dataset.
  • Create tables from a dataset.
  • Lock or unlock a dataset.
  • Delete a dataset, provided that it is not locked by any user.

You can create sub-folders in the Shared folder and move datasets in and out of sub-folders. You can also rename the sub-folders.

You cannot stop sharing a dataset unless you are the owner of the dataset.

Understanding the Datasets panel

The Datasets panel displays all the datasets that you can use. They are:

  • datasets created by you, which are not shared
  • datasets in the folder named Shared, which are created by you or other users

Organizing datasets in folders

In the Datasets panel, you can organize datasets by folders. For example, you can create a folder named Finance for datasets that import data sources from the Finance department, another folder named Engineering for datasets that import data sources from the Engineering department, and so on.

Follow these steps to place datasets in folders:

  1. Right click anywhere in the Datasets panel.
  2. Select New Folder in the pop-up menu.

  3. Click the Edit icon in the pop-up menu and enter a folder name.

    The following list describes the naming conventions:
    • The name is case sensitive.
    • Length must not exceed 255 characters.
    • Characters must be A-Z, a-z, 0-9, hyphen (-), and underscore (_).

    4. Do not specify Shared as the folder name because it is reserved for the folder containing shared datasets.

  4. Press Enter.
  5. Drag and drop any datasets to the newly created folder.

Sorting datasets

The Datasets panel displays a SORT BY menu for you to sort the datasets by type or by name. If you sort by type, the dataset folders are listed before the datasets.

Alternatively, right click anywhere in the Datasets panel to display a pop-up menu. Use Sort on the menu to sort by name or by type.

Understanding dataset locking

Any user can lock and unlock a dataset that is visible in the Datasets panel. A locked dataset is a dataset that cannot be deleted by any user, including the creator. The following icon is for a locked dataset named airlines:

Dataset locking in the Shared folder

For datasets in the Shared folder, users manage their own locks independently. For example, User1 can place a lock on the airlines dataset, but User2 can choose not to place a lock on it. A dataset icon without a lock means that you have not locked the dataset. It does not indicate whether other users have locked it.

You place a lock on a dataset to prevent other users from deleting it. The reason for doing so is that while you currently do not have tables based on the dataset, you want to ensure that it is available for future use.

Dataset locking outside the Shared folder

Datasets not shared are visible only to you. Locking or unlocking a dataset does not have any effects on others. However, locking a dataset prevents you from inadvertently deleting it.

When a dataset becomes locked

After you create a dataset, by default, it is locked. You, as the creator of the dataset, can choose to keep or remove the lock.

Xcalar automatically locks a dataset when a user creates a table from it. As long as you have a table (active or temporary) or an aggregate dependent on a dataset, the dataset is automatically locked for you. You cannot remove the lock unless you remove all tables and aggregates that depend on the dataset.

Determining who has a lock on a shared dataset

Outside the Shared folder, the only user who can place a lock on a dataset is the creator of the dataset.

Inside the Shared folder, to view who has a lock on a dataset, right click the dataset icon and select Get info in the menu. The information displayed also shows who created the dataset.

Understanding dataset deletion

You can delete an unshared dataset if it is not locked.

Any user can delete a shared dataset. However, you can do so only if no users have a lock on the dataset.

In the Datasets panel, the name of the user who created the shared dataset is displayed in parentheses. The name enables you to notify the appropriate user before you delete the dataset.

Effects of workbook deactivation and reactivation on shared datasets

This section describes how dataset availability is affected by workbook deactivation and reactivation.

Effects of sharing and unsharing a dataset

Suppose you share a dataset and another user (User1) creates a table from it. After User1 deactivates the workbook containing the table, the shared dataset is no longer being used by User1. Provided that no other users use the dataset, you can move the dataset out of the Shared folder. As a result, no users but you can use the dataset.

When User1 reactivates the workbook, all users see the dataset displayed in the Shared folder again. Xcalar Design displays a message for the creator of the dataset, indicating that the dataset is moved back to the Shared folder because a workbook dependent on it has been reactivated.

Effects of deleting a dataset

Suppose you share a dataset and another user (User1) creates a table from it. After User1 deactivates the workbook containing the table, the shared dataset is no longer being used by User1. Provided that no other users use the dataset, you can delete the dataset. As a result, it is no longer visible to any user.

When User1 reactivates the workbook, all users see the dataset displayed in the Shared folder again. They can use the dataset in the same way as they did before the dataset deletion.

Effects of pausing a workbook on shared datasets

Pausing a workbook has no effects on datasets. For example, if a user pauses a workbook that uses a shared dataset, that dataset is still being used by the user. The creator of the dataset cannot stop sharing the dataset, and no user can delete the dataset.

Effect of cluster restart on datasets

After the cluster is restarted, some datasets previously in your list of datasets might not be displayed. Only the datasets used by an active workbook on the cluster are shown immediately. This is true of datasets inside or outside the Shared folder.

A dataset appears in the list again in one of these situations:

  • A user activates a workbook containing a table or aggregate that depends on the dataset.
  • You activate the workbook that was active when you created the dataset.
  • A batch dataflow is run against the dataset.
EXAMPLE: You create a dataset called airlines, which is used in workbook1, and a dataset called carrier, which is used in workbook2. Both datasets are visible in the list of datasets, regardless of which workbook is active. After a cluster restart, if workbook1 is active, only the dataset named airlines is displayed. To also display the dataset named carrier, activate workbook2.

Locking and unlocking a dataset

Follow these steps to lock a dataset:

  1. In the Datasets panel, locate the dataset in your folder or the Shared folder.
  2. If the Datasets panel is in grid view, right click the dataset icon and click Lock dataset. If the Datasets panel is in list view, click the closed lock icon for the dataset.

Follow these steps to unlock a dataset:

  1. Verify that you do not have tables or aggregates dependent on the dataset.
  2. In the Datasets panel, locate the dataset in your folder or another user's folder.
  3. If the Datasets panel is in grid view, right click the dataset icon and click Unlock dataset. If the Datasets panel is in list view, click the open lock icon for the dataset.

Deleting a dataset

Follow these steps to delete a dataset:

  1. In the Datasets panel, locate the dataset in your folder or another user's folder.
  2. Right click the icon or name of the dataset and then click Get info in the drop-down list to determine if the dataset is locked by any user.
  3. From the drop-down list, click Delete.
  4. When a confirmation message is displayed, click CONFIRM if you are sure that you no longer need the dataset.
