Using Xcalar Design with Jupyter Notebook

Xcalar Design includes Jupyter Notebook integration, which enables you to write code directly in the browser, interactively analyze data from a table, or create user-defined functions (UDFs).

NOTE: This topic assumes that you are familiar with Jupyter Notebook. For more information about the Jupyter Notebook user interface, such as for running code and generating documents, see https://jupyter-notebook.readthedocs.io).

"The term notebook used in the Xcalar Design online help refers to Jupyter Notebook."

Programming language supported

While Jupyter Notebook can run code that is written in various programming languages, the Jupyter Notebook accessed from Xcalar Design supports only Python 3.6 for helping you write Xcalar user-defined functions (UDF).

Relationship between a notebook and a Xcalar Design workbook

You can create more than one notebook for each workbook. Remember that the purpose of opening a notebook is to use Python to analyze data from a data source file or a Xcalar table.

NOTE: Xcalar recommends that you do not open a notebook containing code that connects the notebook to another workbook, as this can lead to errors.
EXAMPLE: Your current active workbook is named Workbook1. Suppose you created a notebook earlier named MyNotebook2 when the workbook named Workbook2 was active. Using MyNotebook2 for data from Workbook1 might cause errors. To work on data from Workbook1, create a new notebook specifically for Workbook1.

If a notebook does not contain any code that connects to it to another workbook, you can use the notebook.

Naming a notebook

After you create a notebook, follow these Jupyter Notebook naming conventions to name the notebook:

  • The name must contain at least one character.
  • The name is case-sensitive.
  • The name does not contain any of the following:
    • colon (:)
    • forward slash (/)
    • backslash (\).
TIP: Because it is important to open the appropriate notebook for the active workbook, always name your notebook descriptively so that you can easily relate a notebook to a workbook. For example, for a workbook named Airlines, you can create notebooks named AirlinesNB1, AirlinesNB2, and so on. If the notebooks are named untitled1 and untitled2, it is not obvious that the notebooks are created for the Airlines workbook.

Starting the Jupyter Notebook

To open the JUPYTER NOTEBOOK window, do one of the following:

  • When you import a data source, you can start Jupyter Notebook to create an import UDF from the Import Data Source page. For information about importing, see Importing a data source.
  • Click the Jupyter Notebook icon () in the Xcalar Design menu.
  • From the table menu, select Send to Jupyter.

Depending on whether you have an open notebook, the JUPYTER NOTEBOOK window shows one of the following:

  • The Jupyter Notebook dashboard with a file tree, which lists the notebooks created for all workbooks. You can manage your notebooks by opening a notebook for editing or uploading a notebook.

  • The notebook editor with the contents of the open notebook. You can edit the contents as you normally would in Jupyter Notebook.

    IMPORTANT: Do not edit the Xcalar starter code, which is necessary for Xcalar Design to connect to the notebook. The code is in the first cell of the notebook. If you accidentally change the code, you can restore the code by following the instructions in Connecting a notebook to the active workbook.

Opening a notebook not connected to the active workbook

If you open a notebook created from a workbook that is not the active workbook, a warning message is displayed, which shows the name of the expected workbook. Click CLOSE to dismiss the warning message and do one of the following:

  • Close the existing notebook and open the notebook corresponding to the active workbook.
  • Close the existing active workbook and open the correct active workbook for the notebook. For more information, see Workbook Browser.

Connecting a notebook to the active workbook

After opening a notebook, you can connect it to the active workbook, regardless of which workbook you used when the notebook was first created.

You can also use the following procedure to re-establish the connection to the active workbook. For example, if you accidentally modified the Xcalar starter code and you want to restore it to its original form, you can complete the procedure for Xcalar Design to re-enter the code in the notebook. After the code is rewritten, delete the cell in which you made the unintended changes.

NOTE: You can establish a connection between a notebook and the active workbook. You cannot connect a notebook to any other workbook.

To establish a new connection, follow these steps:

  1. At the top of the JUPYTER NOTEBOOK window, click CODE SNIPPETS, which displays a drop-down menu.

  2. Select Connect to Xcalar Workbook, which creates a new cell in the notebook that contains the starter code for connecting the notebook to the active workbook.

    IMPORTANT: If the notebook has a cell with starter code for connecting to a workbook other than the active workbook, you must delete that cell. A notebook with connections to multiple workbooks causes errors.

Creating a UDF with a Xcalar UDF template

Xcalar Design provides you with the following UDF templates:

  • A Map UDF, which is a function for running the Map operator.
  • An Import UDF, which is a function for importing a custom data source format.

The following section describes how to display a UDF template and modify the code.

NOTE: If you want to create an import UDF, start Jupyter Notebook from the Import Data Source window.

The procedure requires that you assign a name to the module containing the function and a name to the function.

The following list describes the naming conventions:
  • Characters must be a-z, 0-9, hyphen (-), and underscore (_).
  • Letters must be lowercase.
  • First character must be a letter or an underscore.

Follow these steps to display a Xcalar UDF template:

  1. Open a notebook and place the insertion point in a notebook cell.

  2. At the top of the JUPYTER NOTEBOOK window, click CODE SNIPPETS, which displays a drop-down menu, as shown in the following screenshot:

Follow these steps to create a Map UDF:

  1. Click Create Map UDF to display the UDF Template modal window.
  2. In the UDF Template modal window, type the module name and function name.
  3. Select the table and table columns on which the UDF will run.
  4. Click CONFIRM. Jupyter Notebook displays a notebook cell for you to enter your code and an additional cell to test your code.

  5. When you are satisfied with the result of your UDF, click Use UDF on table_name. The following list describes the results:

    • Xcalar Design displays the MAP panel with the newly created UDF as the selected function.
    • Xcalar Design displays the table for which the UDF is written. The columns referenced in the UDF are highlighted.
    • The UDF is saved on the cluster.

    You are ready to run the UDF from the MAP panel.

Follow these steps to create an import UDF:

  1. In the Import Data Source window, verify that Custom Format is selected as the data source format. For information about the Import Data Source window, see Importing a data source.
  2. In the Import Data Source window, click WRITE PARSER USING JUPYTER to display the Create Import UDF modal window.

  3. In the Create Import UDF modal window, follow these steps:

    1. Select the data target.
    2. Type the data source path relative to the data target.
    3. Type the module name.
    4. Type the function name.
  4. Click CONFIRM. Jupyter Notebook displays a notebook cell for you to enter your code and an additional cell to show an example of an import UDF.

  5. When you are satisfied with the code, click APPLY UDF ON DATASET PANEL in the Jupyter Notebook cell containing the code. The following list describes the results:

    • The Import Data Source window is displayed with the UDF module name and function name, and the preview section is updated to show the result of parsing the data source with the newly created UDF.
    • The UDF is saved on the cluster.

    If the following error message is displayed, you did not start Jupyter Notebook from the Import Data Source window:

    Please select a dataset to import in the Dataset Panel and select 'Custom Format' as the current format.

    Because the UDF has already been saved, open the Import Data Source window to import the data source for which the UDF was written. In this window, select Custom Format as the format, and select and apply the UDF.

    NOTE: If you do not see the expected results in the preview section of the Import Data Source window, click WRITE PARSER USING JUPYTER to edit the UDF. After editing the UDF, click APPLY UDF in the Import Data Source window.

Publishing data from a table to a notebook

You can publish a table or a partial table to a notebook. Then you can use Jupyter Notebook for data analysis or visualization for the published rows.

RECOMMENDATION: Xcalar recommends that you do not publish more than 100 table rows to a notebook at one time. Publishing more than 100 rows might take a long time.

  1. Open a notebook that is connected to the active workbook.

    NOTE: If a notebook is not open when you publish a table, Xcalar Design creates a notebook for you.
  2. In the workbook, display the worksheet containing the table that you want to publish to the notebook. Locate the active table for publishing.
  3. Click the table title bar or the triangle in the lower-right corner of the table title bar to display a drop-down menu.
  4. Click Publish to Jupyter.
  5. To publish the entire table, select Full Table. To publish a partial table, enter the number of rows in the Partial Table field. If you enter a number, press Enter to submit it. For example, if you enter 10, the first 10 rows are published to the notebook.
  6. If the columns have prefixed names, a modal window is displayed for you to rename your columns before publishing. You can remove the prefix for each column name. After the renaming, click PUBLISH.
