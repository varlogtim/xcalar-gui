Managing dataset references

When you point to a data source, XDP creates a dataset (typically in memory), which is the representation of raw data and metadata. In order for you to interact with the dataset, Xcalar Design creates a dataset reference through which you can preview the data in the dataset and create tables with all or selected fields in the dataset.

Xcalar Design provides a default name for each dataset reference when you point to a new data source. The name is based on the name of the source file or directory. You can enter a different name or accept the default. You cannot rename the dataset reference at a later time.

Dataset references and the datasets they represent are shared by all users on the same cluster. The Dataset Refs list in the DATASTORE/IN panel displays all the dataset references. The following partial screenshot shows an example of the Dataset Refs list.

Organizing dataset references in folders

In the Dataset Refs panel, you can organize dataset references in folders. For example, you can create a folder named Finance for dataset references pointing to data sources in the Finance department, another folder named Engineering for dataset references pointing to data sources in the Engineering department, and so on.

Understanding dataset reference deletion

Your attempt to delete a dataset reference fails if you have a table or aggregate created from the dataset.

If you do not have a table or aggregate created from the dataset, your attempt to delete a dataset reference has various effects depending on various factors.

NOTE: A dropped table that is a part of a batch dataflow is not considered a table dependent on a dataset. As a result, such table does not prevent the dataset reference deletion. For example, after you drop a table named Airlines1#ue3 that is created from a dataset reference named Airlines, you can successful delete Airlines even if Airlines1#ue3 appears in a batch dataflow (provided that no other tables or aggregates use the dataset). For information about dataflows, see Understanding and creating batch dataflows

If you are the creator of the dataset reference

If no other users have tables or aggregates created from the dataset, the dataset reference is deleted immediately.

Otherwise, the dataset is marked for deletion. You cannot take any action for this dataset reference. When all tables and aggregates from this dataset are dropped, this dataset reference is deleted automatically. The associated dataset is also removed from memory.

If you are not the creator of the dataset reference

You cannot delete a dataset reference if it is in another user's folder. However, you can mark it for deletion by selecting Delete when right-clicking its icon. The following list describes the effects of marking another user's dataset reference for deletion:

  • You cannot take any action on the dataset reference.
  • Other users who do not have tables or aggregates created from the dataset cannot start using the dataset reference.
  • Other user who already have tables or aggregates created from the dataset can see that the dataset reference is marked for deletion, but can continue using it.
  • The creator of the dataset reference can see that the dataset reference is marked for deletion. The creator can choose to continue using it or to delete it. The dataset reference continues to exist until the creator deletes it.
  • If the creator chooses to delete the dataset reference, the effect is the same as that described in If you are the creator of the dataset reference

How dataset reference icon changes

The dataset reference icon changes depending on the status of the dataset reference. For more information about how to determine the status of the dataset reference, see Example of dataset reference deletion.

Deleting a dataset reference

Follow these steps to delete a dataset reference:

  1. In the Dataset Refs list, navigate to the dataset reference that you want to delete. It can be located in your folder or another user's folder.
  2. Right click the icon or name of the dataset reference.
  3. In the drop-down menu, click Delete.
  4. When a confirmation message is display, click CONFIRM if you are sure that you no longer need the dataset reference.

What you cannot do to other users' dataset references

You can view or mark other users' dataset references for deletion. You can preview and create tables from other users' dataset references. However, you cannot perform the following tasks:

  • Renaming the folder containing other users' dataset references
  • Moving other users' dataset references to another folder
  • Creating a folder in another user's dataset reference folder

Example of dataset reference deletion

In this example, user1 creates a dataset reference named airlines, from which a table is created. This dataset reference is visible to all other users on the cluster. The following screenshot shows the dataset reference list as seen by user1.

Suppose user2 navigates to user1's dataset reference folder, right-click the icon for airlines, and select Delete. The airlines dataset reference is not deleted immediately but is marked for deletion. User2 can change the status of airlines because user2 does not have any tables or aggregates created from it. From this point on, user2 can no longer use airlines. The following screenshot shows the dataset reference list as seen by user2:

After the airlines dataset reference is marked for deletion by user2, the appearance of the dataset reference icon changes as well for user1, as illustrated in the following screenshot:

Now user1 and other users who have tables and aggregates from the dataset can continue to use airlines. User1 can delete it when all users no longer have tables and aggregates from the dataset. Alternatively, user1 can mark it for deletion. In this way, after the last table or aggregate from the dataset is dropped, the dataset reference is deleted automatically.

NOTE: To see the correct status of the dataset reference, you must click Refresh. In this example, as soon as user2 marks the airlines dataset reference for deletion, its icon disappears. Without clicking Refresh, it seems to user2 that the dataset is deleted, but actually it remains for other users who currently have tables or aggregates from the dataset.

Effect of cluster restart on dataset references

After a cluster is restarted, some dataset references previously in your list of dataset references might no longer be displayed. Only the dataset references used in your active workbook are shown immediately after the restart.

NOTE: The datasets and the dataset references still exist; they are available for you to use again when needed.

A dataset reference appears in the list again when:

  • you activate a workbook that uses the dataset reference, or
  • a batch dataflow is run against the dataset
EXAMPLE: You create a dataset reference called airlines, which is used in workbook1, and a dataset reference called carrier, which is used in workbook2. Both dataset references are visible in the list of dataset references, regardless of which workbook is active. After a cluster restart, if workbook1 is active, only the dataset reference named airlines is displayed. To also display the dataset reference named carrier, activate workbook2.
